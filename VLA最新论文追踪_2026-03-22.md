### 1. LaST-VLA: Thinking in Latent Spatio-Temporal Space for Vision-Language-Action in Autonomous Driving

**基本信息**
- **论文链接**：https://arxiv.org/abs/2603.01928
- **发布时间**：2026-03-12 (v2更新)
- **作者**：Yuechen Luo, Fang Li, Shaoqing Xu, Yang Ji, Zehan Zhang, Bing Wang, Yuannan Shen, Jianwei Cui, Long Chen, Guang Chen, Hangjun Ye, Zhi-Xin Yang, Fuxi Wen
- **机构**：
  - 香港科技大学（广州）
  - 中国科学院深圳先进技术研究院
  - 北京理工大学
  - 清华大学
  - 哈尔滨工业大学
  - 香港中文大学
  - 澳门大学

**核心创新点**

| 创新维度 | 具体内容 |
|---------|---------|
| **问题洞察** | 现有VLA依赖显式文本Chain-of-Thought (CoT)导致语义-感知解耦和感知-符号冲突；而纯隐式潜在推理缺乏物理约束成为"物理不可知表示" |
| **范式突破** | 提出**Latent Spatio-Temporal CoT**范式，将推理从离散符号处理转移到物理基础的潜在时空空间 |
| **技术核心** | 双特征对齐机制：从3D基础模型蒸馏几何约束，从世界模型蒸馏动态前瞻 |

**技术方法**

```
┌─────────────────────────────────────────────────────────────┐
│                    LaST-VLA 架构                            │
├─────────────────────────────────────────────────────────────┤
│  Stage 1: 双特征对齐                                        │
│  ├── 3D几何蒸馏: 从3D基础模型(VGGt)提取空间几何约束          │
│  └── 动态前瞻蒸馏: 从世界模型(Cosmos)提取时序动态特征        │
├─────────────────────────────────────────────────────────────┤
│  Stage 2: 渐进式两阶段SFT                                    │
│  ├── Phase I (物理感知对齐): λ_WM = λ_3D = 1.0 ≫ λ_action   │
│  │   └── 结构化因果掩码: 潜在互掩码 + 视觉瓶颈掩码           │
│  └── Phase II (潜在基础规划): λ_action = 1.0 ≫ λ_WM = λ_3D  │
├─────────────────────────────────────────────────────────────┤
│  Stage 3: GRPO强化学习优化                                   │
│      └── Group Relative Policy Optimization确保安全合规      │
└─────────────────────────────────────────────────────────────┘
```

**关键设计细节**
- **结构化因果掩码**：
  1. *潜在互掩码*：3D和WM token互相掩码以独立学习
  2. *视觉瓶颈掩码*：阻止action token直接访问原始图像embedding，强制模型将视觉信息压缩到潜在思考H中

**实验结果**
- NAVSIM v1: **91.3 PDMS** (新纪录)
- NAVSIM v2: **87.1 EPDMS** (新纪录)
- SURDS和NuDynamics基准：在时空推理任务上表现卓越

**深度分析**
> LaST-VLA代表了VLA推理范式的第三代演进：第一代是直接回归(Direct VLA)，第二代是显式文本CoT (EMMA/AutoVLA)，第三代即LaST-VLA的**物理基础潜在推理**。其核心洞察是：纯连续潜在空间缺乏物理约束会导致模型崩溃，因此需要通过3D基础模型和世界模型的知识蒸馏来"锚定"潜在表示的物理意义。

### 2. LinkVLA: Unifying Language-Action Understanding and Generation for Autonomous Driving

**基本信息**
- **论文链接**：https://arxiv.org/abs/2603.01441
- **发布时间**：2026-03-02
- **作者**：Xinyang Wang, Qian Liu, Wenjie Ding, Zhao Yang, Wei Li, Chang Liu, Bailin Li, Kun Zhan, Xianpeng Lang, Wei Chen
- **机构**：
  - 浙江大学CAD&CG国家重点实验室
  - 理想汽车 (Li Auto)

**核心创新点**

| 创新维度 | 具体内容 |
|---------|---------|
| **问题洞察** | 现有VLA方法存在两个关键限制：(1) 语言指令与动作输出的持续错位；(2) 自回归动作生成的固有低效率 |
| **结构链接** | 将语言和动作token统一到一个共享离散码本中，在单一多模态模型中处理 |
| **语义链接** | 引入辅助的"动作理解"目标，训练模型从轨迹生成描述性字幕，促进双向语言-动作映射 |
| **效率突破** | 两阶段粗到细生成方法(C2F)，节省**86%推理时间** |

**技术方法**

```
┌─────────────────────────────────────────────────────────────┐
│                    LinkVLA 架构                             │
├─────────────────────────────────────────────────────────────┤
│  视觉编码器: InternViT (预训练)                              │
│  语言主干: Qwen2-0.5B                                       │
├─────────────────────────────────────────────────────────────┤
│  核心创新1: 统一码本                                         │
│  ├── 语言token                                              │
│  ├── 导航点token                                            │
│  └── 轨迹token                                              │
│      └── 全部映射到共享离散码本！                           │
├─────────────────────────────────────────────────────────────┤
│  核心创新2: 双向学习                                         │
│  ├── 前向: 视觉+语言 → 动作                                 │
│  └── 反向: 轨迹 → 描述性字幕 (动作理解目标)                  │
├─────────────────────────────────────────────────────────────┤
│  核心创新3: C2F粗到细生成                                    │
│  ├── Step 1: 预测轨迹终点                                   │
│  ├── Step 2: 插值成粗waypoints                              │
│  └── Step 3: 细化为最终平滑轨迹                              │
└─────────────────────────────────────────────────────────────┘
```

**实验结果 (Bench2Drive)**
| 指标 | LinkVLA | SimLingo | 提升 |
|------|---------|----------|------|
| Driving Score | **91.01** | 85.07 | +6.98% |
| Success Rate | **74.55%** | 67.27% | +10.82% |
| Efficiency | **255.84** | - | 显著优于Orion(151.48)和AutoVLA(146.93) |
| Comfort | **34.62** | 33.67 | 优于SimLingo |
| 推理延迟 | 降低86% | - | 相比标准AR方法 |

**多能力评估亮点**
- Merging: **60.00** (提升6.25)
- Overtake: **80.00** (提升11.11)
- Brake: **93.33** (提升11.66)

**深度分析**
> LinkVLA来自工业界(理想汽车)与学术界联合团队，直击VLA部署的两大工程痛点：**对齐问题**和**效率问题**。统一码本的设计强制语言和动作在表示层面一致，而C2F生成策略将轨迹生成从O(n)降低到O(1)+O(n)，这是工程落地的重要突破。理想汽车在2026年3月的发布表明VLA正在从研究阶段向产品化阶段过渡。

### 3. DynVLA: Learning World Dynamics for Action Reasoning in Autonomous Driving

**基本信息**
- **论文链接**：https://arxiv.org/abs/2603.11041
- **发布时间**：2026-03-13
- **作者**：(待确认完整作者列表)
- **机构**：(待确认，推测为学术界/工业界联合)

**核心创新点**

| 创新维度 | 具体内容 |
|---------|---------|
| **问题洞察** | 文本CoT难以捕获细粒度时空关系；视觉CoT需要预测决策无关的背景和纹理细节，增加推理冗余和学习难度 |
| **新范式** | 提出**Dynamics Chain-of-Thought (Dynamics CoT)**，压缩未来动态到紧凑token，先预测动态token再生成动作 |
| **效率优势** | 相比文本/视觉CoT，推理延迟降低**超过一个数量级** |

**技术方法**

```
┌─────────────────────────────────────────────────────────────┐
│                    DynVLA 架构                              │
├─────────────────────────────────────────────────────────────┤
│  Stage 1: Dynamics Tokenizer                                 │
│  ├── 因子分解: 自车中心动态 (ego-centric)                   │
│  ├──        : 环境中心动态 (environment-centric)            │
│  ├── 物理正则化: 将自车动态与自车运动对齐                    │
│  └── 跨视图一致性: 图像和BEV共享相同底层动态表示             │
├─────────────────────────────────────────────────────────────┤
│  Stage 2: SFT on Dynamics CoT                               │
│      └── 训练模型先生成动态token，再生成动作token             │
├─────────────────────────────────────────────────────────────┤
│  Stage 3: RFT (Reinforcement Fine-Tuning)                   │
│      └── 轨迹级奖励 + KL正则化优化策略                       │
└─────────────────────────────────────────────────────────────┘
```

**深度分析**
> DynVLA与LaST-VLA代表了相似的演进方向：远离显式文本推理，转向紧凑的隐式表示。DynVLA的独特之处在于将动态显式分解为自车动态和环境动态，并引入物理正则化解决两者的混淆问题（如"自车前进" vs "前车后退"）。这是世界模型与VLA融合的典型范式。

### 4. DriveMoE: Mixture-of-Experts for Vision-Language-Action Model in End-to-End Autonomous Driving

**基本信息**
- **论文链接**：https://arxiv.org/abs/2505.16278
- **发布时间**：2025-05-22 (CVPR 2026接收)
- **作者**：Zhenjie Yang, Yilin Chai, Xiaosong Jia, Qifeng Li, Yuqian Shao, Xuekai Zhu, Haisheng Su, Junchi Yan
- **机构**：上海交通大学 (Thinklab-SJTU)

**核心创新点**

| 创新维度 | 具体内容 |
|---------|---------|
| **问题洞察** | E2E-AD需要有效处理多视图感知数据，并应对复杂多样的驾驶场景（特别是激进转弯等罕见操作） |
| **MoE应用** | 将Mixture-of-Experts架构从LLM领域迁移到自动驾驶VLA |
| **双MoE设计** | Scene-Specialized Vision MoE + Skill-Specialized Action MoE |
| **基线贡献** | 同时发布了Drive-π₀基线（将π₀ VLA从具身AI领域适配到自动驾驶） |

**技术方法**

```
┌─────────────────────────────────────────────────────────────┐
│                    DriveMoE 架构                            │
├─────────────────────────────────────────────────────────────┤
│  基线: Drive-π₀ (基于π₀ VLA的自动驾驶适配)                   │
├─────────────────────────────────────────────────────────────┤
│  创新1: Scene-Specialized Vision MoE                         │
│  ├── 路由器动态选择相关相机视图                              │
│  └── 模拟人类驾驶员选择性关注关键视觉线索                     │
├─────────────────────────────────────────────────────────────┤
│  创新2: Skill-Specialized Action MoE                         │
│  ├── 路由器激活不同驾驶行为的专业专家模块                     │
│  └── 显式行为特化，避免模式平均问题                           │
└─────────────────────────────────────────────────────────────┘
```

**实验结果**
- Bench2Drive闭环评估：**SOTA性能**
- 验证了Vision MoE和Action MoE结合在自动驾驶任务中的有效性

**深度分析**
> DriveMoE代表了VLA架构从"Dense"向"Sparse"演进的重要尝试。MoE架构在LLM中已证明可实现更强的可扩展性，DriveMoE将其引入自动驾驶领域并解决两个关键问题：*看什么*（Vision MoE）和*怎么做*（Action MoE）。值得注意的是，该工作还发布了Drive-π₀基线，填补了具身AI领域π₀模型在自动驾驶领域的空白。

### 5. A Survey of Reasoning in Autonomous Driving Systems

**基本信息**
- **论文链接**：https://arxiv.org/abs/2603.11093
- **发布时间**：2026-03-11
- **作者**：(多位作者，待确认完整列表)
- **机构**：(推测为多国多机构联合)

**核心贡献**

这是一篇**系统性综述**，而非技术创新论文。其价值在于：

1. **认知层次框架**：将自动驾驶推理划分为三个层次
   - 感知层 (Perceptual Level)
   - 自我中心推理层 (Egocentric Reasoning Level)
   - 社会认知层 (Social-Cognitive Level)

2. **方法分类体系**：
   | 类别 | 代表工作 |
   |------|---------|
   | 推理增强感知 | HiLM-D, ARGUS, DMAD |
   | 推理引导预测 | TRACE, WOMD-Reasoning |
   | 规划与决策推理 | CoT-VLA, Drive-R1, Poutine |
   | 整体架构 | ORION, EMMA, DriveGPT4 |
   | 辅助任务 | ChatSim, TeLL-Drive |

3. **时间线整理**：从2023年Drive Like a Human到2026年VLAAD-TW的完整演进脉络

**深度分析**
> 这篇综述为理解VLA领域的快速发展提供了关键框架。其"社会认知层"概念特别重要——它指出真正的L4/L5自动驾驶需要理解社交常识、交通规则和他者意图，而不仅仅是感知和控制。这与当前VLA向推理增强方向发展的趋势高度一致。

## 技术趋势总结

### 趋势1: 推理范式的三代演进

```
第一代: Direct VLA (直接回归)
    └── UniAD, VAD, SparseDrive
    └── 特点: 高效但缺乏推理，长尾场景弱

第二代: Explicit Textual CoT (显式文本推理)
    └── EMMA, AutoVLA, DriveVLM
    └── 特点: 可解释但高延迟、易产生幻觉

第三代: Physics-Grounded Latent CoT (物理基础潜在推理) ⭐当前热点
    └── LaST-VLA, DynVLA, LinkVLA
    └── 特点: 高效+物理约束+可扩展
```

### 趋势2: 语言-动作对齐成为核心战场

| 方法 | 对齐策略 | 代表工作 |
|------|---------|---------|
| 离散码本统一 | 语言和动作共享码本 | LinkVLA |
| 双向学习目标 | 动作→语言+语言→动作 | LinkVLA |
| 物理约束蒸馏 | 3D几何+动态先验监督 | LaST-VLA |
| 动作理解目标 | 轨迹生成描述 | LinkVLA |

### 趋势3: 效率优化成为工程化关键

| 技术 | 优化目标 | 效果 |
|------|---------|------|
| C2F粗到细生成 | 轨迹生成延迟 | -86% |
| Dynamics Token | 推理token数 | -10x |
| MoE稀疏化 | 计算量 | 动态路由 |
| Token稀疏化 | 上下文长度 | 融合冗余token |

### 趋势4: 世界模型与VLA深度融合

```
融合范式:
├── 预训练阶段: 世界模型提供密集监督 (DriveVLA-W0)
├── 推理阶段: 世界模型token作为中间表示 (DynVLA)
├── 蒸馏阶段: 世界模型提供动态先验 (LaST-VLA)
└── 闭环训练: 世界模型作为仿真器 (GAIA-2)
```

### 趋势5: 强化学习在VLA中的角色转变

| 阶段 | RL作用 | 代表 |
|------|--------|------|
| 早期 | 后 hoc 策略优化 | AlphaDrive |
| 当前 | 与CoT深度融合 | Drive-R1, LaST-VLA(GRPO) |
| 新兴 | 从失败中显式学习 | Learning from Failures (2026.03) |

### 对75篇论文分析框架的验证与扩展

#### 验证的假设
1. **VLA正在取代VLM+E2E的级联架构** ✅
   - LinkVLA、LaST-VLA、DynVLA均验证了这一趋势
   - 理想汽车、小鹏等工业界实践也证实

2. **推理能力是区分VLA代际的关键** ✅
   - 2026年3月论文的核心差异都在"如何推理"
   - 从"是否有推理"转向"如何高效准确推理"

3. **世界模型是VLA的下一个倍增器** ✅
   - LaST-VLA、DynVLA均引入世界模型知识
   - 理想汽车"预测式隐世界模型"概念验证

#### 需要修正的观点
1. **文本CoT的重要性被高估**
   - 原分析可能过于强调文本推理的可解释性
   - 2026.03趋势表明：潜在推理+可视化解释可能是更好路径

2. **VLA效率问题可能比预期更快解决**
   - LinkVLA的86%延迟降低、DynVLA的10x降低
   - 工程优化(如C2F生成)可能比架构改进更有效

1. **物理基础推理 (Physics-Grounded Reasoning)**
   - LaST-VLA的3D几何蒸馏和动态前瞻
