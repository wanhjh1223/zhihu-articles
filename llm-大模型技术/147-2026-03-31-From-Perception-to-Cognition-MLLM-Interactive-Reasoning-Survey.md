# From Perception to Cognition: 多模态大模型视觉语言交互推理全景解析

## 基本信息

- **论文标题**: From Perception to Cognition: A Survey of Vision-Language Interactive Reasoning in Multimodal Large Language Models
- **发表时间**: 2025年
- **论文类型**: arXiv综述论文
- **核心方向**: 多模态大语言模型、视觉语言推理、交互式推理
- **技术特点**: 系统性综述多模态推理技术演进路径

---

## 一、研究背景与核心问题

### 1.1 从感知到认知的鸿沟

多模态大语言模型（MLLMs）在近年来取得了令人瞩目的进展，从早期的图文对齐到如今的复杂推理任务，技术边界不断拓展。然而，**从感知到认知的跨越**仍然是该领域面临的核心挑战。

**感知层面的局限**

传统MLLMs在视觉感知方面取得了长足进步，能够准确识别图像中的物体、场景和属性。然而，单纯的感知能力并不足以支撑复杂的推理任务。例如，模型可以识别出图片中的"苹果"和"桌子"，但难以理解"为什么苹果会放在桌子上"这样的因果推理问题。

当前感知层面的主要瓶颈包括：
- **细粒度视觉理解不足**：难以捕捉细微的视觉线索和纹理信息
- **空间关系理解薄弱**：对物体间的空间位置关系把握不准
- **时序动态建模缺失**：在视频理解等时序任务中表现欠佳

**认知层面的挑战**

认知推理要求模型不仅要"看懂"，更要"想明白"。这涉及：
- **因果推理能力**：理解事件之间的因果关系
- **逻辑演绎能力**：进行严密的逻辑推导
- **常识推理能力**：运用日常知识进行推断
- **多步推理能力**：处理需要多步骤推导的复杂问题

### 1.2 视觉语言交互推理的兴起

为了解决从感知到认知的鸿沟，学术界提出了**视觉语言交互推理（Vision-Language Interactive Reasoning）**这一新兴研究方向。该方向的核心思想是：通过建立视觉与语言之间的动态交互机制，使模型能够在推理过程中持续关注和利用视觉信息。

**传统静态编码的局限**

大多数现有MLLMs采用静态的单遍视觉编码机制，即在推理开始前就将图像转换为固定表示。这种方法存在两个关键缺陷：

1. **视觉信息衰减**：随着文本生成进行，初始视觉信息逐渐衰减
2. **语言先验依赖**：模型过度依赖语言先验，容易产生与视觉事实矛盾的幻觉

**交互式推理的解决方案**

视觉语言交互推理通过构建"思考-行动-观察"推理循环来解决上述问题：
- **动态视觉关注**：模型根据推理需求动态调整对视觉证据的关注
- **迭代信息获取**：支持多轮视觉信息获取和验证
- **视觉证据驱动**：推理过程由视觉证据驱动而非语言先验主导

### 1.3 本文献综述的价值

本综述系统性地梳理了从感知到认知的技术演进路径，涵盖了MLLM推理能力的各个方面：

1. **低层视觉感知增强**：如何提取任务相关的视觉表征
2. **视觉语言对齐优化**：如何建立更好的跨模态对齐
3. **思维链推理训练**：如何训练模型进行显式推理分解
4. **动态证据获取**：如何在推理过程中动态获取视觉证据

## 二、低层视觉感知增强技术

### 2.1 投影层优化

视觉编码器提取的特征需要通过投影层映射到语言模型的输入空间。投影层的设计直接影响模型的视觉理解能力。

**代表性方法**

| 方法 | 会议 | 核心贡献 |
|------|------|----------|
| Honeybee | CVPR'24 | 提出灵活且保持局部性的视觉投影器，在效率与空间理解间取得平衡 |
| Uni-Med | NeurIPS'24 | 提出CMoE模块，通过动态资源分配管理任务间关系 |
| ChartMoE | ICLR'25 | 针对图表场景设计MoE投影器，对齐文本和图形元素 |
| LLaVA-ST | CVPR'25 | 后处理模块压缩视频嵌入，同时保持时空关系 |
| LLaVA-Octopus | arXiv'25 | 指令驱动的自适应投影器 |
| Ovis2.5 | arXiv'25 | 用视觉嵌入表替代MLP投影网络 |

**技术趋势分析**

投影层优化呈现以下发展趋势：

1. **从固定到自适应**：早期方法使用固定的投影层，新方法趋向于根据任务类型动态调整投影策略
2. **从全局到局部**：更加关注局部视觉特征的保留，提升细粒度理解能力
3. **从单模态到多专家**：采用混合专家（MoE）架构，为不同视觉元素类型配备专门处理路径

**关键技术细节**

以Honeybee为例，其核心创新在于：
```
传统MLP投影: 视觉特征 → MLP → 语言空间嵌入
                    ↓
Honeybee: 视觉特征 → 局部性保持编码 → 层次化聚合 → 语言空间嵌入
                ↓
            保留空间位置信息，支持细粒度定位
```

### 2.2 任务特定微调

针对不同任务类型进行专门的微调，以提升特定领域的视觉理解能力。

**主要方向**

- **数学推理（MATCHA）**：针对数学图表和几何图形的专门训练
- **医学影像（LLaVA-Med）**：医学领域的视觉问答和诊断
- **低层感知（Q-Instruct）**：针对图像质量、纹理等低层属性的理解
- **图表解析（ChartInstruct）**：专门针对各类图表的理解和推理

**方法对比**

| 方法 | 目标领域 | 训练数据规模 | 核心创新 |
|------|----------|-------------|----------|
| MATCHA | 数学推理 | 中等 | 数学图表专门表示学习 |
| LLaVA-Med | 医学影像 | 大规模 | 医学知识对齐 |
| Q-Instruct | 低层感知 | 小规模 | 质量评估指标学习 |
| ChartInstruct | 图表解析 | 中等 | 图表结构建模 |

### 2.3 提示学习

通过视觉提示（Visual Prompts）来适配下游任务，无需修改模型参数。

**主要方法**

- **VPT (ECCV'22)**：提出视觉提示用于适配下游视觉任务
- **VPGTrans (NeurIPS'23)**：跨MLLM的可迁移视觉提示生成器
- **TVP (CVPR'24)**：联合学习的视觉和文本提示，实现协调的任务适配

**提示学习的优势**

1. **参数效率**：仅需学习少量提示参数，主模型保持不变
2. **快速适配**：针对新任务快速部署，无需重新训练
3. **可解释性**：提示本身反映了任务的关键视觉特征

## 三、视觉语言对齐优化

### 3.1 指令编码范式改进

如何更好地在指令中编码视觉信息是视觉语言对齐的关键问题。

**坐标编码**

**Shikra**率先将坐标编码到文本指令中：
```
传统指令: "描述图中的主要物体"
            ↓
Shikra指令: "描述[0.2, 0.3, 0.5, 0.7]区域内的主要物体"
            (左上x, 左上y, 右下x, 右下y)
```

这种方法使模型能够精确定位图像中的特定区域，大幅提升了细粒度理解能力。

**边界框与掩码编码**

后续工作进一步扩展了视觉元素的编码方式：

- **LLaVA-Grounding (ECCV'24)**：在提示中编码边界框和掩码，统一视觉对话与检测
- **Kosmos-2 (ICLR'24)**：将ground tokens整合到词表中，使模型能生成边界框作为输出
- **GLaMM (CVPR'24)**：引入mask token到词表，支持像素级掩码预测

**视觉指令融合**

**ViP-LLaVA (CVPR'24)**提出通过视觉特征融合来编码视觉指令提示：
```
用户指令 + 图像特征融合 → 增强的多模态表示 → 模型推理
```

这种方法能够更好地捕捉用户的视觉意图，提升指令跟随能力。

### 3.2 跨模态融合与响应生成

跨模态融合的目标是将视觉和语言信息有效结合，生成连贯的响应。

**融合策略对比**

| 策略 | 代表方法 | 优点 | 缺点 |
|------|----------|------|------|
| 早期融合 | CLIP-like | 特征交互充分 | 计算开销大 |
| 中期融合 | LLaVA | 平衡效率与效果 | 可能丢失细粒度信息 |
| 晚期融合 | Flamingo | 模态独立性高 | 跨模态对齐弱 |

**分割解码器**

最新研究引入分割解码器来增强响应生成：
- 使模型能够输出分割掩码
- 增强像素级理解能力
- 支持更精细的视觉定位

## 四、思维链推理训练范式

### 4.1 问题分解的核心挑战

思维链（Chain-of-Thought, CoT）推理的核心是将复杂问题分解为可管理的子问题。然而，这一过程面临诸多挑战：

**现有方法的局限**

| 核心问题 | 具体表现 | 影响 |
|----------|----------|------|
| 结果监督不足 | 基于结果的监督无法保证问题分解过程的逻辑正确性 | 推理链可能出现逻辑断层 |
| 视觉事实脱节 | 纯文本推理分解与视觉事实解耦 | 产生幻觉或不准确推理 |
| 对抗鲁棒性弱 | CoT增强的模型在对抗攻击下表现不稳定 | 实际部署风险 |
| 数学定理偏离 | 数学问题分解时推理过程不遵循相关定理 | 数学推理错误率上升 |

### 4.2 训练范式演进

**监督微调（SFT）方法**

早期方法主要通过监督微调来注入推理能力：

- **Multimodal-CoT (ICLR'24)**：解决文本推理与视觉事实脱节问题
- **Visual CoT (NeurIPS'24)**：确保问题分解的每一步都基于可验证的局部视觉证据
- **LLaVA-CoT (arXiv'24)**：将问题分解从提示工程能力转化为模型的固有技能
- **Learning Theorem Rationale (AAAI'25)**：在数学问题分解中嵌入定理知识

**强化学习方法**

近期研究开始采用强化学习来优化推理过程：

| 方法 | 技术特点 | 核心创新 |
|------|----------|----------|
| UV-DPO | 直接偏好优化 | 通过SFT学习视觉问题分解的grounding |
| VLM-R1 | SFT+GRPO | SFT不足以找到最优分解策略，需结合RL |
| Visual-RFT | SFT+GRPO | 解决复杂视觉问题分解的奖励信号设计难题 |

**训练范式对比分析**

```
纯SFT范式:
问题 → 人工标注CoT → 模型学习 → 生成推理链
    ↓
局限: 依赖高质量标注数据，难以覆盖所有推理路径

SFT+RL范式:
问题 → 基础CoT能力(SFT) → RL优化 → 更优推理策略
    ↓
优势: 能够探索更多推理路径，找到更优解
```

### 4.3 奖励设计策略

强化学习训练中的奖励设计至关重要：

**格式奖励（Format Reward）**
- 确保模型输出符合预期格式
- 通常用于冷启动阶段
- 示例：奖励包含`<think>`和`<answer>`标签的输出

**准确性奖励（Accuracy Reward）**
- 基于最终答案的正确性给予奖励
- 是最直接的优化目标
- 挑战：稀疏奖励问题，长推理链中信用分配困难

**过程奖励（Process Reward）**
- 对推理过程中的每一步进行评估
- 能够提供更细粒度的优化信号
- 实现难度：需要额外的过程标注或评估模型

**组合策略**

最新研究表明，组合使用多种奖励信号效果更佳：
```
总奖励 = α × 格式奖励 + β × 过程奖励 + γ × 准确性奖励
```

## 五、动态证据获取机制

### 5.1 静态编码的根本局限

传统MLLMs在推理开始前就将图像转换为固定表示，这种静态机制存在严重缺陷：

1. **视觉信息衰减**：随着文本生成进行，初始视觉信息在注意力机制中逐渐稀释
2. **幻觉风险**：模型可能过度依赖语言先验，生成与视觉事实矛盾的内容
3. **注意力分散**：长推理链中难以保持对相关视觉区域的持续关注

### 5.2 内源性视觉证据注入

内源性方法通过模型内部机制实现动态视觉关注，无需调用外部工具。

**核心策略**

**动态注意力重聚焦**：
- 在推理的每个步骤重新关注图像的不同区域
- 通过注意力机制动态调整视觉特征权重
- 保持视觉信息在推理链中的持续流动

**交错视觉-语言思维链**：
- 推理步骤与视觉证据交替出现
- 每个推理步骤都显式关联到视觉证据
- 形成"观察-思考-再观察"的循环

**代表性方法**

| 方法 | 核心机制 | 优势 |
|------|----------|------|
| TACO | 视觉提示 + 注意力重聚焦 | 无需外部工具，实现简单 |
| LLaVA-Plus | 工具使用痕迹学习 | 可学习何时需要额外视觉证据 |
| Refocus | 迭代视觉重聚焦 | 支持多轮视觉证据获取 |

### 5.3 外源性视觉证据注入

外源性方法将MLLM视为智能体，通过调用外部工具动态获取视觉证据。

**"计划-执行"范式**

早期工作如Visual Programming、ViperGPT、HuggingGPT建立了工具辅助推理范式：
```
复杂问题 → 一次性分解为工具调用序列 → 执行器运行 → 生成答案
```

**局限**：缺乏动态反馈，对复杂多步交互鲁棒性差

**交错思维链范式**

新一代方法转向自适应的交错思维链模型：
```
每一步推理:
  1. 模型决定需要何种视觉操作
  2. 选择和执行相关视觉工具
  3. 使用即时观察指导下一步行动
```

**代表性系统**

**基于上下文学习的方法**：
- **ReAct / MM-ReAct**：结合推理和行动，通过精心设计提示引导工具使用
- **Visual Sketchpad**：模拟人类解题时在推理过程中进行素描
- **CLOVA**：引入执行反馈和自我修正机制

**基于微调的方法**：
- **LLaVA-Plus**：在工具使用轨迹数据上进行监督微调
- **VTool-R1 / Visual-ARFT**：引入强化学习，通过试错学习最佳工具调用策略
- **OpenThinking**：将多样化视觉工具封装为统一API

**范式对比**

| 维度 | 内源性方法 | 外源性方法 |
|------|------------|------------|
| 实现复杂度 | 低 | 高 |
| 灵活性 | 受限 | 高 |
| 外部依赖 | 无 | 需要工具库 |
| 推理精度 | 中等 | 高 |
| 部署成本 | 低 | 较高 |

## 六、技术趋势与未来方向

### 6.1 当前技术趋势

**趋势一：从单模态到多模态推理**

推理能力的发展正从纯文本向多模态扩展：
- 视觉推理：图像理解、视觉问答
- 视频推理：时序理解、动作预测
- 音频推理：语音识别、音乐理解
- 统一多模态：跨模态统一推理框架

**趋势二：从静态到动态交互**

视觉信息处理从静态编码向动态交互演进：
- 单次编码 → 多轮迭代
- 固定关注 → 自适应聚焦
- 被动接收 → 主动探索

**趋势三：从监督到强化学习**

训练范式从监督学习向强化学习转变：
- SFT提供基础能力
- RL探索更优策略
- 过程奖励优化推理质量

**趋势四：从单一模型到多智能体**

系统架构从单一模型向多智能体协作发展：
- 专业化分工
- 协作推理
- 工具使用

### 6.2 核心挑战与开放问题

**挑战一：推理效率**

当前多模态推理方法往往涉及多轮迭代，计算开销较大：
- 如何平衡推理深度与效率？
- 如何实现自适应推理深度控制？
- 如何缓存和复用中间结果？

**挑战二：可解释性**

尽管CoT提供了一定的可解释性，但仍存在黑盒问题：
- 如何验证每步推理的正确性？
- 如何定位错误来源？
- 如何提供用户友好的解释？

**挑战三：泛化能力**

现有方法在特定领域表现良好，但泛化能力有限：
- 如何构建通用推理框架？
- 如何实现跨领域知识迁移？
- 如何处理未见过的任务类型？

**挑战四：幻觉问题**

多模态推理中的幻觉问题依然严重：
- 如何减少与视觉事实矛盾的生成？
- 如何实现自我事实核查？
- 如何量化不确定性？

### 6.3 未来研究方向

**方向一：统一多模态推理框架**

构建能够处理文本、图像、视频、音频的统一推理框架：
- 跨模态统一表示学习
- 模态无关的推理机制
- 动态模态选择与融合

**方向二：高效推理架构**

开发更高效的推理架构，降低计算开销：
- 早期退出机制
- 自适应推理深度
- 增量推理与缓存

**方向三：可信多模态推理**

提升多模态推理的可信度和鲁棒性：
- 不确定性量化
- 事实核查机制
- 对抗鲁棒性增强

**方向四：具身多模态智能**

将多模态推理与具身智能结合：
- 视觉-语言-行动统一模型
- 交互式环境探索
- 持续学习与适应

## 七、行业影响与应用前景

### 7.1 对AI产业的影响

**推动多模态AI应用落地**

视觉语言交互推理技术的成熟将推动一系列应用落地：
- **智能客服**：能够理解用户上传的图片并提供准确解答

## 八、总结与展望

### 8.1 核心洞察

本综述系统梳理了从感知到认知的技术演进路径，揭示了以下核心洞察：

1. **交互式推理是突破关键**：静态编码向动态交互的转变是提升多模态推理能力的核心路径

2. **训练范式持续演进**：从SFT到SFT+RL的演进反映了领域对更优推理策略的持续追求
