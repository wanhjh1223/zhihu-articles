**机构**: University of Rochester（罗彻斯特大学）, Microsoft Research（微软研究院）  
**发表时间**: 2026年2月  
**论文链接**: [arXiv:2602.16839](https://arxiv.org/abs/2602.16839)

## 一、研究背景与核心问题

### 1.1 大型推理模型的崛起

大型推理模型（Large Reasoning Models, LRMs）正在成为人工智能领域的新范式。与传统的仅依赖规模和数据量来提升性能的大型语言模型（LLMs）不同，LRMs明确强调推理导向的训练信号和架构设计，使其特别适用于数学、科学推理和编程等需要复杂多步推理的领域。

从DeepSeek-R1到OpenAI的o系列模型，再到Qwen3、Kimi k1.5等，推理模型在各类复杂基准测试上取得了令人瞩目的成绩。然而，随着这些模型在越来越复杂的任务上取得突破，研究的焦点逐渐从追求原始能力转向提高训练和部署的效率。

### 1.2 强化学习训练的效率瓶颈

强化学习（RL）已成为LRMs后训练阶段的标准方法，PPO、GRPO等算法为推理行为提供了精细的控制。然而，RL训练面临一个根本性的效率瓶颈：

**问题本质**：基于结果的奖励是稀疏的，只有在完成长序列的动作后才能获得。在这个自回归解码过程中，内存和计算资源消耗巨大。推理链（Chain-of-Thought, CoT）的长度随任务复杂度增加而增加，导致更长的rollout轨迹。这种扩展的CoT序列显著增加了后训练和推理阶段的内存需求。

### 1.3 现有解决方案的局限性

**滑动窗口缓存策略**：一种自然的内存控制方法是采用动态滑动窗口策略限制KV缓存大小，保持内存使用量大致恒定。然而，激进的token丢弃会显著损害长序列理解和生成能力，因为丢弃中间思维会削弱模型整合长距离上下文的能力。

这种性能下降不仅影响推理时的准确性，还会降低rollout阶段的样本质量，从而阻碍训练效果。实验表明，对Qwen模型应用滑动窗口缓存进行RL训练会导致明显的性能下降。

### 1.4 核心研究问题

这引发了一个关键问题：**能否在严格内存预算下高效训练LRMs，同时不牺牲推理准确性？**

这正是本论文要解决的核心问题。

## 二、技术方法详解

### 2.1 核心思想：渐进式思维编码

**Progressive Thought Encoding（PTE）**是一种参数高效微调方法，旨在解决上述瓶颈。其核心思想是：

> **与其丢弃被驱逐的token，不如先学习这些token，更新少量参数，使测试时策略在给定驱逐策略下能够近似全缓存策略。**

### 2.2 技术框架

#### 2.2.1 注意力机制与KV缓存

在标准的Transformer架构中，KV缓存充当"记忆"，避免自回归解码过程中的冗余计算。对于token $x_t$，其注意力输出为：

$$o_t = \text{softmax}\left(\frac{q_t K_t^\top}{\sqrt{d}}\right) V_t$$

其中 $K_t = [k_1, ..., k_t]$ 和 $V_t = [v_1, ..., v_t]$ 表示到步骤t为止的键和值的缓存。

#### 2.2.2 缓存感知GRPO目标函数

标准的GRPO（Grouped Reinforcement Policy Optimization）在完整缓存策略 $\pi_\theta(\cdot|p)$ 下采样轨迹。而在缓存受限设置中，轨迹由缓存策略 $D$ 生成，该策略在解码过程中在线修剪KV缓存：

$$\pi_\theta^D(y|p) = \prod_{t=1}^T \pi_\theta(y_t | C_t^D)$$

其中 $C_t^D = \text{CachePrune}_D(p, y_{<t})$ 是被修剪的上下文。

缓存感知GRPO目标函数为：

$$\mathcal{L}_{\text{GRPO}}^D(\theta_g; \theta_{\text{ref}}) = \mathbb{E}_{y \sim \pi_{\theta_g}^D(\cdot|p)}[r(y) - \beta \text{KL}(\pi_{\theta_g}^D(\cdot|p) || \pi_{\theta_{\text{ref}}}(\cdot|p))]$$

#### 2.2.3 渐进式思维编码机制

**核心创新**：当KV缓存满时，不是直接丢弃早期token，而是使用这些token更新一个紧凑的潜在表示。

**具体实现**：

1. **编码过程**：对于给定问题 $x$，在rollout阶段，模型基于策略 $\pi_\theta^D(y|p)$ 持续解码下一个思维token $\{y_1, ..., y_l\}$，直到KV缓存满。

2. **驱逐策略**：基于策略 $D$，早期token $\{y_{e_1}, ..., y_{e_m}\}$ 将从缓存中驱逐。

3. **潜在表示更新**：使用全局查询向量 $q_g$（作为所有被驱逐上下文的可学习摘要）帮助更新紧凑的潜在表示。LoRA权重的更新计算为：

$$\Delta W = A \cdot \left(W_Q^a q_g \odot (W_K^a K_e)^\top \odot (W_V^a V_e)\right) \cdot S_e \cdot B$$

其中：
- $q_g$ 是聚合被驱逐信息的全局潜在查询
- $W_Q^a, W_K^a, W_V^a$ 是将全局查询token、被驱逐的键 $K_e$ 和值token $V_e$ 映射到压缩潜在空间的权重矩阵
- $A$ 和 $B$ 是将计算出的被驱逐上下文状态 $S_e$ 映射到模型权重的矩阵

4. **策略更新**：模型在更新后的策略 $\pi_{\theta'}^D(y|p)$ 下继续解码，其中 $\theta' = \theta + \Delta W$。

5. **持续更新**：每次缓存满时，计算新的被驱逐上下文状态 $S_e'$ 并更新 $S_e \leftarrow \text{Normalize}(S_e + S_e')$，然后重新计算 $\Delta W$。

#### 2.2.4 初始化与全局Token

为了启动适应过程，在处理任何被驱逐token之前，使用可学习的全局token初始化上下文状态：

$$S_e = \left(W_Q^a q_g \odot (W_K^a k_g)^\top\right) \odot W_V^a v_g$$

其中 $h_g$ 是全局token，$q_g = W_Q h_g$，$k_g = W_K h_g$，$v_g = W_V h_g$。

这种初始化使 $q_g$ 成为被驱逐上下文信息的显式载体，实现流式适应同时保持内存使用恒定。

### 2.3 训练中的驱逐策略选择

在训练设置中：
- **问题token**：永久保留在缓存中（基于sink-token机制，确保模型即使在思维链变得非常长时也能保持稳定的基础）
- **思维token**：应用简单的滑动窗口驱逐策略
- **批量操作**：这种简单设计支持跨样本的高效批量操作，而基于重要性的更复杂驱逐策略会产生额外计算开销

### 2.4 方法优势总结

| 特性 | 传统方法 | 渐进式思维编码 |
|------|----------|----------------|
| 内存使用 | 随序列长度线性增长 | 恒定内存使用 |
| 长上下文能力 | 滑动窗口导致性能下降 | 通过潜在表示保持 |
| RL训练效率 | 长rollout导致内存瓶颈 | 减少峰值内存使用 |
| 推理性能 | 缓存受限时显著下降 | 在严格缓存预算下保持高性能 |

## 三、实验结果与性能表现

### 3.1 实验设置

#### 3.1.1 模型选择

实验在三个代表性的开源指令微调模型上进行：

1. **Qwen2.5-3B-Instruct**：41亿参数，32层解码器，隐藏维度4096
2. **Qwen2.5-7B-Instruct**：72亿参数，隐藏维度5120，40个注意力头
3. **DeepSeek-R1-Distill-Llama-8B**：80亿参数，从DeepSeek-R1蒸馏到LLaMA-3.1-8B

#### 3.1.2 基准测试

六个数学推理基准，涵盖不同难度和推理深度：

1. **Math500**：500道挑战性文字题，需要符号和多步推理
2. **OlympiadBench**：674道奥林匹克风格问题，测试深度数学推理
3. **Minerva Math**：672道来自arXiv和教科书的题目，强调符号操作
4. **AMC**：40道美国数学竞赛题目，涵盖组合数学、数论和代数
5. **AIME2024**：30道高难度美国邀请数学考试题目
6. **AIME2025**：30道高难度美国邀请数学考试题目

#### 3.1.3 对比方法

1. **Baseline**：RL训练前的原始模型
2. **LoRA**：应用低秩适应的RL训练模型
3. **LoRAc**：LoRA + 滑动窗口缓存进行token驱逐
4. **Ours（PTE）**：提出的方法，显式学习被驱逐的token

#### 3.1.4 训练配置

- 最大序列长度：3072
- 全局批量大小：512
- 训练数据集：DAPO-Math-17K
- 优化器：Adam，学习率 $1 \times 10^{-5}$
- LoRA秩：32
- 滑动窗口缓存大小：设置为当前微批次中的最大问题长度
- 驱逐比例：缓存满时驱逐25%的token
- 全局token数量：32
- 硬件：8× NVIDIA A100 GPUs (40GB)

### 3.2 主要实验结果

#### 3.2.1 整体性能对比

| 模型 | 方法 | 平均准确率 | TFLOPs | 峰值内存 | 平均内存 |
|------|------|------------|--------|----------|----------|
| **Qwen2.5-3B** | Baseline | 23.2% | - | - | - |
| | LoRA | 28.2% | 4.2 | 83.0% | 67.8% |
| | LoRAc | 25.6% | 1.9 | 38.0% | 34.4% |
| | **PTE (Ours)** | **30.1%** | **2.7** | **45.0%** | **41.0%** |
| **Qwen2.5-7B** | Baseline | 33.3% | - | - | - |
| | LoRA | 38.1% | 5.7 | 85.8% | 76.2% |
| | LoRAc | 36.7% | 3.6 | 63.1% | 59.3% |
| | **PTE (Ours)** | **39.6%** | **3.6** | **62.7%** | **56.5%** |
| **DeepSeek-R1-Distill-8B** | Baseline | 34.9% | - | - | - |
| | LoRA | 34.9% | 7.4 | 89.0% | 78.3% |
| | LoRAc | 37.2% | 4.6 | 60.9% | 57.4% |
| | **PTE (Ours)** | **45.6%** | **4.6** | **59.8%** | **55.0%** |

#### 3.2.2 关键发现

**Qwen2.5-3B-Instruct**：
- 全缓存LoRA达到28.2%平均准确率，但需要4.2 TFLOPs和83%峰值内存
- LoRAc将峰值内存降至38%，但准确率降至25.6%
- **PTE达到30.1%（所有方法中最高）**，仅需2.7 TFLOPs和45%峰值内存
- **结论**：朴素的驱逐策略严重损害推理性能，但驱逐感知训练不仅恢复而且提高了准确率

**Qwen2.5-7B-Instruct**：
- 准确率与效率的权衡在更大规模上更明显
- LoRA达到38.1%准确率但内存成本高昂（85.8%峰值）
- LoRAc降低内存至63.1%但准确率降至36.7%
- **PTE实现最佳平均准确率39.6%**，同时相比LoRA几乎减半FLOPs（3.6 vs 5.7）

**DeepSeek-R1-Distill-Llama-8B**：
- 对于最大模型，效率约束占主导
- 全缓存LoRA需要7.4 TFLOPs和89%峰值内存
- **PTE实现45.6%平均准确率**，相比LoRA提升**+10.7%**
- 在极具挑战性的AIME2024上提升**+33.4%**，AIME2025上提升**+23.4%**
- 仅消耗4.6 TFLOPs和59.8%峰值内存

### 3.3 不同计算预算下的评估

#### 3.3.1 KV缓存大小变化实验

在Olympiad、MinervaMath、AMC和Math500基准上，设置最大响应长度为3,072，测试从满容量到高度受限设置的性能变化。

**关键结果**：
- Baseline和LoRA方法随缓存缩小而快速退化，反映它们对完整历史上下文的依赖
- LoRAc通过纳入滑动窗口适应学习在一定程度上缓解了这个问题，但在窗口变窄时效果有限
- **PTE在所有计算预算下始终保持更高准确率**，展示对缓存截断的韧性

**定量分析**：
- 跨所有数据集和缓存设置平均，PTE达到**39.37**准确率
- 相比LoRA的32.99和Baseline的30.31
- 相对改进：**+19.3%相比LoRA**，**+29.9%相比Baseline**
- **重要**：这些增益不需要额外推理时内存，因为PTE无论生成长度如何都保持恒定缓存使用

#### 3.3.2 AIME高难度基准测试

在AIME2024和AIME2025上，允许最多6,072 token生成（超过训练设置），最大缓存大小设为1,536 token加速解码。报告不同缓存大小{768, 1024, 1536}的pass@16分数。

| 模型 | 方法 | AIME2024 | AIME2025 |
|------|------|----------|----------|
| **Qwen2.5-7B** | LoRA | 16.11 | 13.89 |
| | LoRAc | 13.33 | 10.56 |
| | **PTE** | **22.78** | **21.11** |
| **DeepSeek-8B** | LoRA | 13.33 | 13.33 |
| | LoRAc | 3.33 | 7.78 |
| | **PTE** | **41.11** | **31.11** |

**关键发现**：
- 在Qwen2.5-7B上，相比LoRA平均增益**+6.63/+6.70**
- 在DeepSeek-R1-Distill-8B上，相比LoRA平均增益**+27.83/+21.11**
- 相比滑动窗口缓存（LoRAc）的改进同样显著
- **强调朴素上下文截断的局限性**

## 四、深入技术分析

### 4.1 为什么渐进式思维编码有效？

#### 4.1.1 信息压缩与保持的平衡

传统滑动窗口策略通过完全丢弃旧token来限制内存，这导致信息永久丢失。PTE通过将被驱逐token的信息编码到固定大小的潜在向量中，实现了：

1. **信息保持**：关键推理信号被压缩保留
2. **内存恒定**：无论生成长度如何，内存使用保持不变
3. **可学习适应**：通过LoRA参数动态整合历史信息

#### 4.1.2 强化学习训练的特殊优势

在RL训练中，长rollout是必需的，但全缓存训练导致：
- 内存使用量随rollout长度线性增长
- 反向传播需要维护完整的计算图
- 批量大小受限，训练效率低下

PTE通过以下方式解决这些问题：
- 不需要通过完整缓存rollout进行反向传播
- 峰值内存使用大幅减少（最高减少近50%）
- 允许更大的批量大小和更快的训练

#### 4.1.3 推理时的隐性优势

训练好的模型在推理时也受益于PTE：
- 可以在固定缓存大小下生成长推理链
- 保持对早期推理步骤的"记忆"
- 在计算受限环境（如边缘设备）中表现更好

### 4.2 与相关工作的对比

#### 4.2.1 与KV缓存压缩方法的对比

| 方法 | 策略 | 优势 | 局限性 |
|------|------|------|--------|
| **滑动窗口** | 丢弃旧token | 简单，内存恒定 | 严重损害长上下文能力 |
| **H2O** | 保留 Heavy Hitter tokens | 保留重要token | 需要启发式选择 |
| **StreamingLLM** | 保留sink和局部token | 保持注意力汇聚 | 可能丢失中间推理信息 |
| **PTE (本工作)** | 编码为潜在表示 | 学习性保留，内存恒定，训练友好 | 需要额外的LoRA参数 |

#### 4.2.2 与测试时学习方法的对比

测试时学习（Test-Time Learning, TTL）探索LLMs如何在不离线重训练的情况下适应新任务：

- **上下文学习**：通过提示中的演示引出任务特定行为
- **RAG**：在推理时提供任务相关文档
- **梯度更新**：测试时训练、熵最小化等

PTE的独特之处在于：
- 在训练期间学习如何有效利用有限缓存
- 在推理时不需要额外的梯度计算
- 与RL训练范式无缝集成

#### 4.2.3 与参数高效微调的对比

传统PEFT方法（如LoRA）主要关注离线任务适应，而PTE扩展到低秩技术以支持：
- 动态测试时学习
- 流式适应（Streaming Adaptation）
- 持续更新而不增加缓存大小

### 4.3 潜在局限与未来方向

#### 4.3.1 当前局限性

1. **任务专注性**：当前实验主要聚焦于数学推理任务，在其他领域（如代码生成、自然语言推理）的有效性需要验证

2. **驱逐策略简单性**：使用简单的滑动窗口策略，更智能的基于重要性的驱逐策略可能进一步提升性能

3. **全局token开销**：32个全局token引入的额外计算和内存开销在极小规模模型上可能不可忽略

4. **超参数敏感性**：LoRA秩、全局token数量、驱逐比例等超参数的最优值可能因模型和任务而异

#### 4.3.2 未来研究方向

1. **多模态扩展**：将PTE扩展到视觉-语言模型，处理图像和文本的混合推理

2. **自适应驱逐策略**：学习数据驱动的驱逐策略，动态决定保留哪些信息

3. **层次化编码**：设计多层次的编码方案，平衡压缩率和信息保真度

4. **与其他效率技术的结合**：结合量化、剪枝等技术，实现极致的推理效率

## 七、个人理解与深度思考

### 7.1 从"存储"到"理解"的范式转变

PTE代表了一种重要的范式转变：

**传统思维**：缓存的目的是存储所有历史token，以便随时访问。

**PTE思维**：缓存的目的是保持对历史信息的"理解"，而不是原始token。

这与人类认知有有趣的相似性：
- 我们不会记住对话中的每一个词
- 但我们保持对对话脉络的"理解"
- 这种理解是压缩的、结构化的、可随时调用的

### 7.2 效率与能力的重新平衡

深度学习领域长期存在一种隐含假设：能力提升通常需要效率牺牲。PTE挑战了这一假设：

**证据**：PTE不仅提高了效率，还提高了准确率（相比LoRA +19.3%）。

**可能解释**：
1. 驱逐感知训练迫使模型学习更紧凑、更结构化的表示
2. 全局token机制可能起到了"正则化"作用
3. 有限的缓存可能过滤了噪声，保留了关键信号

这对未来研究的启示：**效率优化可能是能力改进的途径，而不仅仅是约束。**

### 7.3 RL训练的新视角

PTE对RL训练的分析提供了重要洞察：

**问题重构**：将内存约束视为优化问题的一部分，而不是外部限制。

**策略学习**：不仅学习如何推理，还学习如何在资源约束下推理。

**元学习元素**：模型学习到的是"如何有效利用有限资源"的元能力。

这与人类专家的直觉相符：
- 专家不仅能解决问题，还能在时间和认知资源受限时有效解决问题
- 这种"资源适应性"是智能的重要维度

### 7.4 对未来架构设计的启示

PTE的成功提示了未来模型架构设计的可能方向：

1. **分层记忆系统**：显式区分工作记忆（缓存）和长期记忆（潜在表示）

2. **自适应计算**：根据任务复杂度动态调整资源分配

3. **压缩感知设计**：在架构层面考虑信息压缩，而不是事后优化

4. **学习性驱逐**：将驱逐策略本身作为可学习参数

### 7.5 对AGI路径的思考

PTE虽然是一个技术性的效率优化方法，但它触及了更深层的问题：

**智能的本质是什么？**
- 是原始计算能力？
- 还是在有限资源下有效工作的能力？

PTE暗示了后者可能更重要。这与以下观点一致：
- 人类大脑在严格的能量和计算约束下运行
- 但实现了惊人的推理能力
- 关键可能在于高效的信息组织和检索

如果这一观点成立，PTE这类方法可能是通往更高效、更强大AI系统的重要一步。

## 八、结论

### 8.1 核心贡献总结

**渐进式思维编码（Progressive Thought Encoding）**是一项突破性的技术创新，成功解决了大型推理模型在强化学习训练中的核心效率瓶颈：

1. **问题识别**：明确识别了RL训练中长rollout导致的内存瓶颈问题，并将其形式化为缓存约束优化问题。

2. **技术创新**：提出了一种参数高效微调方法，通过将被驱逐token编码为固定大小向量，保持长上下文推理能力的同时实现恒定内存使用。

3. **显著效果**：在三个模型、六个基准上的实验表明，PTE相比LoRA提升+19.3%，相比无微调基线提升+29.9%，同时减少近50%内存使用。

4. **实际意义**：不仅提高了训练效率，还改善了推理时的性能，特别是在严格缓存预算下。

### 8.2 对领域的影响

**短期影响**：
- 为现有推理模型的训练和部署提供即时的效率提升
- 为资源受限环境下的模型部署开辟新可能
- 启发新的KV缓存压缩和优化研究

**中期影响**：
- 推动更长推理链、更复杂任务的模型训练
- 促进测试时计算扩展技术的实用化
- 加速边缘设备上的推理模型部署

**长期影响**：
- 可能重塑我们对模型效率和能力的理解
- 为更高效的AGI系统架构提供设计原则
- 促进AI民主化，降低先进AI技术的应用门槛

### 8.3 最终评价

**Progressive Thought Encoding**是一篇高质量的学术论文，具有以下特点：

**科学严谨性**：
- 问题定义清晰，动机充分
- 方法描述详细，可复现性强
- 实验设计全面，结果可信度高

**技术创新性**：
- 提出了新的问题视角（缓存约束下的RL训练）
- 设计了优雅的解决方案（渐进式编码+LoRA适配）
- 实现了效率和性能的双赢

**实用价值**：
- 可直接应用于现有模型和框架
- 显著降低训练和部署成本
- 为多种应用场景铺平道路

**研究影响力**：
- 触及效率和能力关系的基础问题
- 与多个热门研究方向（RL、推理模型、KV缓存优化）交叉
- 为未来研究提供了新的思路和工具
