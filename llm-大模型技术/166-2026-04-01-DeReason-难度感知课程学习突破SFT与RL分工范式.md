# DeReason-难度感知课程学习突破SFT与RL分工范式的通用推理训练新框架

**论文标题**: DeReason: A Difficulty-Aware Curriculum Improves Decoupled SFT-then-RL Training for General Reasoning  
**作者**: Hanxu Hu, Yuxuan Wang, Maggie Huan, Jannis Vamvas, Yinya Huang, Zhijiang Guo, Rico Sennrich  
**机构**: 苏黎世大学、宾夕法尼亚大学、ETH Zurich、香港科技大学(广州)  
**发表时间**: 2026年3月  
**论文链接**: https://arxiv.org/abs/2603.11193  

---

## 一、研究背景与核心问题

### 1.1 RLVR范式的崛起与隐忧

近年来，**可验证奖励强化学习(RLVR, Reinforcement Learning with Verifiable Rewards)**已成为激发大语言模型推理能力的主流范式。从OpenAI的o1系列到DeepSeek-R1，这些突破性工作展示了通过大规模RL训练可以解锁复杂的链式思维推理能力，涌现出自我验证、反思等高级认知模式。

然而，这些成功主要集中在**数学和编程**等具有明确结果验证信号的狭窄领域。当研究者们试图将RLVR扩展到更广泛的**通用STEM推理**领域时，面临着一个根本性挑战：**通用科学推理往往涉及自由形式的解释或定性推理，无法通过基于规则的检查器进行确定性评估**。

### 1.2 被忽视的关键问题

当前学术界和产业界存在一个潜在共识：**RL-based后训练在激发推理能力方面普遍优于监督微调(SFT)**。这一观点驱动了大量研究工作，使得纯RL方法成为大模型训练的主流选择。

然而，这个结论真的站得住脚吗？DeReason研究团队提出了一个尖锐的质疑：

> **"RL在特定领域的成功是否被过度泛化了？在通用STEM推理领域，SFT和RL的真实关系是什么？"**

### 1.3 核心研究问题

本研究聚焦于三个关键问题：

1. **在受控条件下比较SFT和RL**：当两者使用完全相同的训练数据时，它们的相对表现如何？
2. **数据难度对训练效果的影响**：不同难度的训练数据应如何分配给SFT和RL阶段？
3. **SFT和RL的互补机制**：它们各自擅长什么？如何设计最优的协作策略？

## 二、颠覆性发现：SFT比RL更有效

### 2.1 受控实验设计

为了严格评估RL和SFT的相对优劣，研究团队设计了一系列**受控实验**：

- **控制变量**：两者在完全相同的题目集合上训练，仅改变训练数据量
- **SFT设置**：使用中等能力模型（而非前沿模型）生成的响应，确保监督信号不会过强
- **评估基准**：
  - 通用STEM推理：GPQA-Diamond（8次运行的pass@1平均值）
  - 数学推理：AIME24、AIME25和MATH500的pass@1平均值

### 2.2 核心发现：SFT完胜RL

实验结果揭示了一个令人惊讶的事实：

![Figure 1: Scaling behavior of RL and SFT](https://arxiv.org/html/2603.11193/x1.png)

**关键发现**：

| 比较维度 | 数学领域 | 通用STEM领域 |
|---------|---------|-------------|
| **SFT表现** | 在中等质量响应上已达到竞争性或更优性能 | 同样显著优于RL |
| **RL表现** | 即使增加数据量也难以匹敌SFT | 样本效率极低，表现差距明显 |
| **数据效率** | SFT样本效率显著更高 | RL需要更多数据才能达到类似效果 |

这一发现直接挑战了当前"RL优于SFT"的主流认知。

### 2.3 为什么SFT更有效？

研究团队深入分析了SFT优势的根本原因：

**1. 样本效率优势**
- SFT通过直接模仿中等质量解决方案提供更强的学习信号
- RL需要模型通过嘈杂的探索自行发现有效推理路径——对于未经微调的小基础模型来说尤为困难

**2. 领域知识获取**
- 数学和通用STEM推理都需要领域知识（如物理公式、代数恒等式）
- 这些知识难以通过试错单独获取，而SFT提供了更直接的知识整合路径

**3. 学习动态差异**
- SFT：均匀更新所有训练数据
- RL：隐式优先处理基于难度的样本，主要更新中等难度样本（高方差奖励）

### 2.4 互补性洞察

虽然SFT在受控比较中表现更优，但研究也发现：

> **SFT和RL具有互补优势：SFT擅长高效获取知识，而RL可以在足够具有挑战性的问题上推动超越监督信号的性能。**

这一洞察催生了DeReason方法的核心思想：**基于难度的数据解耦策略**。

## 三、DeReason方法详解

### 3.1 核心思想

DeReason（**De**coupled training based on **Reason**ing intensity）是一种基于推理强度的难度感知课程训练策略，其核心创新在于：

**将简单样本分配给SFT进行知识和技能获取，将困难样本保留给RL以推动推理边界。**

### 3.2 三阶段训练流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    DeReason训练流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  阶段1: 难度估计                                                  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ 使用同规模指令模型(Qwen3-4B-Instruct)评估每道题难度        │  │
│  │ 评分标准: 1-5分，基于推理步骤数、所需领域知识、出错可能性   │  │
│  │ • 1分: 直接回忆或单步查找                                   │  │
│  │ • 2分: 1-2个逻辑步骤                                        │  │
│  │ • 3分: 3-5个逻辑步骤                                        │  │
│  │ • 4分: 5-8个逻辑步骤，多变量跟踪                            │  │
│  │ • 5分: 8+逻辑步骤，嵌套推理，假设形成                       │  │
│  └──────────────────────────────────────────────────────────┘  │
│                           ↓                                      │
│  阶段2: 数据分区                                                  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ SFT数据集 (难度≤τ): 知识回忆型问题                         │  │
│  │   └─> 由中等能力教师模型生成参考响应                        │  │
│  │                                                                  │
│  │ RL数据集 (难度>τ): 推理密集型问题                           │  │
│  │   └─> 保留用于RL训练                                        │  │
│  └──────────────────────────────────────────────────────────┘  │
│                           ↓                                      │
│  阶段3: 课程训练                                                  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ Step 1: SFT在D_SFT上训练 → 获得π_SFT                      │  │
│  │ Step 2: 从π_SFT初始化，在D_RL上应用GRPO                   │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 难度评分Prompt设计

DeReason使用精心设计的Prompt让LLM评估题目难度（完整Prompt见原论文附录）：

```
请使用以下量表评估给定问题的推理复杂性：

推理需求量表：
1 - 最小化: 直接回忆或单步查找
   • 答案可立即从记忆/知识中检索
   • 不需要逻辑步骤或分解
   
2 - 简单: 一个或两个逻辑步骤
   • 需要基本推理或简单计算
   • 单一关系或比较
   
3 - 中等: 具有清晰结构的多步骤
   • 需要3-5个逻辑步骤或简单分解
   • 可能需要组合2-3个概念或规则
   
4 - 复杂: 具有依赖关系的多步骤分析
   • 需要5-8个带有中间结论的逻辑步骤
   • 需要跟踪多个变量或关系
   • 可能需要考虑权衡或约束
   
5 - 深度: 具有抽象思维的广泛推理
   • 需要8+个带有嵌套推理的逻辑步骤
   • 必须综合多个框架或领域
   • 涉及假设形成、证明或复杂问题分解

考虑因素：
- 所需逻辑步骤数
- 所需分解深度
- 推理步骤之间的依赖关系
- 中间结论的需求

输出格式：
分析: [逐步分析问题，确定：(1)需要什么知识/操作，
      (2)涉及多少逻辑步骤，(3)步骤之间存在什么依赖关系，
      (4)哪个量表级别最符合这些要求]
推理需求: [1-5]
```

### 3.4 技术细节

**SFT训练设置**：
- 基础模型：Qwen3-4B-Base
- 框架：Llama-Factory
- 批次大小：128
- 学习率：1e-5
- 教师模型：Qwen3-4B-Instruct（避免依赖外部强模型）

**RL训练设置**：
- 框架：VeRL
- 算法：GRPO (Group Relative Policy Optimization)
- 最大响应长度：8192 tokens
- 训练批次大小：128
- 小批次大小：64
- 学习率：1e-6

**验证机制**：
由于通用科学领域难以使用规则验证，采用基于模型的验证器评估语义等价性：
```
R(x, o) = {
    1  if V_θ(Extract(o), a*, x) = True,
    0  otherwise
}
```
其中V_θ是语言模型验证器，可以处理定性解释、近似数值和多部分推导。

## 四、实验结果与性能分析

### 4.1 主要实验结果

研究在两个数据集上验证DeReason：WebInstruct-Verified和Webscale-RL。

#### 表1: 推理基准测试结果

| 模型 | MMLU-Pro | GPQA-D | SuperGPQA | BBEH | AVG |
|------|----------|--------|-----------|------|-----|
| **基线模型** |||||
| GPT-4o | 74.6 | 50.0 | 46.3 | 22.3 | 48.3 |
| DeepSeek-R1 | 84.0 | **71.5** | **59.9** | **34.9** | **62.6** |
| Qwen2.5-7B-Base | 47.7 | 29.3 | 26.7 | 8.0 | 27.9 |
| **4B模型 (WebInstruct-Verified)** |||||
| Qwen3-4B-Base | 51.6 | 26.5 | 25.4 | 8.1 | 27.9 |
| RL only | 62.8 | 42.9 | 32.5 | 12.2 | 37.6 |
| SFT only | 68.6 | 46.8 | 38.4 | 13.5 | 41.8 |
| SFT→随机RL | 68.6 | 47.8 | 39.4 | 15.8 | 42.9 |
| **DeReason (SFT简单+RL困难)** | **68.4** | **50.0** | **40.2** | **16.7** | **43.8** |
| **4B模型 (Webscale-RL)** |||||
| RL only | 55.4 | 34.0 | 30.9 | 10.1 | 32.6 |
| SFT only | 60.7 | 39.2 | 37.3 | 13.4 | 37.7 |

**关键发现**：
1. **SFT-only显著优于RL-only**：在所有基准测试上，使用相同训练数据时SFT表现更好
2. **DeReason达到最佳性能**：难度感知的数据分配策略显著优于随机分配基线
3. **在困难基准上优势更明显**：在BBEH等需要更多推理而非知识检索的基准上，DeReason优势最显著

### 4.3 数据难度分布分析

![Figure 2: Category distribution across difficulty scores](https://arxiv.org/html/2603.11193/x2.png)

**关键洞察**：

- **低难度数据(1-2分)**：分布相对均匀，涵盖历史、生物、商业、心理学等多样化类别
- **高难度数据(4-5分)**：高度集中在数学和物理领域
  - 4分数据：约78%为数学
  - 5分数据：约96%为数学

这验证了研究假设：**简单样本倾向于覆盖广泛的知识导向主题，而困难样本主要是推理密集型**。

### 4.4 训练动态深度分析

#### 4.4.1 奖励分数演化

![Figure 3: Reward score during RL training](https://arxiv.org/html/2603.11193/x3.png)

- **从SFT检查点出发**：初始奖励较高，后续步骤有轻微改善
- **从基础模型出发**：前40步显著改善，之后性能趋于平稳
- **SFT检查点始终表现更好**：验证了SFT作为冷启动机制的有效性

#### 4.4.2 响应长度演化

![Figure 4: Response length during RL training](https://arxiv.org/html/2603.11193/x4.png)

**从SFT检查点出发**：
- 继承冗长生成的行为
- RL逐步缩短响应长度
- 高分响应从约4200 tokens降至3000 tokens

**从基础模型出发**：
- 所有分数级别的响应初始长度相似（约1200 tokens）
- 快速分化：高分问题保持或增长长度，低分问题缩减至500 tokens以下
- 分数依赖的分化在基础模型设置中更为明显

#### 4.4.3 策略熵演化

![Figure 5: Actor's entropy during RL training](https://arxiv.org/html/2603.11193/x5.png)

- **基础模型**：初始熵较高（≈2.0），反映广泛且约束较少的输出分布；前20步急剧下降，之后逐渐稳定至0.10以下
- **SFT初始化模型**：起始熵较低（≈0.30），监督微调已集中策略分布；下降更缓慢、更温和
- **关键发现**：基础模型的熵最终降至SFT模型之下，表明从基础模型开始的RL最终收敛到更确定性的策略

## 五、核心贡献与技术创新

### 5.1 理论贡献

1. **挑战RL主导范式**：首次在受控实验条件下系统证明，对于通用STEM推理，SFT显著优于纯RL
2. **揭示互补机制**：深入分析SFT和RL的不同学习动态，揭示它们在知识获取和推理边界推进方面的互补性
3. **难度感知理论**：建立数据难度与训练阶段匹配的理论基础

### 5.2 方法创新

1. **DeReason课程策略**：首创基于推理强度的数据分区方法
2. **LLM-based难度评估**：利用同规模模型自动评估题目难度，避免依赖外部专有模型
3. **通用验证框架**：扩展RLVR到非规则可验证的通用推理领域

### 5.3 实验贡献

1. **大规模受控对比**：在相同数据集上系统比较SFT和RL
2. **多维度分析**：从性能、数据分布、训练动态、响应长度、策略熵等多角度深入分析
3. **跨数据集验证**：在两个独立数据集上验证方法有效性

## 六、个人理解与行业影响分析

### 6.1 对当前大模型训练的启示

#### 6.1.1 重新审视SFT的价值

DeReason的研究结果对当前大模型训练实践具有重要启示：

**传统认知**：RL是激发推理能力的关键，SFT仅用于冷启动  
**DeReason发现**：SFT在知识获取方面具有不可替代的优势，应被赋予更重要的角色

**实践建议**：
- 不要急于跳过SFT阶段直接进行RL训练
- SFT数据的质量和覆盖范围对最终性能至关重要
- 即使是中等质量的教师模型，其SFT数据也可能优于纯RL探索

#### 6.1.2 数据工程的新维度

DeReason将数据工程提升到一个新高度：

**传统做法**：关注数据质量、多样性、去重  
**DeReason扩展**：增加**数据难度分层**和**阶段适配**维度

**实施建议**：
- 建立系统的数据难度评估流程
- 根据模型规模和目标能力调整难度阈值τ
- 监控训练数据在不同难度级别的分布变化

#### 6.1.3 RL训练的新定位

DeReason的研究促使我们重新思考RL在训练流程中的定位：

**不是**：RL是主要的学习机制  
**而是**：RL是专门的推理边界推进器

**最优配置**：
- RL训练数据量应相对较小但高度精选
- 专注于模型在SFT阶段难以掌握的复杂推理模式
- 利用RL的探索能力突破监督信号的天花板

### 6.2 行业影响预测

#### 6.2.1 短期影响（6-12个月）

1. **训练流程优化**
   - 主要AI实验室将重新评估其SFT/RL数据配比
   - 出现更多基于难度的数据筛选工具
   - SFT阶段的资源投入可能增加

2. **开源生态发展**
   - 社区将出现DeReason的开源实现
   - 难度评估Prompt和流程标准化
   - 更多基于该方法的小型模型训练项目

3. **模型性能提升**
   - 中小规模模型的推理能力将有显著提升
   - 4B-7B模型的STEM推理能力接近当前14B+模型水平

#### 6.2.2 中期影响（1-2年）

1. **新训练范式形成**
   - "SFT-heavy + RL-refinement"可能成为新标准
   - 动态难度调整机制被整合到主流训练框架
   - 自适应课程学习成为标配

2. **多模态扩展**
   - DeReason方法将被扩展到视觉-语言模型
   - 跨模态推理的难度评估标准建立
   - 多模态模型训练效率大幅提升

3. **垂直领域应用**
   - 科学、医学、法律等专业领域模型采用此方法
   - 领域特定的难度评估标准建立
   - 专业知识获取效率显著提升

#### 6.2.3 长期影响（2-5年）

1. **AGI训练策略演进**
   - 难度感知课程学习成为AGI训练的核心组件
   - 与人类认知发展理论结合
   - 建立更科学的模型能力发展路径

2. **数据经济变革**
   - 高质量中等难度数据价值重估
   - 专业领域知识工程服务兴起
   - 数据难度标注成为AI产业链新环节

### 6.3 局限性与未来方向

#### 6.3.1 当前局限

1. **难度评估依赖**：当前使用LLM评分可能存在偏差，需要更客观的评估方法
2. **阈值选择**：难度阈值τ的选择对结果影响较大，需要更系统的调优方法
3. **领域特异性**：当前主要在STEM领域验证，其他领域的适用性有待验证
4. **模型规模**：实验主要在4B规模进行，更大规模模型的行为可能不同

#### 6.3.2 未来研究方向

1. **自适应难度阈值**：开发动态调整τ的机制，根据训练进度自适应调整
2. **细粒度难度建模**：从单一分数扩展到多维度难度画像
3. **在线难度更新**：结合训练过程中的模型表现实时更新难度评估

   • 需要3-5个逻辑步骤或简单分解

   • 示例: "通货膨胀如何影响消费者支出模式？"

   • 可能需要考虑权衡或约束

### 7.2 伪代码实现

```python
class DeReasonTrainer:
    def __init__(self, base_model, teacher_model, difficulty_threshold=3):
        self.base_model = base_model
        self.teacher_model = teacher_model
        self.tau = difficulty_threshold
        
    def estimate_difficulty(self, question):
        """使用LLM评估题目难度"""
        prompt = REASONING_COMPLEXITY_PROMPT.format(question=question)
        response = self.teacher_model.generate(prompt)
        score = extract_score(response)  # 提取1-5分
        return score
    
    def partition_data(self, dataset):
        """基于难度分区数据"""
        sft_data, rl_data = [], []
        
        for item in dataset:
            difficulty = self.estimate_difficulty(item['question'])
            
            if difficulty <= self.tau:
                # SFT数据：生成参考响应
                response = self.teacher_model.generate(item['question'])
                sft_data.append({
                    'question': item['question'],
                    'response': response,
                    'difficulty': difficulty
                })
            else:
                # RL数据：仅保留问题和答案
                rl_data.append({
                    'question': item['question'],
                    'answer': item['answer'],
                    'difficulty': difficulty
                })
        
        return sft_data, rl_data
    
    def train(self, dataset):
        """执行DeReason训练"""
        # 阶段1: 数据分区
        sft_data, rl_data = self.partition_data(dataset)
        
        # 阶段2: SFT训练
        print(f"SFT training on {len(sft_data)} easy samples...")
        sft_model = sft_train(self.base_model, sft_data)
        
        # 阶段3: RL训练
        print(f"RL training on {len(rl_data)} hard samples...")
        final_model = grpo_train(sft_model, rl_data)
        
        return final_model
```

## 八、结论

DeReason研究代表了大模型训练领域的一个重要里程碑。它通过严谨的受控实验，挑战了当前"RL主导推理训练"的主流认知，揭示了**SFT在知识获取方面的不可替代价值**和**数据难度感知分配的关键作用**。

### 核心结论回顾

1. **SFT比RL更有效**：在相同训练数据条件下，SFT在通用STEM推理任务上始终优于纯RL
2. **互补性是关键**：SFT擅长知识获取，RL擅长推理边界推进，两者需要协作而非替代
3. **难度感知分配**：基于推理强度的数据分区策略显著提升训练效果
4. **简单问题给SFT，复杂问题给RL**：这一原则具有广泛的适用性

### 实践价值

- **对于模型开发者**：提供了一套经过验证的高效训练配方
- **对于研究人员**：开辟了数据难度感知训练的新研究方向
- **对于整个社区**：促进了对SFT和RL角色的更深入理解

### 最终思考

DeReason提醒我们：在追求算法创新的同时，不要忽视数据工程的基础价值。有时候，**如何组织数据比使用什么算法更重要**。这一洞察对于正在快速发展的AI领域具有重要的指导意义——在追求更复杂的训练方法之前，先确保我们充分理解了数据的特性和最优利用方式。
