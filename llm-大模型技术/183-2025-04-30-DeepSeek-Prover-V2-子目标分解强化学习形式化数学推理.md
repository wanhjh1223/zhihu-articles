## 一、研究背景与核心问题

### 1.1 形式化定理证明的挑战

大语言模型（LLMs）在自然语言数学推理领域取得了突破性进展，尤其是在竞赛级数学问题解决方面展现出惊人能力。这些进展主要得益于**推理时计算扩展**（Inference-time Scaling）范式，特别是通过自然语言思维链（Chain-of-Thought, CoT）进行多步推理。DeepSeek-R1、OpenAI o1/o3等模型通过"先思考后回答"的模式，在AIME、IMO等数学竞赛中达到了接近人类专家的水平。

然而，将这些能力迁移到**形式化定理证明**领域仍面临根本性挑战：

**根本差异**:
| 维度 | 自然语言推理 | 形式化定理证明 |
|------|------------|--------------|
| **严谨性** | 依赖启发式、近似、数据驱动猜测 | 严格逻辑基础，每步必须明确构造和验证 |
| **容错性** | 允许模糊、隐含假设、省略细节 | 不允许歧义，必须显式表达所有推理步骤 |
| **验证方式** | 人工评估或自动评分 | 证明助手（Lean/Coq/Isabelle）严格形式验证 |
| **知识表示** | 非结构化自然语言 | 结构化形式语言（如Lean 4） |

形式化定理证明系统如Lean 4、Coq、Isabelle等建立在严格逻辑基础之上，要求每个证明步骤都必须被形式化验证。这与LLMs在非结构化自然语言环境中进行启发式推理的方式形成鲜明对比。

### 1.2 子目标分解的启发

人类数学家解决复杂问题时，通常采用**层次化分解策略**：将复杂定理拆分为一系列可管理的子目标（lemmas），逐步推进直至完成证明。这种策略与**层次化强化学习**（Hierarchical Reinforcement Learning, HRL）中的子目标概念高度契合。

**核心洞察**：如果能够训练一个模型，使其具备以下能力，将有望在形式化证明领域取得突破：
1. **高层推理能力**：理解数学问题的整体结构
2. **子目标分解能力**：将复杂问题拆分为可处理的子目标
3. **形式化转换能力**：将自然语言推理转换为严格的形式证明
4. **递归求解能力**：通过解决子目标来构建完整证明

### 1.3 DeepSeek-Prover-V2的核心贡献

DeepSeek-Prover-V2通过以下创新实现了上述目标：

1. **递归定理证明管道**：利用DeepSeek-V3作为统一工具进行子目标分解和形式化
2. **冷启动数据合成**：结合非形式推理和形式证明构建高质量训练数据
3. **双模式架构**：支持高效的无CoT模式和高精度的CoT模式
4. **强化学习优化**：使用GRPO算法进一步提升证明能力

## 二、技术方法详解

### 2.1 递归子目标分解管道

DeepSeek-Prover-V2的核心创新在于构建了一个**递归定理证明管道**，充分利用DeepSeek-V3强大的通用推理能力。

#### 2.1.1 从自然语言到形式化草图

```
输入: 形式化定理陈述（Lean 4）
  ↓
DeepSeek-V3分析数学问题
  ↓
分解为自然语言证明步骤
  ↓
将每步转换为Lean形式化陈述
  ↓
生成高阶证明草图（have语句序列 + sorry占位符）
```

**关键技术细节**：
- DeepSeek-V3同时承担**分解者**和**形式化者**双重角色
- 生成的证明草图模仿人类证明风格：用`have`语句表示中间结论，用`sorry`标记待填充的子目标
- 这种设计利用了通用LLM在数学推理上的优势，同时规避了其生成完整形式证明的弱点

**示例**：
```lean
-- 原始目标: 证明三角恒等式
theorem imo_1963_p5 : 
  Real.cos (Real.pi / 7) - Real.cos (2 * Real.pi / 7) + 
  Real.cos (3 * Real.pi / 7) = 1 / 2 := by
  
  -- DeepSeek-V3生成的证明草图
  have h1 : Real.cos (Real.pi / 7) > 0 := sorry
  have h2 : Real.cos (2 * Real.pi / 7) = Real.cos (2 * Real.pi / 7) := sorry
  have h3 : 关键恒等式 := sorry
  -- ...更多子目标
  sorry  -- 最终合成
```

#### 2.1.2 子目标的递归求解

对于分解出的每个子目标，系统采用递归策略求解：

1. **子目标提取**：从`have`语句中提取子目标表达式
2. **依赖注入**：将前面已解决的子目标作为前提注入当前子目标
3. **本地化处理**：促进更简单的局部依赖结构
4. **轻量级求解**：使用专门优化的7B证明模型处理分解后的引理

**两种子目标类型**（如图3所示）：
- **类型(a)**：替换原始目标状态
- **类型(b)**：纳入前置子目标作为前提（用于递归求解复杂问题）

这种设计使得后续子目标可以利用前面步骤的中间结果，形成**逐步精化**的证明链。

### 2.2 课程学习与数据合成

#### 2.2.1 课程学习框架

形式化证明的训练数据通常来自自然语言数学语料的形式化转换，但这类数据存在**信号稀疏问题**：大量计算尝试无法产生成功证明，导致正奖励信号稀少。

DeepSeek-Prover-V2通过**子目标驱动的课程学习**解决这一问题：

1. **生成猜想定理**：基于分解的子目标生成可处理的猜想
2. **渐进难度提升**：
   - Stage 1：全部简单样本
   - Stage 2：简单:困难 = 1:1
   - Stage 3：简单:困难 = 1:2
3. **专家迭代**（Expert Iteration）：每轮使用当前最优策略生成证明尝试，成功样本加入训练集

这种课程设计类似于AlphaProof的测试时强化学习，通过生成目标问题的变体来增强模型解决挑战性IMO级别问题的能力。

#### 2.2.2 冷启动数据合成

冷启动阶段的核心目标是**统一非形式推理与形式证明**：

```
原始复杂问题
  ↓
DeepSeek-V3分解子目标 + 自然语言推理（CoT）
  ↓
7B模型递归求解所有子目标
  ↓
合成完整形式证明
  ↓
将证明拼接至DeepSeek-V3的CoT → 冷启动训练数据
```

**数据特点**：
- 数百条高质量合成数据
- 每条数据包含：自然语言推理链 + 对应的形式化证明
- 建立了从非形式直觉到形式化验证的映射

**与Kimina-Prover的对比**：
- **DeepSeek-Prover-V2**：先将自然语言证明形式化为结构化草图，再填充细节
- **Kimina-Prover**：收集完整形式证明，再 retrosynthesize 中间推理步骤

### 2.3 双模式架构设计

DeepSeek-Prover-V2支持两种互补的证明生成模式：

#### 2.3.1 无CoT模式（Non-Chain-of-Thought）

- **目标**：快速生成形式Lean证明代码
- **特点**：
  - 推理速度快，token消耗少
  - 适合数据收集和快速迭代
  - 模型可能隐式插入类似推理的自然语言注释

#### 2.3.2 CoT模式（Chain-of-Thought）

- **目标**：高精度、可解释的推理过程
- **特点**：
  - 显式表达中间推理步骤
  - 逻辑透明度高，便于调试和验证
  - 生成token数显著多于无CoT模式

**模式切换**：通过不同引导提示（prompt）控制生成模式，两种模式共享同一基础模型。

### 2.4 强化学习优化

#### 2.4.1 训练流程

```
DeepSeek-V3-Base-671B
  ↓
监督微调（SFT）
  ├── 无CoT数据（专家迭代收集）
  └── CoT冷启动数据
  ↓
强化学习（GRPO）
  ├── 二元奖励：证明通过=1，失败=0
  ├── 一致性奖励（早期阶段）：惩罚结构不对齐
  └── 持续优化
  ↓
DeepSeek-Prover-V2-671B
```

#### 2.4.2 GRPO算法

采用**Group Relative Policy Optimization**（群体相对策略优化）算法：

- **优势**：无需单独critic模型，显著降低内存和计算开销
- **机制**：
  1. 对每个定理提示采样一组候选证明（32个）
  2. 根据相对奖励优化策略
  3. 二元奖励：正确证明=1，错误=0

**关键训练参数**：
- 学习率：5e-6
- 上下文窗口：16,384 tokens（SFT）/ 32,768 tokens（RL）
- 每轮采样：256个不同问题
- 每问题候选数：32个

#### 2.4.3 一致性奖励

在RL早期阶段引入**结构一致性奖励**：
- **问题**：生成证明的结构常与CoT指导的引理分解不一致
- **解决方案**：惩罚结构不对齐，强制要求最终证明包含所有分解的`have`结构引理
- **效果**：显著提升复杂定理（需要多步分解）的证明准确率

### 2.5 7B模型的蒸馏

为了使推理能力可及，团队对7B模型进行了蒸馏：

1. **基础模型**：DeepSeek-Prover-V1.5-Base-7B
2. **上下文扩展**：从4,096扩展到32,768 tokens
3. **数据来源**：
   - 671B模型RL阶段收集的rollout数据
   - 专家迭代阶段的无CoT证明数据
4. **RL增强**：应用与671B模型相同的强化学习阶段

## 三、实验结果与性能表现

### 3.1 MiniF2F基准测试

MiniF2F包含488道形式化竞赛数学题，来源包括AIME、AMC、IMO和MATH数据集，涵盖代数、数论、归纳等核心领域。

#### 3.1.1 SOTA对比（表1）

| 模型 | 参数量 | 策略 | Pass@1 | Pass@32 | Pass@8192 |
|------|--------|------|--------|---------|-----------|
| **DeepSeek-Prover-V2** | **671B** | **CoT** | **63.6%** | **82.4%** | **88.9%** |
| DeepSeek-Prover-V2 | 671B | non-CoT | 59.0% | 75.2% | 82.4% |
| DeepSeek-Prover-V2 | 7B | CoT | 43.4% | 52.9% | - |
| STP | 7B | - | 46.3% | 56.2% | 64.7% |
| Kimina-Prover | 72B | - | 48.0% | 69.7% | 80.7% |
| Goedel-Prover | 7B | - | 39.0% | 52.8% | 67.6% |

**关键发现**：
1. **SOTA突破**：DeepSeek-Prover-V2-671B达到88.9% Pass@8192，创神经定理证明新纪录
2. **7B竞争力**：7B模型超越所有现有开源证明器，展现卓越参数效率
3. **CoT优势**：CoT模式在所有设置下均优于non-CoT模式，验证推理时扩展的有效性
4. **规模效应**：从Pass@1到Pass@8192，671B与7B的性能差距显著扩大，大模型样本效率更高

#### 3.1.2 困难问题攻克（表2）

通过子目标引导的课程学习：
- **miniF2F-valid**：91.0%通过率
- **miniF2F-test**：88.9%通过率
- 子目标分解管道在valid集上达到90.2%成功率，接近完整671B模型性能

**核心洞察**：
- 前沿通用LLM（DeepSeek-V3）能有效支持复杂形式推理任务
- 策略性子目标分解是连接非形式推理与形式证明的有效桥梁

### 3.2 大学级别基准测试

#### 3.2.1 ProofNet

ProofNet包含371道大学纯数学教科书题目，涵盖实/复分析、线性代数、抽象代数、拓扑学。

| 模型 | 设置 | Pass@1024 |
|------|------|-----------|
| **DeepSeek-Prover-V2-671B (CoT)** | **CoT** | **37.1%** |
| DeepSeek-Prover-V2-671B | non-CoT | 32.3% |
| DeepSeek-Prover-V2-7B | CoT | 19.4% |
| Goedel-Prover-SFT | - | 21.5% |

尽管训练数据主要来自高中数学，模型展现出**强大的大学数学泛化能力**。

#### 3.2.2 PutnamBench

PutnamBench包含658道William Lowell Putnam数学竞赛题（1962-2023），是美国最具声望的本科数学竞赛。

| 模型 | 参数量 | 解决问题数 |
|------|--------|-----------|
| **DeepSeek-Prover-V2-671B** | **671B** | **49** |
| DeepSeek-Prover-V2-7B (non-CoT) | 7B | 13 |
| Goedel-Prover | 7B | 8 |
| STP | 7B | 7 |

**意外发现**：7B non-CoT模型解决了13道671B模型未解决的问题，将总解决数从49提升到62。原因在于7B模型独特使用了`Cardinal.toNat`和`Cardinal.natCast_inj`技术处理有限基数问题。

### 3.3 ProverBench新基准

团队贡献了**ProverBench**，包含325道形式化问题：
- 其中15道选自AIME 2024-2025竞赛题
- 用于评估模型在最新竞赛题上的表现

**AIME问题求解结果**：
| 模型 | 解题数/15 |
|------|----------|
| DeepSeek-V3 (自然语言推理) | 8 |
| **DeepSeek-Prover-V2-671B** | **6** |

这一结果具有重要意义：
- **形式与非形式差距缩小**：形式证明模型（6/15）接近自然语言模型（8/15）的表现
- **技术成熟度**：证明了大模型在严格形式化推理领域的实质性进步

## 四、技术深度分析

### 4.1 子目标分解的范式意义

DeepSeek-Prover-V2的核心方法论——**子目标分解**——代表了神经定理证明领域的重要范式转移：

**传统方法**：
- 端到端生成完整证明
- 依赖大规模搜索（MCTS、束搜索等）
- 计算成本高，成功率低

**子目标分解方法**：
- 将指数级搜索空间分解为多项式级子问题
- 每个子目标可独立求解
- 支持模块化、可重用、可解释的证明构建

这与人类数学家的工作方式高度一致：**先理解问题结构，再分解为可管理的步骤，逐步完成证明**。

### 4.2 冷启动策略的创新性

冷启动数据合成是DeepSeek-Prover-V2的关键创新：

**为什么冷启动有效？**
1. **质量优先**：数百条高质量合成数据 > 数万条低质量数据
2. **对齐信号**：建立了自然语言推理与形式证明的显式映射
3. **强化学习基础**：为后续RL提供了良好的初始化点

**对比传统方法**：
- 传统：从随机初始化开始RL，探索效率低
- DeepSeek-Prover-V2：从高质量的CoT+证明数据开始，RL专注于改进已有能力

### 4.3 CoT在形式证明中的价值

实验明确验证了CoT在形式证明中的价值：

1. **性能提升**：CoT模式在所有设置下均优于non-CoT
2. **推理时扩展**：形式证明领域同样受益于推理时计算扩展
3. **隐性推理**：即使在non-CoT模式下，671B模型也常插入类似推理的自然语言注释，暗示高容量模型可能内化中间推理

这引发了一个有趣的问题：**CoT是"训练轮"还是"必要组件"？** 结果表明，至少在复杂形式推理任务中，显式CoT仍是性能的关键。

### 4.4 规模效应与涌现能力

从7B到671B的扩展展现出典型的**规模效应**：

- **Pass@1差距**：7B CoT 43.4% vs 671B CoT 63.6%（+20.2%）
- **Pass@8192差距**：差距进一步扩大
- **样本效率**：大模型在更多采样预算下提升更快

**涌现现象**：
- 671B模型在non-CoT模式下展现出"隐性推理"能力
- 某些复杂证明结构（多步依赖）只在足够大的模型中出现

### 4.5 局限性与开放问题

尽管取得重大突破，DeepSeek-Prover-V2仍存在局限：

1. **AIME差距**：形式证明（6/15）仍落后于自然语言（8/15）
2. **技能发现的不一致性**：7B模型在特定问题（基数处理）上超越671B，暗示多模型集成的潜力
3. **依赖通用LLM**：子目标分解依赖DeepSeek-V3，成本较高
4. **Lean 4限制**：特定版本兼容性问题导致部分Putnam题目无法评估

## 五、行业影响与未来展望

### 5.1 对自动定理证明领域的影响

DeepSeek-Prover-V2的发布标志着**神经定理证明进入新阶段**：

1. **开源SOTA**：首次开源模型在主要基准上达到或超越闭源系统
2. **实用化进展**：88.9%的MiniF2F通过率使自动证明接近实用水平
3. **方法论确立**：子目标分解+冷启动+RL的pipeline可能成为新标准

### 5.2 对LLM推理研究的启示

**形式化推理的重要性**：
- 形式证明是检验LLM真正推理能力的"金标准"
- 非形式推理与形式推理的差距正在快速缩小
- 强化学习在形式推理中的成功可迁移到其他推理任务

**子目标分解的普适性**：
- 不仅是定理证明，任何复杂推理任务都可能受益于层次化分解
- 通用LLM（DeepSeek-V3）+ 专用模型（7B prover）的协作模式值得推广

### 5.3 教育与研究应用前景

1. **数学教育**：自动证明辅助学生理解证明结构
2. **形式验证**：软件/硬件系统的自动形式验证
3. **数学研究**：辅助数学家探索新猜想和证明策略
4. **AI安全**：形式化验证AI系统的行为属性

### 5.4 未来研究方向

基于DeepSeek-Prover-V2的技术路线，以下方向值得探索：

1. **端到端子目标分解**：训练模型自身完成分解，减少对DeepSeek-V3的依赖
2. **跨证明助手迁移**：将Lean中训练的能力迁移到Coq、Isabelle等系统
3. **开放域定理证明**：从固定benchmark向开放数学知识库扩展
4. **人机协作证明**：交互式证明助手，结合人类直觉和机器验证
5. **多模态证明**：结合数学图像、公式的多模态推理

## 六、总结

DeepSeek-Prover-V2代表了**大模型形式化推理能力的重要里程碑**。通过创新的子目标分解管道、高质量的冷启动数据合成、以及强化学习优化，该模型在神经定理证明领域建立了新的SOTA，并在MiniF2F、ProofNet、PutnamBench等基准上展现出卓越性能。

**核心创新回顾**：
1. **递归子目标分解**：将复杂定理拆分为可管理的子目标，大幅降低搜索复杂度
2. **冷启动数据合成**：统一非形式推理与形式证明，为RL提供高质量初始化
3. **双模式架构**：CoT模式追求精度，non-CoT模式追求效率
4. **GRPO强化学习**：无需critic模型的高效RL优化

**关键成果**：
- MiniF2F-test **88.9%** Pass@8192（SOTA）
- ProofNet-test **37.1%** Pass@1024
- PutnamBench **49** 题（+13题由7B模型补充）
- ProverBench-AIME **6/15** 接近自然语言水平

随着形式推理与非形式推理差距的持续缩小，我们有理由期待：**在不久的将来，AI将不仅能辅助人类数学家，还可能独立发现新的数学真理**。DeepSeek-Prover-V2正是朝着这一愿景迈出的坚实一步。
