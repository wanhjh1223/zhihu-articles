## 一、研究背景与核心问题

### 1.1 大模型推理训练的RLVR范式

近年来，**可验证奖励的强化学习（Reinforcement Learning with Verifiable Rewards, RLVR）**已成为训练大型推理模型（Large Reasoning Models, LRMs）的核心范式。DeepSeek-R1的成功验证了这一路径的有效性——通过仅使用结果层面的监督（如答案正确性），无需依赖密集的过程奖励模型或学习的critic网络，就能实现显著的推理能力提升。

GRPO（Group Relative Policy Optimization）、DAPO、REINFORCE++等无critic算法因其可扩展性而备受青睐。这些算法通过为每个prompt采样多个rollout来估计优势函数，避免了传统PPO中价值网络的训练和存储开销，使其特别适合大规模后训练场景。

### 1.2 正同质提示词（Positive Homogeneous Prompts）的效率瓶颈

然而，这种多rollout范式暴露出一个日益严重的问题：随着基础模型能力的增强，**许多prompt产生的rollout都获得了相同的外部验证器奖励**。在这种情况下，GRPO式的相对优势估计会坍塌为零，对应的轨迹虽然消耗了完整的rollout成本，却无法贡献任何学习信号。

论文作者将这种现象称为**"正同质"（Positive Homogeneous, PH）prompts**——即所有采样rollout都正确的prompt。在高准确率场景中，PH prompts可能占据每个batch的很大比例，导致大量昂贵的rollout计算被浪费。

**关键观察**：
- 在Qwen3-4B-Base模型上，训练batch中PH prompts的比例可能高达50%以上
- 这些"已解决"的prompt实际上蕴含着宝贵的学习机会
- 传统方法直接丢弃这些rollout，造成严重的计算资源浪费

### 1.3 现有解决方案的局限

现有解决同质性问题的方法主要分为两类，但都引入了新的权衡：

**数据质量改进路径**：
- 通过预过滤策略筛选训练数据
- 问题：随着模型变强，PH prompts越来越多，预过滤的开销持续增长
- 额外推理成本与训练效率之间的矛盾

**轨迹复用路径**：
- 使用rollout buffer复用历史轨迹
- 问题：引入off-policy元素和分布偏移，影响稳定性和大规模部署

这引出了本文的核心问题：**能否在零边际开销的前提下，从PH prompts中提取学习信号，同时保持主要验证器目标的完整性？**

## 二、Miner框架：技术创新详解

### 2.1 核心洞察：不确定性即信号

Miner的核心理念源于一个简单而深刻的观察：虽然PH rollouts在验证器看来都是"正确"的，但底层策略对它们的"掌握程度"并不相同。

- 许多正确答案是通过**脆弱的低置信度推理路径**生成的
- 这些路径如果得不到优化，将一直是"脆弱的正确"
- 但如果简单地奖励不确定性，又会破坏正确性边界，淹没结果奖励

**核心挑战**：如何在PH prompts上安全、有界地注入不确定性信号，同时不影响非PH prompts的优化？

### 2.2 三大核心组件

Miner框架包含三个紧密耦合的设计：

#### 2.2.1 PH专属的不确定性驱动内在奖励

```
内在奖励 = 基于token级负对数似然的置信度度量
```

**具体机制**：

1. **基于熵的不确定性度量**：对于每个token，使用策略的负对数似然作为内在奖励
   - 低概率（高不确定性）→ 高内在奖励
   - 高概率（高置信度）→ 低内在奖励

2. **正向过滤（Positive Filtering）**：
   - 仅对正确的rollout应用内在奖励
   - 防止强化已经掌握的模式
   - 专注于"脆弱但正确"的推理路径

3. **零额外开销设计**：
   - 复用PPO优化过程中已计算的token概率
   - 无需额外rollout、提示词、replay buffer或辅助奖励模型

#### 2.2.2 Token级焦点信用分配机制（Focal Credit Assignment）

这是Miner的第一个关键创新。传统方法对所有token均匀加权，导致梯度分散在大量平凡token上。

**核心思想**：推理链中存在"瓶颈token"——这些关键位置的决策对整个推理过程至关重要。

**数学实现**：

```
权重(token) = (1 - p(token)) ^ γ
```

其中：
- `p(token)` 是该token的概率
- `γ` 是聚焦参数（通常设为2）

**效果**：
- 低概率token（瓶颈位置）获得更高权重
- 高概率token（常规推理）被抑制
- 梯度集中在真正需要改进的关键决策点

**直观理解**：
想象模型正在解一道数学题。某些步骤（如选择正确的定理、识别关键变量）是推理的"瓶颈"，这些位置的token应该获得更高关注。而简单的代数运算或格式化token则不应占用过多优化资源。

#### 2.2.3 自适应优势校准（Adaptive Advantage Calibration）

这是Miner的第二个关键创新。简单地叠加内在奖励和验证器奖励可能导致：
- 信号层级混乱：内在信号淹没正确性信号
- 优化方向冲突：模型可能为了"探索"而牺牲"正确"

**核心思想**：动态缩放内在优势，确保正确性信号始终优先。

**具体机制**：

1. **参考尺度提取**：从同一batch中的非PH prompts（heterogeneous prompts）提取奖励分布的尺度

2. **自适应缩放**：
   ```
   缩放后的内在优势 = 原始内在优势 × λ_max / 参考尺度
   ```

3. **信号层级保证**：
   - 验证器奖励（正确性）始终保持主导
   - 内在奖励作为"微调"信号，在正确性约束下进行优化

**关键参数 λ_max**：
- 控制内在信号的相对强度
- 在4B和8B模型间平滑迁移（论文中使用统一设置）
- 仅需调整这一个超参数

## 三、实验验证与性能表现

### 3.1 实验设置

**基础模型**：
- Qwen3-4B-Base
- Qwen3-8B-Base

**对比算法**：
- GRPO（标准实现）
- DAPO
- REINFORCE++
- 其他探索增强方法

**评测基准**（6个推理基准）：
1. GSM8K（数学推理）
2. MATH（竞赛级数学）
3. AIME 2024（高难度数学）
4. AMC（数学竞赛）
5. OlympiadBench（奥林匹克题目）
6. Gaokao（高考数学题）

### 3.2 主要实验结果

#### 3.2.1 Pass@1性能提升

| 模型 | 基准 | GRPO | Miner | 提升 |
|------|------|------|-------|------|
| Qwen3-4B | GSM8K | 85.2 | 89.1 | +3.9 |
| Qwen3-4B | MATH | 45.6 | 50.2 | +4.6 |
| Qwen3-4B | AIME24 | 12.3 | 16.9 | +4.6 |
| Qwen3-8B | GSM8K | 88.7 | 92.4 | +3.7 |
| Qwen3-8B | MATH | 52.1 | 56.8 | +4.7 |
| Qwen3-8B | AIME24 | 18.5 | 22.3 | +3.8 |

**平均Pass@1提升**：**+4.58绝对值**

#### 3.2.2 Pass@K性能提升

Pass@K衡量模型在K次尝试中至少成功一次的概率，反映模型的整体推理能力。

| 模型 | 基准 | GRPO | Miner | 提升 |
|------|------|------|-------|------|
| Qwen3-4B | MATH | 62.3 | 68.9 | +6.6 |
| Qwen3-4B | AIME24 | 28.5 | 35.1 | +6.6 |
| Qwen3-8B | MATH | 71.2 | 77.8 | +6.6 |

**平均Pass@K提升**：**+6.66绝对值**

#### 3.2.3 训练效率

- **达到相同峰值性能所需步数**：GRPO的50%
- **PH prompts利用率**：从~0%提升到~80%
- **额外计算开销**：可忽略不计（<1%）

### 3.3 跨领域泛化验证

为了验证Miner的领域无关性，作者在医疗推理领域进行了额外实验：

**训练数据**：85K MedQA/MedMCQA子集  
**评测基准**：MedQA、MedMCQA、PubMedQA、MedXpertQA、MMLU-Pro医疗子集

**结果**：
- Miner在所有五个医疗任务上均超越GRPO
- 即使在PH prompts占比>65%的batch中，Miner仍保持稳定提升
- 超越了需要复杂数据处理和GPT-4o蒸馏的医疗专用模型（HuatuoGPT-o1、MedReason）

**关键发现**：
- **模型架构无关**：在Qwen和Llama架构上均有效（虽然Qwen上提升更明显）
- **领域无关**：数学、医疗等逻辑推理任务均受益
- **数据规模无关**：从4B到8B模型，相同超参数设置即可

### 3.4 消融实验

#### 3.4.1 组件重要性分析

| 配置 | GSM8K | MATH | AIME24 |
|------|-------|------|--------|
| GRPO基线 | 85.2 | 45.6 | 12.3 |
| + 仅内在奖励（无过滤） | 84.1 | 43.2 | 10.1 |
| + PH专属内在奖励 | 86.8 | 47.3 | 14.2 |
| + Token级焦点分配 | 88.2 | 49.1 | 15.8 |
| **完整Miner** | **89.1** | **50.2** | **16.9** |

**结论**：
- 简单的"奖励不确定性"反而有害（-1.1 GSM8K）
- PH过滤是关键（+2.7 GSM8K）
- Token级焦点分配提供额外增益（+1.4 GSM8K）
- 自适应校准确保训练稳定性

#### 3.4.2 探索-利用动态

论文通过熵动态分析展示了Miner的学习过程：

1. **探索阶段**：初期熵值稳步上升，模型积极探索不同推理路径
2. **转换阶段**：中期熵值快速下降，模型开始利用已发现的优质路径
3. **利用阶段**：后期熵值稳定，模型聚焦于稳健的推理策略

**关键观察**：
- Miner比GRPO更快完成从探索到利用的转换
- 性能提升与熵值下降同步发生
- 表明内在奖励有效引导了探索过程

## 四、技术深度分析

### 4.1 与现有工作的对比

#### 4.1.1 vs GRPO/DAPO/REINFORCE++

| 特性 | GRPO | DAPO | REINFORCE++ | Miner |
|------|------|------|-------------|-------|
| 解决PH问题 | ✗ | ✗ | ✗ | ✓ |
| 额外推理成本 | 无 | 中等 | 无 | 无 |
| 存储开销 | 低 | 中 | 低 | 低 |
| 超参数数量 | 少 | 较多 | 少 | 少+1 |
| Pass@1提升 | 基线 | +1-2 | +1-2 | +4.58 |

#### 4.1.2 vs 其他探索增强方法

**与基于多样性奖励的方法对比**：
- ** diversity-based方法**：鼓励生成多样化的推理路径，但可能偏离正确性目标
- **Miner**：仅在已确保正确性的PH rollouts上增强，不干扰正确性优化

**与基于好奇心（curiosity）的方法对比**：
- **好奇心方法**：通常需要额外的预测模型或密度模型
- **Miner**：仅使用策略自身的不确定性，无需辅助模型

#### 4.1.3 vs 数据过滤策略

- **过滤策略**：在训练前筛选数据，避免PH prompts进入训练
- **Miner**：充分利用PH prompts中的信息，变"废"为宝
- **关键差异**：过滤策略随着模型变强，可用数据越来越少；Miner则始终保持高效利用

### 4.2 理论分析

#### 4.2.1 为什么PH prompts有价值？

PH prompts看似"已解决"，但实际上：

1. **脆弱的正确性**：模型可能通过"猜测"或"模式匹配"得到正确答案，而非真正理解
2. **未充分优化的路径**：正确但低置信度的推理路径表明模型"知道但不确信"
3. **知识巩固机会**：强化这些路径可以将其从"脆弱正确"转变为"稳健正确"

**类比人类学习**：
想象一个学生做数学题。有些题他能做对但不太确定（"我好像做对了，但不确定步骤是否正确"），有些题他能轻松做对。前者正是最需要练习的——Miner就像针对这些"不太确定但做对了"的题目的专项训练。

#### 4.2.2 Token级焦点分配的数学直觉

为什么token级加权有效？

假设推理链：A → B → C → D（答案）

传统方法均匀优化：∂L/∂θ_A ≈ ∂L/∂θ_B ≈ ∂L/∂θ_C ≈ ∂L/∂θ_D

焦点分配后：
- 如果p(A)很高（模型很确定），权重↓
- 如果p(B)很低（模型不确定），权重↑

**效果**：梯度集中在"模型不确定但关键"的决策点，类似于课程学习中的"困难样本挖掘"。

### 4.3 局限性与未来方向

#### 4.3.1 当前局限

1. **规模限制**：论文仅在4B和8B模型上验证，未扩展到32B+模型
   - 作者指出：超参数λ_max从4B到8B平滑迁移，暗示方法具有规模可扩展性
   
2. **训练长度限制**：Qwen3-8B模型仅训练一个epoch
   - 作者指出：训练过程稳定，延长训练可进一步提升性能

3. **架构偏好**：在Qwen架构上提升显著，在Llama架构上提升相对较小
   - 原因：Llama在数学推理上的基础能力较弱，PH prompts占比低
   - 随着Llama基础能力提升，Miner效果会相应增强

#### 4.3.2 未来研究方向

1. **与过程奖励模型（PRM）的结合**：
   - Miner目前仅使用结果奖励
   - 结合PRM可能进一步提升训练效率和最终性能

2. **自适应λ_max**：
   - 当前λ_max是固定超参数
   - 探索基于训练动态的自适应调整策略

3. **扩展到更复杂的推理任务**：
   - 代码生成、定理证明、科学推理等
   - 验证Miner在这些领域的有效性

4. **理论理解**：
   - 深入分析内在奖励与验证器奖励的交互机制
   - 建立PH prompts价值的形式化理论

## 五、个人理解与行业影响分析

### 5.1 核心贡献的深层解读

Miner的提出标志着大模型RL训练从"扩大规模"向"提升效率"的重要转变。在DeepSeek-R1展示RLVR范式的强大能力后，社区关注点逐渐从"能否训练"转向"如何更高效地训练"。

**Miner的核心洞察——不确定性即信号——具有深远的启发意义**：

1. **重新定义"有用数据"**：传统观点倾向于筛选"困难"数据，丢弃"已解决"的样本。Miner证明，即使是"全对"的batch，只要处理得当，也能提供丰富的学习信号。

2. **自监督的新范式**：利用模型自身的不确定性作为监督信号，无需外部标注。这与自监督学习的核心理念一致，但应用于RL场景。

3. **精细化的信用分配**：从序列级奖励细化到token级加权，体现了对推理过程本质的更深入理解。

### 5.2 对行业的实际影响

#### 5.2.1 训练成本优化

- **计算资源节省**：达到相同性能仅需50%训练步数
- **显存效率**：无需额外模型或buffer，适合大规模分布式训练
- **实际ROI**：对于训练一次成本数百万美元的大模型，50%的效率提升意味着巨大的经济价值

#### 5.2.2 技术路线影响

Miner的出现可能改变RLVR训练的技术选型：

- **从DAPO到Miner+DAPO**：DAPO提出的动态采样、策略性裁剪等技术与Miner的PH处理机制可以互补
- **开源社区 adoption**：简单的实现、单一超参数、即插即用的特性，使其易于被开源社区采纳
- **商业化应用**：低额外开销特性适合生产环境的持续训练

#### 5.2.3 研究范式启示

Miner的成功验证了几个重要的研究原则：

1. **问题导向的简单方案**：从实际问题（PH效率）出发，而非从复杂技术出发
2. **充分利用已有信息**：token概率本是RL计算的副产品，Miner巧妙地将其转化为学习信号
3. **安全与效率的平衡**：通过PH过滤和自适应校准，确保效率提升不以牺牲稳定性为代价

### 5.3 与当前技术趋势的关联

#### 5.3.1 与测试时计算扩展（Test-Time Scaling）的关系

测试时计算扩展（如o1、o3）通过在推理时投入更多计算来提升性能。Miner从另一角度解决这个问题：

- **测试时扩展**：推理时多采样、多验证
- **Miner**：训练时更高效地利用数据，使模型本身更强，减少测试时所需的额外计算

两者是互补关系：更强的基础模型 + 测试时扩展 = 最佳性能。

#### 5.3.2 与小模型崛起的关联

随着社区对"小模型+RL"组合的关注（如DeepSeek-R1-Distill、TinyR1），训练效率变得更加关键：

- 小模型数据容量有限，必须充分利用每条训练数据
- Miner的PH利用机制正好满足这一需求
- 可能加速高性能小模型的训练流程

#### 5.3.3 与Agent系统的结合

Agent系统通常涉及多轮交互和复杂决策，PH问题可能更加严重：

- 某些子任务模型已经很擅长（PH）
- 其他子任务仍需学习
- Miner的token级焦点分配可能特别适合多步骤Agent任务的细粒度优化

### 5.4 长期展望

Miner代表了大模型训练效率优化的新方向。展望未来：

1. **自适应训练系统**：模型能够自动识别自身的"知识边界"，在掌握不牢固的地方自动加强训练
2. **动态课程学习**：结合Miner的不确定性感知，实现更智能的课程安排
3. **个性化推理模型**：针对特定用户或任务的推理偏好，通过类似机制进行高效微调

## 六、关键公式与实现细节

### 6.1 核心公式汇总

**内在奖励计算**：
```
r_intrinsic = -log p(a_t | s_t)  # token级负对数似然
```

**Token级焦点权重**：
```
w_t = (1 - p(a_t | s_t))^γ
```

**自适应优势校准**：
```
A_calibrated = A_intrinsic × (λ_max / σ_heterogeneous)
```

**最终优势**：

```python
def compute_miner_advantages(rollouts, is_ph_mask, gamma=2.0, lambda_max=1e-3):
    """
    rollouts: 采样轨迹，包含log_probs和rewards
    is_ph_mask: 标记哪些是PH prompts的布尔掩码
    """
    # 1. 计算验证器优势（标准GRPO）
    verifier_advantages = compute_grpo_advantages(rollouts)
    
    # 2. 计算内在奖励（仅PH prompts）
    token_probs = torch.exp(rollouts.log_probs)
    intrinsic_rewards = -torch.log(token_probs + 1e-10)  # 负对数似然
    
    # 3. Token级焦点加权
    focal_weights = (1 - token_probs) ** gamma
    weighted_intrinsic = intrinsic_rewards * focal_weights
    
    # 4. 序列级聚合
    sequence_intrinsic = weighted_intrinsic.sum(dim=-1)
    
    # 5. 自适应校准
    hetero_std = sequence_intrinsic[~is_ph_mask].std()
    calibration_scale = lambda_max / (hetero_std + 1e-10)
    
    # 6. 组合优势
    final_advantages = verifier_advantages.clone()
    final_advantages[is_ph_mask] += sequence_intrinsic[is_ph_mask] * calibration_scale
    
    return final_advantages
```
