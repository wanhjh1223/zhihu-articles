## 一、研究背景与核心问题

### 1.1 测试时计算的困境

大规模语言模型（LLMs）在解决复杂问题（如奥林匹克数学竞赛、软件工程任务）方面取得了令人瞩目的进展，其核心驱动力在于**测试时计算的大幅扩展**。从并行采样、顺序蒙特卡洛到蒙特卡洛树搜索，从验证器引导采样到自一致性机制，这些策略极大地激发了LLMs的高级推理能力。

然而，当前最先进的测试结果往往伴随着**极其高昂的计算成本**。这些低效性主要源于两个根本性问题：

**第一，后训练语言模型的校准失准**。研究表明，经过后训练的LLMs对其推理状态的正确性往往缺乏准确判断。模型在"思考"过程中无法可靠地评估自己是否正确，导致大量无效计算被浪费在错误的推理路径上。

**第二，流行采样技术缺乏校准机制**。现有的测试时扩展策略往往依赖手工设计的参数来平衡样本质量和资源分配——例如并行rollout的数量、验证器的提示词等。这些启发式方法容易受到奖励黑客攻击的影响，且对分布偏移极为脆弱。

### 1.2 现有方法的局限性

**保形预测（Conformal Prediction）**和**校准方法**虽然能提供有限样本覆盖保证，但在测试时扩展的语境下存在关键缺陷：

- **假设固定推理过程**：现有方法假设采样动态在整个推理过程中是均匀的，未能解决两个层面的分布偏移问题：
  - **样本层面**：推理模式可能在长思维链（CoT）的不同位置发生变化
  - **数据集层面**：部署时的提示分布可能与模型开发时所见不同

**测试时训练（Test-Time Training, TTT）**提供了在线适应的自然机制，通过元学习在推理时适应轻量级模块。然而，标准的TTT框架使用重建损失，这些损失与校准或风险控制并不直接对齐。

### 1.3 ORCA的核心贡献

本文提出的**Online Reasoning Calibration (ORCA)**框架，首次将测试时训练与保形预测相结合，实现了具备统计保证的高效测试时扩展：

1. **内循环**：通过TTT层实现评分函数的在线优化，使其能够适应推理的不同阶段
2. **外循环**：元学习共享的"慢权重"（初始化和特征映射），确保在线更新稳定、数据高效且可迁移
3. **风险控制**：通过Learn-then-Test (LTT)框架对完整的自适应算法（包括采样、更新、停止规则）进行校准，提供有限样本保证

## 二、技术方法详解

### 2.1 问题形式化

**符号定义**：
- 输入：$x \in \mathcal{X}$
- 推理步骤$t$时生成的思维前缀：$y_t = [y^{(1)}, ..., y^{(t)}]$
- 当前答案候选：$\text{ans}(y_t)$，可从推理状态推导
- 隐藏表示：$\phi_t \in \mathbb{R}^{d_\phi}$，从基础LLM中提取
- 探针分数：$z_t = f(\phi_t; W_t)$，其中$W_t$是步骤依赖的快速权重
- 最终决策损失：$L(\hat{y}, y) \in \{0, 1\}$，表示过早停止或错误接受答案的风险

### 2.2 核心架构：双层优化框架

ORCA遵循TTT的双层优化结构：

#### 2.2.1 内循环：在线自适应探针

在每个推理步骤$t$，基础LLM产生隐藏表示$\phi_t$。探针维护快速权重$W_t$，沿推理链在线更新，产生置信度分数$s_t \in [0,1]$。

**操作流程**（先评分后更新）：

**步骤1：评分**（使用之前步骤的权重）
```
s_t = f(φ_t; W_{t-1})
```

**步骤2：内循环损失**（Brier分数对抗标签$C_t$）
```
l(W_{t-1}; φ_t) = ||s_t - C_t||²
```

**步骤3：权重更新**（在线梯度下降）
```
W_t = W_{t-1} - η ∇_W l(W_{t-1}; φ_t)
```

其中$C_t \in \{0,1\}$是步骤级标签，指示当前答案尝试的质量，可以来自多种来源：
- **监督模式**：$C_t = \mathbb{I}\{\text{ans}(y_t) \text{ is correct}\}$，需要标准答案
- **一致模式**：$C_t = \mathbb{I}\{\text{ans}(y_t) = \text{ans}(y_T)\}$，与完整预算答案比较，无需标准答案
- **教师/验证器标签**：来自外部验证器的信号

#### 2.2.2 外循环：元学习可泛化的自适应校准

单纯的实例级更新可能过拟合局部噪声，破坏分布偏移下的风险控制。外循环学习适当的初始化和特征接口，使内循环适应可跨任务迁移。

**关键洞察**：
- 内循环学习**校准**（任务特定的在线适应）
- 外循环学习**学习如何校准**（通过元学习跨异构数据集和提示进行训练）

**数学形式化**：

令$\Theta_{\text{outer}} = (\theta_{Q,K}, W_0, \eta)$表示外参数。

外损失定义为：
```
L_outer(x, {W_t}_{t=1}^T; Θ_outer) := Σ_{t=1}^T (s_t - C_t^true)²
```

约束条件：$W_t = W_{t-1} - \eta \nabla_W l(W_{t-1}; x_t)$

通过**截断反向传播**通过内循环更新来优化这一双层目标。

#### 2.2.3 Q/K变体：可学习的投影

受Sun等人(2024)启发，引入可学习投影$\theta_K, \theta_Q \in \mathbb{R}^{d_h \times d_\phi}$，将$\phi_t$映射到低维空间：

```
s_t = f(θ_Q φ_t; W_{t-1}), W ∈ ℝ^{1×d_h}
l(W; φ_t) = ||f(θ_K φ_t; W) - C_t||²
```

投影$\theta_K, \theta_Q$是"慢权重"，与外循环一起学习：
- $\theta_K$决定更新方向
- $\theta_Q$决定评分方向

这允许更新方向和评分方向关注隐藏状态的不同方面。

### 2.3 风险控制的保形推理

#### 2.3.1 Learn-then-Test (LTT)框架

对于阈值$\lambda \in \Lambda$，定义由部署过程产生的分数过程：
```
s_t(x) := f(φ_t; W_{t-1})
```

停止时间：
```
τ_λ(x) := min{t ≤ T : s_t(x) ≥ λ}
```

部署决策规则/过程输出：
```
A_λ(x) := ans(y_{τ_λ(x)})
```

LTT通过在保留的校准集上校准完整过程$A_\lambda$的风险来选择$\lambda^*$：

对每个$\lambda_j$检验零假设：
```
H_j : E[R(y_{τ_{λ_j}})] ≥ δ
```

构建p值：
```
p_j^{BT} := P(Binom(n, δ) ≤ n·R̂_n(λ_j))
```

应用**固定序列检验**控制族错误率(FWER)，选择最激进的被拒绝阈值$\lambda^*$。

**理论保证**：
```
P(E[R(y_{τ_{λ^*}})] ≤ δ) ≥ 1 - ε
```

值得注意的是，这些保证适用于**完整的部署过程**，包括推理链扩展、在线快速权重更新和基于阈值的停止。

### 2.4 推理时部署算法

**算法流程**：

1. **校准阶段**：使用LTT校准停止阈值
2. **推理阶段**：
   - 初始化快速权重$W \leftarrow W_0$
   - 对每个步骤$t = 1, ..., T$：
     - 从推理状态获取当前隐藏表示$\phi_t$
     - 计算探针分数$s_t = f(\phi_t; W)$
     - 如果$s_t \geq \lambda^*$：停止并输出答案
     - 否则：设置伪目标$C_t \leftarrow 0$，执行内循环更新$W \leftarrow W - \eta \nabla_W l(W; \phi_t)$

## 三、实验结果与性能分析

### 3.1 实验设置

**数据集**：
- 训练语料：5K问题（s1K 1K + OpenR1 2K + DeepMath 2K）
- 划分：训练3K : 校准1K : 测试1K
- OOD评估：MATH-500、GPQA-Diamond、AIME 2024/2025/2026

**模型**：
- 主要实验：Qwen2.5-32B-Instruct（$d_\phi = 5,120$）
- 跨模型验证：QwQ-32B、Llama-3.3-70B-Instruct（$d_\phi = 8,192$）

**推理轨迹生成**：DeepSeek-R1-671B

**标签模式**：
- 监督模式（需要标准答案）
- 一致模式（无需标签，自举）

**评估指标**：
1. **节省率** = $1 - \bar{t}_{\text{stop}} / \bar{t}_{\text{total}}$（早停节省的推理步骤比例）
2. **错误率**（在答案仍不正确时停止的问题比例）

### 3.2 分布内结果

| 方法 | 风险水平δ | 监督模式节省 | 一致模式节省 | 错误率 |
|------|----------|-------------|-------------|--------|
| 静态基线 | 0.1 | 38.0% | 34.5% | ~δ |
| TTT-Probe (no-QK) | 0.1 | **47.5%** | **40.7%** | ~δ |
| TTT-Probe (QK) | 0.1 | 41.4% | - | ~δ |

**关键发现**：
- 在$\delta = 0.1$时，监督模式下的TTT-Probe (no-QK)节省了**47.5%**的推理步骤，相比静态基线的38.0%，相对提升**24.9%**
- QK变体达到41.4%，相对提升8.9%
- 一致模式下，no-QK探针节省40.7% vs 基线34.5%，相对提升18.2%，且**无需任何标准答案**
- 所有变体在四个$\delta$水平上都保持错误率在规定的$\delta$预算内或接近

### 3.3 分布外泛化（核心突破）

这是ORCA最关键的贡献——在零样本OOD设置下的表现：

| 方法 | MATH-500节省 | GPQA节省 | 错误率 |
|------|-------------|----------|--------|
| 静态基线 | 24.8% | - | ~δ |
| TTT-Probe (no-QK) | **63.7%** | **71.5%** | < 2.3% |
| TTT-Probe (QK) | **67.0%** | - | < 2.3% |

**革命性成果**：
- 在MATH-500上，监督标签下：
  - no-QK节省63.7%
  - QK节省67.0%
  - 相比基线24.8%，提升**2.6-2.7倍**！
- 在GPQA-Diamond上，no-QK探针达到**71.5%**节省
- 一致模式下，QK变体在MATH-500上达到63.7%（基线的2.7倍）
- 错误率保持在2.3%以下

这表明**标签免费的TTT在OOD场景下也是可行的**！

### 3.4 跨模型泛化

验证发现是否超越单一模型：

| 模型 | 静态基线 | TTT (no-QK) | TTT (QK) | 相对提升 |
|------|---------|-------------|----------|----------|
| Qwen2.5-32B | 38.0% | 47.5% | 41.4% | 8.9-24.9% |
| QwQ-32B | 37.4% | 50.0% | 47.7% | 19.8-33.7% |
| Llama-3.3-70B | 34.5% | 41.3% | 36.8% | 6.8-19.8% |

所有模型的错误率都保持在$\delta = 0.1$预算内或接近，证实了在线适应机制的有效性和可靠性。

## 四、核心创新深度解析

### 4.1 将校准本身作为自适应预测问题

ORCA的核心创新在于**重新定义了校准的目标**。传统方法将校准视为静态过程，而ORCA将其视为一个可以在线优化的动态预测问题：

- **内循环**：实例特定的学习校准更新
- **外循环**：学习初始化和更新动态，跨数据集迁移

这种设计使得校准模块能够：
1. 适应推理的不同阶段（思考初期 vs 思考后期）
2. 对分布偏移具有鲁棒性
3. 在实例级别进行自适应

### 4.2 保形预测与TTT的首次结合

这是**首个将在线适应和测试时训练引入LLM推理校准**的工作。关键突破包括：

1. **重新定义TTT损失**：从重建损失转向校准质量的Brier分数
2. **保形化完整部署过程**：不仅校准原始分数，而是校准包含在线更新的完整决策规则
3. **双层风险控制**：内循环保证实例级适应，外循环保证数据集级泛化

### 4.3 计算效率与统计保证的权衡

ORCA提供了一个优雅的框架来平衡：
- **效率**：通过早期停止减少不必要的测试时计算
- **可靠性**：通过保形预测提供有限样本风险保证
- **泛化**：通过TTT实现OOD鲁棒性

实验表明，ORCA能够在保持低错误率的同时，实现高达**67%**的计算节省——这在实际部署中具有巨大价值。

### 4.4 自举能力：一致模式的突破

一致模式（Consistency Mode）的引入是一个重要创新：
- **无需人工标注**：使用自一致性作为伪标签
- **有效学习信号**：比较中间答案与最终答案
- **OOD有效性**：即使在分布外场景也能工作

这为实际部署提供了可行路径——无需昂贵的人工标注即可实现高效推理。
