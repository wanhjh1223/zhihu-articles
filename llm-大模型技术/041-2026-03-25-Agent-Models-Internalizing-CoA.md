## 一、研究背景与核心问题

### 1.1 OpenAI的AGI五阶段路线图

OpenAI在通往通用人工智能（AGI）的道路上提出了五个渐进阶段：
- **第一阶段：Chatbot** - 以大语言模型（LLMs）如GPT-3.5、GPT-4为代表
- **第二阶段：Reasoner** - 以大型推理模型（LRMs）如o1、o3为代表
- **第三阶段：Agent** - 以Operator和Deep Research为代表（**当前阶段**）
- **第四阶段：Innovator**
- **第五阶段：Organization**

2025年初，OpenAI相继发布了Operator和Deep Research，标志着AI发展正式进入了第三阶段——Agent时代。这些系统不再是简单的问答模型，而是能够结合推理能力与自主工具使用，通过与现实世界交互独立执行多轮工作流的智能体。

### 1.2 当前Agent技术的局限性

目前市面上的开源Deep Research实现（如Hugging Face的Open Deep Research）主要依赖**Agentic Workflows**（如ReAct、Reflexion）来编排任务规划和工具使用。这种架构存在根本性局限：

**"被动式"行为模式**：思维与行动的切换由预设的工作流触发，模型本身并不真正"理解"何时需要使用工具。这是一种基于外部提示的被动行为，而非模型的内在能力。

**缺乏端到端优化**：现有的Agent系统通常将LLM/LRM作为黑盒组件，通过外部提示工程来驱动工具调用，没有实现真正的端到端训练优化。

**推理与行动的割裂**：传统方法中，推理（CoT）和行动（CoA）是两个分离的过程，缺乏内在的连贯性和一致性。

### 1.3 核心研究问题

本论文提出了一个根本性的问题：**能否将工具使用能力内化为推理模型的一部分，使其能够自主决定何时以及如何使用外部工具？**

这个问题的解决方案是**Large Agent Models (LAMs)**——一种新型的大模型范式，它通过端到端的任务导向训练，将Chain-of-Action（行动链）生成能力内化为模型的内在行为。

## 二、核心创新：从Agentic Workflow到Agent Model的范式转变

### 2.1 范式演进的历史脉络

论文作者巧妙地将AI发展的历史脉络与当前的大模型演进进行了类比：

| 发展阶段 | 从...到... | 代表技术 |
|---------|-----------|---------|
| 早期AI | 专家系统 → 机器学习 | 规则系统 → SVM/决策树 |
| 深度学习时代 | 手工特征 → 数据驱动学习 | CNN/RNN/Transformer |
| 推理模型时代 | CoT Prompting → LRM | GPT-4 → o1/DeepSeek-R1 |
| **Agent时代** | **Agentic Workflow → LAM** | **ReAct → AutoCoA** |

这一演进的核心规律是：**从手工设计的规则/提示，逐渐过渡到从数据中学习的模式**。学习得到的模式具有更好的泛化能力和灵活性。

### 2.2 Agent Model的正式定义

论文给出了Agent Model的数学定义：

**定义1**：Agent Model是建立在推理模型基础上的生成模型，通过端到端的任务导向工具增强训练进行强化。它产生推理（Chain-of-Thought）和行动（Chain-of-Action）步骤的交错序列，每个行动调用工具与外部环境交互。这些交互的观察结果指导后续的推理和行动，直到任务完成。

Agent Model的推理可以被形式化为**部分可观察马尔可夫决策过程（POMDP）**：

- **状态**：$s = [s_0, t_c, x_{1:n}]$，包括初始环境状态、任务上下文和已生成的序列
- **策略**：$\pi_\theta$ 参数化的策略模型
- **动作**：生成下一个token $x_{n+1} \sim \pi_\theta(\cdot|s_0, t_c, x_{1:n})$

### 2.3 三种核心Token类型

Agent Model通过三种特殊token实现推理与行动的有机统一：

1. **⟨think⟩** - 开启或继续推理步骤，贡献于Chain-of-Thought (CoT)
2. **⟨action⟩** - 调用工具与环境交互，贡献于Chain-of-Action (CoA)
   - 动作类型：指定工具库中的工具 $\tau \in \mathcal{T}$
   - 动作参数：定义如何执行工具
   - 环境响应：通过工具调用获取的观察结果 $o = \tau(p)$
3. **⟨answer⟩** - 输出最终响应，表示任务完成

这种设计实现了推理与行动的无缝切换，模型能够**自主决定**何时使用工具、使用什么工具、如何使用工具。

### 2.4 与Agentic Workflow的本质区别

| 维度 | Agentic Workflow | Agent Model |
|-----|-----------------|-------------|
| 触发机制 | 外部预设规则/提示 | 模型内在行为 |
| 上下文考虑 | 仅考虑前文 | 考虑前后文全局优化 |
| 训练方式 | 零样本/少样本提示 | 端到端监督+强化学习 |
| 任务完成 | 工作流驱动 | 任务完成为优化目标 |
| 泛化能力 | 受限于提示设计 | 从数据中学习，泛化更强 |

## 三、AutoCoA训练框架详解

### 3.1 核心挑战

开发Agent Model面临两个根本性挑战：

**挑战1：推理与行动的平衡**
- 推理模型擅长生成CoT，但在微调到增强行动执行能力时，存在遗忘CoT生成能力的常见风险
- 更关键的是，需要教会模型在推理过程中的适当时机触发行动

**挑战2：外部环境交互的管理**
- 通过工具调用与外部环境交互可能导致高昂的训练成本、低效率甚至风险
- 外部环境的动态性使训练过程复杂化，特别是在在线强化学习期间

### 3.2 三阶段监督微调（SFT）

AutoCoA框架采用了渐进式的三阶段SFT策略：

#### Stage 1: CoT+A（步骤级行动触发）

**目标**：教会模型**何时**触发行动

**方法**：
- 生成对比数据对：通过修改推理轨迹中的特定点，一个变体启用工具使用，另一个不启用
- 使用**对比损失**进行训练：
  $$\mathcal{L}_{\text{contra}} = -\log \sigma [P(x_{\text{chosen}}|c) - P(x_{\text{rejected}}|c)]$$
- 引入辅助SFT损失防止模型崩溃：
  $$\mathcal{L} = \mathcal{L}_{\text{contra}} + \alpha \cdot \mathcal{L}_{\text{aux}}$$

**核心洞察**：成功的工具调用作为正例，失败的无行动尝试作为负例，通过对比学习让模型学会在知识缺口处自主触发搜索。

#### Stage 2: CoT+CoA（带观察掩码）

**目标**：教会模型**如何**执行行动

**方法**：
- 训练模型在完整决策轨迹上整合CoT和CoA
- **关键设计**：在损失计算中屏蔽外部环境反馈token
- 损失函数：
  $$\mathcal{L} = \frac{1}{|x|} \sum_{i=1}^{|x|} \mathbb{I}_{x_i \neq o} [\log \pi_\theta(x_i|t_c, x_{<i})]$$

这种设计避免了外部反馈对学习过程的干扰，专注于学习"行动"本身的生成模式。

#### Stage 3: CoT+CoA（无观察掩码）

**目标**：学习**内部世界模型**

**方法**：
- 在完整轨迹上包含外部环境反馈token的损失
- 模型同时优化内部推理、行动输出，并预测外部反馈
- 这种联合训练有效地作为内部世界模型的隐式学习

**意义**：为后续的强化学习阶段提供了更稳健的初始化，模型能够"想象"工具调用的结果。

### 3.3 两阶段强化学习（RL）

在SFT基础上，RL阶段使用**Group Relative Policy Optimization (GRPO)** 进一步内化CoA生成能力：

#### 奖励函数设计

- **精确匹配奖励**：仅当最终输出与ground truth完全匹配时给予正奖励
- **格式惩罚**：如果输出不符合要求的格式（如缺少⟨think⟩标签），施加惩罚

这种严格的奖励设计确保了模型在生成CoT和CoA时保持高精度。

#### RL Stage 1: 模拟环境交互

- 模型使用在SFT阶段开发的内部世界模型与模拟环境交互
- 生成假设的工具交互并使用学习到的内部世界模型模拟响应
- 通过基于组的采样和基于规则的奖励，GRPO探索多样化的潜在CoA策略

**优势**：低成本、高效率的沙盒环境，支持大规模探索。

#### RL Stage 2: 真实环境交互

- 模型过渡到真实环境，与实际工具交互获取真实世界反馈
- 将前一阶段学到的策略适应动态和不可预测的真实世界环境

**优势**：确保模型在开放任务中的稳健性能。

### 3.4 数据合成策略

论文设计了三种数据合成方法：

1. **CoT数据**（10,000样本）：纯链式思维过程，不使用外部工具
2. **CoA数据**（10,000样本）：工具辅助推理，包含行动触发、环境观察
3. **CoT+A数据**（1,500对比对）：对比学习的正例和负例对

数据生成使用DeepSeek-R1-Distill-Qwen-32B模型，基于HotpotQA数据集构建。环境使用FlashRAG搭建的本地wiki搜索引擎。

## 四、实验结果与性能分析

### 4.1 实验设置

**基础模型**：R1-Distill-Qwen-7B（小规模推理模型）

**训练框架**：
- SFT：LLaMA-Factory
- RL：verl
- 硬件：单节点Nvidia H20 GPUs

**评测数据集**：
- **单跳QA**：Natural Questions (NQ)、TriviaQA
- **多跳QA**：HotpotQA、2WikiMultihopQA (2WIKI)、MuSiQue、Bamboogle

**评测指标**：
- **Exact Match (EM)**：严格精确匹配
- **LLM-based Accuracy**：使用Qwen2.5-14B-instruct判断语义匹配

### 4.2 主要实验结果

（注：由于论文中部分表格数据在PDF中提取不完整，以下为基于论文描述的主要发现）

#### 发现1：显著超越ReAct基线

AutoCoA训练的Agent Model在任务完成率上**显著优于基于ReAct的工作流**，特别是在需要长期推理和多步行动的任务中。

**原因分析**：
- ReAct依赖外部提示触发行动，缺乏对任务全局的理解
- Agent Model通过端到端训练，能够更好地平衡推理与行动
- 内部世界模型使模型能够进行更有效的规划

#### 发现2：模拟到真实的有效迁移

两阶段RL设计证明了从模拟环境到真实环境的有效迁移：
- 在模拟环境中进行广泛的探索和学习
- 在真实环境中进行少量交互即可适应
- 大幅降低了训练成本，同时保证了真实世界性能

#### 发现3：推理与行动的有效平衡

混合CoT和CoA数据的策略确保了模型：
- 保留了纯推理能力（可处理无需外部知识的任务）
- 获得了工具使用能力（可处理需要外部信息的任务）
- 能够自适应地决定何时使用工具

### 4.3 消融实验（推断）

基于论文描述，可能的消融实验发现包括：

1. **对比学习的有效性**：CoT+A阶段对步骤级行动触发能力的贡献
2. **内部世界模型的作用**：Stage 3对RL阶段快速收敛的影响
3. **数据混合比例**：CoT与CoA数据的最优混合比例
4. **模拟环境的必要性**：直接从真实环境训练 vs 模拟+真实的性能差异

## 五、深度技术解读

### 5.1 Chain-of-Action vs Chain-of-Thought

CoA和CoT的本质区别在于：**CoT是模型与自身的对话，CoA是模型与环境的交互**。

**CoT的局限性**：
- 知识受限于训练数据
- 无法获取实时信息
- 复杂计算容易出错

**CoA的突破**：
- 通过工具调用突破知识边界
- 可访问外部数据库、搜索引擎、计算工具
- 实现真正的"知行合一"

### 5.2 内部世界模型的价值

内部世界模型是AutoCoA的关键创新之一，其价值体现在：

1. **降低训练成本**：在模拟环境中进行大量探索，减少昂贵的真实工具调用
2. **提高探索效率**：模型可以"想象"行动的后果，进行更有针对性的探索
3. **增强规划能力**：基于对世界模型的理解，模型可以进行更长程的规划

这与人类的认知方式高度相似：我们在行动前会在大脑中模拟各种可能性。

### 5.3 从Prompt Engineering到Capability Learning

论文揭示了一个重要趋势：**从提示工程到能力学习的转变**。

**Prompt Engineering的时代**：
- 需要精心设计提示来"诱导"模型行为
- 泛化能力受限于提示的设计
- 每个任务都需要特定的提示调优

**Capability Learning的未来**：
- 模型从数据中学习到内在能力
- 更好的泛化和适应性
- 减少对人类设计的依赖

这与从专家系统到机器学习的演进规律一致：**手工规则/提示 → 数据驱动的学习**。

### 5.4 Agent Roadmap展望

论文提出了Agent发展的三阶段路线图：

#### 短期：应用落地
- **Agentic RPA**：企业工作流的自动化（ERP/CRM系统）
- **个人助手**：移动端的个性化交互，作为用户的"数字孪生"

关键技术：基于推理基础模型+基础工具使用能力的Agent基础模型，通过RFT（强化微调）实现场景适配。

#### 中期：能力扩展
- 更多工具类型的支持
- 跨模态的Agent能力
- 多Agent协作系统

#### 长期：通用Agent
- 接近人类水平的自主决策
- 复杂环境中的长期规划
- 自我改进和自我进化

## 六、行业影响与未来展望

### 6.1 对AI Agent领域的影响

1. **重新定义Agent架构**：从工作流驱动到模型驱动，Agent的架构设计将发生根本性变革

2. **降低Agent开发门槛**：一旦基础Agent Model成熟，开发者无需再设计复杂的提示和工作流

3. **提升Agent可靠性**：端到端训练使得Agent行为更加一致和可预测

### 6.2 对模型训练的启示

1. **多阶段训练的价值**：SFT的三阶段设计展示了渐进式能力注入的有效性

2. **模拟-真实的两阶段RL**：为降低真实环境训练成本提供了可行方案

3. **对比学习在时序决策中的应用**：CoT+A的设计为教授模型"决策时机"提供了新思路

### 6.3 待解决的开放问题

1. **可扩展性**：当前实验基于7B小规模模型，需要验证在更大模型上的有效性

2. **工具多样性**：当前仅测试了搜索工具，需要扩展到更多工具类型（代码执行、API调用、物理环境交互）

3. **安全性**：Agent自主决定工具使用可能带来安全风险，需要有效的安全约束机制

4. **可解释性**：内部世界模型的决策过程需要更好的可解释性支持

### 6.4 与OpenAI Deep Research的关系

论文明确指出，OpenAI的Deep Research正是Agent Model的商业化实践：
- 基于旗舰推理模型o3进行端到端强化微调
- 主要行动是网络搜索
- 环境是公开可获取的网络信息

本论文的开源实现（AutoCoA）为学术界和开源社区提供了探索这一方向的技术路径。

## 七、个人理解与批判性思考

### 7.1 核心贡献评价

这篇论文的核心贡献在于**概念化和系统化**了Agent Model这一新范式：

1. **概念贡献**：明确提出LAM（Large Agent Model）的概念，将Agent能力从"外部编排"提升为"内在能力"

2. **技术贡献**：AutoCoA框架提供了从SFT到RL的完整训练路径，具有较强的可操作性

3. **理论贡献**：将Agent Model形式化为POMDP，为后续研究提供了数学基础

### 7.2 局限性分析

1. **实验规模**：当前实验仅使用7B模型和单一工具（搜索），验证不够充分

2. **比较基线**：与ReAct的比较可能不够全面，缺少与其他先进Agent框架的比较

3. **任务范围**：主要聚焦开放域QA，在其他任务类型上的有效性待验证

### 7.3 未来研究方向

1. **多工具协同**：研究Agent如何自主选择和组合多种工具

2. **长期记忆**：将记忆机制整合到Agent Model中，支持跨会话的持久化知识

3. **多Agent协作**：多个Agent Model之间的协作与竞争

4. **物理世界交互**：将Agent Model扩展到机器人控制、自动驾驶等物理环境

### 7.4 工程实践建议

对于希望应用这一技术的团队：

1. **数据准备是关键**：高质量的工具使用轨迹数据是训练成功的基础

2. **渐进式部署**：先在模拟环境充分验证，再逐步引入真实环境

3. **监控与干预**：初期部署需要完善的监控机制，允许人类干预

4. **成本-性能权衡**：内部世界模型的复杂度需要在训练成本和推理质量之间权衡

## 八、总结

《Agent Models: Internalizing Chain-of-Action Generation into Reasoning Models》这篇论文代表了AI Agent领域的重要范式转变。它提出的**Large Agent Models (LAMs)** 概念，将工具使用能力从外部工作流提升为模型的内在能力，标志着AI从"被动响应"向"主动行动"的跨越。

**核心创新**：
- 提出了Agent Model的数学定义和形式化框架
- 设计了AutoCoA三阶段SFT+两阶段RL的训练框架
- 引入内部世界模型降低真实环境训练成本
- 实现了推理（CoT）与行动（CoA）的无缝切换

**行业意义**：
- 为OpenAI Deep Research等技术提供了学术解释和开源实现路径
- 预示了Agent架构从"工作流驱动"向"模型驱动"的演进
- 展示了大模型训练从"提示工程"向"能力学习"的趋势

**展望未来**：
Agent Model的成熟将带来AI应用的新浪潮。从企业自动化到个人助手，从信息检索到物理世界交互，具备自主行动能力的AI Agent将成为数字经济的新基础设施。而这篇论文，正是这一变革的重要理论基石。
