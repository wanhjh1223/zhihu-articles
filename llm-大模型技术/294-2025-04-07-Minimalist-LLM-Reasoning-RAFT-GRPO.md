## 一、研究背景与核心问题

### 1.1 LLM推理能力的崛起与强化学习的复兴

2024年至2025年，大语言模型（LLM）领域发生了革命性的变化。以OpenAI的o1/o3系列和DeepSeek的R1为代表的大推理模型（Large Reasoning Models, LRMs）展现出了惊人的复杂推理能力。这些模型能够在数学证明、代码生成、科学推理等任务上达到甚至超越人类专家水平，标志着人工智能从"模式匹配"向"深度思考"的关键跨越。

这一突破的核心驱动力是**强化学习（Reinforcement Learning, RL）**的重新应用。与传统的监督微调（SFT）不同，RL允许模型通过试错学习来优化其行为策略，特别适用于那些难以通过标准答案直接监督的开放式推理任务。

在RL for LLM的众多算法中，**Group Relative Policy Optimization (GRPO)** 脱颖而出，成为训练DeepSeek-R1等顶级推理模型的关键技术。GRPO由DeepSeek团队提出，以其简洁的设计和卓越的性能赢得了广泛关注。它摒弃了传统PPO算法中的价值网络（Value Network），改用组内相对奖励来估计优势函数，大大简化了训练流程。

### 1.2 GRPO的成功之谜：知其然不知其所以然

尽管GRPO在实践中表现出色，但其成功的真正原因一直是个谜。研究社区对其核心机制的理解存在诸多误区：

**普遍观点1**：GRPO的组内奖励归一化（Reward Normalization）是关键创新，能够稳定训练过程。
**普遍观点2**：必须使用正负样本（正确和错误答案）的对比学习才能有效训练推理模型。
**普遍观点3**：复杂的RL算法（如PPO、GRPO）显著优于简单的基线方法。

然而，这些观点缺乏系统的实证验证。GRPO的论文虽然提供了实验结果，但对算法组件的消融分析并不充分。这导致研究者在尝试改进或应用GRPO时，往往是在黑暗中摸索。

### 1.3 一个大胆的猜想：也许我们过度复杂化了

本文的作者们提出了一个大胆的假设：也许GRPO的成功并不来自于其复杂的设计，而是来自于一些更简单的机制。如果这一假设成立，那么我们可以通过更简单的算法达到类似甚至更好的效果。

这一猜想的动机来自于机器学习中反复出现的一个现象：**Occam's Razor（奥卡姆剃刀）原则**——在能够解释观测数据的前提下，更简单的模型往往更优。在LLM RL领域，这意味着我们应该认真审视：那些复杂的RL算法是否真的比简单的基线方法更好？

为了回答这个问题，作者们进行了系统性的实验分析，最终带来了这篇具有里程碑意义的论文。

## 二、核心发现：化繁为简的威力

### 2.1 RAFT：简单到令人惊讶的强基线

作者们的第一个核心发现是：**RAFT（Rejection sampling Fine-Tuning）**——一个仅使用正样本（positively rewarded samples）进行训练的简单方法——能够达到与GRPO和PPO相竞争的性能。

**RAFT算法定义**：

```
算法: RAFT (Rejection sampling Fine-Tuning)
────────────────────────────────────────────
输入: 基础模型 π_base, 训练数据集 D, 奖励函数 r
输出: 训练后的模型 π_θ

1. 对于每个查询 x ∈ D:
   a. 使用 π_base 采样多个响应 {y_1, y_2, ..., y_G}
   b. 计算每个响应的奖励 r(x, y_i)
   c. 仅保留奖励为正的响应: Y_pos = {y_i | r(x, y_i) > 0}
   
2. 在过滤后的数据集 {(x, y) | y ∈ Y_pos} 上执行标准SFT训练
   
3. 可选: 使用训练后的模型作为新的基础模型，迭代重复步骤1-2
```

RAFT的核心思想极其简单：
- **只学习"好的"**：只使用能够获得正奖励的样本进行训练
- **拒绝"坏的"**：完全丢弃负奖励样本，不参与梯度更新
- **多次采样**：通过多次采样增加获得正样本的概率

令人惊讶的是，这个看似"过于简单"的方法，在多个标准基准测试上达到了与GRPO相当的性能。

### 2.2 关键洞察：GRPO的优势从何而来？

作者们进一步通过消融实验探究了GRPO各个组件的实际贡献。他们发现了一个**反直觉的结论**：

> **GRPO的主要优势并非来自其奖励归一化机制，而是来自对"全错轨迹"（prompts with entirely incorrect responses）的丢弃。**

具体来说，GRPO的组内采样过程会产生以下效果：

1. **对于某些查询，所有采样响应都是错误的**
2. **GRPO通过组内相对奖励计算，实际上丢弃了这些查询（因为无法获得正相对优势）**
3. **这种隐式的"过滤机制"避免了模型在这些困难样本上浪费训练资源**

为了验证这一假设，作者们对比了以下变体：

| 方法 | 奖励归一化 | 丢弃全错查询 | 使用负样本 | 性能 |
|------|-----------|-------------|-----------|------|
| GRPO | ✓ | ✓（隐式） | ✓ | 基准 |
| GRPO w/o normalization | ✗ | ✓（隐式） | ✓ | 相当 |
| RAFT | N/A | ✓（显式） | ✗ | 相当 |
| SFT on all samples | N/A | ✗ | ✓ | 较差 |

实验结果表明：
- **移除奖励归一化对GRPO性能影响很小**，这挑战了奖励归一化是GRPO关键创新的传统观点
- **RAFT通过显式过滤全错查询**，达到了与GRPO相当的性能
- **而标准的SFT（不过滤负样本）性能明显较差**

### 2.3 Reinforce-Rej：保留精华，去除糟粕

基于上述洞察，作者们提出了**Reinforce-Rej**算法——一种极简的Reinforce变体，明确地对样本进行选择性过滤。

**Reinforce-Rej的核心思想**：

传统的REINFORCE算法使用所有样本进行梯度更新：

$$\nabla_\theta J = \mathbb{E}_{(x,y) \sim \pi_\theta} \left[ \nabla_\theta \log \pi_\theta(y|x) \cdot r(x,y) \right]$$

Reinforce-Rej在此基础上增加了**双重过滤机制**：

$$\nabla_\theta J_{\text{Rej}} = \mathbb{E}_{(x,y) \sim \pi_\theta} \left[ \nabla_\theta \log \pi_\theta(y|x) \cdot r(x,y) \cdot \mathbb{1}_{[\text{filter}(x,y)]} \right]$$

其中过滤函数定义为：

$$\text{filter}(x,y) = \begin{cases} 
\text{True} & \text{if } 0 < r(x,y) < r_{\text{max}} \\
\text{False} & \text{otherwise}
\end{cases}$$

**为什么要过滤"完全正确"的样本？**

这是一个精妙的洞察。作者们发现：
1. **完全正确的样本**（r = r_max）往往已经是最优响应
2. 在这些样本上进行梯度更新可能导致**过拟合**
3. 模型应该学会**区分不同程度的正确性**，而不是仅仅复制完美答案

**Reinforce-Rej的伪代码**：

```python
def reinforce_rej_step(model, queries, num_samples, reward_fn, r_max):
    """
    执行一步Reinforce-Rej更新
    """
    total_loss = 0
    valid_samples = 0
    
    for x in queries:
        # 采样多个响应
        responses = model.sample(x, num_samples=num_samples)
        
        # 计算奖励
        rewards = [reward_fn(x, y) for y in responses]
        
        # 过滤：只保留部分正确的样本
        filtered = [(y, r) for y, r in zip(responses, rewards) 
                    if 0 < r < r_max]
        
        if not filtered:
            continue  # 跳过全对或全错的查询
            
        # 计算策略梯度
        for y, r in filtered:
            log_prob = model.log_prob(y, x)
            loss = -log_prob * r  # 最大化期望奖励
            total_loss += loss
            valid_samples += 1
    
    if valid_samples > 0:
        loss = total_loss / valid_samples
        loss.backward()
        optimizer.step()
```

### 2.4 理论分析：为什么简单方法有效？

作者们从理论上分析了RAFT和Reinforce-Rej的有效性。

**关于负样本的价值**：

传统观点认为，负样本提供了"什么是错的"这一重要信息，有助于模型学习。但作者们的分析表明：

1. **在二元奖励场景**（正确/错误），负样本只告诉模型"这是错的"，但没有告诉它"错在哪里"或"如何改正"
2. **在稀疏奖励场景**，负样本可能占据绝大多数，导致训练信号被稀释
3. **正样本本身就包含了"如何正确推理"的信息**，通过多样化采样可以获得丰富的正样本

**关于KL散度效率**：

一个重要的理论指标是**KL效率**（KL efficiency）——单位KL散度变化带来的奖励提升。

$$\text{KL Efficiency} = \frac{\Delta \mathbb{E}[r]}{\text{KL}(\pi_\theta || \pi_{\text{ref}})}$$

作者们证明了，Reinforce-Rej通过选择性过滤，能够实现更高的KL效率：
- **避免在全错查询上浪费KL预算**：这些查询很难通过当前策略改进
- **避免在完美样本上浪费KL预算**：这些查询已经最优，不需要改变
- **专注于"可改进"的样本**：部分正确的样本提供了最有效的学习信号

## 三、实验验证：数据说话

### 3.1 实验设置

**数据集**：
- **GSM8K**：小学数学文字题，测试基础数学推理能力
- **MATH**：高中竞赛级数学题，测试高级数学推理
- **HumanEval**：代码生成基准，测试算法设计与实现能力
- **MBPP**：基于描述的Python编程任务

**模型**：
- **Qwen2.5-Math-7B**：专注于数学推理的中等规模模型
- **LLaMA-3.1-8B-Instruct**：通用指令微调模型
- **DeepSeek-R1-Distill-Qwen-7B**：蒸馏版推理模型

**对比方法**：
1. **SFT**：标准监督微调，使用所有样本
2. **RAFT**：仅使用正样本的拒绝采样微调
3. **PPO**：近端策略优化（使用价值网络）
4. **GRPO**：组相对策略优化（本文主要对比目标）
5. **Reinforce-Rej**：本文提出的极简方法

**评估指标**：
- **准确率（Accuracy）**：最终答案的正确率
- **KL散度**：与参考模型的偏离程度
- **训练稳定性**：奖励曲线和KL散度的波动情况
- **计算效率**：达到目标性能所需的训练步数和时间

### 3.3 消融实验

#### 3.3.1 负样本到底有没有用？

作者们系统地测试了负样本在不同设置下的作用：

| 配置 | GSM8K | 说明 |
|------|-------|------|
| 仅正样本（RAFT） | 82.8 | 基准 |
| 正样本 + 少量负样本（1:1比例） | 81.5 (-1.3) | 添加负样本反而降低性能 |
| 正样本 + 大量负样本（1:10比例） | 78.2 (-4.6) | 负样本过多显著损害性能 |
| 仅负样本 | 65.1 (-17.7) | 几乎无法学习 |

**结论**：在这个任务设置中，**负样本不仅无益，反而有害**。

作者们分析了可能的原因：
1. **负样本的信号过于稀疏**：只能告诉模型"这是错的"，没有提供改进方向
2. **负样本可能引入噪声**：某些"错误"答案可能包含部分正确的推理步骤
3. **正样本已经包含足够信息**：通过多样化采样，可以获得涵盖各种解题策略的正样本

#### 3.3.2 奖励归一化的真实作用

为了验证奖励归一化是否真的重要，作者们进行了以下对比：

| 变体 | 描述 | GSM8K |
|------|------|-------|
| GRPO (完整) | 使用组内奖励归一化 | 83.5 |
| GRPO w/o normalization | 移除奖励归一化 | 83.2 (-0.3) |
| GRPO + 固定丢弃阈值 | 显式丢弃全错查询 | 83.1 (-0.4) |
| RAFT | 仅使用正样本 | 82.8 (-0.7) |

**结论**：
- **移除奖励归一化对性能影响很小**（仅-0.3）
- **显式丢弃全错查询可以复现大部分GRPO的效果**
- **这表明GRPO的优势主要来自查询过滤，而非奖励归一化**

#### 3.3.3 采样数量的影响

RAFT和GRPO都依赖每组多次采样。作者们测试了不同采样数量对性能的影响：

| 每组采样数 | RAFT | GRPO | 正样本率（RAFT） |
|-----------|------|------|-----------------|
| 4 | 78.5 | 79.2 | 12% |
| 8 | 81.2 | 81.8 | 18% |
| 16 | 82.8 | 83.5 | 28% |
| 32 | 83.9 | 84.1 | 42% |
| 64 | 84.3 | 84.4 | 58% |

**结论**：
- **增加采样数能持续提升性能**，但边际收益递减
- **RAFT对采样数的敏感度与GRPO相似**
- **即使采样数较少（如8），RAFT仍能达到不错的性能**

### 3.4 跨模型和跨任务验证

为了确保结论的普适性，作者们在多个模型和任务上进行了验证：

| 模型 | 任务 | RAFT | GRPO | 差距 |
|------|------|------|------|------|
| Qwen2.5-7B | GSM8K | 82.8 | 83.5 | -0.7 |
| LLaMA-3.1-8B | GSM8K | 79.5 | 80.1 | -0.6 |
| DeepSeek-R1-Distill | MATH | 61.2 | 61.8 | -0.6 |
| Qwen2.5-Coder | HumanEval | 52.1 | 53.2 | -1.1 |

**结论**：RAFT和Reinforce-Rej的优势在不同模型和任务上保持一致，证明了结论的稳健性。

## 四、技术创新点深度解析

### 4.1 范式转变：从"对比学习"到"正例学习"

传统RL for LLM的思路深受对比学习范式影响：
- **PPO**：通过优势函数隐式对比正负样本
- **DPO**：显式对比正样本和负样本的似然比
- **GRPO**：通过组内相对奖励实现对比

本文的核心贡献在于挑战了这一范式，提出了**"正例学习"（Positive-Only Learning）**的新视角：

**核心论点**：在正确的任务设置下，正样本本身就包含了足够的学习信号，无需依赖负样本的对比。

这一范式转变具有深远意义：

1. **简化训练流程**：无需精心构造正负样本对
2. **降低训练不稳定风险**：避免了负样本可能引入的噪声
3. **提升训练效率**：无需在负样本上浪费计算资源
4. **更好的可解释性**：模型明确学习"如何做"，而非"如何避免错"

### 4.2 对GRPO的重新解构

本文的另一大贡献是对GRPO算法的重新解构。作者们指出，研究社区对GRPO的理解存在误区：

**传统理解**：
- GRPO ≈ PPO - Value Network + 组内奖励归一化
- 奖励归一化是GRPO的核心创新
- 组内对比机制至关重要

**本文的新理解**：
- GRPO ≈ RAFT + 隐式查询过滤 + 可选的负样本利用
- **查询过滤**（丢弃全错查询）才是GRPO真正的贡献
- 奖励归一化只是实现查询过滤的一种方式

这一重新解构的意义在于：
1. **指明了改进方向**：未来应专注于更智能的查询过滤策略
2. **简化了实现**：可以用RAFT + 显式过滤实现GRPO的核心效果
3. **深化了理解**：揭示了GRPO效果背后的真正机制

### 4.3 Reinforce-Rej的设计哲学

Reinforce-Rej的设计体现了**"Goldilocks原则"**——既不太热也不太冷，恰到好处：

| 样本类型 | 处理方式 | 理由 |
|---------|---------|------|
| 全错样本（r = 0） | 丢弃 | 学习信号太弱，浪费训练资源 |
| 完美样本（r = r_max） | 丢弃 | 已经最优，无需更新 |
| 部分正确（0 < r < r_max） | 保留并训练 | 最有效的学习信号 |

这种选择性过滤的策略来源于对RL训练动态的深刻理解：

**为什么丢弃全错样本？**
- 这些样本超出了当前策略的能力范围
- 强行训练可能导致策略崩溃
- 更好的做法是等策略改进后再尝试

**为什么丢弃完美样本？**
- 这些样本已经达到了最优
- 继续训练可能导致过拟合
- 浪费本可用于改进其他样本的计算资源

**为什么保留部分正确的样本？**
- 这些样本处于"最近发展区"（Zone of Proximal Development）
- 模型能够通过学习这些样本获得最大进步
- 梯度信号既不过弱也不过强

### 4.4 理论与实践的结合

本文不仅提供了实验验证，还从理论上分析了所提方法的有效性：

**理论分析1：正样本的充分性**

在某些条件下，正样本分布本身就包含了最优策略的信息。形式化地说，如果奖励函数满足某些平滑性条件，那么：

$$\pi^*(y|x) \propto \lim_{n \to \infty} \frac{1}{n} \sum_{i=1}^n \mathbb{1}_{[r(x, y_i) > 0]} \cdot \pi_{\text{base}}(y_i|x)$$

这意味着，通过足够多的正样本采样，我们可以逼近最优策略。

**理论分析2：KL效率的优化**

Reinforce-Rej通过选择性过滤，实际上解决了一个带约束的优化问题：

$$\max_{\pi} \mathbb{E}_{x \sim D, y \sim \pi(\cdot|x)} [r(x,y)] \quad \text{s.t.} \quad \text{KL}(\pi || \pi_{\text{ref}}) \leq \epsilon$$

通过专注于"边际"样本（部分正确的样本），Reinforce-Rej能够以最小的KL散度变化获得最大的奖励提升。

## 五、行业影响与未来展望

### 5.1 对LLM推理研究的影响

这篇论文的发表对LLM推理研究领域产生了深远影响：

**1. 研究范式的简化**

本文证明了简单方法可以达到复杂方法的效果，这将鼓励研究者：
- 更认真地对待简单基线，避免"复杂化"倾向
- 在提出新方法时，首先与简单基线进行充分对比
- 更深入地分析复杂方法中各组件的真实贡献

**2. 对GRPO社区的启示**

本文揭示了GRPO成功的真正原因，为未来改进提供了方向：
- **智能查询选择**：开发更 sophisticated 的查询过滤策略
- **自适应采样**：根据查询难度动态调整采样数量
- **课程学习**：从简单查询逐步过渡到困难查询

**3. 方法论层面的贡献**

本文展示了一套系统性的分析方法：
- **组件消融**：逐一移除算法组件，评估其真实贡献
- **基线对比**：与尽可能简单的基线进行对比
- **理论分析**：结合实验结果进行理论解释

这套方法论可以被应用到其他算法分析中。

### 5.2 实际应用价值

本文的方法在实际部署中具有重要价值：

**1. 降低训练成本**

RAFT和Reinforce-Rej相比GRPO具有更低的计算成本：
- **内存需求降低**：无需维护价值网络或多个策略副本
- **训练速度提升**：1.3-1.8倍的加速
- **调试成本降低**：更稳定的训练过程，减少超参数调优需求

**2. 便于小规模团队采用**

复杂方法（如PPO、GRPO）需要大量的工程经验才能正确实现。简单方法：
- **实现更简单**：几百行代码即可完成
- **超参数更鲁棒**：对超参数变化不敏感
- **更容易调试**：训练过程更可预测

**3. 支持快速迭代**

在研究和产品开发中，快速迭代至关重要。简单方法：
- **单次训练时间更短**，支持更多实验
- **失败成本更低**，鼓励大胆尝试
- **组合更灵活**，易于与其他技术结合

### 5.3 局限性与改进方向

尽管本文取得了重要进展，但仍存在一些局限性：

**1. 任务局限性**

本文主要在数学推理和代码生成任务上验证。在这些任务中：
- 奖励通常是二元的（正确/错误）或易于定义的
- 正样本相对容易识别

但在更开放的任务（如创意写作、开放式对话）中：
- 奖励定义本身就很困难
- "正样本"的界定可能很主观
- 简单方法是否仍然有效需要验证

**2. 负样本的潜在价值**

本文发现在测试的任务中负样本无益甚至有害。但这不意味着负样本在所有场景都无用：
- 在**细粒度奖励**场景（如过程奖励模型），负样本可能提供有价值的信号
- 在**安全对齐**训练（如RLHF），负样本对于定义"不应该做什么"至关重要
- 在**多任务学习**，负样本可能帮助区分不同任务

**3. 与SFT的关系**

RAFT本质上是一种特殊的SFT。这引发了一个问题：
- SFT和RL的边界在哪里？
- 在哪些场景下真正的RL（探索-利用权衡）是必要的？
- 能否通过更 sophisticated 的采样策略完全替代RL？

**4. 扩展性挑战**

虽然简单方法在小到中等规模模型上表现良好，但在超大规模模型（如GPT-4级别）上的表现如何？
- 超大规模模型可能有不同的训练动态
- 简单方法的效率优势是否仍然存在？
- 需要更大规模的实验来验证

### 5.4 未来研究方向

基于本文的发现，可以探索以下研究方向：

**1. 自适应过滤策略**

开发更智能的样本过滤策略：
- 根据当前训练阶段动态调整过滤阈值
- 为不同难度查询设置不同的过滤标准
- 结合不确定性估计进行选择性过滤

**2. 混合学习范式**

探索正样本学习与负样本学习的最佳结合方式：
- 在训练不同阶段采用不同策略（如初期只用正样本，后期引入负样本）
- 对不同查询类型采用不同策略（如简单查询用正样本，复杂查询用对比学习）
- 开发能够自动判断何时使用负样本的机制

**3. 过程级优化**

将本文的思想扩展到过程级奖励：
- 如何定义"部分正确"的推理步骤？
- 如何在每一步进行选择性学习？
- 结合过程奖励模型的极简训练方法

**4. 跨任务迁移**

研究RAFT/Reinforce-Rej训练的模型的跨任务迁移能力：
- 在数学任务上训练的模型能否迁移到代码任务？
- 如何设计任务间的课程学习策略？
- 开发通用的推理能力表示

**5. 理论深化**

进一步深化对正样本学习的理论理解：
- 在什么样的数学条件下正样本学习是最优的？
- 如何量化负样本的潜在价值？
- 建立统一的理论框架来解释不同方法的关系

## 六、总结与评价

### 6.1 核心贡献总结

本文作为2025年LLM推理领域的重要工作，做出了以下核心贡献：

**1. 提出了RAFT这一简单但强大的基线**

证明了仅使用正样本进行拒绝采样微调，就能达到与GRPO等复杂方法相当的性能。这一发现挑战了"必须使用正负样本对比"的传统认知。

**2. 重新解构了GRPO的成功机制**

通过系统性的消融实验，揭示了GRPO的优势主要来自隐式的"全错查询过滤"，而非奖励归一化。这一洞察为未来改进提供了正确方向。

**3. 设计了Reinforce-Rej算法**

一种极简的RL方法，通过双重过滤（丢弃全错和全对样本）实现了更高的KL效率和训练稳定性。

**4. 提供了系统性的分析方法**

展示了如何通过组件消融、基线对比和理论分析来深入理解算法机制。

**5. 推动了研究范式的简化**

证明在正确的问题设置下，简单方法可以匹敌复杂方法，鼓励社区回归"奥卡姆剃刀"原则。

### 6.2 个人理解与思考

作为一名AI技术研究者，我对这篇论文有以下几点深刻感受：

**（1）"简单"的力量**

这篇论文最大的启示是：在机器学习中，简单的方法往往被低估。研究者和工程师常常倾向于设计复杂的系统，认为"复杂等于强大"。但这篇论文提醒我们，应该首先认真评估简单方法的潜力。

GRPO作为一个"黑盒"算法，其内部工作机制一直不甚清晰。这篇论文通过系统性的消融分析，像解剖刀一样精准地定位了其真正有效的组件。这种"解构"的方法论值得学习。

**（4）理论与实践的结合**

本文不仅提供了丰富的实验证据，还给出了理论分析来解释观察到的现象。这种理论与实践结合的研究方式，使结论更加可信和有深度。
