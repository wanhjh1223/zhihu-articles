## 摘要

本文提出了一种突破性的LLM Agent训练范式——**SKILL0**，这是首个将技能内化（Skill Internalization）作为显式训练目标的强化学习框架。与当前主流的推理时技能增强方法不同，SKILL0通过**上下文强化学习（In-Context Reinforcement Learning, ICRL）**机制，在训练阶段逐步减少对技能上下文的依赖，最终实现完全的零样本自主行为。在ALFWorld和Search-QA基准测试上的实验表明，SKILL0相比标准RL基线实现了显著提升（ALFWorld提升9.7%，Search-QA提升6.6%），同时每步上下文token成本控制在0.5k以下，相比技能增强方法降低了超过5倍的推理开销。

## 一、研究背景与核心问题

### 1.1 LLM Agent的技能增强范式

近年来，大型语言模型（LLMs）在复杂真实世界任务中展现出强大的决策能力。随着Claude Code、OpenClaw等Agent脚手架的兴起，**结构化Agent技能**已成为扩展Agent能力的标准机制。这些技能是程序性知识和可执行资源的结构化包，能够动态加载以应对专门化任务。

当前主流范式是**推理时技能增强（Inference-Time Skill Augmentation）**：
- 从技能库中检索相关技能
- 将技能内容以自然语言指导的形式注入模型上下文
- 依赖模型遵循技能描述执行任务

这一方法在实践中取得了成功，催生了日益完善的技能库生态系统、检索管道和进化机制。然而，这种成功掩盖了更根本的局限性。

### 1.2 推理时技能增强的三大瓶颈

**（1）检索噪声污染上下文**

技能检索过程不可避免地引入不相关或误导性的指导，这些噪声会污染Agent的上下文。在高风险的Agent应用场景中，即使少量的错误指导也可能导致灾难性的决策偏差。研究表明，检索噪声会显著降低Agent在复杂多步骤任务中的成功率。

**（2）Token开销限制可扩展性**

注入的技能内容会产生大量的token开销。随着多轮交互的进行，这些开销呈复合增长，严重限制了系统的可扩展性。在长上下文场景下，技能描述可能占据上下文窗口的很大一部分，挤压了实际任务推理的空间。

**（3）知识停留于表面而非内化**

最根本的问题是：**遵循技能描述的模型只是在执行技能，而非真正学习技能**。能力存在于上下文中，而非模型参数中。每次推理都需要重新加载外部知识，模型从未真正将技能转化为内在的能力。

### 1.3 技能内化的必要性

人类技能习得遵循一个熟悉的进程：从显式指导阶段过渡到内化阶段，在内化阶段，相同的行为从记忆中自主执行。推理时技能增强将Agent永久锚定在第一阶段。

强化学习提供了通往第二阶段的自然路径，驱动Agent将有效的策略固化为内在策略，而非从上下文中读取。然而，朴素的RL应用存在两难困境：
- **无技能上下文**：Agent缺乏学习复杂多步行为所需的结构化指导
- **全程技能上下文**：模型保持对外部知识的依赖，从未被要求内化

**关键问题**：如何设计一种训练机制，从技能开始，最终摆脱技能，系统地将能力从上下文转移到参数中？

## 二、SKILL0框架：技术方法详解

### 2.1 核心理念："Skills at training, zero at inference"

SKILL0的核心思想可概括为一句简洁的格言："训练时使用技能，推理时零依赖"。这是首个将技能内化作为显式训练目标的RL框架。

SKILL0通过**上下文强化学习（ICRL）**实现这一理念：
- 训练rollout期间提供技能作为上下文指导
- 推理时完全移除技能
- RL优化直接驱动从上下文依赖执行到自主行为的过渡

### 2.2 框架架构：三大核心组件

![SKILL0框架概览](https://kimi-web-img.moonshot.cn/prod-data/online-image/search-upload/f0e2ba2c0281cf0d6f976386a898a6bc.png)

#### 2.2.1 相关性驱动的技能分组（Relevance-Driven Skill Grouping）

技能被组织在一个层次化的技能库SkillBank中，包含两个层级：

**通用技能**：捕获适用于所有任务类型的通用战略原则，如探索策略和目标跟踪启发式。

**任务特定技能**：存储任务类别k的专门知识，包括领域特定的动作序列和前置条件。

技能按目录结构`skills/{task_name}/{skill_category}.md`组织，每个Markdown文件S_k存储共享相同任务和技能类别的相关技能组。

**离线分组策略**：

在训练前，研究团队基于相关性将验证集划分为N个子任务{T_k}：
- 相关性定义为验证子任务的领域和目标是否与S_k编码的技能类别对齐
- 每个S_k有一个专门的子任务T_k用于评估其效用
- 这种离线分组确保每个技能文件都有评估其有用性的专用子任务

#### 2.2.2 上下文强化学习（In-Context Reinforcement Learning, ICRL）

ICRL结合了技能提示的样本效率和归纳偏置与强化学习的探索能力。

**Agent循环设计**：

给定任务指令I，Agent生成动作序列{a_1, a_2, ..., a_T}。在第t步：
- Agent在结构化环境E中操作，接收文本观察o_t
- Agent从策略π_θ(a_t|I, h_t)采样动作，其中h_t表示历史{o_1, o_2, ..., o_t}
- 环境转换到新状态，返回下一观察o_{t+1} = E(o_t, a_t)

**视觉上下文渲染机制**：

为应对扩展到更多领域时的token成本挑战，SKILL0引入上下文渲染机制，将文本交互上下文（包括历史h_t和检索的技能S）映射到紧凑的RGB图像。

给定压缩比c_t，渲染图像通过视觉编码器Enc编码：

```
V_t = Enc(h_t, S; c_t)
```

其中V_t作为压缩的视觉上下文嵌入输入到策略中，在保留决策所需结构信息的同时显著降低token开销。

**关键创新**：压缩比c_t不是固定超参数，而是由策略在每个步骤自动生成：

```
(a_t, c_t) ~ π_θ(a_t, c_t | I, V_t)
```

**复合奖励函数**：

为激励高效的上下文压缩和技能内化，SKILL0引入复合奖励函数，联合优化任务成功和压缩效率：

```
r_t^comp = {
    ln(c_t), if I_succ(τ) = 1,
    0, otherwise
}
```

```
r̃_t = r_t + λ · r_t^comp
```

其中：
- c_t ∈ (0,1]是第t步的压缩比
- 对数形式反映更高压缩的边际收益递减
- r_t评估Agent是否成功完成任务
- λ ≥ 0控制任务性能和压缩效率之间的权衡

**训练目标**：

对于每个查询q ~ D，π_θ_old采样G条轨迹{τ_i}。训练目标为：

```
L_SKILL0(θ) = E_{τ_i~π_θ_old(q), q~D} [1/Σ|τ_i| · Σ_{i=1}^G Σ_{t=1}^{|τ_i|} clip(r_{i,t}(θ), A_i, ε)] - β · D_KL[π_θ || π_ref]
```

其中优势A_i通过对采样组内的总奖励{r̃(τ_i)}归一化计算。

#### 2.2.3 动态课程学习（Dynamic Curriculum Learning）

为避免上下文空间中的分布偏移，SKILL0设计了一个受控的退火过程。

**线性衰减的技能预算**：

在每个阶段s ∈ {1, ..., N_S}，技能预算M^(s)线性衰减：

```
|S^(s)| ≤ M^(s) = ⌈N · (N_S - s) / (N_S - 1)⌉
```

这约束了技能上下文逐步减少的程度：

```
M^(s) - M^(s+1) ≈ N / (N_S - 1)
```

通过限制活动技能集S^(s)的变化，严格限制渲染视觉上下文V_t^(s) = Enc(h_t, S^(s); c_t)的偏差，确保策略π_θ(a_t, c_t | I, V_t^(s))的分布偏移保持平滑稳定。

**有用性驱动的动态课程**：

训练分为N_S个渐进阶段，技能预算M递减，直到Agent在没有任何检索技能的情况下运行。

每个技能文件S_k的**有用性度量Δ_k**定义为：

```
Δ_k = Acc_k^{w/ skill} - Acc_k^{w/o skill}
```

评估在两种条件下的性能差异：提供S_k和不提供S_k。

**技能选择算法**：

1. **过滤**：仅保留Δ_k > 0的技能（仍有帮助的）
2. **排序**：按Δ_k降序排列
3. **选择**：取前M^(s)个技能文件

这种方法的优雅之处在于**自适应内部化**：只在当前策略不再从中受益时才移除技能，替代了僵化的预设时间表。

### 2.3 与现有方法的对比

| 方法类型 | 训练阶段 | 推理阶段 | 局限性 |
|---------|---------|---------|--------|
| Zero-Shot | 无技能 | 无技能 | 缺乏指导，复杂任务表现差 |
| Few-Shot | 少量示例 | 少量示例 | 样本效率低，难以覆盖复杂场景 |
| Skill Augmentation | 可能有技能 | 检索并注入技能 | 检索噪声、token开销、知识未内化 |
| **SKILL0 (ICRL)** | **渐进减少技能** | **零技能** | **推理高效、知识内化、自主执行** |

## 三、实验结果与性能分析

### 3.1 实验设置

**基准测试**：
- **ALFWorld**：基于ALFRED具身AI基准的文本游戏，包含3,827个任务实例，涵盖6类家庭活动
- **Search-QA**：包含单跳QA（NQ、TriviaQA、PopQA）和多跳QA（HotpotQA、2Wiki、MuSiQue、Bamboogle）

**基线方法**：
- 上下文技能提示方法（Few-Shot with text/OCR history）
- RL方法：GRPO、AgentOCR、EvolveR、SkillRL
- 记忆增强方法：ReAct、Reflexion、Mem0、ExpeL
- 搜索增强方法：Search-o1、Search-R1、ZeroSearch
- 闭源模型：GPT-4o、Gemini-2.5-Pro

**实现细节**：
- 基础模型：Qwen2.5-VL系列（3B和7B）
- 训练：最多180步，4×H800 GPU
- ALFWorld：每批16任务，8 rollouts，最大3072 tokens
- Search-QA：每批128任务，最大4096 tokens
- 课程阶段数：N_S = 3

### 3.2 主要性能结果

| 方法 | ALFWorld (%) | Search-QA (%) | Token/Step (k) |
|-----|-------------|--------------|----------------|
| Zero-Shot | 71.2 | 28.3 | 0.15 |
| Few-Shot | 75.8 | 32.1 | 0.67 |
| AgentOCR | 78.2 | 34.2 | 0.42 |
| SkillRL-3B† | 88.3 | 39.5 | 2.21 |
| **SKILL0-3B** | **87.9** | **40.8** | **0.38** |
| **SKILL0-7B** | **89.8** | **44.4** | **0.47** |

† 使用技能增强的方法

**关键发现**：

1. **相比标准RL基线的显著提升**：
   - ALFWorld：+9.7%（对比AgentOCR）
   - Search-QA：+6.6%（对比AgentOCR）

2. **与技能增强方法的竞争力**：
   - SKILL0-7B在ALFWorld达到89.8%，Search-QA达到44.4%
   - 性能接近或超过SkillRL等技能增强方法

3. **卓越的token效率**：
   - SKILL0-3B：ALFWorld仅0.38k tokens/step，Search-QA仅0.18k
   - 相比SkillRL的2.21k和0.87k，降低了超过5倍的推理开销

### 3.3 与更广泛方法的对比

**ALFWorld上的对比**：

| 方法 | 成功率 (%) |
|-----|-----------|
| ExpeL | 46.3 |
| Mem0 | 54.7 |
| MemRL | 21.4 |
| **SKILL0-7B** | **89.8** |

**Search-QA上的对比**：

| 方法 | 成功率 (%) |
|-----|-----------|
| Search-R1 | 38.5 |
| ZeroSearch | 39.1 |
| EvolveR | 43.1 |
| **SKILL0-7B** | **44.4** |

SKILL0显著超越了记忆增强学习方法和搜索增强方法，证明了技能内化范式的有效性。

### 3.4 消融研究与分析

**课程学习策略的影响**：

- 固定时间表（Fixed Schedule）：预设技能移除时间表
- 动态课程（Dynamic Curriculum）：基于有用性度量Δ_k自适应移除

实验表明，动态课程策略在保持学习稳定性的同时实现了更高效的技能内化。

**视觉压缩比的影响**：

通过让策略自动生成压缩比c_t，SKILL0在不同任务复杂度下实现了：
- 简单任务：更高的压缩比（更小的上下文）
- 复杂任务：较低的压缩比（保留更多细节）

这种自适应性显著优于固定压缩比的基线方法。

**技能数量对性能的影响**：

研究发现，性能提升并非与技能数量线性相关。当技能预算从N降到N/2时，性能下降有限，说明Agent已经内化了相当一部分技能知识。当预算进一步降到0时，经过充分训练的Agent仍能保持强大的零样本能力。

## 四、技术创新点深度解析

### 4.1 上下文强化学习（ICRL）的范式创新

ICRL代表了一种新的RL训练范式，其核心洞察是：

**训练-推理差异的主动利用**：

传统RL追求训练分布与推理分布的一致性。ICRL则主动引入差异——训练时有技能指导，推理时无技能依赖——将这种差异作为优化目标。

**从"使用"到"内化"的转变**：

传统技能增强范式将技能视为工具，Agent的任务是选择和使用正确的工具。ICRL将技能视为学习目标，Agent的任务是理解、吸收并转化为内在能力。

**渐进式复杂性增加**：

通过动态课程，Agent在训练过程中面临的挑战逐步升级：
- 早期：丰富的技能指导，学习基本行为模式
- 中期：有限的技能支持，练习知识整合
- 后期：零技能环境，验证内化效果

### 4.2 有用性驱动课程的理论优势

**与传统课程学习的区别**：

传统课程学习通常基于任务难度或样本复杂度设计。SKILL0的课程基于**知识有用性**，这是一种更精细的粒度。

**有用性度量Δ_k的信息价值**：

Δ_k不仅指示某个技能是否有帮助，还量化了帮助程度。这使得：
- 优先级排序：最有帮助的技能最后移除
- 冗余检测：Δ_k接近零的技能可能是冗余或已被内化
- 个性化适配：不同Agent可能对相同技能有不同的有用性评估

**避免负迁移**：

通过在有用性为负时立即移除技能，SKILL0避免了负迁移现象——即提供错误指导反而降低学习效率。

### 4.3 视觉上下文渲染的工程创新

**压缩-保真权衡的形式化**：

通过复合奖励函数，SKILL0将压缩-保真权衡形式化为可优化的目标：

```
ln(c_t) 当任务成功
0       当任务失败
```

这创建了强激励：成功压缩（低c_t）获得对数奖励，但前提是不能牺牲任务成功率。

**自适应压缩的策略学习**：

允许策略生成c_t意味着：
- Agent学习评估当前任务的信息需求
- 在关键决策点保留更多上下文
- 在常规操作中积极压缩

**与固定压缩策略的对比**：

固定压缩策略（如始终使用50%压缩）无法适应任务复杂度的变化，而SKILL0的自适应压缩实现了效率与性能的最佳平衡。

## 五、局限性与未来研究方向

### 5.1 当前局限性

**技能初始化的依赖**：

SKILL0的训练效果依赖于初始技能库的质量。如果技能库本身存在缺陷（如过时、错误或不完整），内化过程可能传播这些缺陷。

**课程设计的超参数敏感性**：

虽然动态课程比固定时间表更鲁棒，但阶段数N_S、验证间隔d等超参数仍需要针对特定任务调优。

**跨领域泛化的未验证性**：

当前实验集中在ALFWorld（具身AI）和Search-QA（检索增强）。在代码生成、GUI自动化、多模态推理等其他Agent场景中的泛化能力有待验证。

**长程技能依赖的挑战**：

对于需要超长序列（数百步以上）才能展现价值的技能，短期有用性度量Δ_k可能无法准确评估其长期价值。

### 5.2 未来研究方向

**自动技能库构建**：

将SKILL0与技能提取和进化机制结合，形成"发现-内化-进化"的闭环：
- Agent从经验中提取新技能
- 通过SKILL0内化高价值技能
- 移除低效技能，持续优化技能库

**多Agent协作的技能共享**：

探索多个Agent共享内化知识的机制。当多个Agent面对相似任务时，能否通过某种形式的"集体记忆"加速各自的内化过程？

**可解释的内化表示**：

研究内化技能在模型参数中的表示形式。通过可视化或探测技术，理解Agent如何将结构化技能知识编码为神经网络的权重。

**实时技能更新**：

当前SKILL0假设技能库在训练前固定。未来的方向是支持运行时技能更新——Agent在部署后遇到新技能时，能否通过增量学习将其内化，而无需完整重训练？

**多模态技能内化**：

扩展SKILL0以支持视觉技能、代码技能、工具使用技能等多种模态。不同模态的技能可能需要不同的压缩和内化策略。

## 六、个人理解与行业影响分析

### 6.1 对LLM Agent领域的范式意义

**从"外挂"到"内功"的转变**：

SKILL0代表了一种根本性的范式转变。当前Agent生态系统大量依赖外部知识库、检索系统和提示工程，这类似于为AI配备"外骨骼"。SKILL0追求的方向是"内功修炼"——将能力真正转化为模型的一部分。

**效率与性能的双赢**：

传统观点认为，要达到高性能必须付出高成本（计算资源、上下文长度、延迟）。SKILL0证明，通过更好的训练方法，可以实现**更高性能+更低成本**的组合，这是对资源效率 frontier 的重新划定。

**可解释性的潜在提升**：

当知识以外部技能形式存在时，系统行为是技能描述和模型推理的混合产物，难以分析。当技能被内化后，模型行为完全由参数决定，可能更容易通过可解释性技术分析。

### 6.2 对产业应用的启示

**降低部署成本**：

推理成本的5倍降低对大规模部署具有重大商业价值。在客服、内容生成、代码助手等高频场景中，token成本的节省可转化为显著的经济优势。

**提升响应速度**：

消除技能检索步骤和减少上下文长度直接转化为更低的延迟。对于实时交互应用（如语音助手、实时协作工具），这能显著改善用户体验。

**简化系统架构**：

SKILL0减少了对复杂检索管道、技能库管理系统和上下文优化器的依赖。对于希望部署Agent系统的中小企业，这降低了技术门槛和维护成本。

**增强数据隐私**：

当技能被内化后，推理时不再需要访问外部技能库。对于处理敏感数据的应用（医疗、金融），这减少了数据暴露面，提升了隐私安全性。

### 6.3 对研究社区的影响

**强化学习应用的新场景**：

SKILL0展示了RL在"知识内化"领域的独特价值，可能启发RL在迁移学习、持续学习、元学习等领域的应用探索。

**课程学习方法的创新**：

有用性驱动的动态课程为课程学习研究提供了新视角。这种方法可能适用于其他需要从复杂监督信号中学习的场景。

**视觉-语言模型的能力拓展**：

通过视觉上下文渲染机制，SKILL0展示了VLM在压缩和结构化信息处理方面的潜力，可能推动VLM在Agent系统中的更广泛应用。

### 6.4 中长期展望

**通用技能内化Agent**：

未来的方向是构建能够自主发现和内化任意技能的通用Agent。SKILL0是迈向这一目标的重要一步。

## 七、结论

SKILL0通过引入**上下文强化学习（ICRL）**和**动态课程学习**，实现了LLM Agent领域的重大突破。其核心贡献在于将技能内化作为显式训练目标，使Agent能够从依赖外部技能指导的训练起点，逐步过渡到完全自主的零样本推理。

实验结果强有力地证明了这一方法的有效性：
- **性能**：相比标准RL基线显著提升（ALFWorld +9.7%，Search-QA +6.6%）
- **效率**：推理token成本降低超过5倍（<0.5k tokens/step）
- **竞争力**：达到或超越技能增强方法的性能

从更宏观的视角看，SKILL0代表了LLM Agent发展的重要方向——从追求更大模型、更多参数的"规模竞赛"，转向追求更智能训练、更高效推理的"效率竞赛"。在资源受限的现实世界中，后者可能更具决定性意义。

"Skills at training, zero at inference"——这不仅是一句简洁的格言，更是LLM Agent走向实用化、商业化的关键路径。
