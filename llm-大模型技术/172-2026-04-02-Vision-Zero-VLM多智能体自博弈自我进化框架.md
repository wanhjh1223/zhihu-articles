> **论文标题**: Vision-Zero: Scalable VLM Self-Evolution via Multi-Agent Self-Play

## 一、研究背景与核心问题

### 1.1 视觉语言模型训练的瓶颈

近年来，视觉语言模型(Vision-Language Models, VLMs)和多模态大语言模型(Multimodal Large Language Models, MLLMs)取得了显著进展，在图像理解、视觉问答、文档解析等任务上展现出强大能力。然而，当前VLM训练范式面临两个根本性瓶颈：

**数据稀缺问题(Data Scarcity Problem)**：高质量多模态数据的标注成本极其昂贵。以几个代表性数据集为例：
- COCO Attributes: 20万个对象的标注成本高达$60,480
- Ego4D: 消耗超过25万小时的标注工时
- Visual Genome: 动员了3.3万名标注人员

这种高昂的成本严重限制了训练数据的规模和多样性，成为制约VLM性能提升的关键因素。

**知识天花板问题(Knowledge Ceiling)**：传统监督微调(SFT)和基于人类反馈的强化学习(RLHF)使模型能力被人类生成的监督信号所束缚，难以发现超越人类专家水平的策略。模型性能的上限被人为标注的质量和覆盖范围所限制。

### 1.2 自博弈(Self-Play)的启示

自博弈通过竞争性动态消除对人类监督的依赖，已在多个领域超越人类知识上限：
- **TD-Gammon**: 1995年通过自博弈掌握双陆棋
- **AlphaGo/AlphaZero**: 2016-2017年征服围棋，通过自我对弈不断提升
- **OpenAI Five**: 2019年实现复杂团队协作的Dota 2大师级水平

近年来，自博弈开始被引入LLM训练，如SPIRAL(通过Tic-Tac-Toe和Kuhn Poker等游戏增强推理能力)、Absolute Zero(提出者与求解者之间的自博弈)等，在数学和编程任务上取得了SOTA结果。然而，**将自博弈扩展到VLM领域仍然 largely unexplored**，尽管多模态数据的高昂成本使这种方法尤为迫切。

### 1.3 Vision-Zero的核心洞察

理想的自博弈环境应满足四个条件：
1. 智能体为获胜而习得的技能应与目标任务所需技能高度对齐
2. 技能增长应具有可扩展性，环境难度应随自博弈进程持续提升
3. 环境应具有足够的多样性和复杂性，以满足广泛的目标任务需求
4. 环境应无需外部数据或仅需少量低成本数据(如无标签数据)

现有视觉推理游戏无法同时满足以上所有标准。例如，数独满足条件2和4，但不满足1和3。Vision-Zero的创新在于将语言类社交推理游戏(特别是"谁是卧底"这类交替发言和投票的游戏)适配到视觉领域，设计了一种新颖的视觉推理游戏，同时满足上述四个要求。

## 二、Vision-Zero技术框架详解

### 2.1 整体架构概览

Vision-Zero是一个通用、可扩展、高性能的VLM后训练框架，包含三个核心组件：

```
┌─────────────────────────────────────────────────────────────────┐
│                    Vision-Zero Framework                         │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────────┐  ┌──────────────────┐  ┌─────────────────┐ │
│  │ Strategic Game  │  │ Label-Free &     │  │ Iterative-SPO   │ │
│  │ Environment     │  │ Domain-Agnostic  │  │ Training Algo   │ │
│  │ (谁是卧底游戏)   │  │ Data Input       │  │                 │ │
│  └────────┬────────┘  └────────┬─────────┘  └────────┬────────┘ │
│           │                    │                     │          │
│           ▼                    ▼                     ▼          │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │           Zero-Human-in-the-Loop Self-Improvement         │  │
│  └──────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 战略游戏环境设计

Vision-Zero的核心是一个基于"谁是卧底"(Who Is the Spy)的视觉推理游戏，包含以下关键设计：

#### 2.2.1 游戏角色与设定

- **平民(Civilians)**: 观察正常图像，需要通过描述图像内容来帮助同伴识别间谍，同时避免泄露过多信息给间谍
- **间谍(Spy)**: 观察空白图像，必须通过分析平民的线索来推断隐藏的视觉内容，生成与上下文一致的合理线索以避免被识别

这种**非对称信息设计**迫使模型在多个角色之间进行战略推理，同时处理多样化的视觉输入。

#### 2.2.2 游戏阶段

**线索阶段(Clue Stage)**:
- 每轮玩家被告知自己的角色(平民或间谍)
- 每位玩家基于自己的图像和已有线索历史，提供描述图像内容的口头线索
- 玩家按顺序发言，每位玩家的线索对后续玩家可见，但思考过程保持隐藏
- 经过多轮线索阶段后，游戏进入决策阶段

**决策阶段(Decision Stage)**:
- 平民被指示结合所有线索和自己的图像分析来识别间谍
- 间谍知道自己的身份，不参与投票
- 玩家若不确定可以回答"n/a"
- 推理过程和最终投票保持私密

#### 2.2.3 提示词工程

Vision-Zero使用精心设计的提示词来引导模型行为：

**线索阶段提示词关键要素**:
```
你是第{player_id}号玩家，这是第{round_num}轮。你的任务是用一句话描述你的图片。

你的角色: 间谍(你看到的是空白图片) / 非间谍(你的图片与其他大多数人相同)

游戏说明:
- 总共{num_players}名玩家，只有一名玩家看到不同的图片
- 被投票选为异常玩家的将受到严厉惩罚
- 间谍看到完全空白的图像，其他玩家看到正常的图表

你应该做什么:
1. 将你的图片与所有之前的线索进行比较
2. 如果你不是间谍: 描述图片的清晰、具体特征，不重复之前的线索
3. 如果你是间谍:
   (1) 从其他玩家的线索推断其他人看到的图表主题
   (2) 推断图表有哪些属性以及这些属性之间的数值关系
   (3) 描述一个关于其他人看到的图表的正确特征，但不与任何之前的线索相同，以迷惑其他玩家

输出格式:
首先进行你的私密逐步推理——可能包括怀疑、概率、欺骗等。
然后，将你的一句话图片描述放在\boxed{}中。
```

**决策阶段提示词关键要素**:
```
你正在分析一场"谁是卧底"游戏，{num_players}名玩家刚刚提供了关于他们图片的线索。
你的任务是基于他们的线索识别谁是间谍(拥有空白图片的人)。

寻找:
1. 与大多数不一致的线索
2. 故意模糊或笼统的线索
3. 可能试图误导他人的线索
4. 暗示欺骗或不确定性的模式

你看到的图像显示的是原始图片(非间谍玩家看到的)。将线索与这个参考进行比较。

输出格式:
首先进行你的私密逐步推理。
然后，将你的最终答案(玩家编号或N/A)放在\boxed{}中。如果不确定，可以回答N/A。
```

### 2.3 无标签领域无关数据输入

Vision-Zero的输入完全无标签但具有灵活性：每轮环境只需要任意一张图像，原始图像提供给平民，间谍接收空白视觉输入。

论文验证了三种数据类型的通用性：

#### 2.3.1 CLEVR合成数据

使用CLEVR渲染器自动生成2,000张图像：
- 每张图像包含4-6个随机排列的对象
- 对象属性(形状、颜色、大小、材质)通过自动化脚本随机生成
- 渲染过程在NVIDIA A100 GPU上耗时约6小时
- CLEVR场景包含完整的场景图信息，便于程序级监督

#### 2.3.2 图表数据

从ChartQA训练集中随机选择1,000张图像：
- 包含折线图、饼图和柱状图
- 通过GPT-4o解析图表内容为JSON格式
- 随机交换两个属性值生成修改后的图表
- 使用Python绘图模块渲染新图表
- 整个生成过程仅需数十美元成本

#### 2.3.3 真实世界数据

从ImgEdit训练集中随机采样1,000张图像：
- 使用ChatGPT和NanoBanana等图像编辑工具生成差异化图像对
- 即使20%的编辑失败，模型仍能保持性能提升

### 2.4 Iterative Self-Play Policy Optimization (Iterative-SPO)

这是Vision-Zero的核心算法创新，通过交替自博弈和RLVR实现持续性能提升。

#### 2.4.1 数学符号定义

- 每轮有n名玩家：n_c个平民和1个间谍，角色集合 K = {s} ∪ {c₁, ..., c_{n_c}}
- 间谍和平民分别持有图像 I_s 和 I_c
- 线索阶段，每位玩家基于线索历史h提供线索 u_k ~ π_θ^k(·|I_k, h)
- 投票机制返回投票计数 v = (v_s, v_{c₁}, ..., v_{c_{n_c}})

#### 2.4.2 线索阶段的自博弈策略优化

**零和奖励设计(Zero-Sum Reward)**:

间谍和平民构成对立双方，根据零和博弈原理设计奖励：

```
r_s^clue = -β(v_s - v̄_c)
r_{c_j}^clue = (β/n_c)(v_s - v̄_c) - λ(v_{c_j} - v̄_c)
```

其中：
- v̄_c = (1/n_c)∑v_{c_j} 表示所有平民平均得票数
- β > 0 控制间谍与平民之间的竞争强度
- λ > 0 调节平民之间行为不一致性的惩罚

这个设计确保间谍和平民之间的总奖励为零，得票越多的玩家获得越低的奖励。

**角色优势估计(Role-Advantage Estimation, RAE)**:

为缓解非对称角色信息导致的胜率不平衡，应用RAE：

```
b_s = α·b_s + (1-α)·r_s^clue
b_c = α·b_c + (1-α)·(1/n_c)∑r_{c_j}^clue
A_k^clue = r_k^clue - b_k
```

其中α是衰减率，优势值通过从原始奖励中减去RAE来消除信息非对称性。

**优化目标**：

```
L^clue(θ) = -E[(1/n)∑A_k^clue·log π_θ^k(u_k|I_k, h)] + τ_clue·E[(1/n)∑D_KL(π_θ^k || π_ref^k)]
```

KL项约束更新接近参考策略，稳定学习并防止退化性发言。

#### 2.4.3 决策阶段的RLVR

**离散奖励设计**：

```
r_{c_i}^dec = {
  +1   if ŝ_{c_i} = s*      (正确识别)
  -0.5 if ŝ_{c_i} = ∅       (不确定)
  -1   else                 (错误识别)
}
```

这种奖励设计鼓励玩家做出推理充分的判断，即使在极具挑战性的条件下，也激励承认不确定性而非坚持错误答案。

**组归一化与优化目标**：

为消除轮次特定难度，应用组归一化：

```
μ_r = mean[r_{c_i}^dec]
σ_r = std[r_{c_i}^dec]
A_{c_i}^dec = (r_{c_i}^dec - μ_r) / (σ_r + ε)
```

优化目标：

```
L^dec(θ) = -E[(1/n_c)∑A_{c_i}^dec·log q_θ(ŝ_{c_i}|H)] + τ_dec·E[(1/n_c)∑D_KL(q_θ(·|H) || q_ref(·|H))]
```

#### 2.4.4 迭代阶段训练机制

纯自博弈设置通常达到局部均衡，限制新推理路径的探索；而纯RLVR方法在掌握可用问题集后面临知识饱和。**Iterative-SPO通过两阶段交替训练解决这些问题**。

**阶段切换逻辑**：

定义决策阶段的平均预测准确率 acc_t 和"n/a"率 na_t：

```
acc_t = (1/B)∑1[arg max_y q_θ(y|H_i) = s_i*]
na_t = (1/B)∑q_θ(∅|H_i)
```

维护指数移动平均：

```
acc̄_t = ρ·acc̄_{t-1} + (1-ρ)·acc_t
nā_t = ρ·nā_{t-1} + (1-ρ)·na_t
```

**切换规则**：

```
Decision → Clue: if m_t=0 and acc̄_t ≥ τ_acc↑ and nā_t ≤ τ_na↓ and d ≥ K_min
Clue → Decision: if m_t=1 and (1-acc̄_t ≥ τ_err↑ or nā_t ≥ τ_na↑) and d ≥ K_min
```

典型阈值设置：
- τ_acc↑ = 0.9 (准确率上阈值)
- τ_err↑ = 0.4 (错误率上阈值)
- τ_na↑ = 0.5 (n/a率上阈值)
- τ_na↓ = 0.1 (n/a率下阈值)
- K_min = 5 (每阶段最小轮数)

这种交替机制提供两个主要优势：
1. 通过动态切换防止模型在战略均衡或知识高原停滞
2. 交替自博弈与RLVR引入监督信号，稳定训练并防止角色崩溃或发散

## 三、实验结果与性能分析

### 3.1 实验设置

**模型**：在Qwen2.5-VL-7B、InternVL3-8B和InternVL3-14B上进行评估

**评估基准**(14个任务，涵盖推理、图表分析和视觉中心领域)：
- 数学推理：MathVista, MathVision, WeMath, MathVerse, LogicVista, DynaMath
- 图表理解：ChartXiv, FunctionQA, PaperQA, ReachQA
- 视觉中心任务：RealWorldQA, MMVP, BLINK, MuirBench

**基线方法**：
- R1-OneVision-7B：基于Qwen2.5-VL的跨模态推理模型
- MM-Eureka-Qwen-7B：基于规则RL的多模态推理
- VLAA-Thinker-7B：SFT+RL两阶段训练
- OpenVLThinker-7B：迭代SFT→RL训练
- ViGaL(Snake+Rotation)：纯RL游戏训练

### 3.2 主要实验结果

#### 3.2.1 持续性能增长

Vision-Zero在训练过程中实现了持续的性能提升。在CLEVR数据上：
- Qwen2.5-VL-7B的胜率从50%提升至71%
- 平均token长度显著增加，特别是在决策阶段(InternVL3-8B和InternVL3-14B从250增长到约400个token)
- 这表明Iterative-SPO促进了推理能力的增强

#### 3.2.2 强大的任务泛化能力

| 模型 | MathVista | MathVision | WeMath | MathVerse | LogicVista | DynaMath | Avg |
|------|-----------|------------|--------|-----------|------------|----------|-----|
| Qwen2.5-VL-7B | 68.2 | 25.4 | 36.1 | 49.0 | 47.2 | 20.9 | 41.1 |
| VisionZero-Qwen-7B(CLEVR) | **72.2** | **28.4** | **39.2** | **53.2** | **49.8** | **22.9** | **44.3** |
| VisionZero-Qwen-7B(Chart) | **72.4** | **28.9** | **38.8** | **52.5** | **51.2** | **22.1** | **43.3** |
| VisionZero-Qwen-7B(Real-World) | **73.1** | **28.5** | **40.1** | **52.1** | **50.8** | **22.5** | **44.5** |

关键发现：
- Vision-Zero模型在各类基准上持续超越SOTA基线方法
- VisionZero-Qwen-7B(CLEVR)和VisionZero-Qwen-7B(Chart)相比基线模型提升约3%
- 相比之下，最先进的基线方法仅提升约1.9%
- **重要的是**：Vision-Zero环境没有显式包含任何数学特定任务训练，而是通过自然语言环境中的战略游戏逻辑推理能力，有效泛化到更广泛的数学和推理任务

#### 3.2.3 跨能力负迁移缓解

VLM后训练的一个关键挑战是跨能力负迁移——在特定任务上训练的模型在其他任务上往往表现更差。

**图表理解任务表现**：

| 模型 | ChartXiv_RQ | FunctionQA | PaperQA | ReachQA |
|------|-------------|------------|---------|---------|
| Qwen2.5-VL-7B | 42.5 | 82.3 | 68.4 | 50.8 |
| VisionZero-Qwen-7B(Chart) | **45.8** | **85.5** | **73.7** | **53.4** |

VisionZero-Qwen-7B(Chart)在四个图表数据集上平均准确率提升+3.9%。

**视觉中心任务表现**：

| 模型 | RealWorldQA | MMVP | BLINK | MuirBench |
|------|-------------|------|-------|-----------|
| Qwen2.5-VL-7B | 68.1 | 76.8 | 55.2 | 58.2 |
| VisionZero-Qwen-7B(CLEVR) | **68.4** | **79.5** | **56.0** | **58.2** |

VisionZero-Qwen-7B(CLEVR)将MMVP准确率从76.8%提升至79.5%。

这表明Vision-Zero的战略性多能力训练环境显著缓解了传统单一能力训练范式中常见的负迁移问题。

#### 3.2.4 模型泛化性

| 模型 | MathVista | MathVision | WeMath | MathVerse | LogicVista | DynaMath | Avg |
|------|-----------|------------|--------|-----------|------------|----------|-----|
| InternVL3-8B | 60.4 | 21.3 | 26.8 | 32.2 | 40.5 | 26.8 | 34.7 |
| MM-Eureka-InternVL-8B | 62.4 | 22.1 | 26.8 | 32.1 | 38.9 | 28.7 | 35.2 |
| VisionZero-InternVL3-8B | **62.2** | **24.2** | **28.7** | **32.9** | **41.8** | **29.2** | **36.5** |

尽管QwenVL和InternVL模型系列在视觉编码器、预训练策略和训练程序上存在差异，Vision-Zero仍能持续提升这些模型的性能，凸显其强大的泛化能力。

#### 3.2.5 Iterative-SPO的优越性

对比三种训练设置：
1. **纯线索阶段训练**：决策阶段冻结，仅通过自博弈训练线索阶段
2. **纯决策阶段训练**：线索阶段冻结，仅通过RLVR训练决策阶段
3. **Iterative-SPO**：交替训练

**结果**：Iterative-SPO显著优于两种单模式方法：
- 在LogicVista数据集上，相比纯自博弈提升2%准确率
- 相比纯RLVR训练提升1%准确率

纯自博弈缺乏直接可验证的奖励——奖励信号来自决策者，当决策质量不足以有效区分角色时，模型性能会过早达到平台期。交替训练缓解了这一限制，实现可持续的性能改进。

| **VisionZero-Qwen-7B** | **批量渲染** | **2k** | **0** | **Alternating Self-play+GRPO** | **✓** | **127 A100-Hours** |

Vision-Zero：
- **标签成本为0**：完全依赖无标签数据
- 仅需127 A100-Hours，显著低于其他GRPO方法
- 由于样本效率高，以显著更少的训练迭代实现更优的模型性能

#### 3.3.2 训练效率

Vision-Zero虽然涉及多轮交互，但并未引入大量训练开销：

1. 采用固定交互模式(两轮线索后跟一轮决策)，支持跨多个游戏完全并行化的前向和后向传播，无异步延迟或梯度冲突
2. 每个样本生成多个动作，提供比标准单轮RL设置更密集的学习信号和更高样本效率

实验对比(相同硬件设置)：
- Qwen2.5-VL-7B：Vision-Zero实现**3.3倍**整体训练效率提升
- InternVL3-8B：Vision-Zero实现**6.4倍**整体训练效率提升

## 四、个人理解与行业影响分析

### 4.1 核心创新点评析

#### 4.1.1 游戏化自博弈范式突破

Vision-Zero的最大创新在于将**社交推理游戏**与**视觉语言模型训练**相结合。"谁是卧底"这个游戏选择极具巧思：

1. **非对称信息设计**天然适合训练VLM的多视角理解能力
2. **竞争性动态**提供了明确的奖励信号，无需人工标注
3. **多轮交互**强制模型发展长程推理和策略规划能力
4. **角色扮演**使单一模型能够同时学习"描述"和"识别"两种互补技能

这与传统的对比学习或单任务RLVR形成鲜明对比。Vision-Zero创造的是一个**开放-ended的技能发展环境**，而非针对特定任务的优化。

#### 4.1.2 Iterative-SPO算法的战略价值

Iterative-SPO解决了自博弈训练中两个经典难题：

**均衡问题**：纯自博弈往往收敛到局部均衡，模型学习到的策略相互适应但可能并非最优。通过引入RLVR阶段，算法注入外部验证信号，打破均衡陷阱。

**探索vs利用**：纯RLVR在掌握现有问题集后面临知识饱和。通过回到自博弈阶段，模型被迫探索新的推理路径以应对不断进化的对手。

这种**动态难度调整机制**类似于AlphaZero的蒙特卡洛树搜索，但应用于语言生成领域，是一个极具前景的方向。

#### 4.1.3 零人类监督的规模化潜力

Vision-Zero的"零标注成本"特性具有战略意义：

1. **数据获取成本**：仅需任意图像，可使用互联网上近乎无限的未标注图像
2. **领域适应性**：通过简单修改图像生成方式即可适配特定领域(医学影像、卫星图像、工业检测等)
3. **持续学习**：模型可以无限期地自博弈，理论上可以持续改进而没有数据瓶颈

这为构建**真正自主进化的AI系统**迈出了重要一步。

### 4.2 局限性与未来方向

#### 4.2.1 当前局限

1. **图像编辑依赖**：Vision-Zero的实现依赖于图像编辑器生成差异化图像对。在高度专业化或资源受限的领域(如医学成像、科学图表、遥感)，这种编辑数据可能不易获得。

2. **单图像限制**：当前框架围绕单图像观察和成对编辑设计。扩展到更丰富的模态(包括扩展视频、复杂多图像上下文或交互式3D环境)可能需要对游戏机制和训练算法进行重大重新设计。

3. **奖励设计复杂度**：零和奖励和RAE机制虽然有效，但引入了多个超参数(β, λ, α等)，需要针对新领域进行调优。

#### 4.2.2 未来研究方向

1. **多模态扩展**：将Vision-Zero扩展到视频理解(时间维度推理)、3D场景理解(空间推理)、音频-视觉联合推理等领域。

2. **更复杂的社交游戏**：探索更复杂的社交推理游戏，如"狼人杀"(多角色、多阵营)、"阿瓦隆"(任务机制)等，可能进一步提升模型的策略推理和Theory of Mind能力。

3. **与真实世界对齐**：如何将Vision-Zero中习得的能力更好地对齐到实际应用任务，可能需要开发更有效的迁移学习机制。

4. **理论分析**：对Iterative-SPO的收敛性、最优性进行更深入的理论分析，为算法设计提供指导。

### 4.3 对行业的影响

#### 4.3.1 降低VLM训练门槛

Vision-Zero展示的"无标注训练"路径可能显著降低开发高性能VLM的成本：
- 初创企业和研究机构可以用更少资源训练有竞争力的模型
- 特定领域(医疗、法律、科学)可以更容易地构建领域专用VLM
- 可能催生更多开源VLM，加剧行业竞争

#### 4.3.2 自博弈成为标准工具

随着Vision-Zero和类似方法的发展，自博弈可能从"研究 curiosity"转变为VLM训练的标准组件：
- 类似于SFT和RLHF，自博弈阶段可能成为标准训练流程的一部分
- 可能出现专门的"游戏设计"研究方向，为不同能力训练设计最优游戏环境

#### 4.3.3 对AGI路径的启示

Vision-Zero代表了一种重要的AGI发展范式：**通过自我交互而非人类监督实现能力提升**。这与LeCun提出的"自监督学习是世界模型的基础"观点相呼应。

如果Vision-Zero的核心理念被证明可扩展到更复杂的认知能力，它可能成为构建真正自主学习和进化的AI系统的关键组件。

### 4.4 与其他前沿工作的联系

Vision-Zero与当前多个研究热点形成呼应：

1. **Test-Time Scaling**：Vision-Zero本质上是将test-time compute扩展到训练阶段，通过多轮交互提升推理能力。

2. **Agentic AI**："谁是卧底"可以看作是多智能体交互的简化版本，为更复杂的AI Agent协作奠定基础。

3. **Reasoning Models**：Vision-Zero为VLM提供了类似DeepSeek-R1的推理能力增强路径。

4. **Self-Improvement**：与OpenAI的" Constitutional AI"、DeepMind的"Self-Improving AI"等方向一致，探索AI系统的自主改进机制。

## 五、总结

Vision-Zero代表了视觉语言模型训练范式的重要突破。通过将"谁是卧底"这一经典社交推理游戏适配为VLM自博弈环境，配合创新的Iterative-SPO算法，Vision-Zero实现了：

1. **零人工标注**：完全依赖无标签数据，消除昂贵的人工标注成本
2. **持续性能提升**：通过交替自博弈和RLVR避免均衡陷阱和知识饱和
3. **强大泛化能力**：习得的推理能力广泛泛化到数学、图表、视觉理解等任务
4. **训练效率优势**：相比传统方法实现3-6倍的训练效率提升

这项工作不仅提供了一个实用的VLM训练框架，更重要的是展示了**游戏化自博弈作为AI能力发展通用范式**的巨大潜力。

在通往AGI的道路上，Vision-Zero提示了一个重要方向：或许真正的智能不是从人类标注中学来的，而是通过**自我对抗、自我验证、自我进化**的过程自然涌现的。正如AlphaZero在围棋中超越人类千年积累，Vision-Zero可能预示着VLM能力的下一次飞跃——**不是模仿人类，而是超越人类**。
