## 一、研究背景与核心问题

### 1.1 Test-Time Compute 时代的效率挑战

大规模语言模型（LLM）的发展正在经历一次范式转变。随着 Chain-of-Thought（CoT）推理、迭代优化等技术的普及，**Test-Time Compute（测试时计算）** 已经成为推动 LLM 性能提升的关键驱动力。Snell 等人（2024）的研究表明，通过增加推理时的计算量，模型可以解锁全新的能力。与此同时，并行化、Agentic 工作流的快速崛起进一步加剧了对高效推理和部署的需求。

这一范式转变使得**推理效率**变得至关重要。AI 系统的实际影响力现在严重依赖于其在部署期间执行大规模推理的能力。模型架构设计在决定推理效率方面扮演着根本性角色，因为架构选择直接决定了生成过程中的计算和内存需求。

### 1.2 Transformer 的瓶颈

当前，基于 Transformer 的模型是行业标准，但它们存在根本性瓶颈：

| 问题维度 | 具体表现 | 影响 |
|---------|---------|------|
| **内存需求** | KV Cache 随序列长度线性增长 | 长上下文场景内存爆炸 |
| **计算复杂度** | Self-Attention 的二次方计算复杂度 O(n²) | 长序列推理速度急剧下降 |
| **硬件效率** | 高算术强度需求 | 实际部署成本高昂 |

这些缺陷推动了**次二次方模型（Sub-Quadratic Models）**的研究，如状态空间模型（SSMs）和线性注意力机制。这类模型保持恒定内存和线性计算复杂度，同时达到与 Transformer 相当甚至更优的性能。

### 1.3 现有线性模型的局限

尽管 Mamba-2 和 Gated DeltaNet 等线性模型已经被整合到大规模混合架构中（如 Kimi K2.5、NVIDIA Nemotron-H、腾讯混元），但仍存在显著局限：

1. **质量与效率的权衡**：Mamba-2 为了提升训练速度和简化性，牺牲了一定的表达能力，导致推理匹配模型的性能下降
2. **状态追踪能力不足**：现有线性模型在简单状态追踪任务（如奇偶校验）上表现糟糕
3. **硬件效率不佳**：理论上高效的推理在实际中并未充分利用硬件能力，解码阶段算术强度低

### 1.4 Mamba-3 的研究目标

基于以上背景，Mamba-3 旨在从**推理优先（Inference-First）**的视角出发，通过状态空间模型（SSM）的理论框架，解决现有线性模型的三大核心问题：

1. **提升模型质量**：改进 Pareto 前沿，在不牺牲效率的前提下提升性能
2. **增强模型能力**：克服状态追踪等关键能力的缺失
3. **优化硬件效率**：提高解码阶段的硬件利用率

## 二、核心技术创新详解

Mamba-3 引入了**三大核心方法论改进**，每项创新都深深植根于状态空间模型的理论基础：

### 2.1 创新一：指数-梯形离散化（Exponential-Trapezoidal Discretization）

#### 2.1.1 理论基础：连续到离散的转换

状态空间模型天然定义为连续时间动态系统，通过常微分方程（ODE）描述：

```
ẋh(t) = A(t)h(t) + B(t)x(t)  （状态方程）
y(t) = C(t)ᵀh(t)              （输出方程）
```

在序列建模中，数据仅在离散时间点观测，因此需要**离散化**将连续动态转换为离散递推。

#### 2.1.2 现有方法的局限

Mamba-1 声称使用了零阶保持（ZOH）离散化，但实际实现采用了一种启发式近似，缺乏理论证明。Mamba-2 沿用了这一方法，但同样缺乏严格的数学基础。

#### 2.1.3 指数调整离散化框架

Mamba-3 提出了一个通用的**指数调整离散化框架**，核心思想来源于线性 ODE 的闭式解 x(t) = e^(tA)x(0)。研究团队分析了**指数调整系统** e^(-At)x(t) 的动态，推导出统一的离散化形式：

**命题 1（指数-梯形离散化）**：
使用广义梯形规则近似状态输入积分，得到递推式：

```
h_t = e^(Δt·At)·h_{t-1} + (1-λt)·Δt·e^(Δt·At)·B_{t-1}·x_{t-1} + λt·Δt·B_t·x_t

简记为：
h_t = α_t·h_{t-1} + β_t·B_{t-1}·x_{t-1} + γ_t·B_t·x_t
```

其中 λt ∈ [0,1] 是数据依赖的标量参数。

#### 2.1.4 统一现有方法

这一框架具有强大的统一性：

| 方法 | 参数设置 | 说明 |
|-----|---------|------|
| **经典梯形规则** | λt = 1/2 | 均匀平均端点 |
| **指数-Euler（Mamba-1/2）** | λt = 1 | 仅使用右端点 |
| **指数-梯形（Mamba-3）** | λt ∈ [0,1] 数据依赖 | 数据依赖的凸组合 |

**重要性质**：
- **表达能力**：指数-梯形规则是经典梯形规则和 Mamba-2 Euler 规则的推广
- **误差阶数**：二阶离散化，误差为 O(Δt³)（当 λt = 1/2 + O(Δt) 时）

#### 2.1.5 隐式卷积解释

指数-梯形离散化等价于对 SSM 状态输入应用**数据依赖的 2-宽度卷积**：

```
标准 SSM：v_t = B_t·x_t → h_t = α_t·h_{t-1} + γ_t·v_t
Mamba-3：  先对 v_t 应用卷积（权重 β, γ），再传入递推
```

这与 Mamba/GDN 使用的标准短卷积有本质区别：
- 标准短卷积：在核心递推外独立应用于 x_t（和 B_t, C_t）
- Mamba-3 卷积：在核心递推内应用于状态输入 B_t·x_t

#### 2.1.6 并行形式与 SSD 框架

在状态空间对偶（SSD）框架下，Mamba-3 对应一个更 expressive 的掩码 L：

```
L = [衰减矩阵] ⊙ [2-带状卷积矩阵]
```

这是一个 **1-半可分矩阵与 2-带状矩阵的组合**，相比 Mamba-2 的对角矩阵（仅含 γt）表达能力更强。

### 2.2 创新二：复数值状态空间模型（Complex-Valued SSMs）

#### 2.2.1 动机：状态追踪的局限

近期研究（Grazzi 等人 2025；Sarrof 等人 2024）表明，将状态转移矩阵限制为实数、非负特征值会严重削弱模型在简单状态追踪任务上的能力。具体而言：

**奇偶校验问题**：对于二进制输入 {0,1}，计算 ⊕_t x_t mod 2
- 理论解法：使用 2D 旋转矩阵 R(π·x_t) 更新状态
- 实数限制：无法表达这种"旋转"动态

**定理 1**（Grazzi 等人 2024）：将转移矩阵特征值限制为实数会阻碍模型解决奇偶校验等可解群问题（TC⁰ 复杂度）。

#### 2.2.2 复数 SSM 的引入

Mamba-3 引入**复数值状态空间模型**，恢复状态追踪能力：

```
复数 SSM：
ẋh(t) = Diag(A(t) + iθ(t))·h(t) + (B(t) + iB̂(t))·x(t)
y(t) = Re[(C(t) + iĈ(t))ᵀ·h(t)]
```

#### 2.2.3 复数到实数的等价转换

**命题 2（复数-实数 SSM 等价性）**：
在指数-Euler 离散化下，复数 SSM 等价于一个实数 SSM：

```
h_t = e^(Δt·At)·R_t·h_{t-1} + Δt·B_t·x_t
y_t = C_tᵀ·h_t
```

其中：
- 状态维度翻倍：h_t ∈ ℝᴺ（原为 ℂ^(N/2)）
- 转移矩阵为块对角旋转矩阵：R_t = BlockDiag{R(Δt·θt[i])}ᵢ₌₁^(N/2)

#### 2.2.4 RoPE 技巧：高效实现

**命题 3（复数 SSM 与数据依赖 RoPE 的等价性）**：
上述实数 SSM 的输出等价于在标准标量转移 SSM 的 B、C 组件上应用**数据依赖的旋转位置编码（RoPE）**：

```
h_t = e^(Δt·At)·h_{t-1} + (Πᵢ₌₀ᵗ Rᵢᵀ)·(Δt·B_t·x_t)
y_t = [(Πᵢ₌₀ᵗ Rᵢᵀ)·C_t]ᵀ·h_t
```

其中旋转矩阵的累积乘积可以高效计算（Su 等人 2023 的 RoPE 技巧）。

**关键洞察**：
- 复数 SSM 中的数据依赖旋转对应于 SSD 框架中的 Query（C）和 Key（B）
- 标准 RoPE 使用固定频率的旋转角度 θ[i] = θ_base^(-2i/d)
- Mamba-3 使用**数据依赖**的旋转角度，通过 θ_t 动态调整

#### 2.2.5 与指数-梯形的结合

**命题 4（指数-梯形下的 RoPE 实现）**：
使用指数-梯形规则离散化复数 SSM，得到：

```
h_t = α_t·h_{t-1} + β_t·(Πᵢ₌₀^(t-1) Rᵢᵀ)·B_{t-1}·x_{t-1} + γ_t·(Πᵢ₌₀ᵗ Rᵢᵀ)·B_t·x_t
y_t = [(Πᵢ₌₀ᵗ Rᵢᵀ)·C_t]ᵀ·h_t
```

实验验证表明，这种复数 SSM 实现（通过数据依赖 RoPE）能够解决实数 SSM（无论是否使用标准 RoPE）无法解决的状态追踪任务。

### 2.3 创新三：多输入多输出（MIMO）SSM

#### 2.3.1 动机：解码阶段的硬件效率

随着 Test-Time Compute 的扩展，Agentic 工作流等应用使得推理在整体计算预算中占比越来越高。尽管 SSM 比 Transformer 有更低的延迟，但其解码阶段严重受限于**内存带宽**而非计算能力，导致硬件利用率低下。

**算术强度（Arithmetic Intensity）**：
- Mamba 单次生成的算术强度：约 2.5 ops/byte
- NVIDIA H100 bfloat16 matmul 的算术强度：约 295 ops/byte
- **差距**：SSM 解码远未达到计算饱和状态

#### 2.3.2 从 SISO 到 MIMO 的转换

标准 SSM 使用**单输入单输出（SISO）**递推：

```
SISO：h_t ← α_t·h_{t-1} + Δt·B_t·x_t
       （其中 B_t ∈ ℝᴺ, x_t ∈ ℝ，外积计算）
```

Mamba-3 引入**多输入多输出（MIMO）**递推：

```
MIMO：B_t ∈ ℝ^(N×R), x_t ∈ ℝ^(P×R)
      h_t ← α_t·h_{t-1} + Δt·B_t·x_tᵀ
      （矩阵-矩阵乘法，利用 Tensor Core）
```

#### 2.3.3 硬件效率分析

| 指标 | SISO | MIMO |
|-----|------|------|
| 内存流量 | 主要由状态 h_t 决定 | 相近（R 较小） |
| 计算量（FLOPs） | ∝ NP | ∝ N·P·R（增加 R 倍） |
| 算术强度 | ~2.5 ops/byte | 随 R 线性增长 |
| 实际延迟 | 基准 | 几乎不变 |

**核心洞察**：
- 外积 B_t·x_tᵀ 变为硬件高效的**矩阵-矩阵乘法（matmul）**
- 使用快速 Tensor Core 计算
- 仅带来边际延迟成本
- 内存流量不显著增加（R 较小，通常 R << N）

#### 2.3.4 MIMO SSM 的训练

尽管 MIMO 动机来自推理效率，但训练算法需要适配：

**关键观察**：MIMO SSM 可表示为 **R² 个 SISO SSM** 的组合：

```
对于 i, j ∈ {0, ..., R-1}：
  h_t^(j) ← α_t·h_{t-1}^(j) + Δt·B_t^(j)·x_t^(j)
  h_t ← Σⱼ₌₀^(R-1) h_t^(j)
  y_t^(i) ← (C_t^(i))ᵀ·h_t
```

因此：y_t^(i) = Σⱼ SSM(α, Δ, B^(j), C^(i), x^(j))_t

**训练策略**：
1. **黑盒复用**：将 SISO 训练算法作为黑盒并行调用 R² 次
2. **分块算法优化**：利用 chunked algorithm 将训练成本降至 R 倍（而非 R²）

## 三、Mamba-3 架构整合

### 3.1 完整架构图

```
输入 x_t
    ↓
[线性投影] → 生成 Δ_t, B_t, C_t, A_t（数据依赖参数）
    ↓
[离散化] → 指数-梯形规则计算 α_t, β_t, γ_t
    ↓
[复数状态更新] → 应用数据依赖 RoPE（旋转矩阵 R_t）
    ↓
[MIMO 递推] → h_t = α_t·h_{t-1} + β_t·(旋转后的 B_{t-1}·x_{t-1}) + γ_t·(旋转后的 B_t·x_t)
    ↓
[输出投影] → y_t = (旋转后的 C_t)ᵀ·h_t
    ↓
输出 y_t
```

### 3.2 两种变体

| 变体 | 特性 | 适用场景 |
|-----|------|---------|
| **Mamba-3 (SISO)** | 使用 SISO 递推，复数状态，指数-梯形 | 平衡质量与效率 |
| **Mamba-3 (MIMO)** | MIMO 递推，额外 FLOPs，不增加延迟 | 追求最高性能 |

## 四、实验结果与性能分析

### 4.1 语言建模性能

#### 4.1.1 下游任务准确率（1.5B 规模）

| 模型 | 平均下游准确率 | 相比 Transformer | 相比 Mamba-2 | 相比 GDN |
|-----|--------------|-----------------|-------------|---------|
| Transformer | 基准 | - | - | - |
| Mamba-2 | - | - | - | - |
| Gated DeltaNet (GDN) | 次优基准 | - | +0.2 | - |
| **Mamba-3 (SISO)** | **+0.6** | +1.6 | +1.9 | +0.6 |
| **Mamba-3 (MIMO)** | **+1.8** | +2.2 | +2.5 | +1.8 |

**关键发现**：
- Mamba-3 (SISO) 比次优模型 GDN 提升 0.6 个百分点
- Mamba-3 (MIMO) 进一步比 GDN 提升 1.8 个百分点
- 相比 Transformer 优势达 2.2 个百分点

#### 4.1.2 状态尺寸效率

| 配置 | 状态尺寸 | 困惑度 |
|-----|---------|-------|
| Mamba-2 | 128 | PPL_128 |
| **Mamba-3 (MIMO)** | **64** | **≈ PPL_128** |

**结论**：Mamba-3 (MIMO) 以**一半的状态尺寸**达到 Mamba-2 相当的困惑度，显著提升内存效率。

### 4.2 状态追踪能力

#### 4.2.1 合成任务测试

在奇偶校验等状态追踪任务上：

| 模型 | 奇偶校验准确率 | 评论 |
|-----|--------------|------|
| Mamba-2 | ~50%（随机） | 无法学习旋转动态 |
| Mamba-2 + 标准 RoPE | ~50%（随机） | 固定旋转不足够 |
| **Mamba-3（复数 SSM + 数据依赖 RoPE）** | **≈100%** | 成功学习旋转动态 |

**理论验证**：复数状态更新使模型能够表达旋转动态，数据依赖 RoPE 提供了实现这种能力的计算框架。

### 4.3 推理效率

#### 4.3.1 解码 FLOPs 与延迟

| 指标 | Mamba-2 | Mamba-3 (MIMO) | 提升 |
|-----|---------|----------------|-----|
| 解码 FLOPs | 基准 | **4×** | +300% |
| 墙钟延迟 | 基准 | **相近** | ~0% |
| 困惑度 | PPL_base | **PPL_improved** | 改善 |

**核心成果**：Mamba-3 (MIMO) 在几乎不增加延迟的前提下，将解码计算量提升 4 倍，同时改善困惑度和下游性能。

#### 4.3.2 硬件利用率

MIMO 设计使 SSM 解码的算术强度随 R 线性增长，更接近计算饱和状态，有效利用 GPU Tensor Core。

## 五、理论贡献与方法论洞察

### 5.1 SSM 视角的独特价值

Mamba-3 的三大创新（指数-梯形离散化、复数状态、MIMO）都自然源于**SSM 中心视角**，但从其他流行视角（如线性注意力或 Test-Time Regression）并不明显：

| 创新点 | SSM 视角 | 其他视角 |
|-------|---------|---------|
| 指数-梯形 | 连续系统离散化 | 不明显 |
| 复数状态 | 特征值复数化 | 与 RoPE 的意外联系 |
| MIMO | 信号处理 SISO→MIMO 推广 | 矩阵乘法优化 |

### 5.2 与控制理论的深度联系

Mamba-3 的离散化框架将深度学习中的 SSM 与经典控制理论重新连接：

1. **ZOH（零阶保持）**：经典 LTI 系统方法，被证明适用于 LTV
2. **指数-Euler**：理论化 Mamba-1/2 的启发式方法
3. **指数-梯形**：新的二阶方法，统一并超越现有方案

### 5.3 与注意力机制的关系

通过 SSD 框架，Mamba-3 与注意力机制建立了清晰联系：

```
SSD 框架：Y = (L ⊙ CBᵀ)X
          ↓ 设置 Q=C, K=B, V=X
        → 注意力的一般化形式
```

Mamba-3 的掩码 L 是 Mamba-2 的推广，因此也是线性注意力的推广。

## 六、行业影响与未来展望

### 6.1 对 LLM 架构设计的启示

#### 6.1.1 混合架构趋势加强

Mamba-3 的发布进一步验证了**混合架构**（Hybrid Architecture）的趋势：

| 模型/平台 | 架构选择 | 发布时间 |
|----------|---------|---------|
| Jamba (AI21) | 1:7 Attention:Mamba + MoE | 2024 |
| Mamba-Llama (Together AI) | 75% Mamba 替换 | 2024 |
| Falcon Mamba (TII) | 纯 Mamba | 2025 |
| Nemotron-H (NVIDIA) | 92% Mamba-2 | 2025 |
| Kimi K2.5 | Gated DeltaNet + Attention | 2026 |
| **Mamba-3 潜在应用** | 新一代混合架构 | 2026+ |

#### 6.1.2 推理优先设计范式

Mamba-3 明确倡导**推理优先（Inference-First）**的设计范式：

- **传统**：先优化训练，再适配推理
- **Mamba-3**：从推理效率出发，反向推导架构设计

这一范式转变对于 Test-Time Compute 时代至关重要。

### 6.2 对 Agentic AI 的影响

#### 6.2.1 长上下文处理能力

Agentic 工作流需要处理：
- 多轮对话历史
- 工具调用序列
- 检索增强的上下文

Mamba-3 的恒定内存和线性计算特性使其特别适合这些场景。

#### 6.2.2 高效推理支持并行工作流

Anthropic (2026) 和 OpenAI (2026) 的研究表明，并行、Agentic 工作流正在兴起。Mamba-3 的硬件高效解码支持大规模并行推理。

### 6.3 与测试时计算扩展的关系

#### 6.3.1 Test-Time Compute 的效率瓶颈

Test-Time Compute 扩展（如 CoT、迭代优化、Agent 工作流）带来新挑战：

```
总计算 = 训练计算 + 推理计算
              ↓
随着 Test-Time Compute 扩展，推理计算占比急剧上升
              ↓
推理效率成为关键优化目标
```

#### 6.3.2 Mamba-3 的定位

Mamba-3 并非要与 Test-Time Compute 竞争，而是为其提供**高效的基础设施**：

- 相同推理预算下支持更多推理步骤
- 相同延迟下支持更大规模的 Agent 工作流
- 恒定内存使长上下文 Agent 会话成为可能

### 6.4 局限与未来方向

#### 6.4.1 当前局限

1. **规模验证**：当前实验主要在 1.5B 规模，更大规模（10B+）的验证有待进行
2. **长上下文评估**：虽然理论上支持无限上下文，但实际长上下文（100K+）评估有限
3. **多模态扩展**：当前聚焦于语言建模，视觉-语言等多模态扩展是开放问题

#### 6.4.2 未来研究方向

1. **大规模训练**：验证 Mamba-3 在 10B+ 参数规模的性能
2. **与 MoE 结合**：探索 Mamba-3 + MoE 的混合架构
3. **硬件协同设计**：针对 Mamba-3 的专用硬件加速器设计
4. **多模态 Mamba**：将 Mamba-3 扩展到视觉-语言模型

## 七、个人理解与深度分析

### 7.1 为什么 Mamba-3 重要？

#### 7.1.1 理论深度

Mamba-3 不是简单的工程改进，而是有着深厚理论基础的突破：

1. **控制理论**：将经典控制理论的离散化方法引入深度学习
2. **信号处理**：SISO→MIMO 的推广带来硬件效率提升
3. **复数分析**：复数状态与旋转动态的联系

这种理论深度使 Mamba-3 的改进具有**可解释性**和**可扩展性**。

#### 7.1.2 实践价值

Mamba-3 解决了次二次方模型的**关键痛点**：

| 痛点 | Mamba-3 解决方案 |
|-----|-----------------|
| 状态追踪能力差 | 复数状态 + 数据依赖 RoPE |
| 硬件利用率低 | MIMO 设计增加 FLOPs 不增延迟 |
| 表达能力受限 | 指数-梯形离散化 |

#### 7.1.3 范式意义

Mamba-3 代表了**后 Transformer 时代**架构设计的新思路：

- 不再追求纯 Attention 或纯 SSM
- 而是根据**任务需求**和**硬件约束**灵活组合
- 推理优先的设计范式可能成为新标准

### 7.2 与相关工作的关系

#### 7.2.1 与 Mamba-2 的关系

Mamba-3 是 Mamba-2 的**进化**而非**革命**：

- 保留 SSD 框架和硬件高效训练
- 在离散化、状态空间、输出维度三个维度扩展
- 向后兼容：Mamba-2 是 Mamba-3 的特例（指数-Euler，实数状态，SISO）

#### 7.2.2 与 Gated DeltaNet 的关系

Gated DeltaNet（Yang 等人 2025）是同期竞争方法：

| 方面 | Gated DeltaNet | Mamba-3 |
|-----|---------------|---------|

| 状态追踪 | 有限 | 通过复数状态解决 |
| 硬件效率 | 良好 | MIMO 进一步优化 |
| 性能 | 强劲基准 | 超越 GDN 0.6-1.8 点 |

两者代表了次二次方架构的不同技术路线，Mamba-3 在性能和理论统一性上略胜一筹。

#### 7.2.3 与混合 Transformer 的关系

```
         ↓                      ↓                      ↓

```

### 7.3 技术实现的关键洞察

#### 7.3.1 指数-梯形的实现技巧

实现指数-梯形需要小心处理历史依赖（x_{t-1}）：

```
核心挑战：需要访问前一时间步的输入
解决方案：
1. 在并行计算中使用偏移操作
2. 边界处使用零填充
3. 或通过递推形式自然处理
```

#### 7.3.2 复数状态的实现技巧

复数 SSM 通过 **RoPE 技巧** 高效实现：

```
复数乘法：(a + bi)(c + di) = (ac - bd) + (ad + bc)i
RoPE 技巧：转换为 2D 旋转矩阵乘法
          [cosθ  -sinθ] [a]   [a·cosθ - b·sinθ]
          [sinθ   cosθ] [b] = [a·sinθ + b·cosθ]

效率：复数运算 → 实数矩阵乘法（Tensor Core 优化）
```

#### 7.3.3 MIMO 的训练效率

MIMO 训练的关键优化：**分块算法（Chunked Algorithm）**

```
序列分块长度 C = 状态维度 N = 头维度 P
↓
块内并行计算：SSD 的矩阵形式
块间递推计算：标准 SSM 递推
↓
总 FLOPs：8TN²（线性于序列长度 T）
```

Mamba-3 通过三大创新推动了次二次方序列建模的前沿：
