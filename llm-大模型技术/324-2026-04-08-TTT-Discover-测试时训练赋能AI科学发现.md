## 一、研究背景与核心问题

### 1.1 科学发现与AI的交汇点

人类在面对复杂科学问题时，往往需要经历**尝试-失败-学习-再尝试**的循环过程。回想我们第一次解决艰难编程作业的经历：尽管我们阅读了教科书并完成了课后练习，但当真正面对超出课本范围的问题时，简单的猜测往往只能产生微小的进展。正是通过从失败尝试中学习，我们的后续尝试变得更加智能，最终掌握解决问题的新方法。

将这一思维模式应用于AI驱动的科学发现，面临着一个根本性挑战：**科学发现问题的定义本身就要求突破人类已有的全部知识边界**。这意味着模型的目标分布不仅超出了训练数据，甚至超出了人类文明的集体知识储备。对于AI而言，这种分布外泛化（Out-of-Distribution Generalization）的难度不亚于甚至超过人类面临的挑战。

### 1.2 现有方法的局限性

**测试时扩展（Test-Time Scaling）**作为近年来大模型研究的重要方向，已经在多个领域展现了惊人潜力。以AlphaEvolve为代表的进化搜索方法，通过提示一个**冻结的LLM**来生成大量候选解，并利用手工设计的启发式规则筛选和复用历史尝试。

然而，这种"搜索但不学习"的范式存在一个根本缺陷：**模型本身无法从尝试中获得真正的能力提升**。这就像一个学生只能通过阅读参考答案的提示来改进下一次尝试，而永远无法内化问题背后的核心原理。虽然提示可以引导模型改进先前方案，但模型权重始终保持不变，无法真正"学会"如何解决这类问题。

### 1.3 从搜索到学习的历史启示

AI发展史反复验证了一个规律：**学习最终会超越纯搜索**。无论是围棋（AlphaGo到AlphaZero）还是蛋白质折叠（AlphaFold），最初的搜索方法虽然有效，但最终都被端到端学习所取代。Sutton在其著名的《苦涩的教训》中总结道："计算能力的扩展始终是最终获胜的因素，而学习是最高效的计算扩展方式。"

TTT-Discover正是基于这一历史洞见提出的：**如果我们在测试时继续训练模型，让模型从针对特定问题的尝试中学习，会怎样？**

## 二、核心创新：测试时强化学习（Test-Time Training）

### 2.1 发现问题与标准RL的本质区别

TTT-Discover将科学发现问题形式化为马尔可夫决策过程（MDP），但与标准强化学习存在两个**关键差异**：

#### 差异一：单问题专注 vs 泛化需求
标准RL的目标是找到一个能泛化到同环境所有状态的策略；而在发现问题中，**我们只需要解决这一个特定问题**，不需要考虑策略在其他问题上的表现。

#### 差异二：单次最优 vs 平均性能
标准RL优化期望回报，关注平均表现；而在发现问题中，**我们的目标只有一个：找到一个突破现有最优解的单一方案**。策略本身的平均表现并不重要，重要的是它能否至少一次发现更优解。

这两个差异意味着，我们可以设计专门针对"发现"这一目标的训练方法和搜索策略。

### 2.2 形式化定义

论文首先给出了清晰的数学形式化：

- **问题描述** $d$：以自然语言描述的科學问题
- **状态** $s$：候选解（如GPU内核代码、数学证明步骤函数等）
- **奖励函数** $R(s) \in \mathbb{R}$：连续标量，如内核运行时间的倒数
- **最优已知解** $s_{sota}$ 及其奖励 $r_{sota} = R(s_{sota})$

**发现（Discovery）的正式定义**：找到一个状态 $s$ 使得 $R(s) > r_{sota}$。奖励差距越大，发现的显著性越高。

### 2.3 状态复用与动作复用

搜索方法的核心设计是**如何有效复用历史尝试**。论文定义了三种层次的复用：

**基础Best-of-N**：独立采样N个候选解，选择奖励最高的。初始状态 $s$ 可以是最优已知解 $s_{sota}$ 或空状态 $\langle \text{empty} \rangle$。

**状态复用（State Reuse）**：维护一个历史解缓冲区 $\mathcal{H}_i$，从中采样初始状态 $s_i$。复用启发式 favor 高奖励解但仍保留对低奖励解的非零采样概率。

**状态-动作复用（State-Action Reuse）**：不仅复用历史解，还复用生成这些解的思考过程（thinking tokens）和中间结果。这对应于进化搜索中的交叉和变异操作。

## 三、TTT-Discover方法详解

### 3.1 朴素测试时RL的局限

将标准RL算法（如PPO、GRPO）直接应用于测试时训练存在三个问题：

**目标函数不匹配**：标准RL优化期望回报，对是否突破最优解无偏好。例如，在GPU内核工程中，当前最优2000μs，达到1900μs或1950μs获得几乎相同的奖励，但前者代表重大突破而后者只是小幅改进。

**有效视野受限**：每次都从空状态开始限制了单次尝试能探索的解空间深度。复用历史解相当于延长有效时间步长。

**探索-利用平衡**：优化期望回报可能导致策略坍缩到安全但平庸的解，而非冒险寻找可能带来突破的解。

### 3.2 熵正则化目标函数

TTT-Discover的核心创新之一是**熵正则化目标函数（Entropic Objective）**：

$$J_\beta(\theta) = \mathbb{E}_{s \sim \text{reuse}(\mathcal{H})} \left[ \log \mathbb{E}_{a \sim \pi_\theta(\cdot|s)} \left[ e^{\beta \cdot R(s,a)} \right] \right]$$

这一设计的直觉是：
- 当 $\beta > 0$ 时，指数函数 $e^{\beta R}$ 强烈偏袒高奖励动作
- 对数-期望-指数结构对应softmax操作，天然关注最大值的期望
- 梯度会引导策略朝向能产生突破性高奖励的方向

对比标准RL的目标 $\mathbb{E}[R]$，熵目标对突破最优解有强烈偏好，这正符合科学发现的需求。

### 3.3 训练算法流程

```python
# TTT-Discover 算法概述
Input: 问题描述 d, 初始策略 π_θ₀
1. 根据 d 构建环境 (R, T)
2. 初始化缓冲区 H₀ = {(empty, R(empty), {})}
3. for i = 0, 1, ..., N-1:
4.   s_i, c_i ~ reuse(H_i)          # 复用启发式采样
5.   a_i ~ π_θi(·|d, s_i, c_i)     # 策略生成动作
6.   s_i' = T(a_i)                  # 环境转移
7.   r_i = R(s_i')                  # 评估奖励
8.   H_{i+1} = H_i ∪ {(s_i, a_i, s_i', r_i)}  # 更新缓冲区
9.   θ_{i+1} = train(θ_i, (d, s_i, c_i, a_i, r_i))  # 策略更新
10. return argmax_i r_i 的状态
```

### 3.4 复用启发式设计

复用启发式是TTT-Discover的第二个核心组件。给定缓冲区 $\mathcal{H}$，复用函数需要决定：
- 以何种概率选择哪个历史状态作为下一轮的初始状态
- 对于状态-动作复用，还需要决定附带哪些上下文信息 $c_i$

论文采用了一个简单的温度调节softmax：
$$P(s) \propto e^{\tau \cdot R(s)}$$

其中 $\tau$ 是温度参数。高温度偏向探索（均匀采样），低温度偏向利用（集中在高奖励解）。

## 六、局限与未来方向

### 6.1 当前局限

**依赖连续奖励**：当前实现聚焦于具有连续奖励函数的问题，对于离散奖励（如代码正确性通过/不通过）的直接应用需要适配。

**特定问题训练**：每个问题需要独立的训练过程，无法在不同问题间迁移学习到的策略（这正是TTT-Discover的设计选择，但对于相关问题的序列学习仍有探索空间）。

**计算资源需求**：虽然比调用闭源模型便宜，但相比单次推理仍有额外开销。
