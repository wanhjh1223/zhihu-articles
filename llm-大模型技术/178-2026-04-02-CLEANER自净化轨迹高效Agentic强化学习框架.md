## 一、研究背景与核心问题

### 1.1 Agentic AI的崛起与挑战

2025-2026年，大语言模型正在经历从"被动文本生成器"到"自主智能体"的范式转变。以OpenAI的o系列模型、DeepSeek-R1、Kimi K2.5为代表的新一代推理模型，已经展现出强大的工具使用能力和自主决策能力。这一趋势标志着**Agentic AI时代**的全面到来。

在Agentic AI的技术栈中，**工具增强推理（Tool-Augmented Reasoning）** 是核心能力之一。其中，Python代码解释器因其图灵完备性和确定性执行语义，成为最重要的工具模态。正如代码框架CodeAct所倡导的："代码应被视为智能体推理中的一等公民"。

### 1.2 小模型Agentic RL的致命瓶颈

然而，当前Agentic强化学习（Agentic RL）面临一个根本性挑战：**参数受限模型（4B-7B）在探索阶段频繁遭遇执行失败**。

具体而言：

1. **高失败率问题**：小模型在RL训练初期生成无效代码的概率极高，导致"错误→反馈→重试"的恶性循环
2. **上下文污染**：执行错误产生的冗长traceback消耗宝贵的上下文窗口，干扰后续推理
3. **信用分配困境**：在稀疏奖励设置下（如GRPO），最终成功的轨迹会将错误动作和正确动作等同强化

**核心洞察**：研究发现，轨迹中的执行错误数量与模型性能呈现强负相关——错误峰值出现时，AIME25准确率直接停滞或下降。

### 1.3 现有方案的困境

针对上述问题，学术界提出了多种解决方案，但各有缺陷：

| 方案类型 | 代表方法 | 核心缺陷 |
|---------|---------|---------|
| **密集奖励** | Process Reward Model | 易引发奖励作弊（Reward Hacking），模型优化中间指标而非最终结果 |
| **超采样过滤** | rStar2-Agent | 计算成本过高（rollout占RL训练>80%时间），资源受限场景不可行 |
| **课程学习** | 渐进式难度 | 难以适应动态变化的错误模式 |

**CLEANER的突破口**：与其在外部过滤或增加计算，不如利用模型**内在的自我修正能力**，直接在数据层面净化训练轨迹。

## 二、CLEANER框架：核心技术详解

### 2.1 整体架构概览

CLEANER（**C**ontext-**L**evel **E**rror **A**voidance for **N**oisy **E**xecution in **R**L）是一个基于**轨迹自净化**的Agentic RL训练框架。

**核心思想**：当模型在rollout过程中生成错误代码但随后自我修正成功时，不保留"错误历史"，而是构造一条"虚拟的、流畅的"成功轨迹用于训练。这样，模型学习的是正确的推理模式，而非错误恢复循环。

```
传统方法轨迹: [Query] → [推理1] → [错误代码] → [Traceback] → [重试推理] → [正确代码] → [结果] ✓
                                                                   ↓
                                                               全部强化学习

CLEANER轨迹:   [Query] → [推理1] → [正确代码] → [结果] ✓  (回溯替换错误历史)
                                                                   ↓
                                                            仅强化正确路径
```

### 2.2 SAAR机制：相似性感知的自适应回滚

**SAAR（Similarity-Aware Adaptive Rollback）** 是CLEANER的核心创新，它通过两个阶段实现轨迹净化：

#### 阶段一：错误触发与前向修正（Phase I: Error Trigger and Lookahead Correction）

当环境返回执行错误 `o_t^-` 时：

1. **状态冻结**：冻结当前历史 `h_t`，不立即提交错误
2. **上下文扩展**：构造增强上下文 `h̃_t = h_t ⊕ (r_t, c_t, o_t^-)`，暴露错误信息
3. **修正生成**：从 `h̃_t` 采样修正响应 `(r'_aux, c'_t)`
4. **验证循环**：执行修正代码，若成功（`o'_t = o^+`）进入阶段二；否则最多重试K次

**关键设计**：不直接接受错误结果，而是给模型一次"改过自新"的机会。

#### 阶段二：相似性感知的自适应替换（Phase II: Similarity-Aware Adaptive Replacement）

获取成功修正 `c'_t` 后，需要决定如何替换原轨迹中的错误部分。SAAR的核心创新在于**根据语义相似度自适应选择替换粒度**：

**相似度计算**：
```python
sim = difflib.SequenceMatcher(None, c_t, c'_t).ratio()
```

**Case A: 实现级修复（高相似度，sim ≥ γ = 0.5）**
- **场景**：语法错误、变量名拼写错误等浅层错误
- **策略**：保留原推理 `r_t`，仅替换错误代码为 `c'_t`
- **净化结果**：`(r_t, c'_t, o'_t)`

**Case B: 推理级修复（低相似度，sim < γ = 0.5）**
- **场景**：逻辑错误、算法选择错误等深层错误
- **策略**：原推理 `r_t` 与错误代码 `c_t` 一并删除，采用修正推理 `r'_aux`
- **净化结果**：`(r'_aux, c'_t, o'_t)`

**为什么需要自适应粒度？**

- 高相似度表明原推理是正确的，只是实现细节出错——应保留推理，避免过度删除
- 低相似度表明原推理本身有问题——必须彻底替换，防止语义不一致

### 2.3 训练流程与优化细节

#### Logit重计算与RadixAttention

净化后的轨迹存在分布偏移问题：修正代码 `c'_t` 最初是从错误增强上下文 `h̃_t` 采样的，但在净化轨迹中它是在干净上下文 `h_t ⊕ r_final` 中出现的。

为保证策略更新的因果正确性，需要**重计算 `c'_t` 在净化上下文中的log概率**。

CLEANER采用SGLang的**RadixAttention**机制高效解决这一问题：
- 复用不变历史前缀的KV缓存
- 仅重新计算修改后后缀的logits
- 将额外开销降至最低

#### 课程混合策略（Curriculum Mixing）

为防止模型完全"遗忘"错误恢复能力，CLEANER采用随机混合策略：
- **70%轨迹**：应用SAAR净化
- **30%轨迹**：保留原始状态（含错误历史）

这一平衡确保模型既优先学习正确生成，又保留调试和自我修正的韧性。

## 三、实验结果：性能与效率的双重突破

### 3.1 实验设置

**基础模型**：
- Qwen3-4B-Instruct-2507
- Qwen2.5-7B-Instruct

**数据集**：
- Agentic SFT数据集（冷启动）
- DAPO-Math (17K)
- Skywork-or1 (数学4,902 + 代码3,586)
- MegaScience (科学3K)

**评测基准**：
- **AIME24/25**：高难度数学竞赛
- **GPQA**：研究生级科学问答
- **LiveCodeBench**：代码能力评测

**训练配置**：
- RL算法：DAPO（字节跳动开源大规模RL系统）
- Batch Size：128
- Group Size：16
- 学习率：4B模型2e-6，7B模型1e-6

### 3.2 主实验结果

#### 性能对比：以小博大

| 方法 | AIME24 | AIME25 | GPQA | LiveCodeBench | RL步数 |
|------|--------|--------|------|---------------|--------|
| **Self-Contained Reasoning** |
| Qwen2.5-7B-Instruct | 16.7 | 10.0 | 31.3 | 15.2 | - |
| Qwen3-4B-Instruct | 63.3 | 47.4 | 52.0 | 35.1 | - |
| DeepSeek-V3 (671B) | 39.2 | 28.8 | 59.1 | 16.1 | - |
| DeepSeek-R1-Zero (671B) | 71.0 | 53.5 | 53.5 | - | - |
| **Agentic Reasoning** |
| ToRL-7B | 43.3 | 30.0 | - | - | 550 |
| ReTool-32B | 72.5 | 54.3 | - | - | 1200 |
| rStar2-Agent-14B | 80.6 | 69.8 | 60.9 | - | 500 |
| DemyAgent-4B (SOTA) | 72.6 | **70.0** | 58.5 | 51.7 | 750 |
| DAPO-baseline-4B | 66.7 | 59.4 | 56.9 | 49.5 | 250 |
| **CLEANER-4B** | **72.7** | 67.1 | **60.2** | **54.9** | **250** |

**核心发现**：

1. **性能卓越**：CLEANER-4B在AIME24上达到72.7%，超越DemyAgent-4B，接近14B参数的rStar2-Agent
2. **效率革命**：仅用**1/3训练步数**（250 vs 750）即可匹配SOTA性能
3. **全面领先**：在GPQA（+1.7%）和LiveCodeBench（+3.2%）上超越DemyAgent-4B

#### 消融实验：验证各组件贡献

**Qwen3-4B-Instruct消融**：

| 配置 | AIME24 Pass@1/16 | AIME25 Pass@1/16 | GPQA | LiveCodeBench |
|------|------------------|------------------|------|---------------|

| + Tools | 66.7/84.4 | 59.4/84.2 | 56.9 | 26.6 |
| + SAAR (CLEANER) | **72.7/87.6** | **67.1/84.1** | **60.2** | **26.8** |

**提升幅度**：
- AIME：+6% Pass@1
- GPQA：+3%
- LiveCodeBench：+5%

**Qwen2.5-7B-Instruct消融**：

| 配置 | AIME24 Pass@1/16 | AIME25 Pass@1/16 | GPQA | LiveCodeBench |
|------|------------------|------------------|------|---------------|
| RL w/o Tools | 15.4/30.5 | 14.4/24.4 | 32.3 | 1.1 |
| + Tools | 40.2/59.1 | 27.3/46.3 | 35.9 | 13.0 |
| + SAAR (CLEANER) | **44.6/64.3** | **31.0/54.7** | **40.0** | **13.1** |

**关键洞察**：工具集成本身带来巨大提升（7B模型AIME24 +25%），而SAAR在此基础上进一步显著优化（+4.4%）。

### 3.3 训练动态分析

#### 错误抑制与性能提升的关联

实验数据显示，CLEANER能有效抑制轨迹中的错误工具调用：

```
训练过程中工具错误数/轨迹对比：
- DAPO-baseline：初期峰值>2次错误/轨迹，后期波动在0.5-1.0之间
- CLEANER：快速收敛至接近0次错误/轨迹

对应AIME25准确率：
- DAPO-baseline：与错误数负相关，错误峰值时准确率停滞
- CLEANER：错误持续走低，准确率稳步提升
```

**关键结论**：减少错误工具调用不是副作用，而是性能提升的直接原因。

#### 从次优策略恢复

一个有趣的实验：在DAPO-baseline训练到第200步（此时已出现严重不稳定，平均0.6次错误/轨迹）时引入SAAR。

**结果**：
- 训练动态迅速稳定
- AIME24提升5.2%，AIME25提升1.0%
- 但**未能达到从头训练SAAR的水平**

**启示**：SAAR机制需要在训练全周期内使用，事后补救效果有限。

## 四、技术创新深度解读

### 4.1 从信用分配到轨迹净化：范式转变

传统RL解决工具执行错误的思路集中在**奖励设计**：
- 密集奖励：区分正确/错误步骤 → 奖励作弊
- 结果奖励：只看最终结果 → 错误动作被连带强化

CLEANER提出了完全不同的思路：**不调整奖励函数，而是调整训练数据本身**。

**理论优势**：
- 规避了奖励设计的内在矛盾
- 将"学习正确行为"转化为"只展示正确行为"
- 符合"模仿学习优于强化学习"的直觉（当专家数据可用时）

### 4.2 SAAR的生物学启发

SAAR机制与**人类学习模式**高度相似：

1. **错误识别**：意识到自己的错误（错误触发）
2. **尝试修正**：利用反馈信息找到正确方案（前向修正）
3. **经验整合**：将正确的做法内化，"忘掉"失败的尝试（自适应替换）

这种"试错-修正-内化"的循环正是认知科学中**程序性学习**的核心机制。

### 4.3 相似度阈值γ的深层含义

γ=0.5的阈值选择不仅是工程调参，更反映了**错误分类理论**：

- **高相似错误（≥0.5）**：属于" slips "（失误）——意图正确，执行出错
- **低相似错误（<0.5）**：属于" mistakes "（错误）——意图本身有误

不同错误类型需要不同的学习策略：
- 失误：强化正确执行，保留原意图
- 错误：彻底替换，重新建立因果联系

## 五、行业影响与未来展望

### 5.1 对Agentic AI发展的意义

CLEANER的成功验证了几个关键趋势：

1. **小模型+Agentic RL可以挑战大模型**：4B参数通过高效训练可以达到32B甚至更大模型的推理水平
2. **数据质量>数据量**：自净化轨迹提供了高质量学习信号，大幅降低训练成本
3. **工具使用是推理的关键杠杆**：CodeAct范式下，代码能力是推理能力的放大器

### 5.2 实际应用价值

**企业级部署**：
- 计算成本降低70%（1/3训练步数+更少错误重试）
- 小模型可私有化部署，满足数据安全需求
- 更稳定的训练过程，减少超参调优成本

**研究方向启发**：
- 轨迹净化思想可扩展至其他工具类型（搜索、API调用等）
- 相似度自适应机制可应用于多模态场景（视觉-语言对齐）
- SAAR可作为测试时增强技术（仅需8.8%额外延迟）

### 5.3 局限性与开放问题

**当前局限**：
1. 仅验证于Python代码工具，其他工具模态待验证
2. 依赖规则化的代码执行环境，开放域工具（如Web搜索）挑战更大
3. 负样本利用尝试失败，如何有效利用失败经验仍是开放问题

**未来方向**：
1. **多工具协同**：将SAAR扩展至多工具场景（代码+搜索+API）
2. **在线自适应**：动态调整γ阈值，根据模型能力进化净化策略
3. **理论分析**：建立轨迹净化与策略优化收敛性的形式化联系

## 六、核心要点总结

### CLEANER的关键贡献

1. **问题诊断**：首次系统分析了小模型Agentic RL中的"错误污染"问题，揭示了执行噪声与性能瓶颈的因果关系

2. **技术创新**：提出SAAR机制，通过语义相似度实现自适应的轨迹净化，在保持训练稳定性的同时消除错误干扰

3. **效率突破**：仅1/3训练步数即可匹配SOTA，为小模型高效训练开辟新路径

4. **开源贡献**：完整开源代码和训练流程，推动社区复现和创新

### 关键数据

- **性能提升**：AIME +6%，GPQA +3%，LiveCodeBench +5%
- **效率提升**：训练步数减少67%
- **错误抑制**：工具执行失败率趋近于0
- **相似度阈值**：γ=0.5平衡修复精度与语义一致性

### 一句话总结

**CLEANER证明：在Agentic RL中，学会"忘记错误"比"从错误中学习"更有效。**
