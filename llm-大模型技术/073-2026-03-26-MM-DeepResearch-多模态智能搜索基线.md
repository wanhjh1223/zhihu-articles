## 一、研究背景与核心问题

### 1.1 多模态深度研究的兴起

近年来，推理型多模态大语言模型（Reasoning MLLMs）在解决复杂任务方面展现出惊人的能力，代表性工作如OpenAI的o1/o3系列、DeepSeek-R1、Kimi K2等。这些模型通过显式生成中间推理过程（Chain-of-Thought），能够在数学、代码、逻辑推理等任务上达到接近甚至超越人类专家的水平。

然而，一个根本性的限制始终存在：**参数容量受限的模型知识是固定的、有界的**。当面对超出其内在知识库的信息密集型任务时，即使是强大的推理模型也会"力不从心"。这尤其体现在需要实时信息、专业领域知识或开放世界事实的场景中。

### 1.2 从RAG到Agentic Search的演进

为应对这一挑战，学术界和工业界提出了多种解决方案：

**第一阶段：传统RAG（Retrieval-Augmented Generation）**
- 预定义的信息检索流水线
- 检索与生成解耦，缺乏动态适配能力
- 单次检索即推理，无法迭代优化

**第二阶段：Prompt-based Workflow**
- 通过精心设计的提示工程编排搜索行为
- 工具使用模式在推理时固定
- 仍受限于刚性流程，难以处理复杂查询

**第三阶段：Agentic Search（智能体搜索）**
- 检索与推理深度耦合
- 支持多轮迭代、自适应策略调整
- 能够主动调用多种工具进行深度研究

OpenAI的DeepResearch是第三阶段的开创性工作，展示了AI代理自主进行深度研究的巨大潜力。然而，现有工作主要局限于文本模态，多模态领域的深度研究仍面临重大挑战。

### 1.3 三大核心挑战

本文作者深入分析了构建多模态深度研究智能体的三大核心挑战：

**挑战一：搜索密集型多模态QA数据稀缺**
- 公开可用的、需要多轮搜索和多工具调用的多模态问答数据集非常有限
- 现有数据不足以有效激发MLLM的智能搜索能力
- 构建此类数据成本高昂、技术复杂

**挑战二：有效的搜索轨迹缺失**
- 传统的基于提示的搜索轨迹合成方法主要针对单轮搜索设计
- 多轮场景需要多轮迭代和各类搜索工具的交互，现有方法难以胜任
- 缺乏高质量的监督信号来指导模型学习复杂搜索行为

**挑战三：在线搜索API的训练成本过高**
- 大多数现有方法依赖SerpAPI、Jina等在线搜索API进行训练
- 每次训练运行可能产生数千美元的费用
- 高昂成本严重限制了广泛实验和系统探索的可能性

## 二、核心方法论详解

为应对上述挑战，作者提出了三个互补的核心技术创新：**Hyper-Search**（超图搜索数据生成）、**DR-TTS**（分解-重组工具树搜索）和**离线多模态搜索引擎**。

### 2.1 Hyper-Search：基于超图的搜索密集型QA生成

Hyper-Search是首个将超图（Hypergraph）概念引入搜索密集型问答数据合成的工作。其核心思想是：将网络视觉和文本内容建模为图中的节点，并通过超边捕获这些信息之间的复杂多对多关系。

#### 2.1.1 超图构建

**节点定义**：
- **图像节点（Image Nodes）**：对应从在线源收集的图像
- **文本节点（Text Nodes）**：代表网页的完整内容

每个节点都利用MLLM生成描述：图像节点生成标题（caption），文本节点生成摘要（summary）。

**节点扩展策略**：

对于图像节点，设计两种扩展操作：
1. **图像反向搜索（Image Reverse Search）**：查询图像获取K个相关网页内容，生成新的文本节点
2. **图像视觉搜索（Image Visual Search）**：返回K个视觉上相似的图像，生成新的图像节点

对于文本节点，同样设计两种扩展操作：
1. **文本节点扩展**：从原生页面内容中提取前K个信息性网页URL
2. **图像节点扩展**：识别网页内容中前K个相关图像链接

**超边连接**：
每个节点扩展都会诱导一个超边，将父节点与所有新生成的节点连接起来，显式建模跨模态检索关系。超边的形式化定义为：

```
e_{d,k} = {i_{d,k}^{i_{d-1,k}}, i_{d+1,1}^{i_{d,k}}, ..., i_{d+1,K}^{i_{d,k}}, t_{d+1,1}^{i_{d,k}}, ..., t_{d+1,K}^{i_{d,k}}}
```

通过持续扩展，Hyper-Search构建深度为D的超图，为下游多模态QA生成提供结构化的关系单元。

#### 2.1.2 多模态搜索QA生成

基于构建好的超图，Hyper-Search采用两种生成策略：

**Level 1：超边内QA生成（Intra-hyperedge）**
- 在单条超边内生成QA对
- 选择图像节点作为查询图像
- 答案必须从超边内的其他图像标题和网页内容中推导
- 鼓励模型同时使用图像和文本工具进行搜索

**Level 2：跨超边QA生成（Inter-hyperedge）**
- 整合多个超边的信息
- 增加搜索深度和难度
- 需要多轮搜索，鼓励更深层次的探索和更广泛的信息聚合

#### 2.1.3 数据过滤

使用GPT-5和Seed1.8等强模型进行多轮过滤，去除：
- 低质量图像
- 重复的QA对
- 与视觉无关或无需搜索的问题
- 无法从证据中验证的答案

最终构建包含**3K搜索密集型多模态QA对**的数据集，称为Hyper-Search-3K。

### 2.2 DR-TTS：分解-重组工具树搜索

DR-TTS（Decompose-Recompose Tool Tree Search）是一种用于合成搜索工具轨迹的树搜索方法。其核心创新在于**先分解任务训练专业工具专家，再重组专家进行联合树搜索**。

#### 2.2.1 搜索工具定义

作者定义了两大类搜索工具：

**信息型搜索工具**（基于外部网络源检索）：
- **Text-to-Text检索**：基于文本查询返回相关网页
- **Text-to-Image检索**：基于文本查询返回相关网络图像
- **Image-to-Image检索**：基于图像查询返回视觉相似图像

**知识型搜索工具**（基于专家模型生成）：
- 将复杂查询提交给领域特定专家模型
- 接收专家模型生成的知识密集型内容
- 适用于需要专业知识但无法直接通过网络检索的任务

#### 2.2.2 工具专家的分解训练

直接使用预训练的智能体基础MLLM进行树搜索往往性能不佳，原因包括：
- 模型频繁产生无效的工具调用
- 整合检索信息的能力有限
- 有效探索能力不足

**解决方案**：
1. 使用GPT按工具类型对搜索相关训练数据进行分类
2. 每个工具特定子集用于训练对应的专家（通过RL策略）
3. 最终得到M个专业工具专家，每个专家精通单一搜索工具

这种分解设计**降低了工具学习复杂度，增强了每工具熟练度**。

#### 2.2.3 工具专家的重组与树搜索

将训练好的工具专家重组，通过树搜索协同探索有效搜索轨迹：

**搜索过程**：
1. 从根节点（输入问题Q = {T, I}）开始
2. 扩展M个子节点，每个子节点由一个工具专家生成
3. 每个节点包含：推理规划步骤t、工具调用c、工具响应r
4. 在不同工具专家上分支，递归进行搜索
5. 当专家选择生成最终答案时，使用LLM评估正确性
6. 正确答案终止搜索，错误答案剪枝继续

**轨迹提取**：
对于成功产生正确答案的搜索轨迹，提取探索路径构建SFT数据：
```
τ = {t_1, c_1, r_1, t_2, c_2, r_2, ..., t_n, a_n}
```

通过DR-TTS，树结构搜索在多样化工具专家间实现平衡的工具使用探索，显著提高了轨迹探索多样性和成功率。最终收集**10K SFT轨迹**，称为DR-TTS-10K。

### 2.3 离线多模态搜索引擎

为解决在线搜索API成本高昂的问题，作者构建了一个支持多工具的离线搜索引擎。

#### 2.3.1 语料库构建

**文本语料**：
- 使用GPT为每个问题生成多个候选搜索查询
- 预先获取对应的网页内容（使用SerpAPI和Jina Reader）
- 额外整合大规模英文Wikipedia数据（2018）
- 将网页内容分段为512词的段落
- 使用e5-base-v2作为文本检索器

**图像语料**：
- 使用GPT生成多个候选图像搜索查询
- 使用SerpAPI检索相关图像URL
- 使用jina-embeddings-v4作为多模态检索器

#### 2.3.2 训练与推理策略

**训练时**（使用离线搜索）：
- 文本检索：返回top-3最相似的文本段落
- 图像检索：仅保留嵌入相似度>0.7的图像
- 每个工具调用查询只保留最相似的1张图像（避免上下文过长）

**推理时**（使用在线搜索）：
- 文本搜索扩展至top-5结果
- 每个查询检索最多3张图像
- 使用辅助LLM验证和总结工具响应（防止过长输出超出上下文限制）

#### 2.3.3 成本与效率对比

| 搜索方式 | 训练成本 | 训练时间（每样本） |
|---------|---------|------------------|
| 在线搜索 | ~$640/百步 | ~60秒 |
| 离线搜索 | $0 | ~1秒 |

离线搜索引擎实现了**零额外搜索成本**和**60倍速度提升**，使得大规模强化学习训练成为可能。

## 三、模型训练框架

基于上述三个核心组件，作者设计了MM-DeepResearch的两阶段训练流程：

### 3.1 阶段一：SFT冷启动（监督微调）

使用DR-TTS探索的搜索轨迹进行多轮SFT，训练模型学习工具调用模式和跨模态信息整合能力。

**训练目标**：
```
L_SFT = -E_{τ~D} Σ_{i=1}^n log p_θ(t_i, c_i, a_n | Q, t_{<i}, c_{<i}, r_{<i})
```

其中：
- 最大化模型生成的推理规划步骤t_i、搜索工具调用c_i和最终答案a_n的似然
- 工具响应r_i从损失中屏蔽（作为观察条件用于下一步预测）
- 最终答案仅在最后一步生成和监督

**SFT配置**：
- 框架：LLaMA-Factory
- 批次大小：128
- 学习率：5e-6
- 训练轮数：3 epoch

### 3.2 阶段二：多轮强化学习

使用离线搜索引擎，采用GRPO（Group Relative Policy Optimization）算法进行多轮RL训练。

**奖励计算**：
```
R = α * R_acc + (1-α) * R_format
```

- **格式奖励R_format**：鼓励结构化多轮推理和有效工具调用
  - 符合预期交互格式：奖励1
  - 违反格式或产生无法解析的工具调用：奖励0

- **准确性奖励R_acc**：评估最终答案是否正确
  - 使用强LLM作为评判器，将模型答案与标准答案对比
  - 相比规则化评估更加灵活准确

**组相对优势计算**：
```
A_g = R_g - (1/G) * Σ_{k=1}^G R_k
```

**GRPO优化目标**：
```
J_GRPO(θ) = E[(1/G) * Σ_{i=1}^G min( (π_θ(τ_i|I,T) / π_θ_old(τ_i|I,T)) * A_i, clip(...) ) - β * D_KL(π_θ || π_ref)]
```

**RL配置**：
- 框架：VeRL
- 全局批次大小：128
- 每提示rollouts数：5
- 学习率：1e-6
- 每轨迹最大工具调用数：5
- 最大训练上下文长度：70,000 tokens（支持长多模态工具响应）

**计算资源**：
- 7B/8B模型：8×NVIDIA H100
- 32B模型：32×NVIDIA H100
- 离线搜索引擎和评判模型：8×NVIDIA H100

## 四、实验结果与性能分析

### 4.1 评测基准

作者在6个信息密集型基准上评估MM-DeepResearch：

1. **SimpleVQA**：评估MLLM回答基于图像的短自然语言问题时的事实准确性
2. **MMSearch**：多模态浏览基准，要求整合视觉和文本信号与外部检索
3. **LiveVQA**：大规模视觉问答数据集，关注最新视觉知识
4. **FVQA**：需要外部常识知识才能正确回答的视觉问答数据集
5. **InfoSeek**：专注于信息寻求问题的视觉问答数据集

## 五、个人理解与行业影响分析

### 5.1 技术创新点评析

#### 5.1.1 Hyper-Search的超图建模思想

Hyper-Search的核心创新在于将**超图理论引入多模态数据合成**。传统图结构仅支持边连接两个节点，而超边可以连接任意数量的节点，这天然适合建模复杂的多模态信息依赖关系。

这种建模方式的优势在于：
1. **跨模态关系显式化**：图像与文本、图像与图像之间的复杂关系被显式编码
2. **多跳推理自然支持**：超图的多层扩展天然支持多跳信息检索
3. **数据质量可控制**：通过超边内/跨超边的QA生成策略，可精确控制问题复杂度

**启示**：超图建模不仅适用于数据合成，也可能成为未来多模态知识表示的重要方向。

#### 5.1.2 DR-TTS的分而治之策略

DR-TTS采用的"分解-重组"策略体现了经典的**分而治之（Divide and Conquer）**思想。这一策略的巧妙之处在于：

1. **降低学习难度**：单一专家只需精通一种工具，避免了"样样通、样样松"的问题
2. **提高探索效率**：重组后的专家在树搜索中分工协作，覆盖更广泛的决策空间
3. **可扩展性强**：新增工具类型只需训练新的专家，不影响已有专家

这种思想与Mixture of Experts (MoE)有异曲同工之妙，但在工具学习这一特定场景下更具针对性。

#### 5.1.3 离线搜索引擎的工程创新

离线搜索引擎看似是工程层面的优化，实则具有重要的研究价值：

1. **降低研究门槛**：使得更多研究团队能够承担多模态智能体训练
2. **加速迭代周期**：60倍的速度提升意味着更快的实验反馈循环
3. **可控的训练环境**：离线环境便于调试和分析模型行为

**潜在风险**：离线搜索引擎的检索质量依赖于预构建语料库的覆盖度和时效性，可能无法完全模拟真实网络环境的动态性。

### 5.2 与相关工作对比

#### 5.2.1 与MMSearch-R1的对比

MMSearch-R1是首个使用端到端RL赋予MLLM图像和文本搜索工具能力的工作。

| 维度 | MMSearch-R1 | MM-DeepResearch |
|-----|-------------|-----------------|
| 数据生成 | 未公开详细方法 | Hyper-Search超图方法 |
| 轨迹合成 | 未详细说明 | DR-TTS分解-重组策略 |
| 训练成本 | 依赖在线API | 离线搜索引擎降低成本 |
| 性能（7B） | 49.0（平均） | 56.1（平均） |
| 开源程度 | 部分开源 | 完全开源（代码+数据） |

MM-DeepResearch在方法论透明度和性能上均有优势，为社区提供了更强的基线。

#### 5.2.2 与WebWatcher的对比

WebWatcher通过合成搜索密集型QA对并转换为VQA数据进行训练。

| 维度 | WebWatcher | MM-DeepResearch |
|-----|------------|-----------------|
| 核心方法 | QA合成+VQA转换 | 超图QA+工具轨迹+RL |
| 数据规模 | 未公开 | 3K QA + 10K轨迹 |
| 训练范式 | 主要为SFT | SFT + RL两阶段 |
| 性能（7B） | SimpleVQA: 54.3, MMSearch: 49.1 | SimpleVQA: 62.0, MMSearch: 61.4 |

MM-DeepResearch通过引入RL训练阶段，显著提升了模型在复杂搜索任务上的表现。

### 5.3 行业影响预测

#### 5.3.1 对学术研究的影响

1. **开源基线推动社区发展**：MM-DeepResearch的完全开源（包括代码、数据和训练细节）将为后续研究提供坚实的起点，可能催生一系列改进工作。

2. **方法论范式的确立**：
   - 超图建模可能成为多模态数据合成的标准方法之一
   - 分解-重组的训练策略可能被应用到更广泛的工具学习场景
   - 离线搜索引擎的设计思路可能被其他研究团队借鉴

3. **评测基准的演进**：BrowseComp-VL等复杂基准的引入，将推动多模态智能体向更深度的研究能力发展。

#### 5.3.2 对工业应用的影响

1. **企业级搜索增强**：
   - 企业内部文档、图像、视频的多模态检索与问答
   - 结合私有知识库的定制化深度研究智能体
   - 降低对昂贵在线API的依赖，降低部署成本

2. **内容创作辅助**：
   - 基于多源信息的长文写作
   - 跨模态内容的事实核查
   - 研究报告的自动生成

3. **教育与研究**：
   - 学术研究文献的深度检索与综述
   - 教育领域跨模态知识的问答与解释
   - 专业领域（医疗、法律、金融）的知识检索

#### 5.3.3 潜在挑战与风险

1. **离线搜索引擎的局限性**：
   - 语料库的时效性难以保证
   - 特定领域知识覆盖可能不足
   - 无法获取最新信息（如新闻、股市等）

2. **幻觉与错误传播**：
   - 多轮搜索可能累积错误信息
   - 工具调用失败或返回低质量结果时的降级策略
   - 跨模态信息整合中的语义偏差

3. **安全与伦理考量**：
   - 智能体可能被用于生成虚假信息
   - 深度研究能力的滥用（如隐私侵犯）
   - 自动化内容生产的版权与归属问题

### 5.4 未来研究方向

基于MM-DeepResearch的工作，以下方向值得进一步探索：

1. **动态语料更新机制**：研究如何高效更新离线搜索引擎的语料库，平衡成本与时效性。

2. **多智能体协作**：扩展至多智能体场景，多个专业智能体协作完成更复杂的深度研究任务。

3. **长上下文优化**：当前最大上下文已达70K tokens，随着搜索轮次增加，更高效的上下文管理机制将至关重要。

4. **跨语言能力**：当前工作主要关注英文，多语言多模态深度研究能力值得探索。

5. **可解释性增强**：提升智能体决策过程的可解释性，使用户能够理解信息来源和推理链条。

## 六、总结

MM-DeepResearch是多模态深度研究领域的一项里程碑式工作。通过Hyper-Search、DR-TTS和离线搜索引擎三个核心创新，作者成功解决了该领域面临的数据稀缺、轨迹缺失和训练成本高昂三大挑战。

该工作的主要贡献可总结为：

1. **方法论创新**：首次将超图引入搜索密集型QA数据合成，提出分解-重组的工具学习策略
2. **工程突破**：构建成本低廉、效率高效的离线多模态搜索引擎
3. **性能领先**：在多个基准上实现SOTA性能，且完全开源
4. **社区价值**：为多模态智能体研究提供了简单有效的基线

从技术演进的角度看，MM-DeepResearch代表了从静态RAG到动态Agentic Search的重要进步，也是多模态大模型向自主智能体演进的关键一步。随着类似工作的不断涌现，我们可以期待在不久的将来，AI智能体将具备真正意义上的深度研究能力，成为人类知识探索的有力助手。
