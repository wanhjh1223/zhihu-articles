## 一、研究背景与核心问题

### 1.1 深度研究智能体的兴起

随着大型语言模型(LLM)能力的飞速提升，配备网络搜索功能的智能体已经展现出在深度研究任务中的巨大潜力。从OpenAI的Deep Research到Google Gemini的深度研究功能，再到Grok3的DeeperSearch，这些系统能够在不同领域进行综合信息检索、代码编写执行以及迭代式调查研究。这标志着AI系统正在从简单的问答助手向能够自主完成复杂研究任务的智能体演进。

然而，当前大多数深度研究智能体都依赖于人工设计的提示工程(prompt engineering)工作流。这些预定义的工作流虽然能够在特定场景下表现良好，但存在显著的局限性：严格的行为模式导致指令遵循能力不足、推理一致性差、对新任务的泛化能力有限，并且需要大量人工调优才能达到可靠性能。

### 1.2 现有技术路线的瓶颈

当前深度研究智能体的技术路线主要分为两类，但都面临根本性挑战：

**第一类：基于提示工程的智能体**

这类方法如OpenResearcher、AirRAG、IterDRAG、Plan*RAG等，通过精心设计的人工工作流来指导LLM如何与外部知识源交互。尽管这些方法在搜索能力方面取得了显著进展，但其依赖人工设计的提示和交互模式导致了严格的行为限制，难以适应复杂多变的研究场景。

**第二类：基于RAG环境的强化学习方法**

近期研究如Search-R1、R1-Searcher、ReSearch等尝试将强化学习(RL)与信息检索相结合。然而，这些方法主要局限于使用静态、本地文本语料库的检索增强生成(RAG)环境。这种受控的RAG设置存在三个关键问题：

1. **信息时效性衰减**：固定知识库无法获取最新信息
2. **领域适应性差**：无法处理训练时未覆盖的领域
3. **根本性假设缺陷**：假设所有必要信息都存在于固定知识库中，这在真实场景中往往不成立

更重要的是，RAG系统无法模拟真实网络搜索环境中的噪声、搜索质量变化以及处理多样化网页内容格式的挑战。

### 1.3 DeepResearcher的核心突破

DeepResearcher是首个在真实网络环境中进行端到端强化学习训练的深度研究智能体框架。与现有方法相比，它的核心创新在于：

- **真实世界环境**：直接与实时搜索引擎交互，而非使用静态知识库
- **端到端训练**：无需人工先验，让智能体自主发现问题解决策略
- **多智能体架构**：专门的浏览智能体从各种网页结构中提取相关信息
- **规模化RL训练**：成功解决了真实网页环境中的API限制、网络延迟、反爬虫机制等技术挑战

## 二、技术方法详解

### 2.1 整体架构设计

DeepResearcher采用了一种专门设计用于处理动态真实网络环境的多智能体架构。该架构包含三个核心组件：

**1. 主研究智能体 (DeepResearcher)**

主智能体负责整体的研究流程控制。它接收用户问题，进行推理，决定何时使用搜索工具，何时浏览网页，以及何时生成最终答案。每个推理过程都被包裹在`<think>`标签中，这一设计遵循DeepSeek-R1的设置，使推理过程可追踪和可解释。

**2. 网络搜索工具 (Web Search Tool)**

当主智能体需要搜索信息时，它会生成一个JSON格式的请求，包含搜索查询。搜索工具返回结构化的结果，包括每个网页的标题、URL和摘要片段。当前实现采用固定的top-k值（如10）来检索搜索结果。

**3. 网页浏览智能体 (Web Browsing Agent)**

这是DeepResearcher架构中最具创新性的组件。浏览智能体专门负责从网页中提取相关信息。它维护一个短期记忆库，接收网页浏览请求后，按顺序处理URL的页面片段。基于查询、历史记忆和新获取的网页内容，浏览智能体执行两个关键动作：

- 决定是否继续阅读下一个URL/片段或停止
- 将相关信息追加到短期记忆中

这种设计模仿人类浏览网页的行为：如果URL的初始片段主要包含无关信息，则该网页可能不值得继续阅读，从而能够更有效地分配资源并提高整体信息提取效率。

### 2.2 深度研究轨迹 (Deep Research Trajectory)

DeepResearcher的研究轨迹体现了迭代式推理和工具使用的结合：

```
用户问题 → 推理 → 搜索查询 → 获取结果 → 浏览网页 → 信息提取 → 推理 → ... → 最终答案
```

这个循环会持续进行，直到智能体确定已经收集了足够的信息来回答问题。在整个过程中，智能体需要做出一系列复杂决策：

- **查询生成**：根据当前对问题的理解，生成有效的搜索查询
- **结果筛选**：从搜索结果中识别最相关的网页
- **深度浏览**：决定是否深入浏览某个网页获取更多细节
- **信息综合**：将来自多个来源的信息整合成连贯的理解
- **终止判断**：确定何时已经收集到足够的信息

### 2.3 强化学习训练框架

DeepResearcher采用Group Relative Policy Optimization (GRPO)算法进行训练。GRPO通过利用参考策略和由现有策略生成的一组轨迹来优化当前策略。

**GRPO的核心机制**：

给定G个轨迹 τ = {y_i}_{i=1}^G ~ π_{θ_old}(·|x)，GRPO使用这些轨迹估计基线，而不是训练单独的critic网络。当前策略通过最大化以下目标函数进行优化：

```
J(θ) = E_{x~D, {y_i}_{i=1}^G ~ π_{θ_old}(·|x)} [
    (1/G) * Σ_{i=1}^G [ min( (π_θ(y_i|x) / π_{θ_old}(y_i|x)) * A_i,
                          clip(π_θ(y_i|x) / π_{θ_old}(y_i|x), 1-ε, 1+ε) * A_i ) ]
    - β * D_KL(π_θ || π_{θ_ref}) ]
]
```

其中A_i是优势函数，β是KL散度惩罚的系数。

**观测掩码 (Observation Masking)**：

工具输出的是观测(observation)，而非模型预期生成的结果。因此，DeepResearcher应用掩码机制，防止观测参与训练，仅允许模型的响应部分贡献到训练损失中。这一设计确保了模型学习的是如何生成合适的动作，而不是简单地复制观测内容。

### 2.4 奖励设计

奖励在训练过程中起着关键作用，引导智能体不断提升性能。DeepResearcher采用F1分数作为主要奖励指标，这是因为训练使用了带简短答案的开放域QA数据集。

**奖励结构**：

```
reward = {
    -1,  如果格式不正确
    F1分数, 如果格式正确
}
```

- **格式惩罚**：如果输出格式不正确（如缺少标签或结构错误），智能体获得-1的惩罚
- **F1奖励**：如果格式正确，奖励基于词级别的F1分数，衡量生成答案与参考答案的匹配程度

这种简单但有效的奖励设计使模型能够通过结果反馈自主学习复杂的搜索和推理策略。

### 2.5 真实世界环境的技术挑战与解决方案

在开放的实时网络环境中训练RL智能体面临诸多独特挑战，DeepResearcher提出了针对性的解决方案：

**挑战一：高并发请求的延迟问题**

GRPO的实现导致大量采样迭代，产生大量的搜索查询和网页爬取操作（如4096个），造成严重延迟。

**解决方案**：构建分布式CPU服务器集群（50个节点），专门管理RL rollout过程中产生的工具请求。每个服务器负责处理部分请求、处理搜索结果，并根据语言模型标识的URL进行网页爬取。

**挑战二：网页爬取和API限制**

在爬取阶段，系统经常遇到网站服务器的反爬措施，可能返回无关内容或完全不响应。同时，搜索引擎或LLM API的提供商限制（如每秒200次）也是一大障碍。

**解决方案**：
- 实现鲁棒的重试机制，有效处理API调用或网页爬取过程中遇到的异常
- 引入搜索结果缓存策略：如果在预定时间内（如7天）发出相同的搜索查询，系统从缓存中检索结果
- 这一策略不仅减少API调用频率，还有助于管理相关成本（特别是对于Google Search API等昂贵服务）

**挑战三：多智能体信息提取优化**

许多网页篇幅较长但相关内容有限，直接处理整个网页效率低下。

**解决方案**：采用多智能体框架，将网页分割成较小的片段。浏览智能体按顺序从第一页开始处理内容，假设URL的初始片段如果主要包含无关信息，则该网页可能不具有生产力，可以跳过。

### 2.6 数据构建与污染检测

**训练数据来源**：

尽管对深度研究能力的兴趣日益增长，但目前不存在专门为这一目的设计的开源训练数据集。DeepResearcher利用现有的开放域问答数据集，包含从单跳到多跳的各种检索复杂度问题：

- **单跳场景**：NaturalQuestions (NQ)、TriviaQA (TQ) - 答案通常可在单个网页文档中找到
- **多跳场景**：HotpotQA、2WikiMultiHopQA (2Wiki) - 需要整合多个来源的信息

**数据污染检测**：

为了确保模型真正学习利用网络搜索工具，而非简单地回忆记忆信息，DeepResearcher实施了全面的两阶段过滤方法：

1. **低质量问题过滤**：排除可能产生不可靠搜索结果的问题，包括：
   - 时间敏感问题（如"苹果现任CEO是谁？"）
   - 高度主观查询（如"最好的智能手机是什么？"）
   - 潜在有害或违反政策的内容

2. **污染检测**：对于每个候选问题，从将要用于训练的基础模型中随机采样10个响应，检查是否有响应包含正确答案（pass@10）。如果模型在不使用搜索的情况下就能产生正确答案，则该问题被排除在训练集之外。

经过质量控制后，构建了包含80,000个示例的最终训练数据集，分布比例为NQ:TQ:HotpotQA:2Wiki = 1:1:3:3。这一比例刻意强调多跳场景（75%的示例），因为这些更好地反映了深度研究所需的复杂信息寻求行为。

## 三、实验结果与性能表现

### 3.1 实验设置

**基础模型**：Qwen2.5-7B-Instruct  
**训练框架**：verl框架  
**训练参数**：
- 每步采样256个prompt
- 每个prompt采样16个rollout
- 每个rollout最多10次工具调用 + 最终答案步骤
- mini-batch大小：4,096

### 3.2 评估基准

为了全面评估模型在分布内(ID)和分布外(OOD)设置下的性能，DeepResearcher构建了一个多样化的基准测试套件：

**分布内评估**：
- NaturalQuestions (NQ) Dev集
- TriviaQA (TQ) Dev集
- HotpotQA Dev集
- 2WikiMultiHopQA (2Wiki)

**分布外评估**：
- MuSiQue：测试模型超越训练分布的泛化能力
- Bamboogle：问题风格和信息分布显著不同的数据集
- PopQA：评估模型在处理不同知识类型时的鲁棒性

为确保公平和平衡的评估，从各数据集开发集中随机采样512个示例（Bamboogle使用全部125个样本）。

### 3.3 基线方法

DeepResearcher与以下基线方法进行比较：

1. **CoT Only**：使用思维链(Chain-of-Thought)推理生成答案，无外部参考上下文
2. **RAG**：结合思维链推理与检索参考上下文指导答案生成
3. **Search-o1**：多步推理基线，模型生成搜索查询或中间答案，上下文限于检索器检索的片段
4. **Search-o1+Web Search**：与Search-o1相比，允许模型访问开放网络，可通过API发送实时搜索查询并访问URL浏览网页
5. **Search-r1**：使用强化学习进行问答的方法，在训练和推理阶段使用检索器搜索Wikipedia信息
6. **R1-Searcher**：给定搜索查询时，在查询后附加"site:en.wikipedia.org"，通过Bing搜索并总结前三个搜索结果页面

### 3.4 主要实验结果

**分布内性能**（表1）：

| 方法 | NQ (F1) | TQ (F1) | HotpotQA (F1) | 2Wiki (F1) |
|------|---------|---------|---------------|------------|
| CoT Only | 0.204 | 0.587 | 0.282 | 0.325 |
| RAG | 0.363 | 0.706 | 0.336 | 0.367 |
| Search-o1 | 0.151 | 0.503 | 0.187 | 0.176 |
| Search-o1+Web | 0.284 | 0.650 | 0.318 | 0.352 |
| Search-r1-base | 0.393 | 0.721 | 0.370 | 0.414 |
| R1-Searcher | 0.360 | 0.674 | 0.338 | 0.376 |
| **DeepResearcher** | **0.412** | **0.756** | **0.438** | **0.521** |

DeepResearcher在所有四个数据集上的MBE（模型-based评估）指标均优于所有基线方法，在F1指标上表现出强劲性能，特别是在TQ和2Wiki上表现卓越。

值得注意的是，Search-r1-base在本地RAG环境中训练和评估，可直接访问相关Wikipedia语料库，而DeepResearcher必须在整个互联网中导航寻找信息，尽管面临更真实和更具挑战性的环境，仍取得了优秀结果。

**与提示工程方法的比较**：

相比提示工程基线（Search-o1+Web），DeepResearcher实现了**最高28.9分**的显著提升。这表明直接交互真实搜索环境对于开发鲁棒研究能力至关重要。

**与RAG-based RL方法的比较**：

相比RAG-based RL智能体（Search-r1-base），DeepResearcher实现了**最高7.2分**的改进。这证明了在真实网络环境而非受控RAG环境中进行训练的价值。

### 3.5 涌现的认知行为

定性分析揭示了DeepResearcher端到端RL训练产生的几个重要认知行为：

**行为一：规划能力**

在处理多跳问题时，DeepResearcher能够制定计划并在推理过程中动态调整。值得注意的是，模型可以在适当时合并步骤，表明规划能力可以在没有显式规划数据SFT的情况下自然涌现。

**案例示例**：
```
问题：谁是首届诺贝尔物理学奖得主的最著名学生的出生地？

规划过程：
1. 首先找出首届诺贝尔物理学奖得主 → 伦琴
2. 然后找出伦琴最著名的学生 → 可能需要在多个来源中确认
3. 最后确定该学生的出生地

动态调整：
- 在搜索过程中发现伦琴的著名学生信息较为分散
- 决定扩大搜索范围，查找"伦琴 学生 著名"
- 综合多个来源确认是某位特定科学家
```

**行为二：交叉验证**

DeepResearcher在第一次工具调用时就识别出了正确答案。然而，它没有立即提交这个结果，而是通过后续步骤验证其准确性。这种谨慎的方法增强了模型响应的可靠性，确保最终预测的更大鲁棒性。

**案例示例**：
```
第一次搜索：直接找到答案"巴黎"

但模型没有立即输出答案，而是：
- 进行第二次搜索，验证该信息
- 查找其他来源确认
- 检查是否有矛盾信息
- 确认后才给出最终答案

这种自我质疑和验证的行为类似于人类研究者的严谨态度。
```

**行为三：自我反思与重定向**

当遇到搜索困难或发现当前路径无效时，DeepResearcher能够识别问题并调整搜索策略。

**案例示例**：
```
初始搜索方向未能找到相关信息

模型行为：
- 识别当前搜索策略的低效
- 反思："我的查询可能过于具体，需要更一般化"
- 调整：使用更宽泛的关键词重新搜索
- 成功找到相关信息
```

**行为四：诚实性**

当无法找到确切答案时，DeepResearcher会诚实地承认限制，而不是编造答案。这是深度研究智能体的重要品质，体现了系统的可靠性和可信度。

**案例示例**：
```
经过多次搜索后仍无法找到明确答案

模型输出：
"基于我的搜索，我无法找到这个问题的确切答案。多个来源提供了不同的信息，但没有足够的权威来源可以确认。我建议查阅[特定专业数据库]获取更准确的信息。"
```

## 四、核心创新点分析

### 4.1 真实世界RL训练的开创性

DeepResearcher的最大创新在于它是首个成功在真实网络环境中规模化应用强化学习的深度研究框架。此前的工作要么局限于静态RAG环境，要么依赖人工设计的提示工程。

**技术突破**：

1. **分布式架构**：50节点CPU集群处理高并发搜索和爬取请求
2. **鲁棒性机制**：重试策略、缓存系统应对API限制和网络异常
3. **多智能体设计**：专门的浏览智能体处理多样化网页结构

**研究意义**：

证明了在真实环境而非受控模拟中进行端到端训练不仅是实现细节上的改进，而是开发能够在真实应用中有效执行的鲁棒研究能力的根本要求。

### 4.2 端到端学习范式

DeepResearcher摒弃了人类设计的工作流，采用端到端学习方法。这种范式转变带来几个优势：

**自主性**：智能体自主发现问题解决策略，而非遵循预设规则
**适应性**：能够根据具体问题和环境动态调整策略
**可扩展性**：随着RL训练的扩展，系统能力持续提升
**简洁性**：仅需结果奖励，无需复杂的中间步骤奖励设计

### 4.3 多智能体协作架构

浏览智能体的引入是架构设计上的精妙之处。它解决了以下问题：

**效率问题**：无需将整个网页内容输入主模型，浏览智能体只提取相关信息
**专注问题**：主模型可以专注于高层推理和决策，细节处理交由专业智能体
**可扩展性**：可以独立优化浏览策略，而不会影响主模型

### 4.4 涌现能力的发现

论文详细记录了RL训练产生的多种涌现认知行为，这是对LLM能力的深刻洞察：

**规划无需显式教导**：模型通过结果奖励自主学习多步规划
**自我纠正自然形成**：交叉验证行为在训练中出现，无需专门设计
**元认知能力**：模型能够评估自己的知识边界，知道何时应该继续搜索，何时应该承认无法回答

这些发现表明，强化学习能够解锁LLM固有的、但未被传统监督学习方法发掘的能力。

## 五、局限性与未来方向

### 5.1 当前局限性

**评估指标限制**：

当前主要使用F1分数和MBE进行评估，适用于短答案QA任务。对于长格式答案，需要更复杂的奖励机制，如OpenAI Deep Research系统卡片中提到的评估方法。

**搜索空间限制**：

虽然相比RAG环境已大幅扩展，但仍受限于搜索引擎的索引范围。某些专业数据库或付费墙后的内容可能无法访问。

**计算成本**：

50节点的分布式集群要求对于资源有限的研究者来说门槛较高。如何降低训练成本是未来需要考虑的问题。

**安全与伦理**：

在真实网络环境中训练带来了内容安全的新挑战，如可能接触到有害信息或偏见内容。论文提到进行了内容过滤，但详细的安全评估需要更多研究。

### 5.2 未来研究方向

**长答案生成**：

扩展框架以支持深度研究报告的生成，而非仅短答案。这可能需要开发更复杂的奖励模型来评估长文本质量。

**多模态研究**：

整合图像、视频等多模态信息的检索和理解能力，使智能体能够处理更丰富的研究内容。

**持续学习**：

开发机制使智能体能够从每次研究任务中学习并更新知识，逐步提升研究能力。

**协作研究**：

多个专业智能体协作完成复杂研究任务，如一个负责文献综述，一个负责数据分析，一个负责综合结论。

**效率优化**：

探索更高效的训练方法，如课程学习、迁移学习等，降低计算资源需求。

## 六、个人理解与行业影响分析

### 6.1 对Agentic AI的深远影响

DeepResearcher代表了AI智能体发展的重要里程碑。它证明了：

**真实环境训练的必要性**：

此前许多研究在受控环境中取得了好结果，但在真实场景中表现不佳。DeepResearcher明确表明，要构建真正实用的研究智能体，必须在真实环境中训练和评估。

**RL的普适性**：

继DeepSeek-R1在数学和代码领域成功后，DeepResearcher将RL的应用扩展到了开放域研究任务。这表明RL可能是解锁LLM在各种复杂任务中潜力的通用方法。

**涌现能力的可靠性**：

论文详细记录的涌现行为表明，通过适当的训练方法，LLM可以自主发展出复杂的认知能力，而无需显式编程。这为未来AI系统的能力发展提供了新思路。

### 6.2 对研究范式的启示

**从Prompt Engineering到Learning**：

DeepResearcher的成功标志着AI研究范式从精心设计提示向端到端学习的转变。这种转变可能适用于更多AI应用场景。

**评估标准的演进**：

真实世界性能应成为AI系统评估的核心标准，而非仅关注在受控基准测试上的表现。

**开源与透明**：

论文完全开源训练框架，这对于推动领域发展和建立可信AI至关重要。相比OpenAI Deep Research的封闭性，这种开放态度值得赞赏。

### 6.3 产业应用前景

**企业研究助手**：

DeepResearcher类型的系统可以显著加速企业市场调研、竞品分析、技术追踪等工作。

**学术研究支持**：

辅助研究人员进行文献综述、跨学科信息整合、假设验证等工作。

**内容创作**：

为记者、作家、分析师提供深度背景调研支持。

**教育与培训**：

作为学习辅助工具，帮助学生在复杂课题上进行深度探索。

### 6.4 竞争格局变化

DeepResearcher的开源发布可能改变深度研究AI的竞争格局：

- 降低了进入门槛，更多研究机构和企业可以开发自己的深度研究智能体
- 推动行业从封闭系统向开放协作转变
- 可能加速相关技术的迭代和创新

### 6.5 技术挑战与机遇

**挑战**：

- 如何在更大规模上扩展训练
- 如何处理更复杂的多模态信息
- 如何确保生成内容的准确性和可靠性

**机遇**：

- 结合更多工具（代码执行、数据分析、可视化）构建更强大的研究智能体
- 开发个性化研究助手，适应特定领域或用户偏好
- 探索人机协作的新模式，让AI和人类研究者各自发挥优势

## 七、结论

DeepResearcher是深度研究智能体领域的重要突破。它首次成功实现了在真实网络环境中端到端训练LLM智能体，通过规模化强化学习让智能体自主发展出规划、交叉验证、自我反思等复杂认知能力。

该研究不仅在技术上取得了显著进展（相比基线方法提升最高28.9分），更重要的是，它证明了真实环境训练对于开发鲁棒AI系统的根本重要性。这为未来Agentic AI的发展指明了方向。

随着该框架的开源，我们可以期待看到更多基于真实环境训练的深度研究智能体涌现，推动AI在复杂知识工作领域的应用迈上新台阶。
