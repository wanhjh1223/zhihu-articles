## 一、研究背景与核心问题

### 1.1 大模型推理能力的发展困境

随着DeepSeek-R1、OpenAI o系列等推理模型的成功，**可验证奖励强化学习（Reinforcement Learning with Verifiable Rewards, RLVR）**已成为提升大语言模型（LLM）推理能力的核心范式。RLVR通过二元奖励信号（正确+1/错误-1）直接优化模型，在数学推理、代码生成等可验证领域取得了显著成果。

然而，RLVR的**内在机制**长期以来是一个"黑箱"。传统认知认为：
- **正样本强化（Positive Sample Reinforcement, PSR）**是提升性能的主要驱动力
- **负样本（错误响应）**主要起辅助作用，帮助模型避免错误

这种认知源于监督学习（SL）的直觉——我们需要告诉模型"什么是正确的"。但RLVR与SL有着本质区别：它使用策略梯度同时学习正确和错误样本。

### 1.2 核心研究问题

本研究提出了一个挑战传统认知的问题：

> **在RLVR中，负样本强化（NSR）究竟扮演什么角色？仅使用负样本训练能否有效提升模型推理能力？**

这个问题看似简单，实则触及强化学习本质：当我们只惩罚错误而不奖励正确时，模型能否学会推理？

### 1.3 研究动机

近期的研究发现了一些令人困惑的现象：

1. **探索天花板问题**：Yue等人（2025）的研究表明，RLVR主要提升采样效率（Pass@1），而非真正扩展模型的推理能力边界（Pass@256）

2. **多样性下降**：经过RLVR训练的模型往往比基模型具有更少的语义多样性和操作多样性（Dang等，2025）

3. **过拟合风险**：PSR可能导致输出分布过度集中，损害模型生成多样化正确响应的能力

这些现象暗示：**我们可能过度关注"教模型正确答案"，而忽视了"让模型自己发现正确答案"的重要性**。

## 二、技术方法详解

### 2.1 RLVR的数学分解

#### 2.1.1 标准RLVR目标函数

RLVR的目标函数基于REINFORCE算法：

```
L_RLVR(θ) = -E_{x~D}[Σ_y r(x,y) · π_θ(y|x)]
```

其中：
- `r(x,y) ∈ {-1, +1}` 是可验证奖励
- `π_θ(y|x)` 是策略模型生成响应y的概率

#### 2.1.2 分解为PSR和NSR

本研究的核心创新是将RLVR目标函数分解为两个组成部分：

```
L_RLVR(θ) = L_PSR(θ) + L_NSR(θ)

其中：
L_PSR(θ) = -E_{x~D}[Σ_{y:r(x,y)=1} π_θ(y|x)]    (正样本强化)
L_NSR(θ) = -E_{x~D}[Σ_{y:r(x,y)=-1} -π_θ(y|x)]  (负样本强化)
```

这种分解的直觉是：
- **PSR** 增加正确响应的似然
- **NSR** 减少错误响应的似然

### 2.2 独立训练实验设计

为了隔离PSR和NSR的效果，研究团队设计了独立的训练设置：

| 训练设置 | 使用的样本 | 目标函数 |
|---------|-----------|---------|
| PSR-only | 仅正确响应 | `L = -E[Σ_{正确} π_θ(y|x)]` |
| NSR-only | 仅错误响应 | `L = -E[Σ_{错误} -π_θ(y|x)]` |
| PPO | 正确+错误 | 标准PPO |
| GRPO | 正确+错误 | 组相对策略优化 |

### 2.3 梯度分析：理解NSR的作用机制

#### 2.3.1 NSR的梯度公式

对于NSR，其对参数θ的梯度为：

```
∇_θ L_NSR = E_{x~D}[Σ_{y:r(x,y)=-1} ∇_θ log π_θ(y|x)]
```

关键洞察：**NSR梯度会抑制错误响应的概率质量，但不会指定这些概率质量应该去向何方**。

#### 2.3.2 概率质量重分配机制

通过token级别的梯度分析，研究揭示了NSR的工作机制：

1. **抑制错误路径**：NSR降低生成错误推理步骤的概率
2. **重分配概率质量**：被抑制的概率质量根据模型的**先验信念**重新分配
3. **保留探索能力**：由于NSR不强制特定正确路径，模型保留了探索其他可能解决方案的能力

这与PSR形成鲜明对比：
- **PSR** 将概率质量集中到已采样的正确路径上
- **NSR** 允许模型基于先验知识"自主发现"正确路径

### 2.4 Weighted-REINFORCE算法

基于上述分析，研究团队提出了**Weighted-REINFORCE**，通过上调NSR权重来平衡PSR和NSR：

```
L_Weighted-REINFORCE = L_PSR + α · L_NSR
```

其中α > 1是NSR的权重系数。实验发现，适当增加NSR权重可以：
- 保持高Pass@1性能
- 显著提升高k值的Pass@k性能
- 平衡准确性与多样性

## 三、实验设计与结果分析

### 3.1 实验设置

#### 3.1.1 模型
- **Qwen2.5-Math-7B**：数学专用模型
- **Qwen3-4B**：通用多语言模型
- **Llama-3.1-8B-Instruct**：指令微调模型

#### 3.1.2 数据集
- **MATH**：7,500道竞赛级数学题
- **AIME 2025**：美国数学邀请赛题目
- **AMC 12/10**：美国数学竞赛题目

#### 3.1.3 训练配置
- Prompt batch size: 1,024
- 每prompt生成rollouts: 8
- 采样温度: 1.0
- 最大上下文长度: 4,096 (Qwen2.5-Math-7B), 32,768 (Qwen3-4B)
- Mini-batch size: 256
- 学习率: 1e-6

### 3.2 核心实验结果

#### 3.2.1 NSR-only训练的惊人效果

**主要发现**：仅使用负样本训练（NSR-only）能够：

1. **在全Pass@k谱上持续改进基模型**：从Pass@1到Pass@256，NSR-only均优于基模型
2. **匹配或超越PPO和GRPO**：在多个数据集上，NSR-only的性能与完整RL算法相当
3. **保持输出多样性**：在高k值下性能不退化

具体数据（Qwen2.5-Math-7B在MATH上）：

| 方法 | Pass@1 | Pass@64 | Pass@256 |
|------|--------|---------|----------|
| 基模型 | 52.3% | 78.5% | 85.2% |
| PSR-only | 58.7% | 72.1% | 74.3% |
| **NSR-only** | **55.4%** | **80.2%** | **87.1%** |
| PPO | 59.2% | 78.9% | 84.6% |
| GRPO | 60.1% | 79.5% | 85.8% |

#### 3.2.2 PSR的多样性代价

PSR-only训练显示出明显的**多样性-准确性权衡**：

- **Pass@1提升显著**：PSR-only在Pass@1上表现最好
- **高k值性能下降**：随着k增加，PSR-only性能增长缓慢，最终低于基模型
- **输出分布坍塌**：PSR过度集中概率质量于早期正确响应

这一现象在AIME 2025上尤为明显：

| 方法 | Pass@1 | Pass@256 |
|------|--------|----------|
| 基模型 | 15.2% | 48.6% |
| PSR-only | 22.8% | 38.4% |
| **NSR-only** | **18.5%** | **52.3%** |

**关键洞察**：PSR虽然提升了贪婪解码准确率，但损害了模型的探索能力和生成多样性。

#### 3.2.3 推理时扩展趋势

研究还观察到了不同的**推理时扩展（Inference-Time Scaling）**行为：

- **NSR**：随着采样预算增加，性能持续提升，与基模型保持竞争力
- **PSR/PPO/GRPO**：在中等k值后性能增长放缓，甚至下降

这表明NSR更好地保留了模型的**推理时扩展能力**——这是实现"思考更久、答案更好"的关键。

### 3.3 训练动态分析

#### 3.3.1 学习曲线

训练过程中的Pass@1变化：

- **PSR**：快速提升，但很快饱和
- **NSR**：提升较慢但稳定，最终达到相当水平
- **PPO/GRPO**：结合两者优势，早期快速+长期稳定

#### 3.3.2 输出多样性量化

通过衡量响应的语义多样性（嵌入空间中的分散程度）：

| 方法 | 语义多样性 | 操作多样性 |
|------|-----------|-----------|
| 基模型 | 1.00 (基准) | 1.00 (基准) |
| PSR-only | 0.62 | 0.58 |
| NSR-only | 0.89 | 0.85 |
| GRPO | 0.71 | 0.68 |

NSR-only显著保留了更多的多样性。

### 3.4 Weighted-REINFORCE效果

在标准RLVR目标中增加NSR权重（α=2.0）：

| 数据集 | 基线GRPO | Weighted-REINFORCE | 提升 |
|--------|----------|-------------------|------|
| MATH | 60.1% | 62.8% | +2.7% |
| AIME 2025 | 28.4% | 31.2% | +2.8% |
| AMC 23 | 68.5% | 71.3% | +2.8% |

这种简单修改在多个基准上带来一致改进。

## 四、深入分析与技术洞察

### 4.1 为什么NSR如此有效？

#### 4.1.1 "纠正式学习" vs "发现式学习"

传统监督学习是**纠正式学习**（Corrective Learning）：
- 直接告诉模型"正确答案是什么"
- 模型被动接受知识

NSR代表**发现式学习**（Discovery Learning）：
- 只告诉模型"这个答案不对"
- 模型主动探索其他可能性
- 最终答案来自模型先验知识的重组

#### 4.1.2 概率质量重分配的数学解释

考虑一个简单的多选题场景：

假设基模型对选项A、B、C、D的概率分布为：
```
P(A)=0.4, P(B)=0.3, P(C)=0.2, P(D)=0.1
```

假设正确答案是B，但模型最初不知道。

**PSR的训练效果**：
- 大幅推高P(B)，压制其他选项
- 最终分布可能变成：P(B)=0.95, 其他=0.016
- 问题：如果采样时没选到B，后续也很难选到

**NSR的训练效果**（假设采样到A是错误的）：
- 压低P(A)
- 释放的概率质量按原始比例分配给B、C、D
- 新分布：P(A)=0.1, P(B)=0.45, P(C)=0.30, P(D)=0.15
- B的概率自然上升，同时保留探索C、D的能力

这就是为什么NSR能够保持多样性——**它不强制正确答案，只是排除错误答案**。

#### 4.1.3 与贝叶斯学习的联系

NSR可以看作一种**贝叶斯更新**：
- 先验：基模型的初始分布
- 证据："这个响应是错误的"
- 后验：排除错误响应后的条件分布

这与人类学习过程相似：当被告知"这个解法不对"时，我们会回退到已知的其他方法尝试，而非被直接告知正确答案。

### 4.2 PSR的隐性代价

#### 4.2.1 模式坍塌（Mode Collapse）

PSR-only训练会导致：
1. **早期正确响应被过度强化**
2. **其他可能的正确路径被抑制**
3. **模型陷入局部最优**

实验中观察到的现象：
- PSR-only模型在Pass@1上很高
- 但增加采样预算（k>8）后性能反而不如基模型
- 说明模型失去了生成多样化正确响应的能力

#### 4.2.2 探索能力退化

这与Yue等人（2025）发现的"探索天花板"现象一致：
- RLVR训练后，模型的输出轨迹主要分布在基模型的预训练分布内
- 很少发现全新的解题策略
- PSR加剧了这一问题，因为它加速了"已知正确路径"的概率集中

### 4.3 NSR的先验依赖

NSR的效果高度依赖基模型的**先验质量**：

- **强先验模型**（如Qwen2.5-Math-7B）：NSR-only效果显著，因为模型已经"知道"正确答案，只是混杂在错误选项中
- **弱先验模型**：NSR-only可能效果有限，因为排除错误后，剩余选项中正确答案的概率可能仍然不高

这解释了为什么NSR在数学推理任务上特别有效——这些模型的预训练已经编码了大量数学知识。

### 4.4 与现有工作的联系

#### 4.4.1 对比：RLVR-Decomposed vs 传统RL

| 方面 | 传统RL (PPO/GRPO) | RLVR-Decomposed |
|------|------------------|-----------------|
| 学习信号 | 混合使用正负样本 | 可独立分析PSR/NSR |
| 主要关注点 | 最大化奖励 | 理解学习机制 |
| 多样性 | 中等 | NSR保留更多多样性 |
| 实用性 | 成熟稳定 | 需要调参（NSR权重）|

#### 4.4.2 与相关研究的互补性

- **RLOO (Ahmadian等, 2024)**：改进基线估计，可与NSR权重调整结合
- **DAPO (Yu等, 2025)**：动态采样过滤无效prompt，可与NSR-only训练结合
- **Dr. GRPO (Liu等, 2025)**：分析GRPO的优化偏差，NSR提供了新的视角

## 五、行业影响与启示

### 5.1 对RLVR训练范式的重新思考

#### 5.1.1 从"奖励正确答案"到"惩罚错误答案"

传统RLVR训练强调"奖励正确响应对齐模型行为"。本研究表明：

> **"惩罚错误响应"可能是同样甚至更重要的学习信号**

这对RLVR训练实践有直接影响：
1. **负样本不应被浪费**：即使一个prompt的所有rollout都是错误的，仍然可以通过NSR学习
2. **零方差组的问题有解**：GRPO在处理全对或全错的组时梯度为零，但NSR可以单独处理错误响应
3. **数据效率提升**：可以从错误样本中提取学习信号

#### 5.1.2 多样性与准确性的平衡

NSR-only训练展示了**不牺牲多样性也能提升准确性**的可能性：

- 这对需要**推理时扩展**的场景特别重要
- 如o1-style的"思考更久"范式，依赖模型生成多样化推理路径
- PSR-only的多样性损失可能限制模型的长期推理潜力

### 5.2 对模型评估的启示

#### 5.2.1 Pass@k指标的重要性

本研究强调了**Pass@k（尤其是高k值）**的重要性：

- Pass@1只反映采样效率
- Pass@256反映模型的**推理能力边界**
- 一个模型可能在Pass@1上领先，但在Pass@256上落后（如PSR-only）

建议的评估标准：
```
综合性能 = w1·Pass@1 + w2·Pass@64 + w3·Pass@256
```

其中w1+w2+w3=1，且w3应该足够大以反映真实推理能力。

#### 5.2.2 多样性的量化指标

除了准确率，还应该评估：
- **语义多样性**：响应嵌入的分散程度
- **解法多样性**：不同解题策略的比例
- **探索能力**：新测试集上的零样本表现

### 5.3 实际应用建议

#### 5.3.1 何时使用NSR-only训练？

NSR-only训练特别适合以下场景：

1. **高质量基模型**：当基模型已经具备较强先验时
2. **数据受限**：当正确标注数据稀少，但错误响应容易获得时
3. **需要保留多样性**：当应用场景需要多样化输出（如创意写作、头脑风暴）
4. **推理时扩展**：当计划使用大规模采样（k>64）时

#### 5.3.2 混合策略

更实用的方案是**PSR+NSR的加权组合**：

```python
# 推荐的训练配置
nsr_weight = 1.5  # 上调NSR权重
psr_weight = 1.0  # 标准PSR权重

# 动态调整
if epoch < warmup_epochs:
    # 早期更多PSR，快速提升Pass@1
    nsr_weight = 1.0
else:
    # 后期增加NSR，保持多样性
    nsr_weight = 2.0
```

#### 5.3.3 数据策略

基于NSR的洞察，可以优化数据收集：

1. **错误样本也有价值**：不要丢弃所有错误的rollout
2. **困难负样本挖掘**：关注那些"接近正确"的错误响应
3. **多样性采样**：确保错误响应覆盖不同的错误类型

### 5.4 对未来研究的启发

#### 5.4.1 研究方向1：自适应NSR权重

开发根据训练动态自动调整NSR权重的算法：
- 当Pass@1提升停滞时，增加PSR权重
- 当Pass@k（k>64）下降时，增加NSR权重

#### 5.4.2 研究方向2：错误响应的质量评估

并非所有错误响应都有同等学习价值：
- "思路正确但计算错误" > "完全错误的思路"
- 开发评估错误响应"质量"的指标
- 优先学习"高质量错误"

#### 5.4.3 研究方向3：NSR在其他领域的应用

将NSR思想扩展到其他领域：
- **代码生成**：从编译错误/测试失败中学习
- **科学推理**：从实验失败中学习
- **创造性任务**：从"不够好"的作品中学习

#### 5.4.4 研究方向4：理论分析

更深入的理论理解：
- NSR收敛性的形式化证明
- PSR导致模式坍塌的理论解释
- 最优PSR/NSR权重的理论预测

## 六、局限性与未来工作

### 6.1 研究局限性

#### 6.1.1 任务范围

本研究主要聚焦于**数学推理**任务：
- 数学领域有明确的对错判断
- 其他领域（如开放域问答、创意写作）的可验证性较弱
- NSR在这些领域的有效性需要进一步验证

#### 6.1.2 模型规模

实验使用的模型规模（7B-8B）相对较小：
- 更大模型（70B+）上的NSR效果需要验证
- 模型规模与NSR效果的关系尚不清楚
- 超大规模模型可能表现出不同的学习动态

#### 6.1.3 先验依赖

NSR的效果高度依赖基模型的先验质量：
- 弱基模型上NSR可能效果有限
- 如何量化"先验质量"与NSR效果的关系是开放问题

#### 6.1.4 训练稳定性

NSR-only训练在某些设置下可能不稳定：
- 错误响应的梯度信号可能噪声较大
- 需要仔细调整学习率和batch size

### 6.2 待解决问题

#### 6.2.1 最优权重如何确定？

Weighted-REINFORCE中的NSR权重α是一个超参数：
- 不同任务/模型的最优α可能不同
- 需要系统性的超参数搜索方法
- 可能实现α的自适应调整

#### 6.2.2 错误响应的选择策略

所有错误响应都有同等学习价值吗？
- "接近正确"的错误 vs "完全错误"
- 系统性错误 vs 随机错误
- 可能需要优先学习某些类型的错误

#### 6.2.3 与过程奖励的结合

本研究使用**结果奖励**（最终答案对错）：
- 结合**过程奖励**（中间步骤评估）可能进一步提升效果
- NSR可以扩展到过程级别的负样本学习

#### 6.2.4 长链推理（Long-CoT）场景

在极长推理链（如DeepSeek-R1的数万token推理）中：
- NSR的效果如何？
- 如何定位错误发生的具体步骤？
- 局部NSR vs 全局NSR的选择

## 七、个人理解与思考

### 7.1 哲学层面：什么是"学习"？

这项研究促使我重新思考"学习"的本质：

**传统观点**：学习就是获取正确答案  
**新视角**：学习是排除错误答案后的自然涌现

这与人类学习经验相似：
- 学习骑自行车时，没有人在每一步告诉你"正确的姿势是什么"
- 而是通过无数次"这个姿势会摔倒"的负反馈，身体自然找到平衡
- 最终掌握的平衡感不是被"教"出来的，而是错误被排除后的剩余

NSR-only训练展示了AI学习的一种新可能：**模型可能不需要被告知"什么是正确的"，只需要被告知"什么是错误的"，就能自主发现正确路径**。

### 7.2 实践层面：训练策略的再平衡

在实践中，我建议采用**"NSR优先"**的策略：

```
推荐训练流程：
1. 预训练（获得强先验）
2. NSR-only微调（保留多样性，筛选错误）
3. 少量PSR微调（精确对齐已知正确路径）
4. 持续NSR+PSR混合训练（平衡准确性与多样性）
```

这种"先NSR后PSR"的顺序可能优于同时进行：
- NSR阶段帮助模型"清理"错误认知
- PSR阶段在此基础上"精炼"正确路径

### 7.3 技术层面：与Curriculum Learning的联系

NSR-only训练可以看作一种特殊的**课程学习**（Curriculum Learning）：

- **传统课程学习**：从简单样本到复杂样本
- **NSR课程学习**：从"排除错误"到"强化正确"

这种视角提示我们：
- 可以设计更复杂的NSR课程（如先排除明显错误，再排除细微错误）
- 错误响应的排序策略可能影响学习效果

### 7.4 行业层面：开源社区的机遇

这项研究对开源社区特别有价值：

1. **降低数据门槛**：不需要大量正确标注数据，错误响应也能训练
2. **计算效率**：NSR-only可能收敛更快（不需要同时优化正负样本的平衡）
3. **可解释性**：分解PSR/NSR帮助理解模型学到了什么

期待看到更多基于NSR的开源实现和复现。

### 7.5 长远展望：通向AGI的学习范式

如果将视野放得更远，NSR-only训练可能代表了一种更接近人类智能的学习方式：

**人类学习特点**：
- 大部分学习时间是在"试错"中度过
- 成功的定义往往是"没有失败"
- 发现新知识的时刻通常是"排除了所有错误选项后"

**AGI的可能路径**：
- 不是给模型灌输更多"正确答案"
- 而是给模型更好的"错误检测能力"
- 让模型在排除错误的过程中自主发现真理

NSR-only训练的成功是这一方向的重要信号。

## 八、结论

《The Surprising Effectiveness of Negative Reinforcement in LLM Reasoning》是一项**颠覆传统认知**的重要研究。其核心贡献包括：

### 8.1 主要贡献

1. **理论贡献**：首次系统分解RLVR为PSR和NSR，揭示了负样本强化的独立价值

2. **实证贡献**：证明NSR-only训练在全Pass@k谱上有效，甚至匹配/超越PPO/GRPO

3. **方法贡献**：提出Weighted-REINFORCE，通过简单调整NSR权重获得一致性能提升

4. **洞察贡献**：揭示PSR的多样性代价和NSR的概率质量重分配机制

### 8.2 核心启示

- **"惩罚错误"可能和"奖励正确"同等重要**
- **保持多样性对推理时扩展至关重要**
- **基模型先验质量是NSR效果的关键**
- **简单的目标函数修改可以带来显著改进**

### 8.3 对行业的影响

这项研究将：
1. **改变RLVR训练实践**：更多关注负样本的价值
2. **影响模型评估标准**：Pass@k指标比Pass@1更全面
3. **启发新的研究方向**：自适应权重、错误质量评估等
4. **降低训练门槛**：错误数据也能有效训练

### 8.4 最终评价

这是一篇**方法简单但洞察深刻**的论文。它没有提出复杂的算法或架构，而是通过细致的分解和分析，揭示了RLVR中一个被忽视的重要维度。

在LLM推理能力快速提升的今天，理解"模型如何学习"与"模型学到了什么"同等重要。这项研究为我们提供了一个新的视角：**也许我们不需要教模型更多正确答案，只需要帮助它更好地排除错误选项，正确答案就会自然涌现**。

对于正在探索RLVR训练的研究者和工程师，这篇论文**必读**。
