## 一、研究背景与核心问题

### 1.1 从静态推理到战略推理的范式跃迁

大语言模型（LLMs）在数学推理、代码生成和逻辑推理等静态任务上取得了令人瞩目的成就。以OpenAI的o1、DeepSeek-R1、Kimi k1.5为代表的推理模型，通过大规模强化学习训练，在具有明确规则和确定性结果的问题上展现出接近甚至超越人类专家的能力。然而，这些成功掩盖了一个关键局限：**真实世界的复杂性远非静态任务所能涵盖**。

商业谈判、政策设计、多方博弈等场景要求模型具备**战略推理（Strategic Reasoning）**能力——这是一种在动态环境中导航、在不确定性中管理长期目标、并随条件变化持续适应的高级认知能力。与静态推理不同，战略推理具有以下核心特征：

- **动态交互**：环境状态随多方行为持续演化，不存在固定的解题路径
- **信息不完全**：参与者的目标和策略相互保密，需要基于有限信息进行推断
- **长期规划**：短期决策必须服务于长期目标，需要权衡即时收益与未来可能性
- **开放动作空间**：可能的行动选择几乎无限，而非预定义的有限集合

### 1.2 现有方法的根本局限

现有提升LLM战略推理能力的方法可分为三类，但均存在明显缺陷：

**（1）提示工程方法**（如ReAct、Chain-of-Thought）
- 依赖模型固有的推理能力，难以实时适应环境反馈
- 缺乏针对长期目标优化的机制，容易陷入局部最优
- 无法有效处理多轮交互中的策略调整

**（2）监督微调与强化学习**（如直接对LLM进行RL训练）
- 在特定领域训练后难以泛化到新场景（领域过拟合）
- 过度优化行为输出，损害模型的通用语言能力
- 训练成本高，且需要大量标注数据

**（3）推理时搜索**（如MCTS、Best-of-N）
- 计算开销巨大，难以扩展到开放动作空间
- 针对每轮交互独立优化，缺乏长期一致性
- 在实时交互场景中响应延迟过高

### 1.3 EPO的核心洞见

EPO（Explicit Policy Optimization，显式策略优化）的核心创新在于**解耦战略推理与行为执行**：

> 与其训练单一模型同时负责"思考策略"和"执行动作"，不如专门训练一个"战略推理模型"（LLM_s）负责生成高层策略，而保持"行为执行模型"（LLM_d）不变，仅通过策略引导其行为。

这种架构设计带来三重优势：

1. **专业化分工**：战略模型专注于长期目标对齐，行为模型保持通用语言能力
2. **即插即用**：战略模型可无缝集成到任意LLM代理，无需修改被辅助模型
3. **持续优化**：通过强化学习不断优化策略生成，而不影响行为模型的泛化能力

## 二、技术方法详解

### 2.1 架构设计：双模型协作系统

EPO采用双模型架构，清晰定义了战略推理与行为执行的分工：

```
┌─────────────────────────────────────────────────────────────┐
│                     EPO 系统架构                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────┐         策略 a_t         ┌──────────────┐  │
│  │  战略推理模型 │ ───────────────────────→ │  行为执行模型 │  │
│  │   LLM_s      │                         │   LLM_d      │  │
│  │  (可训练)    │                         │  (冻结权重)   │  │
│  └──────────────┘                         └──────────────┘  │
│         ↑                                       │            │
│         │      交互历史 h_t, 观察 x_t           │            │
│         └───────────────────────────────────────┘            │
│                          ↓                                  │
│                    环境反馈 (奖励 r_t)                        │
└─────────────────────────────────────────────────────────────┘
```

**战略推理模型（LLM_s）**：
- **输入**：目标G、交互历史h₁:t₋₁、先前策略a₁:t₋₁、当前观察x_t
- **输出**：自然语言形式的高层策略a_t
- **公式化**：a_t = LLM_s(s_sys, G, h₁:t₋₁, a₁:t₋₁, x_t)

**行为执行模型（LLM_d）**：
- **输入**：目标G、交互历史h₁:t₋₁、策略a_t、系统提示d_sys
- **输出**：具体行为响应y_t
- **公式化**：y_t = LLM_d(d_sys, G, h₁:t₋₁, a_t, x_t)

关键设计在于：LLM_d**选择性采纳**LLM_s生成的策略，允许其在策略不合理时进行覆盖，这种机制既保证了策略引导，又保留了行为模型的自主判断能力。

### 2.2 多轮强化学习训练框架

EPO将战略推理建模为**部分可观察马尔可夫决策过程（POMDP）**，设计了轻量级多轮RL训练流程：

#### 2.2.1 问题建模

- **状态**：交互历史h_t（包含过往观察与行为）
- **动作**：自然语言策略a_t
- **奖励**：过程奖励r_t（评估策略对目标推进的关键性）
- **目标**：最大化期望累积折扣回报

$$J(θ) = \mathbb{E}_{τ∼π_θ}[R(τ)] = \mathbb{E}_{π_θ}\left[\sum_{t'=t}^{T} γ^{T-t'} r_{t'}\right]$$

其中γ∈[0,1)为折扣因子，控制模型对长期回报的偏好程度。

#### 2.2.2 REINFORCE策略优化

EPO采用经典的REINFORCE算法进行策略梯度优化：

$$\mathcal{L}(θ) = -\mathbb{E}_{π_θ} \left[ \frac{1}{T} \sum_{t=1}^{T} R_t \frac{1}{|k_t|} \sum_{i=0}^{k_t} \log π_θ(a_{t,i} | h_{1:t-1}, a_{t,1:i-1}, x_t) \right]$$

其中：
- a_{t,i}表示策略a_t的第i个token
- k_t为策略a_t的token总数
- R_t为从t到T的累积折扣奖励

为稳定训练，EPO采用**最大绝对值归一化**处理R_t，确保不同长度轨迹的梯度尺度一致。

#### 2.2.3 过程奖励模型（PRM）

与仅关注最终结果的终端奖励不同，EPO引入**过程奖励**评估每个策略的关键性：

$$r_t = \begin{cases} 1 & \text{若}a_t\text{被判定为关键策略} \\ 0 & \text{否则} \end{cases}$$

PRM通过另一个LLM（LLM_p）实现，输入完整交互历史h₁:T和策略序列a₁:T，输出关键策略索引集合：

$$[S_{idx}] = LLM_p(p_{sys}, G, h_{1:T}, \text{score}, a_{1:T})$$

过程奖励的设计使得战略模型能够在**短期战术合理性**与**长期策略一致性**之间取得平衡。

### 2.3 迭代自博弈训练

为扩展RL训练规模，EPO采用**迭代自博弈**机制：

```
迭代 0: 使用CoT生成的策略初始化LLM_s
    ↓
迭代 n: 两个EPO实例作为不同角色进行交互
    ↓
记录完整轨迹（交互历史 + 策略序列）
    ↓
PRM评估策略 → 获得过程奖励
    ↓
基于Eq.4更新LLM_s策略
    ↓
若策略未收敛，进入迭代 n+1
```

**关键设计**：在自博弈过程中，LLM_d的权重始终保持冻结，确保其通用能力不因特定领域训练而退化。这种"只训练策略生成器、不训练行为执行器"的范式，从根本上避免了传统RL方法的过优化问题。

### 2.4 算法无关性与扩展性

虽然EPO采用REINFORCE作为基础算法，但其框架具有**算法无关性**：
- 可替换为PPO等在线RL方法
- 可结合DPO等离线偏好优化方法
- 可引入基线函数减少梯度方差
- 可结合搜索技术（如采样多个候选策略并rerank）

## 三、实验结果与性能表现

### 3.1 实验设置

EPO在三个具有代表性的战略推理环境中进行评估：

| 环境 | 类型 | 任务描述 | 评估指标 |
|------|------|----------|----------|
| **SOTOPIA** | 社交对话 | 多轮社交交互场景，参与者有隐藏目标 | 目标完成度(0-10)、综合得分 |
| **WebShop** | 网络导航 | 电商网站购物任务 | 平均奖励(0-1) |
| **ALFWorld** | 具身智能 | 家庭环境中的指令执行任务 | 平均奖励(二值) |

**基线方法**：
- Vanilla：标准提示方法
- ReAct：CoT推理+行动生成
- PPDPP：对话行为规划（基于预定义动作集）
- DAT：连续动作向量规划

**实现细节**：
- 基础模型：Llama3-8B-Instruct
- 学习率：1e-6，余弦退火调度
- Batch size：32
- 折扣因子γ：0.99
- 训练步数：19k（SOTOPIA）/ 20k（跨域训练）

### 3.2 主要结果

#### 3.2.1 SOTOPIA社交对话任务

| 方法 | 目标完成度 | 综合得分 |
|------|-----------|---------|
| Vanilla | 4.8 | 5.2 |
| ReAct | 5.1 | 5.5 |
| PPDPP | 5.5 | 6.1 |
| DAT | 5.7 | 6.0 |
| **EPO-Llama3-8B (SFT)** | 5.8 | 6.3 |
| **EPO-Llama3-8B (RL)** | **7.3** | **7.6** |
| EPO-Claude-3.5-Sonnet | 6.5 | 6.9 |
| EPO-GPT-4o | 6.8 | 7.1 |

**关键发现**：
1. 即使是未经额外训练的现成LLM（EPO-Claude/GPT-4o），作为战略推理模型也显著优于传统提示方法
2. 经过RL训练的EPO-Llama3-8B（仅8B参数）超越所有基线，甚至超过使用更大模型的方案
3. RL训练相比SFT带来显著性能提升（+0.5目标完成度）

#### 3.2.2 WebShop与ALFWorld跨域评估

| 方法 | WebShop Seen | WebShop Unseen | ALFWorld Seen | ALFWorld Unseen |
|------|-------------|----------------|---------------|-----------------|
| ReAct | 58.2 | 52.1 | 68.5 | 55.3 |
| PPDPP | 59.5 | 54.8 | 70.2 | 57.8 |
| **EPO-Llama3-8B (RL)** | **65.7** | **61.3** | **77.4** | **66.8** |
| EPO + GPT-4o | 67.2 | 63.1 | 79.1 | 68.5 |

**关键发现**：
1. 在未见任务（Unseen）上，EPO展现出强大的**策略迁移能力**
2. 即使是较弱的Llama3-8B行为模型，在EPO策略引导下也能在ALFWorld-Unseen上取得66.8%的成功率（相比ReAct提升11.5%）
3. 将EPO策略模型接入GPT-4o，达到该任务领域的SOTA水平

### 3.3 消融实验与分析

#### 3.3.1 迭代自博弈的扩展效应

实验显示，随着RL迭代轮次增加，性能稳步提升：
- 第1轮：目标完成度6.2
- 第3轮：目标完成度6.9
- 第5轮：目标完成度7.3
- 第6轮后：边际收益递减，策略趋于稳定

这表明：**多轮迭代自博弈是策略优化的关键**，但过度训练会导致收益饱和。

#### 3.3.2 显式策略优化的必要性

对比实验表明，将策略与行为合并训练（单一模型同时输出策略和行为）会导致：
- 目标完成度下降至6.5（-0.8）
- 行为输出过拟合训练场景，泛化能力下降

这验证了**解耦架构的核心价值**：分离关注允许战略模型专注于长期目标对齐，而不干扰行为模型的通用能力。

### 3.4 涌现的协作机制

定性分析揭示了EPO中涌现的多种协作模式：

1. **策略引导型**：LLM_s提供详细行动建议，LLM_d严格遵循执行
2. **意图对齐型**：LLM_s阐明高层次目标，LLM_d自主决定具体行动
3. **动态调整型**：LLM_d根据环境反馈向LLM_s请求策略更新

这种多样化的协作机制表明，EPO框架允许不同能力的行为模型以最适合其能力的方式利用战略指导。

## 四、核心创新点深度解析

### 4.1 战略推理的形式化建模

EPO的核心贡献之一是将战略推理从模糊的"能力"概念转化为可优化的**显式策略学习问题**。通过POMDP建模，战略推理被形式化为：

> 在部分可观察的动态环境中，学习一个映射函数π_θ: (G, h, x) → a，使得长期累积回报最大化。

这种形式化使得：
- **可衡量**：战略质量可通过回报量化评估
- **可优化**：可使用RL算法持续改进策略
- **可迁移**：学到的策略函数可应用于不同行为模型

### 4.2 过程奖励与长期目标对齐

传统RL方法关注终端结果（如是否完成任务），而EPO的过程奖励设计体现了对**长期规划能力**的培养：

- **关键策略识别**：PRM不仅评估"是否成功"，更评估"哪些决策对成功至关重要"
- **中间步骤监督**：允许在长时间跨度的交互中提供密集反馈
- **信用分配**：明确每个策略对最终结果的贡献度

这种设计尤其适合长时程任务，避免了稀疏奖励带来的学习效率低下问题。

### 4.3 开放动作空间的策略生成

与先前工作依赖预定义动作集不同，EPO允许LLM_s在**自然语言空间**中自由生成策略：

- **表达能力**：可表达复杂、创造性的策略（如"先展示紧迫感，再强调价值"）
- **领域无关**：无需为每个新领域重新定义动作空间
- **可解释性**：人类可理解的策略描述，便于调试和信任建立

表10展示了一个谈判案例的对比：
- ReAct：僵化地坚持初始报价
- SFT策略：结构化让步但缺乏创新
- **RL策略**：创造性地提出紧迫感制造、价值说服等高级策略

### 4.4 行为模型冻结策略

EPO的关键设计决策是**在RL训练过程中冻结LLM_d**：

| 训练策略 | 领域性能 | 通用能力 | 迁移性 |
|----------|---------|---------|--------|
| 联合训练 | 高 | 下降 | 受限 |
| 仅训练LLM_s | 高 | 保持 | 强 |

这一决策的理论基础是：**行为能力是基础模型的通用属性，不应因特定任务优化而退化**。通过仅优化策略生成层，EPO实现了：
- 保持行为模型的多领域泛化能力
- 避免过拟合特定交互模式
- 允许策略模型在不同行为模型间灵活迁移

## 五、个人理解与行业影响分析

### 5.1 EPO的范式意义

EPO代表了一种从**端到端训练**向**模块化强化学习**的范式转变：

```
传统范式：单一模型 ← RL ← 环境反馈
              ↓
EPO范式：  策略模型 ← RL ← 环境反馈
              ↓（策略指导）
         行为模型（冻结）← 交互环境
```

这种转变的深层含义是：**并非模型的所有组件都需要针对特定任务优化**。将通用能力（语言理解、行为生成）与专用能力（特定领域的策略规划）分离，可能是一种更高效的 scaling 路径。

### 5.2 对Agent系统的启示

当前Agent系统的设计往往面临两难：
- **通用Agent**：适应性强但任务完成率低
- **专用Agent**：完成率高但难以迁移

EPO提供了一条"两全其美"的路径：
- 保持通用Agent的行为模型不变
- 为特定任务训练专用的策略插件
- 通过组合实现专业化能力

这种"基础Agent + 策略插件"的架构可能成为未来Agent系统的主流设计模式。

### 5.3 战略推理能力的行业应用

EPO的技术框架可直接应用于以下场景：

**（1）商业谈判助手**
- 输入：谈判目标、对方历史行为、当前报价
- 输出：策略建议（"先建立信任，再逐步透露底线"）
- 价值：提升谈判成功率，减少培训成本

**（2）智能客服升级**
- 输入：客户问题、情绪状态、解决目标
- 输出：应对策略（"先共情，再提供方案，最后确认满意度"）
- 价值：提升客户满意度，降低人工介入率

**（3）游戏AI与虚拟角色**
- 输入：角色目标、游戏状态、其他玩家行为
- 输出：角色策略（"暂时结盟，收集资源，后期反击"）
- 价值：创造更具挑战性和真实感的游戏体验

**（4）自动驾驶决策**
- 输入：目的地、交通状况、其他车辆行为预测
- 输出：驾驶策略（"保持车距，准备变道，预留制动距离"）
- 价值：提升复杂交通场景下的安全性和效率

### 5.4 局限性与未来方向

尽管EPO取得了显著进展，但仍存在以下局限：

**（1）奖励设计依赖**
- 过程奖励依赖PRM的质量，而PRM本身需要精心设计
- 复杂场景中的"关键策略"定义可能存在歧义

**（2）计算开销**
- 迭代自博弈需要大量交互采样
- 多轮RL训练的计算成本仍较高

**（3）理论理解不足**
- 为何解耦架构优于联合训练仍需更深入的理论分析
- 策略模型的最优容量与行为模型的匹配关系尚不明确

**未来研究方向**：
1. **自适应策略粒度**：根据任务复杂度动态调整策略的抽象层次
2. **多策略融合**：允许同时生成多个候选策略，由行为模型选择
3. **元学习增强**：通过元学习提升策略模型在新领域的快速适应能力
4. **人机协作策略**：将人类专家知识整合到策略学习过程中

### 5.5 对LLM研究趋势的判断

EPO的出现反映了LLM研究的几个重要趋势：

**（1）从通用到专用**
基础模型的通用能力已接近饱和，未来竞争焦点将转向**特定领域的高效适配**。EPO的模块化设计恰好适应这一趋势。

**（2）从训练到推理时计算**
与Test-Time Scaling类似，EPO代表了"通过推理时计算提升性能"的范式。但EPO更进一步：**将推理时计算转化为可学习的策略**。

**（3）从单模型到系统**
单一LLM的能力存在瓶颈，多模型协作系统将成为突破方向。EPO的双模型架构是这一趋势的早期例证。

---

## 六、总结

EPO通过显式策略优化为LLM战略推理能力的发展开辟了新路径。其核心贡献包括：

1. **架构创新**：提出战略推理模型与行为执行模型的解耦设计，实现了专业化分工与即插即用
2. **方法创新**：将战略推理形式化为POMDP，设计了基于过程奖励的多轮RL训练框架
3. **实验验证**：在SOTOPIA、WebShop、ALFWorld等多个领域取得SOTA性能，验证了策略迁移能力
4. **深度洞察**：揭示了多样化协作机制的涌现现象，证明了开放动作空间策略生成的有效性

EPO的范式意义超越了具体技术细节，为**模块化Agent系统**的设计提供了理论框架和实践路径。在LLM能力边际收益递减的背景下，通过架构创新实现能力的"组合式爆发"，可能成为推动领域发展的关键驱动力。

对于研究者而言，EPO提示了一个重要方向：**并非所有问题都需要通过扩大模型规模解决，精巧的架构设计和训练策略同样能带来质的飞跃**。对于从业者而言，EPO的"策略插件"模式为垂直领域Agent的低成本定制提供了可行方案。

随着多Agent系统、长期规划任务、复杂交互场景的需求持续增长，EPO所代表的战略推理优化范式有望在更多领域展现其价值，推动LLM从"静态推理器"向"动态决策者"的演进。
