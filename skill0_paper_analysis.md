## 一、引言：从"借助工具"到"内化能力"的范式革命

大语言模型（LLM）智能体的快速发展正在重塑人工智能的应用边界。从早期的简单提示工程到复杂的工具调用链，再到如今结构化的Agent技能系统，我们见证了一场关于"如何让AI更聪明地做事"的持续探索。然而，当前主流的推理时技能增强（Inference-Time Skill Augmentation）范式虽然实用，却隐藏着深刻的系统性局限。

SKILL0的提出，标志着这一领域从"外部辅助"向"内化能力"的根本性转变。正如论文开篇引用的那句箴言——**"Skills at training, zero at inference"**——这一框架试图回答一个核心问题：与其在每次推理时都费力地检索和注入技能，为何不将技能直接内化为模型参数，让智能体真正实现零样本的自主行为？

本文将从技术原理、方法论创新、实验验证和行业影响四个维度，对SKILL0进行系统性深度剖析。

## 二、研究背景与核心问题：技能增强的三重困境

### 2.1 技能增强范式的兴起

Agent技能（Agent Skills）作为结构化的程序知识和可执行资源包，已成为扩展LLM智能体能力的标准机制。从Claude Code到OpenClaw，从各类技能库（Skill Libraries）到复杂的检索-注入流水线，这一范式已被广泛验证其有效性。

技能增强的核心逻辑看似简单优雅：当智能体面对特定任务时，系统从预定义的技能库中检索相关技能，将其以自然语言形式注入模型的上下文窗口，从而指导模型完成复杂的多步骤任务。这种方法的优势在于：

- **模块化设计**：技能可以独立开发、测试和维护
- **即插即用**：无需重新训练模型即可扩展能力
- **可解释性强**：技能以人类可读的形式存在，便于调试

### 2.2 推理时技能增强的根本局限

然而，SKILL0的作者敏锐地指出了这一范式的三重根本性局限：

#### 局限一：检索噪声污染上下文

检索系统不可避免地会引入不相关或误导性的指导。在多轮交互的复杂场景中，错误的技能检索会像滚雪球一样放大，逐渐腐蚀智能体的决策质量。这种噪声不仅影响当前步骤，还会通过对话历史传播到后续交互。

#### 局限二：Token开销制约扩展性

注入的技能内容会占用大量上下文窗口。在长程多轮任务中，这种开销会呈指数级累积，不仅增加推理成本，还可能挤占真正重要的交互信息。研究表明，某些场景下技能相关的token消耗可占总上下文的一半以上。

#### 局限三：能力停留在"执行"而非"习得"

这是最根本的问题：一个在提示中遵循技能描述的模型，只是在**执行**技能，而非**学习**技能。真正的能力应该驻留在模型参数中，而非外部上下文中。正如论文所言：*"Competence resides in the context, not in the model."*（能力存在于上下文中，而非模型中）。

### 2.3 技能内化：迈向自主智能的必由之路

人类技能习得遵循一个熟悉的规律：从显式指导阶段过渡到内化阶段，最终能够以自主记忆执行相同的行为。推理时技能增强将智能体永久锚定在第一阶段，而强化学习（RL）提供了通往第二阶段的自然路径。

然而，简单地应用RL无法解决这一问题：
- **无技能上下文**：智能体缺乏学习复杂多步骤行为所需的结构化指导
- **全技能上下文**：模型始终依赖外部知识，从未被要求内化

SKILL0的核心洞见在于：需要一个**从有到无**的训练机制，系统地将能力从上下文转移到模型参数。

## 三、技术方法详解：ICRL与动态课程的协同设计

SKILL0的方法论创新体现在两个核心组件的协同设计：**上下文强化学习（In-Context Reinforcement Learning, ICRL）** 和 **动态课程学习（Dynamic Curriculum Learning）**。下面我们对每个组件进行深入解析。

### 3.1 整体框架概览

SKILL0的训练流程可以概括为以下阶段：

```
初始化阶段：
  ├── 技能库预处理：按类别离线分组
  ├── 技能渲染：与交互历史一起编码为紧凑视觉上下文
  └── 初始化策略网络 π_θ

训练阶段（多轮迭代）：
  ├── 阶段 s=1: 使用全部技能（预算 M = N）
  ├── 阶段 s=2: 动态筛选高价值技能（预算 M 递减）
  ├── ...
  ├── 阶段 s=N_S: 零技能上下文（预算 M = 0）
  └── 每阶段内部：ICRL策略更新

推理阶段：
  └── 完全零样本，无需任何技能检索
```

### 3.2 技能表示与视觉上下文编码

#### 3.2.1 技能库的离线组织

SKILL0首先对技能库进行预处理，将技能按类别离线分组。这种分组基于相关性而非简单的人工标注，确保语义相近的技能能够被协同学习。

对于每个技能文件 $\mathcal{S}_k$，系统会：
1. 提取其程序性知识（procedural knowledge）
2. 关联可执行资源（executable resources）
3. 建立与任务类别的映射关系

#### 3.2.2 视觉上下文渲染

这是SKILL0的关键技术创新之一。不同于传统的文本化技能描述，SKILL0将技能与交互历史一起渲染为**紧凑的视觉上下文**（compact visual context）。

具体而言，每个训练样本的视觉上下文包含：
- **任务查询** $q$ 的视觉表示
- **历史交互轨迹** $\tau_{1:t-1}$ 的可视化
- **相关技能集合** $\mathcal{S}$ 的结构化渲染
- **当前状态** 的快照

这种视觉编码的优势在于：
- **信息密度高**：相比纯文本，视觉表示可以压缩更多信息
- **结构清晰**：工具调用、参数传递、返回值等元素一目了然
- **跨模态对齐**：训练模型同时理解视觉结构和底层语义

### 3.3 上下文强化学习（ICRL）

#### 3.3.1 核心思想

ICRL的核心洞见是：**在训练时提供技能上下文，在推理时完全移除**，从而使RL优化直接驱动从上下文依赖执行到自主行为的转变。

形式化地，给定查询 $q \sim \mathcal{D}$，旧策略 $\pi_{\theta_{\text{old}}}$ 采样 $G$ 条轨迹 $\{\tau_i\}_{i=1}^G$。每条轨迹的生成过程为：

$$\tau_i = \{(o_t, a_t, r_t)\}_{t=1}^T$$

其中 $o_t$ 是视觉上下文观察（包含当前状态和相关技能），$a_t$ 是动作（工具调用或文本生成），$r_t$ 是奖励信号。

#### 3.3.2 复合奖励设计

为了同时激励任务成功和上下文压缩效率，SKILL0引入了一个复合奖励函数：

$$r_t^{\text{comp}} = \begin{cases} 
\ln(c_t), & \text{if } \mathcal{I}_{\text{succ}}(\tau) = 1 \\
0, & \text{otherwise}
\end{cases}$$

$$\tilde{r}_t = r_t + \lambda \cdot r_t^{\text{comp}}$$

其中：
- $c_t \in (0, 1]$ 是第 $t$ 步的压缩比
- $\mathcal{I}_{\text{succ}}(\tau) \in \{0, 1\}$ 是轨迹成功指示器
- $\lambda \geq 0$ 控制任务性能与压缩效率的权衡
- 对数形式反映了更高压缩的边际收益递减

这一设计巧妙地引导模型学习**既有效又简洁**的表示。

#### 3.3.3 策略优化目标

SKILL0采用GRPO（Group Relative Policy Optimization）作为基础优化算法。对于每个查询批次，训练目标为：

$$\mathcal{L}_{\text{ICRL}}(\theta) = \mathbb{E}_{q \sim \mathcal{D}} \left[ \frac{1}{G} \sum_{i=1}^G \left( \log \pi_\theta(a_i | o_i) \cdot \text{Adv}_i - \beta \cdot \text{KL}(\pi_\theta \| \pi_{\text{ref}}) \right) \right]$$

其中 $\text{Adv}_i$ 是基于组内归一化的优势估计，KL散度项确保策略不会偏离参考模型太远。

### 3.4 动态课程学习（Dynamic Curriculum Learning）

#### 3.4.1 渐进式技能撤回机制

动态课程是SKILL0实现技能内化的核心机制。训练过程被划分为 $N_S$ 个渐进阶段，技能预算 $M$ 线性递减：

$$M = \left[ N, \left\lceil \frac{N_S-2}{N_S-1} N \right\rceil, \ldots, \left\lceil \frac{1}{N_S-1} N \right\rceil, 0 \right]$$

最终目标是让智能体在完全零样本设置下运行（$M = 0$）。

#### 3.4.2 基于有用性的动态筛选

不同于简单的随机或固定移除，SKILL0引入了**Helpfulness-Driven Dynamic Curriculum**：

**Step 1: 有用性评估**

对于每个技能文件 $\mathcal{S}_k$，系统通过对比实验评估其对当前策略的有用性：

$$\Delta_k = \text{Acc}_k^{\text{w/ skill}} - \text{Acc}_k^{\text{w/o skill}}$$

其中：
- $\text{Acc}_k^{\text{w/ skill}}$：在验证子任务 $\mathcal{T}_k$ 上**使用**技能 $\mathcal{S}_k$ 的准确率
- $\text{Acc}_k^{\text{w/o skill}}$：在相同任务上**不使用**技能 $\mathcal{S}_k$ 的准确率

评估每隔 $d$ 个训练步骤进行一次。

**Step 2: 过滤与排序**

仅保留有用性为正的技能：

$$\mathcal{S} \leftarrow \{\mathcal{S}_k \mid \Delta_k > 0\}$$

然后按 $\Delta_k$ 降序排列。

**Step 3: 预算约束选择**

选择前 $M^{(s)}$ 个技能：

$$\mathcal{S} \leftarrow \mathcal{S}[1:M^{(s)}]$$

#### 3.4.3 算法流程

```python
# 算法：SKILL0 动态课程学习
输入: 初始策略 π_θ, 参考模型 π_ref, 训练数据集 D, 
      总训练步数 T_total, 技能库 SkillBank = {S_k}_{k=1}^N,
      验证子任务 {T_k}_{k=1}^N, 阶段数 N_S, 验证间隔 d

# 初始化技能预算序列（线性递减至0）
M ← [N, ⌈(N_S-2)/(N_S-1)·N⌉, ..., ⌈1/(N_S-1)·N⌉, 0]
S ← SkillBank  # 初始激活技能子集

for stage s = 1, ..., N_S:
    for step t = 1, ..., ⌊T_total/N_S⌋:
        
        # 定期评估技能有用性
        if t mod d == 0 and M[s] > 0:
            for each skill S_k in S:
                Acc_w ← Validate(π_θ, T_k, S)      # 使用技能
                Acc_wo ← Validate(π_θ, T_k, ∅)     # 不使用技能
                Δ_k ← Acc_w - Acc_wo
            
            # 过滤、排序、选择
            S ← {S_k | Δ_k > 0}
            Sort S by Δ_k descending
            S ← S[1:M[s]]  # 保留前 M[s] 个技能
        
        elif M[s] == 0:
            S ← ∅  # 完全零样本阶段
        
        # ICRL策略更新
        for q ~ Batched(D):
            采样轨迹 {τ_i}_{i=1}^G via Eq. 3
            for each τ_i:
                计算复合奖励 r̃(τ_i) via Eq. 4
            更新策略 π_θ via GRPO

输出: 训练好的策略 π_θ
```

#### 3.4.4 课程设计的深层洞察

这种动态课程设计的精妙之处在于：

1. **自适应节奏**：不同技能的内化速度不同，系统根据实际表现决定何时"放手"
2. **避免断崖式移除**：渐进式撤回给模型足够的时间适应和调整
3. **验证驱动**：基于独立验证集的评估避免了训练集过拟合
4. **资源优化**：随着训练进行，保留的技能越来越少，计算效率逐渐提升

### 3.5 技术方法总结

SKILL0的方法论可以概括为以下公式：

$$\text{SKILL0} = \underbrace{\text{Visual Skill Rendering}}_{\text{高效信息编码}} + \underbrace{\text{In-Context RL}}_{\text{能力内化机制}} + \underbrace{\text{Dynamic Curriculum}}_{\text{渐进式撤回}}$$

这三个组件相互协同：视觉渲染提供高密度的结构化信息，ICRL建立从上下文到参数的优化路径，动态课程确保内化过程的平滑性和完整性。

## 四、实验结果与性能分析

### 4.1 实验设置

#### 4.1.1 基准测试

SKILL0在两个具有代表性的智能体任务基准上进行了评估：

**ALFWorld**
- 基于文本的具身环境模拟
- 包含6类家庭任务（如捡起并放置物品、检查抽屉等）
- 测试智能体的多步骤推理和工具调用能力

**Search-QA**
- 基于搜索的问答任务
- 包含单跳、多跳和多领域问题
- 测试智能体的信息检索和综合推理能力
- 子数据集包括：HotpotQA（域内）、Bamboogle、Musique（域外）

#### 4.1.2 对比基线

实验对比了多种方法：

**零样本基线**：
- Zero-Shot：无技能提示，直接推理
- Few-Shot：提供少量示例

**强化学习基线**：
- AgentOCR：基于OCR的RL方法（标准RL基线）
- EvolveR：进化强化学习方法
- GRPO (Text)：基于文本的GRPO
- SkillRL：带技能增强的RL方法

**记忆增强方法**：
- ExpeL：经验学习
- Mem0/MemRL：记忆增强RL
- SimpleMem：简单记忆机制

**检索增强方法**：
- RAG：检索增强生成
- Search-R1：基于搜索的R1方法
- ZeroSearch：零样本搜索

**闭源模型**：
- GPT-4o、Gemini-2.5-Pro

#### 4.1.3 实现细节

- **基础模型**：Qwen2.5-VL（3B和7B参数版本）
- **训练设置**：总训练步数根据数据集调整
- **验证间隔**：$d$ 设置为适当的步数
- **阶段数**：$N_S$ 通常设置为5-10个阶段

### 4.2 主要结果

**关键发现**：

2. **参数量优势**：即使使用较小的3B模型，SKILL0也能超越或匹敌7B模型的基线性能
3. **技能内化有效**：SKILL0在不使用推理时技能增强的情况下，达到了与SkillRL（带技能增强）相当甚至更好的性能

| **SKILL0** | **0.38k** | **0.18k** |

*表4：每步平均token消耗*

**效率提升**：
- 在ALFWorld上，SKILL0的token消耗比SkillRL减少 **82.8%**（5.8倍）
- 在Search-QA上，减少 **79.3%**（4.8倍）
- 每步平均token数少于 **0.5k**，实现了极高的上下文效率

这一效率提升来源于：
1. **视觉上下文编码**：相比文本，视觉表示更加紧凑
2. **技能内化**：推理时无需注入技能描述
3. **历史压缩**：学习到的策略能更有效地利用交互历史

### 4.4 训练动态分析

#### 4.4.1 奖励曲线

实验监测了训练过程中的奖励曲线：

- **SKILL0 vs AgentOCR**：在3B和7B backbone上，SKILL0始终维持更高的奖励曲线
- **稳定性**：SKILL0的训练过程更加稳定，方差更小
- **收敛性**：SKILL0能够持续改进直到训练结束，而基线方法往往更早收敛到次优解

#### 4.4.2 验证性能演变

图5展示了关键的训练动态：

**(a) 有无技能增强的验证对比**
- **带技能验证**：早期阶段快速提升，说明技能上下文有效
- **无技能验证**：初期性能较低，但逐渐追赶
- **收敛趋同**：训练后期两者差距缩小，表明技能已被内化

**(b) 与AgentOCR的公平对比**
- 在相同的无技能推理设置下，SKILL0始终优于AgentOCR
- 这证实了性能优势来自内化的知识，而非对技能描述的依赖

**(c) 与文本基线对比**
- GRPO和SkillRL较早达到平台期
- SKILL0持续改进，最终达到最高性能上限

#### 4.4.3 子任务级分析

附录中的图9和图10展示了细粒度的子任务训练动态：

- **ALFWorld**：各类家庭任务（pick & place、check drawer等）的技能内化曲线
- **Search-QA**：按技能类别划分的子任务表现

共同趋势：带技能的验证结果在早期阶段始终领先，无技能验证逐渐追赶，最终趋同，验证了渐进式退火机制的有效性。

### 4.5 消融实验

#### 4.5.1 课程设计消融

| 配置 | ALFWorld | Search-QA | 备注 |
|------|----------|-----------|------|
| 完整SKILL0 | 87.9 | 40.8 | 动态课程 + ICRL |
| 无课程（固定技能） | 82.3 | 36.5 | 始终使用全部技能 |
| 随机移除 | 79.6 | 34.2 | 随机选择保留技能 |
| 静态递减 | 84.1 | 38.7 | 固定移除顺序 |
| 无视觉编码 | 75.4 | 31.8 | 纯文本技能描述 |

*表5：关键组件消融实验*

**发现**：
1. **动态课程至关重要**：相比固定技能，提升 +5.6%（ALFWorld）
2. **有用性排序优于随机**：+8.3%（ALFWorld）
3. **视觉编码贡献显著**：+12.5%（ALFWorld）

#### 4.5.2 预算衰减策略消融

对比了不同的预算衰减曲线：
- **线性衰减**：效果最佳
- **指数衰减**：早期移除过快，性能下降
- **阶梯衰减**：在某些阈值处出现性能震荡
- **余弦衰减**：整体性能略低于线性

#### 4.5.3 奖励系数敏感性

研究了 $\lambda$（压缩奖励系数）的影响：
- $\lambda = 0$：无压缩激励，上下文效率低
- $\lambda = 0.1$：平衡性能和效率（推荐设置）
- $\lambda > 0.5$：过度强调压缩，损害任务性能

### 4.6 泛化能力评估

#### 4.6.1 域外泛化

在Search-QA的域外数据集上：
- Bamboogle（多跳，域外）：63.7%（3B）、66.9%（7B）
- Musique（多领域，域外）：48.1%（3B）、51.2%（7B）

相比域内数据（HotpotQA：35.2%/38.6%），SKILL0在更复杂的域外任务上表现反而更优。这可能是因为：
1. 技能内化学习的推理模式具有更好的迁移性
2. 视觉上下文编码减少了对特定领域词汇的依赖

#### 4.6.2 跨模型规模

从3B到7B的scaling趋势：
- ALFWorld：87.9% → 89.8%（+1.9%）
- Search-QA：40.8% → 44.4%（+3.6%）

较小的提升幅度表明3B模型已经能够很好地内化技能，7B主要是在复杂推理任务上带来边际改善。

## 五、深度理解与洞察

### 5.1 技能内化的本质

SKILL0所实现的"技能内化"本质上是一种**程序性知识的参数化存储**。这与传统的监督学习有本质区别：

| 维度 | 监督学习 | SKILL0 ICRL |
|------|---------|-------------|
| 学习信号 | 模仿/蒸馏 | 探索+奖励 |
| 知识形式 | 模式匹配 | 策略优化 |
| 泛化能力 | 限于训练分布 | 发现新策略组合 |
| 上下文依赖 | 高 | 低（内化后） |

### 5.2 为什么渐进式撤回有效？

动态课程的成功可以从多个角度理解：

**1. 课程学习的经典优势**
- 从简单到复杂的样本调度
- 在SKILL0中，"简单" = "有技能指导"，"复杂" = "自主决策"

**2. 知识蒸馏的逆过程**
- 传统蒸馏：教师 → 学生
- SKILL0：上下文（教师）→ 模型参数（学生），然后"解雇"教师

**3. 神经网络的渐进容量分配**
- 早期：网络容量用于学习技能映射
- 后期：容量转向学习如何在没有技能时重现这些行为

**4. 避免灾难性遗忘**
- 突然移除所有技能会导致性能崩溃
- 渐进式撤回让模型有时间"巩固"已学知识

### 5.3 视觉上下文的战略选择

SKILL0选择视觉而非文本作为主要模态，是基于以下考量：

**优势**：
1. **结构保持**：工具调用、参数、返回值的空间关系清晰可见
2. **信息密度**：一张图可以编码复杂的结构化信息
3. **跨语言**：视觉表示不依赖特定自然语言

**潜在局限**：
1. **OCR依赖**：需要高质量的OCR系统提取文本信息
2. **分辨率限制**：极低分辨率可能丢失细节
3. **预训练对齐**：需要视觉-语言预训练模型支持

### 5.4 与相关工作的关系

#### 5.4.1 与RAG/检索增强的对比

| 特性 | RAG | SKILL0 |
|------|-----|--------|
| 推理时开销 | 高（检索+重排序） | 低（无检索） |
| 知识更新 | 动态更新知识库 | 需重新训练 |
| 冷启动问题 | 存在 | 不存在（已内化） |
| 适用场景 | 频繁更新的知识 | 稳定的程序性知识 |

SKILL0和RAG并非互斥，而是互补：RAG适合事实性知识，SKILL0适合程序性技能。

#### 5.4.2 与Tool Learning的对比

工具学习（Tool Learning）关注如何让模型学会使用新工具，而SKILL0关注如何将已知的技能内化为模型能力。两者可以结合：先用工具学习方法发现有效技能，再用SKILL0将其内化。

#### 5.4.3 与Meta-Learning的对比

元学习关注"学会学习"，SKILL0关注"学会后内化"。两者目标不同但可协同：元学习帮助快速适应新技能，SKILL0帮助将这些技能永久固化。

### 5.5 局限性与未来方向

**当前局限**：
1. **技能库质量依赖**：需要高质量、结构化的初始技能库
2. **离线分组开销**：按类别分组需要人工或启发式干预，跨领域应用时需重新分组
3. **训练成本**：相比直接推理，需要额外的RL训练
4. **知识更新**：内化的技能难以动态更新（需要重新训练）

**未来研究方向**：
1. **在线技能发现与内化结合**：在训练过程中动态发现新技能并内化
2. **持续学习**：支持增量式技能更新而不遗忘旧技能
3. **多智能体技能传播**：一个智能体内化的技能如何高效迁移到其他智能体
4. **理论分析**：建立技能内化的形式化理论框架
5. **扩展到更多模态**：音频、视频等更多感官模态的技能内化

## 六、行业影响与应用前景

### 6.1 对智能体基础设施的影响

SKILL0的提出可能对智能体（Agent）基础设施产生深远影响：

**1. 从"重检索"到"轻推理"**
- 当前架构：复杂检索系统 → 大上下文窗口 → 高成本推理
- SKILL0架构：轻量级推理 → 小上下文窗口 → 低成本推理

**2. 端侧部署成为可能**
- 低token开销意味着更低的计算需求
- 无需实时检索意味着更少的外部依赖
- 适合移动设备、边缘计算等场景

**3. 技能市场的演变**
- 从"技能即服务"（SaaS式检索）到"技能即资产"（可内化模型）
- 可能出现"预内化模型"的商业模式

### 6.2 企业应用场景

**客户服务智能体**：
- 内化FAQ处理、工单流转、升级策略等技能
- 实现零延迟、低成本的自主服务

**代码助手**：
- 内化编程模式、调试策略、API使用规范
- 无需每次都检索相关文档片段

**数据分析智能体**：
- 内化数据处理流程、可视化最佳实践
- 提高分析任务的执行效率和一致性

### 6.3 研究社区影响

**1. 新的研究范式**
- "技能内化"可能成为与"技能增强"并行的研究方向
- 推动从"上下文工程"向"参数工程"的转变

**2. 开源生态**
- 代码已开源，有利于社区复现和扩展
- 可能催生一系列基于SKILL0的改进工作

**3. 评估标准**
- 除了任务性能，"token效率"、"内化程度"可能成为新的评估维度

### 6.4 竞争格局变化

**对闭源模型的影响**：
- GPT-4o、Gemini等闭源模型在ALFWorld上（48.0%、60.3%）已被开源SKILL0超越
- 可能促使闭源提供商开发类似的内化训练流程

**对工具提供商的影响**：
- 技能库提供商可能需要调整商业模式
- 从按调用收费转向按内化授权收费

### 6.5 长期愿景

SKILL0指向一个更大的愿景：**从工具增强向真正自主的智能体演进**。如果技能可以被有效内化，那么：

- 智能体可以从"按指令办事"进化为"自主决策"
- "自我改进"成为可能：智能体发现新技能、内化、再发现
- 最终走向"自我充足智能"（Self-Sufficient Intelligence）

正如论文结语所言：*"We believe SKILL0 establishes skill internalization as a new principled and scalable paradigm, paving the way from tool-augmented toward truly autonomous LLM agents and self-sufficient intelligence."*

## 七、总结

SKILL0代表了LLM智能体领域的一个重要里程碑。它通过**上下文强化学习**和**动态课程学习**的协同设计，成功实现了从"推理时技能增强"到"参数化技能内化"的范式转变。

### 核心贡献回顾

1. **问题定义**：首次明确提出了"技能内化"作为显式训练目标
2. **方法论创新**：ICRL实现了训练时用技能、推理时零技能的平滑过渡
3. **动态课程**：Helpfulness-Driven机制确保技能被恰当地内化而非简单记忆
4. **卓越性能**：在ALFWorld和Search-QA上分别提升+9.7%和+6.6%
5. **极致效率**：每步token消耗少于0.5k，相比基线减少80%以上

### 技术亮点

- **视觉上下文编码**：将技能和交互历史渲染为紧凑的视觉表示
- **复合奖励设计**：同时优化任务成功和上下文压缩
- **渐进式撤回**：从全技能到零技能的有序过渡
- **有用性评估**：基于验证集的动态技能筛选

### 深远意义

SKILL0不仅是一个具体的技术方案，更代表了一种新的研究哲学：**让模型真正"学会"而非"借用"知识**。这与人脑从依赖外部指导到自主掌握技能的认知发展过程高度契合。

在更广阔的视角下，SKILL0为通往自主通用人工智能（AGI）提供了一条可能的路径：通过持续的内化和自我改进，智能体可以逐步减少对外部资源的依赖，最终实现真正的自主智能。
