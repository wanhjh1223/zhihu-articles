> **arXiv**: 2510.15446  
> **发布时间**: 2026年2月  
> **关键词**: VLA, Diffusion Policy, Reinforcement Learning, End-to-End Autonomous Driving

## 一、引言：当VLA遇见扩散策略

近年来，端到端自动驾驶领域呈现出百花齐放的发展态势。从早期的模块化感知-预测-规划流水线，到基于Transformer的统一架构，再到如今融合大语言模型(LLM)视觉理解能力的Vision-Language-Action (VLA)模型，技术迭代的速度令人目不暇接。

然而，当我们仔细审视现有的VLA方案时，会发现一个共同的瓶颈：**视觉-语言理解与动作生成之间的模态鸿沟**。现有的VLA模型往往将VLM作为特征提取器或推理引擎，然后通过一个独立的策略头生成动作，这种分离式架构难以实现真正的端到端优化。

**VDRive的破局之道**：本文提出的VDRive框架首次将**强化VLA**与**扩散策略(Diffusion Policy)**深度融合，构建了一个统一的端到端自动驾驶系统。该系统的核心思想是：利用VLA的世界知识和推理能力进行高层语义理解，同时借助扩散策略强大的连续动作分布建模能力生成精细的驾驶轨迹。

## 二、研究背景与动机

### 2.1 VLA在自动驾驶中的困境

VLA模型在机器人操作领域取得了巨大成功（如Google的RT-2、Physical Intelligence的π-0），但在自动驾驶场景中面临独特挑战：

| 挑战 | 具体问题 |
|------|----------|
| **动作空间维度** | 驾驶轨迹是高维连续空间（多步未来轨迹点），远复杂于机器人关节控制 |
| **安全性约束** | 物理不可行的动作可能导致灾难性后果，需要严格的安全边界 |
| **实时性要求** | 车载平台需要在有限算力下实现低延迟推理 |
| **数据分布** | 驾驶数据高度不平衡，正常驾驶样本占绝大多数 |

现有VLA方案如DriveVLM、AutoVLA等，大多采用自回归方式生成动作token，这种方法虽然简单直接，但在建模复杂多模态驾驶行为分布时存在局限性。

### 2.2 扩散策略的优势

扩散模型(Diffusion Model)在图像生成领域已经证明了其强大的分布建模能力。Diffusion Policy将其引入机器人控制领域，展现出以下优势：

1. **多模态动作建模**：能够捕捉数据集中隐含的多模态行为分布
2. **高频动作生成**：支持高频连续控制信号生成
3. **稳定训练**：避免了GAN训练中的模式崩溃问题

然而，扩散策略缺乏高层语义理解能力，这正是VLA的强项。

### 2.3 VDRive的核心洞察

VDRive的核心洞察是：**VLA负责"理解世界"，扩散策略负责"精细执行"**。具体而言：

- **VLA部分**：通过视觉token预训练和偏好数据集强化微调，赋予模型强大的场景理解和推理能力
- **扩散策略头**：作为动作生成器，将VLA的高层语义指导转化为具体的轨迹点
- **层次化结构**：先生成几何层次的宏观动作，再细化为具体的轨迹

### 3.2 VLA预训练阶段

#### 3.2.1 视觉Token预训练

VDRive首先对VLA进行视觉token预训练，目标是让模型能够预测未来的视觉观测：

$$
\mathcal{L}_{pre} = \mathbb{E}_{(o_t, a_t, o_{t+1}) \sim \mathcal{D}} \left[ \| f_{VLA}(o_t, a_t) - \phi(o_{t+1}) \|^2 \right]
$$

其中：
- $o_t$ 是当前视觉观测
- $a_t$ 是执行的动作
- $o_{t+1}$ 是下一时刻的观测
- $\phi(\cdot)$ 是视觉编码器

这种预训练方式类似于世界模型的训练，让VLA隐式地学习环境的动态特性。

#### 3.2.2 数据集构建

预训练数据来自：
- **nuScenes**：1000个驾驶场景，包含多摄像头图像和标注轨迹
- **Bench2Drive**：大规模闭环驾驶数据集，覆盖多种复杂场景

数据预处理包括：
1. 图像归一化和数据增强
2. 轨迹点平滑处理
3. 场景难度标注（简单/中等/困难）

### 3.3 强化微调阶段

#### 3.3.1 偏好数据集构建

VDRive的核心创新之一是构建了**偏好数据集**用于强化学习微调。对于每个场景，构建 $(o, a^+, a^-)$ 三元组：

- **Chosen action $a^+$**：专家驾驶员的动作或经过验证的安全轨迹
- **Rejected action $a^-$**：随机采样或碰撞轨迹

偏好数据的标注策略：
1. **规则基础**：基于碰撞检测、舒适性指标、交通合规性
2. **专家VLM评分**：使用GPT-4V等强VLM对轨迹进行质量评估

#### 3.3.2 强化微调目标

采用类似于RLHF的训练方式，使用Bradley-Terry模型建模偏好：

$$
\mathcal{L}_{RL} = -\mathbb{E}_{(o, a^+, a^-) \sim \mathcal{D}_{pref}} \left[ \log \sigma \left( r_{VLA}(o, a^+) - r_{VLA}(o, a^-) \right) \right]
$$

其中 $r_{VLA}$ 是VLA隐式学习出的奖励函数。

### 3.4 扩散策略头设计

#### 3.4.1 层次化动作生成

VDRive采用**两阶段生成策略**：

**第一阶段 - 几何动作生成**：
- 生成高层次的动作指令（如：左转、保持车道、减速）
- 动作空间离散化，包含12个基础动作类别
- 使用VLA的自回归输出直接预测

**第二阶段 - 轨迹细化**：
- 基于第一阶段的动作指导，使用扩散模型生成连续轨迹
- 扩散过程条件化于VLA的语义特征
- 输出：未来3秒的轨迹点序列（10Hz，共30个点）

#### 3.4.2 扩散模型架构

```python
# 扩散策略核心逻辑（伪代码）
class DiffusionPolicyHead(nn.Module):
    def __init__(self, vla_dim, action_dim, num_steps=10):
        self.vla_encoder = MLP(vla_dim, hidden_dim)
        self.noise_pred_net = UNet1D(action_dim, hidden_dim)
        self.num_steps = num_steps
        
    def forward(self, vla_features, noisy_action, timestep):
        # 条件化于VLA特征
        cond = self.vla_encoder(vla_features)
        # 预测噪声
        noise_pred = self.noise_pred_net(noisy_action, timestep, cond)
        return noise_pred
    
    def sample(self, vla_features):
        # DDPM采样过程
        action = torch.randn(batch_size, action_dim)
        for t in range(self.num_steps, 0, -1):
            noise_pred = self.forward(vla_features, action, t)
            action = denoise_step(action, noise_pred, t)
        return action
```

#### 3.4.3 训练目标

扩散策略的训练采用标准的DDPM目标：

$$
\mathcal{L}_{diff} = \mathbb{E}_{t, \epsilon} \left[ \| \epsilon - \epsilon_\theta(a_t, t, c) \|^2 \right]
$$

其中：
- $a_t = \sqrt{\bar{\alpha}_t} a_0 + \sqrt{1 - \bar{\alpha}_t} \epsilon$ 是加噪后的动作
- $c$ 是VLA提供的条件特征
- $\epsilon_\theta$ 是噪声预测网络

### 3.5 端到端训练流程

完整的训练流程分为三个阶段：

```
Stage 1: 视觉Token预训练 (100 epochs)
    ├── 数据：nuScenes + Bench2Drive
    ├── 目标：预测未来视觉观测
    └── 学习率：1e-4

Stage 2: 扩散策略预训练 (50 epochs)
    ├── 数据：专家驾驶轨迹
    ├── 目标：重建专家动作
    └── 冻结VLA部分参数

Stage 3: 强化微调 (30 epochs)
    ├── 数据：偏好数据集
    ├── 目标：优化偏好对齐
    └── 学习率：5e-5
```

## 四、实验结果与性能评估

### 4.1 开环规划评估（nuScenes）

在nuScenes验证集上的开环规划性能：

| 方法 | L2误差(m) ↓ | 碰撞率(%) ↓ | 说明 |
|------|------------|------------|------|
| ST-P3 | 1.33 | 45 | 基线方法 |
| UniAD | 0.71 | 23 | 端到端感知-规划 |
| VAD | 0.65 | 20 | 向量化场景表示 |
| DriveVLM | 0.52 | 28 | 早期VLA方法 |
| AutoVLA | 0.41 | 22 | 自适应推理VLA |
| **VDRive (Ours)** | **0.29** | **18** | VLA+Diffusion |

**关键发现**：
- L2误差降至0.29m，相比AutoVLA提升29%
- 碰撞率降低至18%，显著优于其他VLA方法
- 在复杂交互场景（路口、变道）中优势更明显

### 4.2 闭环驾驶评估（Bench2Drive）

**分析**：VDRive在复杂交互任务（变道、超车）中提升最显著，这验证了扩散策略在建模多模态行为方面的优势。

### 4.4 定性分析

#### 4.4.1 复杂场景案例

**案例1：多车交互路口**
```
场景描述：四向路口，VDRive车辆需要左转，同时面对对向直行车辆和右侧来车

VDRive推理过程：
1. [VLA] 识别场景："四向路口，需要让行对向车辆"
2. [VLA] 决策："等待对向白色轿车通过后左转"
3. [Diffusion] 生成轨迹：先减速停车，等待安全间隙，平滑左转

- 未能准确判断安全间隙，过早启动导致紧急制动
```

**案例2：拥堵路段变道**
```
场景描述：高速公路拥堵，需要从右侧变道至左侧快车道

VDRive推理过程：
1. [VLA] 分析："左侧车道速度更快，但间隙较小"
2. [VLA] 策略："等待前方大型货车通过后变道"
3. [Diffusion] 生成轨迹：减速拉开距离，加速并入目标车道

对比方法：
- 基线方法倾向于强行变道，舒适性差
```

## 五、技术深度解析

### 5.1 为什么VLA+Diffusion有效？

#### 5.1.1 模态互补性

| 能力 | VLA | Diffusion Policy |
|------|-----|------------------|
| 语义理解 | ✅✅✅ | ✅ |
| 常识推理 | ✅✅✅ | ❌ |
| 连续控制 | ✅ | ✅✅✅ |
| 多模态建模 | ✅✅ | ✅✅✅ |
| 分布外泛化 | ✅✅ | ✅✅ |

VLA提供**语义指导**和**高层决策**，扩散策略负责**精细执行**和**多模态动作采样**。

#### 5.1.2 训练稳定性

VLA的自回归训练与扩散的去噪训练具有天然的兼容性：
- 两者都采用最大似然目标
- 都可以受益于大规模预训练
- 都可以通过偏好数据进行对齐

### 5.2 强化微调的关键设计

#### 5.2.1 偏好数据质量

实验表明，偏好数据的质量比数量更重要：
- 10K高质量偏好对 > 100K低质量对
- 困难场景的偏好对价值更高
- 专家VLM评分比规则标注更可靠

#### 5.2.2 奖励黑客问题

VDRive通过以下方式缓解奖励黑客：
1. **混合奖励**：规则奖励 + 学习奖励
2. **保守策略更新**：限制策略偏离参考策略的程度
3. **多轮验证**：在仿真中验证策略的实际表现

### 5.3 扩散策略的改进

相比原始Diffusion Policy，VDRive的改进包括：

1. **条件增强**：引入VLA的语义特征作为强条件
2. **多尺度去噪**：不同时间步采用不同的噪声水平
3. **轨迹约束**：在扩散过程中加入运动学约束

## 六、局限性与未来方向

### 6.1 当前局限

1. **计算成本**：7B参数VLA需要较大显存
2. **数据依赖**：偏好数据标注成本较高
3. **场景覆盖**：在极端天气和罕见场景表现待验证
4. **可解释性**：扩散过程的决策依据较难解释

### 6.2 未来研究方向

1. **模型轻量化**：探索LoRA、蒸馏等技术降低计算成本
2. **在线学习**：实现部署后的持续适应能力
3. **多智能体协同**：扩展到多车协同场景
4. **世界模型融合**：将VDRive与驾驶世界模型深度整合

## 七、总结与行业影响

### 7.1 核心贡献回顾

VDRive为端到端自动驾驶领域带来了以下重要贡献：

1. **架构创新**：首次成功融合VLA与扩散策略
