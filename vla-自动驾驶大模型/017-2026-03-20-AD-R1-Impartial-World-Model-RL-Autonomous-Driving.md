## 一、引言：当世界模型开始"自欺欺人"

自动驾驶领域正在经历一场深刻的范式变革。从传统的模块化感知-预测-规划流水线，到端到端的大模型，再到如今融合视觉-语言-动作（VLA）的世界模型架构，技术的演进速度令人目不暇接。然而，在这场技术狂欢中，一个根本性的问题长期被忽视：**我们训练的世界模型，可能正在"自欺欺人"**。

想象一下这样的场景：一辆自动驾驶汽车正行驶在城市道路上，前方突然窜出一个行人。如果此时车辆的世界模型被输入一条"撞向行人"的轨迹，它会预测什么？令人震惊的是，当前绝大多数世界模型会"幻觉"出一个安全的未来——行人凭空消失，或者车辆神奇地避开了碰撞。这种**"乐观偏差"（Optimistic Bias）**不是模型的优点，而是致命的缺陷。

这正是AD-R1团队发现并试图解决的核心问题。来自澳门大学、中山大学、理想汽车等机构的研究者们，在2025年11月发表的这篇论文中，首次系统性地诊断了世界模型中的乐观偏差问题，并提出了**Impartial World Model（公正世界模型）**的概念——一个能够诚实面对危险、准确预测失败的智能体。

## 二、研究背景：强化学习在自动驾驶中的困境

### 2.1 端到端自动驾驶的演进之路

端到端自动驾驶（End-to-End Autonomous Driving）代表了自动驾驶技术的最高追求：从传感器原始数据直接映射到车辆控制信号，无需人工设计的中层表征。这一范式经历了三代演进：

**第一代：模仿学习时代**

早期的端到端方法主要依赖模仿学习（Imitation Learning, IL），让模型学习人类驾驶员的行为。代表性的工作如NVIDIA的PilotNet、Waymo的ChauffeurNet等。这类方法在常规场景表现良好，但在长尾事件（Long-tail Events）中表现糟糕——因为它们从未在训练数据中见过这些罕见情况。

**第二代：世界模型兴起**

为了解决模仿学习的分布外泛化问题，研究者们引入了世界模型（World Model）。世界模型能够"想象"未来：给定当前状态和候选动作，预测未来的场景演变。GAIA-1、DriveDreamer、WorldDreamer等工作展示了世界模型在生成可控驾驶视频方面的强大能力。

**第三代：强化学习闭环**

最新的趋势是将世界模型与强化学习（Reinforcement Learning, RL）结合，实现闭环策略优化。Think2Drive、Raw2Drive等工作在CARLA仿真器中验证了这条路径的可行性。然而，这些工作主要依赖外部仿真器，存在明显的sim-to-real鸿沟。

### 2.2 世界模型的致命缺陷：乐观偏差

AD-R1团队指出了一个被长期忽视的问题：**当前的世界模型存在系统性的乐观偏差**。

这个问题的根源在于训练数据。现有的自动驾驶数据集（如nuScenes、Waymo Open Dataset）主要由专业驾驶员的安全驾驶记录构成。世界模型在这些数据上训练后，学会了"补全"场景，但这种补全总是趋向于安全的结果。当输入一条危险的轨迹时，模型无法准确预测碰撞或失控的后果，而是会"脑补"出一个安全的未来。

这种偏差有多严重？AD-R1团队在论文中展示了一个典型案例：当输入一条"驶向绿化带"的轨迹时，标准世界模型预测的 Occupancy 中，绿化带竟然消失了！这种"眼不见为净"的预测方式，使得世界模型完全无法作为强化学习的可靠评判者（Critic）。

### 2.3 强化学习的根本挑战

强化学习的核心在于**试错学习**（Trial and Error）。智能体需要尝试不同的动作，根据奖励信号调整策略。在自动驾驶中，这意味着：

1. **探索问题**：智能体需要尝试各种动作，包括危险动作，才能学会什么是安全的
2. **奖励塑造**：需要准确的奖励函数来评判动作的好坏
3. **安全性约束**：不能让智能体在真实世界中执行危险动作来学习

世界模型本应解决这些问题：它提供了一个"想象空间"，让智能体在安全的环境中尝试各种动作。但如果世界模型本身对危险"视而不见"，整个学习过程就失去了意义。

## 三、AD-R1核心创新：教会模型"想象失败"

AD-R1的解决方案可以概括为一句话：**让世界模型学会诚实地面对危险**。这通过三大核心创新实现：

### 3.1 反事实合成（Counterfactual Synthesis）

AD-R1提出了一个系统性的数据合成管道，专门用于生成**反事实的失败场景**。这里的"反事实"（Counterfactual）指的是"与事实相反"——既然真实数据都是安全的，我们就主动合成危险的场景。

**场景分解**

合成过程首先将真实场景分解为三个部分：
- **静态环境**：道路、建筑物、植被等
- **动态智能体**：其他车辆、行人、骑行者等
- **自车状态**： ego-vehicle 的位置、速度、朝向等

这种分解利用了nuScenes等数据集的3D标注信息，为后续的轨迹生成提供了基础。

**危险轨迹生成**

基于分解后的场景，AD-R1使用一个统一的运动学模型生成危险的 ego-trajectory。轨迹生成的核心是一个**目标导向的混合模型**：

```
v_target = (p_target - p_t) / ||p_target - p_t||
p_{t+k} = p_t + α_k · Δd_k · h_hist + β_k · Δd_k · v_target + N(0, σ²)
```

其中，`p_target`是目标点，`h_hist`是历史航向，`α_k`和`β_k`是时变混合系数。这个模型能够生成平滑、物理合理的危险轨迹。

**分层课程设计**

AD-R1设计了三层难度的失败课程：

1. **自车状态失败（Ego-State Failure）**：驶离可行驶区域（Off-Road）
2. **静态环境交互（Static Environment Interaction）**：撞向建筑物、护栏、植被等
3. **动态智能体交互（Dynamic Agent Interaction）**：与车辆、行人发生碰撞

通过选择不同的目标点`p_target`，可以系统性地生成覆盖各种失败模式的训练数据。

**占用网格合成**

生成新的轨迹后，需要合成对应的未来Occupancy序列。AD-R1使用**刚性变换**（Rigid-body Transformation）将原始的未来Occupancy映射到新的自车坐标系：

```
O'_{t+k}(v') = O_{t+k}(T_k(v'))
```

这种方法保持了场景中所有复杂空间关系的物理一致性，确保合成的Occupancy在几何上是合理的。

### 3.2 公正世界模型（Impartial World Model）

在反事实合成数据的基础上，AD-R1构建了**Impartial World Model（IWM）**——一个能够公正预测安全和危险动作后果的世界模型。

**基础架构：I²-World**

IWM基于I²-World的架构，采用两阶段处理流程：

1. **场景Token化（Scene Tokenization）**：使用I²-Scene Tokenizer将高维Occupancy序列压缩为紧凑的隐式Token序列
2. **自回归预测（Autoregressive Forecasting）**：使用Transformer-based的I²-Former自回归地预测未来Token序列，再解码为Occupancy

**模型中心化改进**

仅靠数据改进还不够，AD-R1引入了两个关键的模型架构改进：

**（1）轨迹感知门控（Trajectory-Aware Gating, TAG）**

这是一个轻量级的即插即用模块。首先将输入的ego-trajectory光栅化为2D空间掩码，通过小型CNN处理得到门控图`A_traj`。然后，这个门控图通过元素乘法动态调节中间场景特征：

```
F_gated = F_scene ⊙ A_traj
```

这强制模型关注命令路径上的特征，防止模型"忽视"障碍物而 invent 一个安全的结果。

**（2）自车轨迹保真度损失（Ego-Trajectory Fidelity Loss）**

这是一个创新的损失函数，直接惩罚预测的自车位置与命令轨迹的偏差。对于每个预测的未来时间步`k`，从预测的Occupancy中解码出自车的几何中心`p̂_ego^{t+k}`，然后计算与命令轨迹位置的L2距离：

```
L_fidelity = Σ_{k=1}^K w_k ||p̂_ego^{t+k} - p_ego^{t+k}||₂²
```

这个损失函数起到了强正则化的作用，**直接阻止模型幻觉出一个"自动避让"的未来**。它强制执行一个核心原则：世界模型的职责是预测，而不是行动。

**最终训练目标**

IWM在混合数据集上训练，结合原始的安全场景和合成的失败场景：

```
L_total = L_baseline + λ · L_fidelity
```

### 3.3 四维奖励建模（4D Reward Modeling）

拥有了能够诚实预测危险的IWM后，AD-R1进一步设计了**四维奖励建模**框架，充分利用3D几何和时序信息。

**体积碰撞惩罚（Volumetric Collision Penalty）**

传统的碰撞检测通常是二元的（碰撞/未碰撞）。AD-R1提出了**体积碰撞IoU（VC-IoU）**来量化碰撞的严重程度：

```
VC-IoU_k = |V_ego ∩ V_agent,k| / |V_ego ∪ V_agent,k|
```

碰撞惩罚不仅惩罚接触，还根据重叠程度加权：

```
R_coll = -Σ_{t,k} [C_base · I(VC-IoU_k > 0) + C_iou · VC-IoU_k]
```

**离路和人行道惩罚（Off-Road and Sidewalk Penalty）**

利用语义预测，惩罚自车占用非可行驶区域：

```
R_offroad = -Σ_t |{v ∈ V_footprint | Sidewalk ∈ O_t(v) ∨ Vegetation ∈ O_t(v)}|
```

**垂直净空惩罚（Vertical Clearance Penalty）**

这是3D感知带来的独特优势。2D BEV表示无法感知的高度信息，在3D Occupancy中变得可计算：

```
R_clearance = -Σ_t |{v ∈ V_bubble | Building ∈ O_t(v)}|
```

这防止了与天桥、隧道等低矮结构的碰撞。

**可行驶表面稳定性奖励（Drivable Surface Stability Reward）**

2D BEV中标记为"可行驶"的区域，在3D中可能是陡峭的路缘或不平的路面。AD-R1通过惩罚自车下方可行驶表面体素Z坐标（高度）的高方差，鼓励选择平坦、稳定的表面：

```
R_stability = -Σ_t Var(Z(v)) for v ∈ {voxels below V_footprint | Drivable ∈ O_t(v)}
```

**综合奖励函数**

最终的奖励函数是多方面奖励的加权组合：

```
R_total = w_coll·R_coll + w_offroad·R_offroad + w_clear·R_clearance + w_stab·R_stability + w_prog·R_progress + w_vel·R_velocity
```

## 四、技术详解：闭环强化学习框架

### 4.1 问题形式化

AD-R1的目标是精炼一个预训练的端到端驾驶策略`π`，该策略将传感器观测`O_t`和导航指令`G`映射到未来自车轨迹`T_ego`。这被形式化为一个强化学习问题：

```
π* = argmax_π E_{T_ego ~ π(O_t, G)} [R(T_ego, O_t)]
```

其中，奖励函数`R`通过世界模型`M`计算：

```
{Ô_{t+k}}_{k=1}^K = M(O_{≤t}, T_ego)
```

### 4.2 基于GRPO的策略优化

AD-R1采用**组相对策略优化（Group Relative Policy Optimization, GRPO）**进行策略精炼。这是DeepSeek-R1成功应用的算法，特别适合大模型的强化学习。

**扩散策略公式化**

对于基于扩散的驾驶策略，AD-R1将去噪过程形式化为有限视野马尔可夫决策过程（MDP）。策略从高斯噪声`T_T ~ N(0, I)`开始，逐步去噪生成完整轨迹`T_ego = T_0`。

**群体采样与优势估计**

为了探索超越模仿学习的策略空间，AD-R1采样一组`G`条轨迹候选`{T_ego^(i)}_{i=1}^G`。每条轨迹通过世界模型"想象"其后果，得到未来Occupancy序列`Ŝ^(i) = M(O_t, T_ego^(i))`，并计算标量奖励`r_i = R(Ŝ^(i))`。

关键的创新在于**组标准化优势**（Group-Standardized Advantage）的计算：

```
Â_i = (r_i - mean({r_j}_{j=1}^G)) / std({r_j}_{j=1}^G)
```

这种归一化将奖励中心化并按方差缩放，聚焦于样本组内的相对质量。

**策略损失函数**

最终的训练目标包含两个部分：

1. **RL损失**：鼓励奖励寻求行为
   ```
   L_RL = -(1/G) Σ_{i=1}^G log π_θ(T_ego^(i) | O_t) · Â_i
   ```

2. **行为克隆损失**：防止策略偏离预训练模型的合理先验
   ```
   L_BC = E_{T_ref ~ π_ref} [-log π_θ(T_ref | O_t)]
   ```

3. **总损失**：
   ```
   L_total = L_RL + λ · L_BC
   ```

通过这种方式，驾驶策略能够在完全离线的设置中学习到更安全、更舒适的轨迹，有效地从想象的失败中学习。

## 五、实验验证：AD-R1的实力如何？

### 5.1 实验设置

**数据集**

- 训练数据：nuScenes（80%真实安全数据 + 20%合成失败数据）
- 评测基准：NavSim（标准化闭环评测环境）

**评测指标**

AD-R1在两个层面进行评测：

1. **世界模型保真度**：使用新的**风险预见基准（Risk Foreseeing Benchmark, RFB）**
2. **驾驶策略性能**：使用NavSim的标准指标（NC、DAC、TTC、Comfort、EP、PDMS）

### 5.2 风险预见基准（RFB）

RFB是AD-R1团队贡献给社区的重要工具，专门用于量化世界模型预测失败的能力。

**评测结果**

| 世界模型 | G-IoU (%) ↑ | f-IoU ↑ | DAF ↑ |
|---------|-------------|---------|-------|
| DOME | 21.01 | 14.21 | 8.75 |
| I²-World | 30.91 | 21.44 | 14.21 |
| **IWM (Ours)** | **40.21** | **45.91** | **25.10** |

结果清晰地展示了乐观偏差的问题：仅在安全数据上训练的标准世界模型（DOME、I²-World）在预测危险轨迹后果时表现糟糕。相比之下，经过反事实合成数据和模型中心化改进的IWM，在f-IoU和DAF指标上实现了**2-3倍的提升**。

### 5.3 闭环驾驶性能

**主要结果**

| 方法 | 方案 | WM | NC ↑ | DAC ↑ | TTC ↑ | Comfort ↑ | EP ↑ | PDMS ↑ |
|-----|-----|-----|------|-------|-------|-----------|------|--------|
| DiffusionDrive | IL | | 98.2 | 96.2 | 94.7 | 100 | 82.2 | 88.1 |
| **+ AD-R1 (Ours)** | **RL** | **✓** | **98.5** | **97.5** | **94.6** | **100** | **84.8** | **89.8** (+1.7) |
| Recogdrive-stage2 | IL | | 98.1 | 94.7 | 94.2 | 100 | 80.9 | 86.5 |
| **+ AD-R1 (Ours)** | **RL** | **✓** | **98.74** | **97.8** | **94.8** | **100** | **87.5** | **91.9** (+1.1) |

AD-R1展示了即插即用的有效性：

- 对于基于扩散策略的DiffusionDrive，PDMS提升**1.7个百分点**
- 对于基于VLA的Recogdrive，PDMS提升**1.1个百分点**，达到**91.9的SOTA水平**

更重要的是，这种提升主要来自安全性的改善（DAC、EP指标提升），而非牺牲舒适性的激进驾驶。

### 5.4 消融研究

| 配置 | G-IoU ↑ | PDMS ↑ |
|-----|---------|--------|
| Baseline (DiffusionDrive) | - | 88.1 |
| + i²-world | 30.91 | 85.3 (下降!) |
| + CS (反事实合成) | 37.22 | 89.1 |
| + CS + MR (模型改进) | 40.21 | 89.3 |
| + CS + MR + R (4D奖励) | 40.21 | **89.8** |

消融研究揭示了几个关键洞察：

1. **乐观偏差是真实存在的**：直接使用标准世界模型（i²-world）进行RL精炼，反而导致性能下降（88.1 → 85.3）
2. **反事实合成是最关键的组件**：仅添加CS就将PDMS提升到89.1，证明教会模型识别失败是最重要的
3. **各组件协同作用**：模型改进（MR）和4D奖励（R）各自贡献额外的性能提升

### 5.5 定性分析

**失败想象的可视化**

论文展示了令人印象深刻的可视化对比：当输入相同的"驶向绿化带"危险轨迹时：

- **标准世界模型**：幻觉出安全的未来，绿化带在中间道路消失
- **IWM**：忠实预测off-road场景，生成提供清晰惩罚信号的想象未来

**策略精炼的可视化**

在关键场景中，基线智能体与精炼后的智能体表现出明显不同的行为。精炼后的策略能够更早地识别风险，采取更平滑的避让动作。

## 六、深度解读：AD-R1的技术洞察

### 6.1 为什么"乐观偏差"如此重要？

乐观偏差问题的核心在于**因果忠实性**（Causal Fidelity）的缺失。当前的世界模型本质上是被训练为"场景补全器"——它们学会了填充缺失的未来帧，但这种填充总是趋向于训练数据的分布（即安全驾驶）。

当输入分布外的危险动作时，模型无法维持因果链条的忠实性。它不会预测"撞向行人 → 发生碰撞"，而是 invent 一个"撞向行人 → 行人消失"的虚假因果链。

这种偏差对于强化学习是致命的，因为：

1. **探索失效**：智能体无法通过探索学会什么是危险的
2. **奖励欺骗**：虚假的奖励信号导致策略优化走向错误方向
3. **安全幻觉**：模型对危险视而不见，造成虚假的安全感

### 6.2 反事实合成的数据效率

AD-R1的反事实合成方法展示了一种**数据-centric**的解决方案。与其收集大量真实的危险驾驶数据（这在现实中几乎不可能），不如利用现有的3D标注和物理约束，程序化地合成危险场景。

这种方法的优势在于：

1. **安全性**：不需要在真实世界中执行危险动作
2. **多样性**：可以系统性地覆盖各种失败模式
3. **可扩展性**：可以生成任意数量的训练样本
4. **物理一致性**：通过刚性变换保持几何关系的正确性

### 6.3 从"预测"到"评判"的范式转变

传统世界模型的定位是"预测器"——预测未来的场景演变。AD-R1将其重新定位为"评判者"（Critic）——评判动作的好坏。

这个范式转变带来了新的设计约束：

- **预测器**只需要在训练数据分布内表现良好
- **评判者**必须在整个动作空间内都表现良好，包括危险的、从未见过的动作

这就是为什么IWM需要额外的模型中心化改进（TAG、Fidelity Loss）来强制执行因果忠实性。

### 6.4 4D奖励的物理基础

AD-R1的4D奖励建模展示了**物理基础**（Physical Grounding）在奖励设计中的重要性。传统的2D BEV方法无法感知：

- **垂直空间**：无法检测与天桥、隧道的碰撞
- **表面几何**：无法区分平坦道路和陡峭路缘
- **碰撞严重程度**：无法量化碰撞的冲击力

通过3D Occupancy表示，AD-R1能够设计更精细、更物理合理的奖励信号，从而实现更安全、更舒适的行为。

## 七、行业影响与未来展望

### 7.1 对自动驾驶行业的意义

AD-R1的发表标志着自动驾驶世界模型研究进入了一个新的阶段：

**从"生成好看的视频"到"准确预测后果"**

早期世界模型研究主要关注生成视觉质量高的驾驶视频，这种"好莱坞式"的评价标准忽视了因果忠实性。AD-R1提出了新的评估维度：**能否准确预测危险**。

**强化学习的真正落地**

AD-R1证明了基于学习的内部评判者（Learned Internal Critic）是可行的，为强化学习在自动驾驶中的大规模应用铺平了道路。这将带来：

- **更高的数据效率**：从想象的失败中学习，而非真实世界的试错
- **更好的长尾泛化**：通过反事实合成覆盖罕见的危险场景
- **更强的安全性**：在离线环境中学习安全策略

**产业界的认可**

值得注意的是，AD-R1的研究团队包括**理想汽车**（Li Auto）的工程师（Xia Zhou, Xueyang Zhang, Yifei Zhan, Kun Zhan）。这表明中国自动驾驶公司正在积极参与前沿学术研究，将产业界的实际问题反馈给学术界。

### 7.2 局限性与开放问题

尽管AD-R1取得了显著进展，仍存在一些局限性：

**依赖3D标注**

当前的反事实合成流程依赖nuScenes等数据集的3D标注。作者指出，未来可以用高保真感知系统替代这一依赖，但这本身也是一个开放问题。

**合成数据的真实性**

虽然刚性变换保持了物理一致性，但合成的Occupancy可能缺乏真实世界传感器噪声、光照变化等因素。如何在合成数据中注入这些真实世界的复杂性，是一个值得研究的方向。

**计算成本**

4D Occupancy预测和群体采样（Group Sampling）带来了显著的计算开销。如何在保持性能的同时提高效率，是工程落地的关键挑战。

**sim-to-real的终极问题**

虽然AD-R1减少了对外部仿真器的依赖，但学习得到的策略仍然需要在真实世界中验证。如何设计有效的真实世界评估和持续学习机制，仍是未解之谜。

### 7.3 未来研究方向

基于AD-R1的基础，可以展望几个激动人心的研究方向：

**多模态世界模型**

当前的IWM主要基于Occupancy表示。未来可以扩展到多模态输出，包括图像、点云、甚至声音，提供更丰富的预测能力。

**分层强化学习**

AD-R1展示了在轨迹层面的RL精炼。未来可以探索分层架构：高层做决策（如"变道"、"超车"），低层做轨迹生成，每层都有自己的世界模型和RL优化。

**交互式世界模型**

当前的世界模型主要关注自车动作的预测。未来可以建模多智能体的交互，学习其他车辆、行人对自车动作的反应，实现更真实的闭环仿真。

**持续学习与自适应**

如何让世界模型在部署后持续学习，适应新的场景和交通规则？这需要解决灾难性遗忘、在线学习稳定性等挑战。

---

## 八、总结：走向真正安全的自动驾驶

AD-R1是一项里程碑式的工作，它首次系统性地诊断并解决了世界模型中的乐观偏差问题。通过反事实合成、公正世界模型和4D奖励建模三大核心创新，AD-R1实现了：

1. **能够想象失败的世界模型**：IWM在RFB基准上实现了2-3倍的性能提升
2. **更安全、更鲁棒的驾驶策略**：在NavSim上达到91.9的SOTA PDMS分数
3. **即插即用的RL框架**：可以无缝集成到现有的端到端系统中

更重要的是，AD-R1提供了一种新的方法论：**教会模型诚实地面对危险**。这不仅是技术层面的进步，更是哲学层面的转变——从追求"好看的结果"到追求"真实的结果"。

在通往完全自动驾驶的道路上，安全性是唯一的北极星。AD-R1证明了，只有当我们敢于让模型"想象危险"，才能真正学会避免危险。这或许正是通向安全自动驾驶的关键一步。
