# HERMES：首个统一3D场景理解与生成的一体化自动驾驶世界模型

## 一、引言：世界模型的新纪元

自动驾驶技术正处在一个关键的转折点上。从传统的模块化架构到端到端学习范式，再到如今备受瞩目的世界模型（World Model），每一次技术跃迁都在重新定义"智能"在自动驾驶中的含义。然而，现有的世界模型存在一个根本性的局限：它们大多专注于**未来场景生成**，却忽视了**场景理解**这一同等重要的能力。

2025年3月，来自**华中科技大学**、**旷视科技**、**Mach Drive**和**香港大学**的研究团队提出了**HERMES**——这是一个革命性的统一驾驶世界模型，它首次将**3D场景理解**与**未来场景生成**无缝融合在一个框架中。HERMES的名字源自希腊神话中的信使神，象征着信息传递与沟通，这也恰如其分地反映了该模型在感知与预测之间架起的桥梁。

HERMES的核心突破在于：它不仅能像传统世界模型那样预测未来场景演化，还能像大型视觉语言模型那样深入理解当前场景，并通过世界查询（World Queries）机制将两种能力相互增强。实验结果显示，HERMES在nuScenes数据集上将生成误差降低了**32.4%**，在OmniDrive-nuScenes数据集上的CIDEr理解指标提升了**8.0%**，展现出了卓越的性能优势。


## 二、研究背景：割裂的理解与生成

### 2.1 世界模型的演进脉络

世界模型（World Model）的概念源于强化学习领域，旨在让智能体建立对环境动态的认知模型，从而能够在"脑海"中推演不同行为的后果。在自动驾驶领域，世界模型通过预测未来场景演化，为规划决策提供了重要的前置信息。

现有的驾驶世界模型，如ViDAR、4D-Occ、DriveWorld等，主要聚焦于**未来点云生成**或**未来帧预测**。它们通过学习从当前观测到未来状态的映射，实现了对场景演化的建模。然而，这些模型存在一个明显的短板：**它们只是"预测器"，而非"理解者"**。

### 2.2 场景理解的关键价值

场景理解能力对于自动驾驶系统至关重要。它不仅包括对场景中物体的识别和定位，更涉及对交通规则、驾驶意图、潜在风险等高层语义信息的推理。近年来，大型视觉语言模型（VLM）在场景理解方面取得了显著进展，如OmniDrive、DriveVLM等工作展示了VLM在驾驶场景问答、场景描述生成等任务上的潜力。

然而，**纯粹的VLM缺乏对未来演化的预测能力**。它们可以描述"现在发生了什么"，但难以推断"接下来会发生什么"。这种理解与生成的割裂，限制了自动驾驶系统的整体性能。

### 2.3 统一框架的必要性

从认知科学的角度来看，人类的驾驶行为依赖于对当前场景的深度理解和对未来演变的准确预判。这两种能力并非孤立存在，而是相互促进、协同工作的。理解指导预测，预测反过来又验证和修正理解。

因此，构建一个能够**同时进行场景理解和未来生成**的统一框架，不仅是技术发展的自然延伸，更是实现真正智能驾驶的必由之路。HERMES正是朝着这一方向迈出的关键一步。


## 三、HERMES技术架构深度解析

### 3.1 整体框架设计

HERMES采用了一个简洁而优雅的端到端架构，如下图所示：

```
多视角图像输入 → BEV Tokenizer → 展平BEV特征 → 大型语言模型（LLM）
                                                          ↓
                    文本回答 ← 场景理解      World Query ← 因果注意力
                                                          ↓
                    当前到未来链接（Current-to-Future Link）
                                                          ↓
                    共享渲染器（Shared Render）→ 当前点云 + 未来点云序列
```

这一架构的核心设计哲学是**"统一而非堆砌"**。不同于简单地将理解模块和生成模块拼接在一起，HERMES通过World Query机制实现了两个任务的深度耦合和相互增强。

### 3.2 BEV Tokenizer：从多视角到统一表征

HERMES首先使用基于BEV（Bird's-Eye View，鸟瞰图）的Tokenizer将多视角相机输入转换为统一的场景表征。这一设计具有以下优势：

**几何一致性保持**：BEV表征天然地将不同视角的信息投影到统一的空间坐标系中，避免了透视失真带来的几何畸变问题。对于自动驾驶场景中的距离判断、相对位置关系推理等任务，BEV表征提供了更稳定的基础。

**计算效率优化**：相较于处理原始高分辨率图像，BEV表征大幅降低了后续处理的计算复杂度。HERMES采用的BEV网格尺寸经过精心设计，在保证空间分辨率的同时控制了计算开销。

**语义信息丰富**：HERMES的Tokenizer不仅仅是几何投影，还融合了丰富的语义信息。通过预训练的视觉骨干网络（如InternVL2），Tokenizer能够提取包含物体类别、道路结构、交通标志等语义内容的特征。

### 3.3 World Query：理解与生成的桥梁

World Query是HERMES最核心的创新之一。它的设计灵感来自于信息检索中的查询机制，旨在将LLM中的"世界知识"有效地注入到场景生成过程中。

#### 3.3.1 为什么需要World Query？

在HERMES之前，一个直观的统一思路是：让LLM处理BEV特征进行场景理解，同时用单独的未来生成器基于相同的BEV特征预测未来。这种"共享特征+独立处理"的方式（如图1c所示）虽然简单，但存在明显缺陷：

1. **知识孤岛**：理解和生成两个任务各自为政，LLM学习到的世界知识无法有效指导未来生成。
2. **优化冲突**：两个任务的训练目标可能产生冲突，导致次优的性能表现。
3. **交互缺失**：当前场景理解的结论无法直接作用于未来预测，反之亦然。

#### 3.3.2 World Query的工作机制

HERMES的解决方案是引入一组可学习的**World Query**，它们在模型中扮演着"知识搬运工"的角色：

**初始化阶段**：World Query首先由原始BEV特征（未经LLM处理）初始化。这确保了Query携带了丰富的空间信息。

**知识注入阶段**：在LLM的处理过程中，World Query通过**因果注意力机制**与文本Token进行交互。这里的"因果"意味着Query可以"阅读"用户的问题和场景描述，从中提取与生成任务相关的世界知识。

**特征增强阶段**：被世界知识增强后的World Query，通过"当前到未来链接"（Current-to-Future Link）与LLM处理后的BEV特征进行交互。这一步骤确保了生成的未来场景不仅具有几何合理性，还符合语义逻辑。

用数学语言表达，设$B_t$为当前时刻的BEV特征，$Q$为World Query，$T$为文本Token，则：

$$Q_{enhanced} = \text{Attention}(Q, T, T)$$

$$B_{future} = \text{CurrentToFutureLink}(B_t, Q_{enhanced})$$

其中$\text{Attention}$表示因果注意力操作，$\text{CurrentToFutureLink}$是一个轻量级的特征变换模块。

### 3.4 统一的理解与生成训练

HERMES的训练目标是同时优化场景理解和未来生成两个任务：

**场景理解任务**：给定用户关于当前场景的自然语言问题，模型需要生成准确的文本回答。这部分采用标准的语言建模损失：

$$\mathcal{L}_{understand} = -\sum_{i} \log P(w_i | w_{<i}, B_t)$$

**未来生成任务**：基于当前观测，预测未来若干时刻的点云。这部分采用Chamfer Distance作为损失函数：

$$\mathcal{L}_{generate} = \sum_{\Delta t} \text{CD}(P_{t+\Delta t}, \hat{P}_{t+\Delta t})$$

其中$P$和$\hat{P}$分别表示真实点云和预测点云。

**联合训练**：最终的训练损失是两个任务的加权和：

$$\mathcal{L}_{total} = \lambda_1 \mathcal{L}_{understand} + \lambda_2 \mathcal{L}_{generate}$$

通过联合训练，模型在两个任务上实现了相互促进。实验表明，单独训练每个任务会获得略微更好的专项性能，但统一训练带来的综合优势（尤其是生成质量）更为显著。


## 四、实验验证：性能的全面提升

### 4.1 数据集与评测指标

HERMES在以下两个主流数据集上进行了全面评测：

**nuScenes**：包含1000个驾驶场景的多模态数据集，每个场景约20秒，提供相机、激光雷达、雷达等多种传感器数据，以及详细的3D目标标注和地图信息。

**OmniDrive-nuScenes**：基于nuScenes构建的端到端自动驾驶评测基准，包含丰富的场景描述和视觉问答（VQA）标注。

评测指标分为两类：

**生成任务**：采用Chamfer Distance（CD）衡量预测点云与真实点云的几何差异。CD越小表示生成质量越高。

**理解任务**：采用自然语言生成领域的标准指标，包括METEOR、ROUGE-L、CIDEr等。这些指标从词汇重叠、语义相似性等角度评估生成文本的质量。

## 五、深入分析：HERMES的技术洞察

### 5.1 理解如何促进生成？

HERMES的一个核心洞见是：**场景理解能够显著提升未来生成的质量**。这种促进作用体现在多个层面：

**语义一致性**：通过World Query注入的世界知识，生成的未来场景在语义上更加合理。例如，模型能够理解"前车正在刹车"这一语义信息，从而在生成未来点云时正确预测其减速行为。

**物理合理性**：理解模块学习到的交通规则和物理常识，通过World Query传递到生成模块。这使得HERMES生成的场景更符合物理规律，减少了不合理的"跳跃"或"穿透"现象。

**长程依赖性**：传统的生成模型往往难以捕捉长程时空依赖。HERMES通过LLM的因果注意力机制，能够更好地建模场景中的长程交互关系（如交叉路口车辆的博弈行为）。

### 5.2 生成如何反哺理解？

生成任务对理解能力的反哺同样重要：

**时序推理能力**：预测未来需要理解场景的动态演化规律。这种时序推理能力反过来增强了模型对当前场景的解析能力。

**反事实推理**：生成模型可以在"脑海"中推演不同决策的后果。这种反事实推理能力是高级场景理解的重要组成部分。

**不确定性建模**：生成任务要求模型对场景演化的不确定性进行建模。这种不确定性意识使得理解输出更加可靠和鲁棒。

### 5.3 BEV表征的关键作用

HERMES选择BEV作为中间表征，这一决策背后有深刻的考量：

**跨模态对齐**：BEV提供了一个统一的空间参考系，使得相机、激光雷达等不同模态的信息能够有效融合。

**几何直观性**：相比抽象的隐空间表征，BEV与人类驾驶员的视角更为接近，便于注入先验知识和进行可视化分析。

**计算友好性**：BEV的规整网格结构非常适合现代深度学习框架的高效计算，有利于实际部署。


## 六、局限性与未来展望

### 6.1 当前局限性

尽管HERMES取得了显著进展，但仍存在一些局限性：

**预测时长限制**：当前HERMES主要关注未来3秒内的场景演化。对于更长时段的预测，误差会累积增长。

**开环评测**：目前的实验主要在开环设置下进行，即模型根据真实历史观测进行预测。闭环评测（模型根据自家预测进行滚动预测）将更具挑战性。

**计算开销**：虽然HERMES采用了轻量级设计，但LLM的推理仍然存在一定的延迟，可能限制其在高实时性场景中的应用。

**数据依赖**：HERMES的性能依赖于大规模高质量的训练数据。在数据稀缺的场景（如极端天气、罕见路况）下，性能可能下降。

### 6.2 未来研究方向

基于HERMES的突破性工作，未来研究可以朝以下方向拓展：

**动作条件生成**：将HERMES扩展为动作条件世界模型，即给定规划动作序列，预测对应的场景演化。这将为规划模块提供更直接的反馈信号。

**不确定性量化**：显式建模生成结果的不确定性，为决策模块提供更丰富的信息。这对于安全关键场景尤为重要。

**多智能体交互**：更深入地建模多智能体（车辆、行人、骑行者）之间的复杂交互，提升在拥堵场景的预测准确性。

**跨场景迁移**：提升模型在不同城市、不同驾驶风格之间的泛化能力，降低对大规模标注数据的依赖。


## 七、个人理解：HERMES的行业意义

### 7.1 范式转变的标志

HERMES的出现标志着自动驾驶世界模型研究进入了一个新的阶段。在此之前，世界模型主要被视为"高级预测器"，其任务是生成合理的未来场景。HERMES则展示了世界模型作为"认知引擎"的潜力——它不仅能够预测，还能够理解、推理和解释。

这种范式的转变与端到端自动驾驶的大趋势是一致的。正如特斯拉的FSD、理想的MindVLA等方案所展示的，未来的自动驾驶系统将是统一的、端到端的、数据驱动的大型模型。HERMES在这一方向上迈出了关键一步。

### 7.2 对端到端自动驾驶的启示

HERMES对端到端自动驾驶研究具有重要的启示意义：

**表征学习的价值**：HERMES的成功再次验证了高质量中间表征的重要性。BEV作为一种结构化的、人类可理解的表征，在连接感知、理解和决策方面发挥着关键作用。

**任务协同的必要性**：多个相关任务的联合训练能够带来相互促进的效果。这为端到端系统的设计提供了重要参考——与其训练独立的专家模型，不如构建统一的框架。

**世界知识的注入**：通过World Query将LLM的世界知识注入到生成过程中，展示了如何利用预训练大模型的能力。这一思路可以推广到规划、决策等其他模块。

### 7.3 工程落地的考量

从工程落地的角度来看，HERMES具有以下优势：

**简洁性**：相比堆砌多个独立模块的方案，HERMES的端到端架构更加简洁，有利于降低系统复杂度和维护成本。

**可扩展性**：BEV表征和LLM backbone的选择都具有较好的可扩展性。随着计算能力的提升，可以采用更大规模的模型获得更好的性能。

**可解释性**：HERMES生成的文本描述和点云预测都具有一定的可解释性，有助于系统调试和安全验证。

当然，实际部署中还需要解决延迟优化、模型压缩、边缘部署等工程挑战。


## 八、总结

HERMES是自动驾驶世界模型领域的一项里程碑式工作。它首次实现了3D场景理解与生成的统一，通过创新的World Query机制将大型语言模型的世界知识有效地注入到场景预测中。

从技术贡献来看，HERMES的主要创新包括：
1. 提出了首个统一的驾驶世界模型框架，同时支持场景理解和未来生成
2. 设计了World Query机制，实现理解与生成的深度耦合
3. 在nuScenes和OmniDrive-nuScenes数据集上取得了SOTA性能

从行业影响来看，HERMES代表了自动驾驶技术向更统一、更智能、更可解释的方向发展的重要趋势。它展示了大模型时代下，如何通过巧妙的架构设计，将不同能力整合到一个高效的整体中。

展望未来，随着计算能力的提升和算法的持续优化，类似HERMES的统一世界模型有望成为下一代自动驾驶系统的核心组件，推动自动驾驶技术从"可用"走向"好用"，最终实现真正的安全、可靠、智能的自动驾驶。
