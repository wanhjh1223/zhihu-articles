## 一、引言：世界模型——自动驾驶的下一个圣杯

在自动驾驶技术的发展历程中，我们见证了一个又一个技术范式的更迭。从早期的规则驱动系统，到基于深度学习的模块化感知-预测-规划管线，再到近年来端到端大模型的崛起，每一次技术跃迁都带来了性能的显著提升。而当下，**世界模型（World Model）**正在成为学术界和工业界共同追逐的下一个技术制高点。

世界模型的核心思想源于人类认知科学：人类驾驶员并非简单地根据当前感知做出反应，而是在大脑中构建了一个关于周围世界的动态模型，能够预测未来的场景演变，并基于这些预测做出决策。正如深度学习先驱Yann LeCun所预言的，世界模型是实现真正自主智能的关键路径。

2025年6月，来自地平线机器人联合清华、北大、南大、港科大、NTU、腾讯等顶尖机构的研究团队发布了**Epona**——一个革命性的**自回归扩散世界模型**。这篇论文不仅在技术上实现了多项突破，更重要的是，它首次成功地将**长时序视频生成**与**实时轨迹规划**统一在一个端到端框架中，为下一代自动驾驶系统的实现描绘了一幅清晰的蓝图。

## 二、背景：现有世界模型的困境

在深入解析Epona之前，我们需要先理解当前世界模型领域面临的根本性挑战。现有方法主要沿着两条技术路线演进，但各自都存在难以逾越的瓶颈。

### 2.1 扩散模型的困境：时空耦合的局限

以Vista、DriveDreamer等为代表的扩散模型方法，虽然在视觉质量上取得了令人瞩目的成就，但其核心问题在于**全局联合分布建模**的范式。这些方法将过去和未来的固定长度帧序列作为一个整体进行建模：

```
p({O_{T+i}}_{i=1}^n, {O_t, a_t}_{t=1}^T)
```

这种建模方式破坏了历史观测与未来预测之间的**因果时序结构**。就像试图一次性预测未来10秒的所有画面，而不是逐步推演，这不仅限制了模型对真实世界渐进动态的理解能力，更致命的是无法支持**可变长度的长期预测**。想象一下，如果世界模型只能生成固定15秒的未来视频，对于需要持续决策的自动驾驶系统来说，这显然是不够的。

此外，扩散模型缺乏显式的**多模态预测机制**，使得轨迹规划变得异常困难。在真实驾驶场景中，车辆面临的是一个高度不确定的环境：前车可能减速也可能变道，行人可能等待也可能突然穿行。一个好的世界模型需要能够捕捉这些多种可能的未来，而传统的扩散模型在这方面显得力不从心。

### 2.2 自回归模型的困境：离散化的代价

另一条技术路线是以GAIA-1、DrivingWorld为代表的GPT风格自回归模型。这些方法通过将图像离散化为token序列，然后像生成文本一样逐token预测：

```
∏_{i=1}^L p(t_i | t_{<i}, {O_t, a_t}_{t=1}^T)
```

这种方法确实解决了时序灵活性的问题，可以生成任意长度的序列。但代价是巨大的——**量化过程严重损害了视觉质量和轨迹精度**。想象一下，将一张高清驾驶图像压缩成几百个离散token，道路标线、远处车辆、交通信号灯的精细细节都会在量化过程中丢失。

此外，自回归Transformer的因果特性限制了其只能预测**下一个动作**，而非规划**长时程轨迹**。这就像人类驾驶员每次只能思考下一秒要做什么，而无法提前规划接下来几秒钟的行驶路线。

### 2.3 两条路线的融合：一个诱人的方向

扩散模型擅长连续空间的高保真生成，但缺乏时序分解能力；自回归模型擅长时序建模，但离散化损失了精度。能否将两者的优势结合起来？这正是Epona试图回答的核心问题。

## 三、Epona核心创新：三个关键设计

Epona的名字来源于凯尔特神话中的马女神，象征着速度、力量与优雅。正如其名，Epona框架在设计上体现了对速度（实时规划能力）、力量（长时序生成能力）与优雅（高保真视觉质量）的追求。

### 3.1 解耦时空分解：让时间与空间各司其职

Epona的第一个核心创新是**解耦时空分解（Decoupled Spatiotemporal Factorization）**。这一设计的核心思想是：与其让一个模型同时处理时间和空间，不如将这两个任务解耦，让每个专门的模块各司其职。

具体来说，Epona采用了一个分层的架构设计：

**第一层：时空变换器（Multimodal Spatiotemporal Transformer, MST）**

MST负责处理历史驾驶上下文，将过去的观测 {O_t}_{t=1}^T 和轨迹 {a_{t-1→t}}_{t=1}^T 编码成一个紧凑的潜在表示 F。这里的关键设计是**交错的多模态空间注意力层**和**因果时序注意力层**。

通过einops库的优雅表达，这个过程可以描述为：

```
E ← rearrange(E, '(b t) l c → (b l) t c')
E ← CausalTemporalLayer(E, CausalMask)
E ← rearrange(E, '(b l) t c → (b t) l c')
E ← MultimodalSpatialLayer(E)
```

这种设计巧妙地实现了：
- **因果性**：通过三角因果注意力掩码确保模型只能看到过去，不能"偷看"未来
- **效率**：相比全序列注意力显著降低内存消耗
- **灵活性**：天然支持可变长度的历史上下文

**第二层：双流扩散变换器（Dual-Stream DiT）**

基于MST编码的历史潜在表示F，Epona使用两个专门的扩散变换器分别处理不同模态的输出：

- **轨迹规划DiT（TrajDiT）**：预测未来N帧的车辆轨迹
- **视觉生成DiT（VisDiT）**：生成下一帧的图像

这种解耦设计带来了几个关键优势：
1. **专业化**：每个DiT可以针对特定任务进行优化
2. **并行性**：两个DiT可以异步生成，支持灵活的应用场景
3. **效率**：如果只关心规划，可以关闭VisDiT，实现实时推理

### 3.2 异步多模态生成：轨迹与视觉的和谐共舞

Epona的第二个创新是**异步多模态生成机制**。这一设计的精妙之处在于：它不是简单地将轨迹和图像生成捆绑在一起，而是让两者在共享潜在表示的基础上，通过并行的去噪过程独立生成。

在训练过程中，TrajDiT和VisDiT都使用**流匹配目标（Flow Matching Objective）**进行优化。这是一种比传统扩散模型更高效的训练方法，通过定义一条直线概率路径连接数据分布和噪声分布：

```
x_(t) = (1-t)x_(0) + t·ε,  ε ~ N(0, I)
L_RF = E[||v_Θ(x_(t), t) - (x_(0) - ε)||²]
```

轨迹规划的损失函数为：

```
L_traj = E[||v_traj(ā_(t), t) - (ā - ε)||²]
```

视觉生成的损失函数为：

```
L_vis = E[||v_vis(Z_{T+1(t)}, t) - (Z_{T+1} - ε)||²]
```

总损失是两者的简单相加：

```
L = L_traj + L_vis
```

这种设计的关键洞察是：**视觉理解和动作规划是同一驾驶动态的两个不同视角**。通过共享MST编码的潜在表示F，模型被迫学习一种既包含场景语义信息又包含动态变化规律的统一表示。实验表明，这种联合训练显著提升了下游规划任务的性能。

### 3.3 Chain-of-Forward训练：破解自回归的误差累积魔咒

自回归模型的一个长期困扰是**误差累积问题**。在训练时，模型使用真实的过去帧作为输入（教师强制）；但在推理时，模型必须使用自己生成的帧作为下一步的输入。这种训练与推理之间的域差距导致误差随时间迅速累积，长序列生成的质量迅速恶化。

Epona提出了一个简单但极为有效的解决方案——**Chain-of-Forward训练策略**。其核心思想是：在训练过程中周期性地执行多次前向传播，使用模型自己预测的帧来增强对推理噪声的鲁棒性。

具体实现上，Epona不是从纯噪声中采样下一帧的潜在表示，而是利用模型预测的速度v_Θ在一步内估计去噪后的潜在表示：

```
x̂_(0) = x_(t) + t·v_Θ(x_(t), t)
```

然后，这个估计的x̂_(0)连同之前的条件帧一起用于下一次前向传播，以自回归方式生成后续帧。

这个策略的美妙之处在于：
1. **直接模拟推理误差**：模型在训练时就暴露于自己的预测错误中
2. **计算效率高**：只使用单步估计，而非完整的去噪过程
3. **通用性强**：可以应用于任何自回归生成模型

实验结果显示，使用Chain-of-Forward训练的模型在长序列生成上的FID分数显著优于基线方法，在NuPlan测试集上生成40秒视频时，FID差距尤为明显。

## 四、技术细节：深挖Epona的工程实现

### 4.1 深度压缩自编码器（DCAE）的时序增强

为了在保持高视觉质量的同时降低计算开销，Epona采用了DCAE（Deep Compression Autoencoder）作为图像编解码器。与传统的8倍下采样自编码器不同，DCAE将下采样倍数提高到32倍，潜在token数量减少了16倍。

然而，DCAE作为图像自编码器缺乏时序交互能力，逐帧解码时会出现闪烁问题。Epona的解决方案是：**在DCAE解码器之前引入时空自注意力层**，同时保持编码器固定进行微调。

这个设计充分体现了工程上的权衡艺术：
- **最大化预训练参数**：利用DCAE在大规模图像数据上预训练的知识
- **最小化架构改动**：只添加必要的时序模块
- **显著提升**：大大增强了生成视频的时序一致性

实验表明，使用时序感知DCAE解码器后，NuPlan测试集上的FVD分数从52.95/76.46/100.11（10/25/40帧）显著提升到50.77/61.46/74.88。

### 4.2 模型规模与训练配置

Epona的模型规模设计体现了实用主义的哲学：

- **总参数量**：25亿
  - MST：12层，13亿参数
  - VisDiT：12层，12亿参数
  - TrajDiT：2层，5000万参数

- **训练数据**：
  - NuPlan公开视频
  - NuScenes 700个场景
  - 图像分辨率：512×1024

- **训练配置**：
  - 48张NVIDIA A100 GPU
  - 约2周训练时间
  - 60万迭代
  - batch size：96
  - Chain-of-Forward：每10步执行，每次3次前向传播
  - 优化器：AdamW，学习率1×10^-4，权重衰减5×10^-2

- **推理速度**（单张NVIDIA 4090）：
  - MST：~0.02秒
  - TrajDiT（10步）：~0.03秒
  - TrajDiT（100步）：~0.3秒
  - VisDiT（10步）：~0.3秒
  - VisDiT（100步）：~2秒

值得注意的是，当只使用MST和TrajDiT时，Epona可以实现**高达20Hz的实时轨迹规划**，这对于自动驾驶的实时性要求至关重要。

### 4.3 轨迹控制的实现机制

Epona支持基于轨迹控制的视频生成，这对于模拟极端驾驶场景和闭环测试非常重要。在VisDiT中，除了历史潜在表示F作为条件外，还引入了一个额外的调制分支用于动作控制a_{T→T+1}。

这种设计允许模型：
1. **遵循预测轨迹**：使用TrajDiT生成的轨迹作为条件
2. **遵循给定轨迹**：使用用户提供的轨迹作为条件
3. **生成多样化场景**：通过不同的轨迹条件生成不同的未来视频

在NuScenes验证集上，Epona取得了当前驾驶世界模型中的最佳FVD分数：

|------|-------|-------|--------------|

## 六、深层洞察：Epona告诉我们什么

### 6.1 世界知识可以从预测中学习

Epona最引人注目的发现之一是：**模型仅仅通过自监督的下一帧预测任务，就学会了基本的交通规则和世界知识**。

如图1(C)所示，当遇到红灯时，Epona生成的视频显示车辆会停止；当绿灯亮起，车辆继续行驶。这种对交通信号的理解完全是从数据中自动涌现的，而非显式编程。这强有力地证明了世界模型范式的一个核心假设——预测未来是学习世界知识的有效途径。

### 6.2 视觉与规划的统一表示

传统自动驾驶系统通常将感知和规划作为两个独立的模块。感知模块输出场景表示（如3D边界框、占用栅格），规划模块基于这些表示进行决策。这种分离带来了信息损失和错误累积。

Epona展示了一种新的可能性：**视觉理解和动作规划可以共享同一个潜在表示**。MST编码的F既包含场景语义信息（道路结构、其他车辆位置），也包含动态变化规律（速度、加速度、交互模式）。这种统一表示使得规划器能够直接访问原始视觉信息，而非经过压缩的结构化表示。

### 6.3 生成与规划的等价性

Epona的架构揭示了一个深刻的等价性：**生成未来场景和规划未来轨迹本质上是同一个问题的两个视角**。给定历史上下文，生成"场景会如何演变"和规划"我应该如何行动"是紧密耦合的。

这一洞察对自动驾驶系统的设计具有深远意义。与其分别训练一个世界模型和一个规划器，不如训练一个统一的模型同时完成这两个任务。这不仅提高了效率，更重要的是让规划器具备了对场景演变的深度理解能力。

---

## 七、局限性与未来方向

尽管Epona取得了令人瞩目的成果，但仍存在一些值得关注的局限性：

### 7.1 视觉质量的进一步提升空间

虽然Epona在FVD指标上取得了SOTA，但FID分数（7.5）略高于Vista（6.9）。这可能与使用的视频编码器有关——Epona使用DCAE（专为效率设计），而Vista使用专门优化的3D-VAE。未来工作可以探索更先进的视频编解码器。

### 7.2 多视角支持

当前Epona主要使用前视相机输入。虽然论文展示了单视角的强性能，但融合多视角信息（前视、侧视、后视）对于全面的环境理解仍然重要。将Epona扩展到多视角设置是一个自然的下一步。

### 7.3 闭环交互建模

Epona目前的训练和评估主要基于开环数据集。在真实驾驶中，车辆的行为会影响周围环境（其他车辆对自车动作的反应），形成闭环交互。将Epona扩展到闭环设置， modeling这种双向交互，是未来研究的重要方向。

### 7.4 计算效率优化

尽管Epona可以实现20Hz的实时规划，但生成高质量视频仍需要较长时间（100步DiT采样约2秒）。探索更高效的采样策略（如一致性模型、流匹配的少步采样）可以进一步提升实用性。

## 八、行业影响与展望

Epona的发布标志着自动驾驶世界模型领域的一个重要里程碑。它不仅是一个技术上先进的系统，更重要的是它展示了一种新的可能性：**一个真正理解世界、能够预测未来、并基于这些预测做出决策的自动驾驶系统**。

### 8.1 对学术界的启示

Epona为驾驶世界模型的研究设定了一个新的基准：
- **长时序生成**：2分钟的生成时长远超之前的方法
- **统一框架**：首次成功统一了视频生成和轨迹规划
- **开源承诺**：代码将公开发布，促进社区发展

### 8.2 对工业界的意义

对于自动驾驶公司而言，Epona提供了一条值得探索的技术路径：
- **数据闭环**：世界模型可以生成无限量的合成数据，用于训练和测试
- **场景仿真**：能够生成可控的极端场景，提升系统鲁棒性
- **端到端简化**：减少对手工设计模块的依赖，降低系统复杂度

### 8.3 对未来的展望

Epona所代表的技术方向——自回归扩散世界模型——可能预示着一个新的范式转变。在这个范式中：
- 自动驾驶系统不再是被动的反应器，而是主动的预测者
- 规划和生成不再是分离的任务，而是统一的过程
- 世界知识不再依赖手工标注，而是从预测任务中自动学习

正如论文作者所言："Epona显著推进了驾驶世界模型的SOTA，为构建下一代自动驾驶系统揭示了巨大潜力。"

---

## 九、总结

Epona是一个划时代的驾驶世界模型，它通过三个关键创新——解耦时空分解、异步多模态生成、Chain-of-Forward训练——成功地将自回归模型和扩散模型的优势结合在一起。实验结果表明，Epona在长时序视频生成和实时轨迹规划两个任务上都取得了SOTA性能。

更重要的是，Epona展示了一种构建自动驾驶系统的新范式：通过预测未来来学习世界，通过生成场景来验证规划。这种范式不仅在技术上具有优势，也更符合人类驾驶的认知机制。

随着代码的即将开源，我们可以期待Epona成为驾驶世界模型研究的重要基石，推动整个领域向着更智能、更安全的自动驾驶未来迈进。
