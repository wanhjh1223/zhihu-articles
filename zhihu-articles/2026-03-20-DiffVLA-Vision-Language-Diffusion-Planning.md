**作者**: Anqing Jiang*, Yu Gao, Zhigang Sun, Yiru Wang, Jijun Wang, Jinghao Chai, Qian Cao, Yuweng Heng, Hao Jiang, Yunda Dong, Zongzheng Zhang, Xianda Guo, Hao Sun, Hao Zhao

## 一、研究背景：端到端自动驾驶的三重困境

端到端自动驾驶（End-to-End Autonomous Driving）近年来已成为智能驾驶领域最炙手可热的研究方向。从早期的模块化设计（感知→预测→规划→控制）到如今的一体化端到端架构，学术界和工业界都在追求一个终极目标：让自动驾驶系统像人类驾驶员一样，能够直接从传感器输入学习驾驶策略，在复杂多变的真实交通环境中做出安全、舒适、高效的驾驶决策。

### 1.1 端到端范式的演进与瓶颈

传统的模块化自动驾驶系统虽然在特定场景下表现稳定，但存在明显的短板：模块间的信息传递会造成误差累积，规则驱动的设计难以应对长尾场景，系统优化目标分散导致整体性能受限。为此，以UniAD、VAD为代表的端到端方法应运而生，它们将感知、预测、规划整合到一个完全可微的模型中，通过大规模人类驾驶演示数据进行端到端训练。

然而，纯粹的数据驱动方法也面临着根本性挑战：

**第一重困境：计算成本高昂**。现有的端到端方法严重依赖BEV（鸟瞰图）特征表示，需要处理高分辨率的密集特征图，导致计算开销巨大，难以在车载嵌入式平台实时运行。

**第二重困境：行为多样性不足**。人类驾驶行为本质上是多模态的——面对同一场景，不同的驾驶员可能采取不同的合理策略。传统的回归方法倾向于输出"平均化"的轨迹，无法捕捉驾驶行为的内在不确定性，在复杂交互场景中往往表现僵化。

**第三重困境：复杂场景决策次优**。端到端模型虽然能够从数据中学习驾驶策略，但缺乏对世界知识的显式理解和推理能力。在遇到罕见的长尾场景时，模型往往无法做出合理的语义解释和因果推断，导致决策失误。

### 1.2 VLM与扩散模型的潜力

面对上述挑战，研究者们开始探索两种有前景的技术路线：

**视觉语言模型（VLM）**的引入为自动驾驶系统带来了世界知识和语义理解能力。通过在海量互联网图文数据上预训练，VLM能够理解复杂的交通场景、识别交通标志、理解导航指令，甚至进行常识推理。然而，VLM的3D空间推理能力有限，直接输出的控制指令可能缺乏物理可行性。

**扩散模型（Diffusion Model）**在生成高质量、多样化样本方面展现出卓越能力。将扩散模型应用于轨迹规划，可以自然地建模驾驶行为的多模态分布，生成丰富多样的候选轨迹。但扩散模型本身缺乏对场景语义的理解，难以处理需要高层次推理的复杂决策。

**一个自然的想法是：能否将VLM的语义理解能力与扩散模型的生成能力相结合，构建一个既具备世界知识、又能生成物理可行轨迹的自动驾驶系统？**

DiffVLA正是基于这一思路提出的创新框架。

## 二、DiffVLA架构解析：三位一体的协同设计

DiffVLA的核心设计理念是**融合认知推理与端到端规划**，通过三个关键模块的协同工作，实现从感知到决策再到规划的完整闭环。

### 2.1 VLM引导模块：赋予系统"认知大脑"

VLM引导模块是DiffVLA的"认知核心"，负责理解场景语义、解析驾驶意图、生成高级驾驶指令。该模块基于Senna-VLM框架构建，包含四个核心组件：

**视觉编码器（Vision Encoder）**：采用CLIP ViT-L/14作为骨干网络，处理来自车载多摄像头（前视、侧视、后视等）的图像序列。与单目方法不同，DiffVLA采用多图像编码策略，能够从不同视角获取全面的场景信息。

**驾驶视觉适配器（Driving Vision Adapter）**：这是一个关键的创新设计。原始CLIP编码器针对自然图像优化，直接应用于自动驾驶场景存在域差距。驾驶视觉适配器通过轻量级的投影层，将CLIP特征空间映射到驾驶专用特征空间，生成紧凑的图像Token（E_img ∈ R^{N_img×M_img×C}）。

**文本编码器（Text Encoder）**：将用户的自然语言指令（如"请跟随前车"、"准备右转"）和导航命令编码为文本Token，与视觉Token共同输入大语言模型。

**大语言模型（LLM）**：采用Vicuna-v1.5-7B作为推理引擎，接收多模态输入后，生成结构化的驾驶决策。值得注意的是，VLM的输出不仅是文本描述，而是被编码为离散的控制指令（横向控制：变道、转向；纵向控制：加速、制动）。

这种设计的精妙之处在于：VLM不再只是"解释"场景，而是**直接参与到规划流程中**，成为下游扩散规划器的语义引导信号。这解决了纯端到端方法缺乏可解释性的问题，同时也避免了VLM直接输出轨迹可能导致的物理不可行问题。

### 2.2 混合感知模块：稀疏与密集的互补融合

感知是自动驾驶的基石。DiffVLA提出了一种创新的**混合稀疏-密集感知架构**，兼顾计算效率和信息丰富性。

**密集感知分支（Dense Perception Branch）**：
- 采用BEVDet的方法，将多视角图像特征投影到BEV空间
- BEV网格设置为128×128，覆盖自车周围64m×64m的范围
- 生成密集的BEV特征图，作为隐式场景表示（z_i）
- 这种隐式表示包含了丰富的上下文信息，但计算成本较高

**稀疏感知分支（Sparse Perception Branch）**：
- 基于Sparse4D v3的采样策略进行3D目标检测
- 在线地图生成模块提取车道线、边界线、停止线等矢量地图元素
- 每个地图元素用20个采样点表示，形成结构化矢量表示
- 输出显式目标框（3D bounding boxes）和地图矢量，经MLP编码为显式特征（z_e）

**双分支融合策略**：
```
规划条件 z = {z_i（隐式BEV特征）, z_e（显式实例特征）}
```

这种设计的优势在于：密集分支提供丰富的上下文和纹理信息，有助于理解复杂场景；稀疏分支提供精确的几何信息和实例级理解，便于进行碰撞检测和可行驶区域判断。两者互为补充，共同构建对驾驶场景的全面感知。

### 2.3 扩散规划模块：生成式轨迹规划

扩散规划模块是DiffVLA的"动作执行器"，负责在VLM语义引导和感知特征支持下，生成多样化、物理可行的驾驶轨迹。

**轨迹词汇表构建**：
不同于直接回归连续坐标，DiffVLA采用**离散化轨迹词汇表**策略：
1. 从NavTrain数据集中采样所有人类驾驶轨迹
2. 使用K-means聚类算法将轨迹分组，构建离散轨迹词汇表 V = {v_i}_{i=1}^N
3. 每个轨迹词汇 v_i 包含一系列路点 τ = {(x_t, y_t, θ_t)}_{t=1}^{T_h}
4. 规划时域 T_h = 4秒，涵盖短期驾驶决策

这种离散化表示的优势在于：降低了动作空间的复杂度，同时保留了人类驾驶行为的多模态特性。

**截断扩散策略（Truncated Diffusion）**：
DiffVLA采用DiffusionDrive提出的截断扩散策略，而非完整的扩散过程：
1. 从轨迹词汇表中采样32个锚点轨迹 A = {a_k}_{k=1}^{32}
2. 向锚点添加高斯噪声，启动扩散过程
3. 采用**缩短的扩散调度**（truncated schedule），大幅减少去噪步数
4. 结合隐式特征 z_i 和显式特征 z_e 作为条件引导

扩散模型的目标函数为：
```
{ŝ_k, τ̂_k}_{k=1}^{N_anchor} = f_θ({τ_k^i}_{k=1}^{N_anchor}, z)
```

其中 f_θ 是扩散轨迹解码器，ŝ_k 是轨迹分类分数，τ̂_k 是预测的目标路点。

**层次化信息编码**：
为了有效融合异构信息源，DiffVLA设计了层次化编码策略：
- 底层：BEV空间特征提供几何约束
- 中层：实例级特征（障碍物、车道线）提供交互约束
- 高层：VLM指令提供语义引导

这种多层级条件机制确保了生成的轨迹既符合物理规律，又满足驾驶任务要求。

### 2.4 后处理优化：细节决定成败

DiffVLA还包含一个实用的后处理模块。团队观察到，在高速场景下，预测轨迹的碰撞率略高于其他场景。经过分析，他们认为这是由于预测轨迹分布与实际数据分布存在偏差。

为此，他们实施了一个简单而有效的策略：**沿Y轴施加2%的减速偏移**。这一微调显著降低了高速场景下的碰撞风险，同时不影响其他场景的性能表现。这种工程优化体现了学术研究中对实际部署细节的考量。

## 三、技术创新点深度解读

### 3.1 认知-执行分离架构

DiffVLA最值得称道的创新之一是采用了**认知-执行分离**的架构设计：

- **VLM负责"思考"**：进行场景理解、意图推理、语义决策
- **扩散模型负责"行动"**：生成具体、物理可行的轨迹

这种分离解决了VLA模型面临的核心矛盾：VLM擅长抽象推理但不擅长精确控制；扩散模型擅长生成多样化动作但缺乏语义理解。通过明确分工，两者各尽其能，协同工作。

### 3.2 混合稀疏-密集表示

传统的端到端方法要么完全依赖密集BEV表示（计算量大），要么完全依赖稀疏查询表示（信息损失）。DiffVLA的混合架构实现了"鱼与熊掌兼得"：

- 稀疏分支提供**实例级精确几何信息**，便于碰撞检测和交通规则检查
- 密集分支提供**丰富的上下文和隐式特征**，有助于处理复杂交互场景

这种设计哲学与近年来多模态学习领域的趋势一致：不同粒度、不同模态的信息源应该互补融合，而非相互替代。

### 3.3 扩散模型与VLM的深度耦合

现有的VLA方法大多采用简单嫁接策略：VLM输出文本指令，然后传递给下游控制器。DiffVLA实现了更深层次的耦合：

- VLM的输出被编码为**结构化控制指令**（横向+纵向），而非自由文本
- 这些指令与感知特征一起，作为**条件信息**输入扩散模型
- 扩散模型在每一步去噪过程中，都能访问VLM的语义引导

这种深度交互确保了VLM的决策意图能够真正影响到最终的轨迹生成，而非流于形式。

### 3.4 多模态锚点先验

DiffVLA采用32个多模态锚点作为扩散过程的先验，这一设计有深意：

- **多模态性**：锚点覆盖了不同的驾驶行为模式（跟车、变道、停车等），确保生成轨迹的多样性
- **数据驱动**：锚点从真实人类驾驶数据中提取，保留了自然的驾驶风格
- **截断扩散**：以锚点为起点的截断扩散过程，既保留了扩散模型的生成能力，又大幅提高了推理效率

## 四、实验验证：闭环测试的硬核证明

### 4.1 实验设置

**数据集**：NavSim v2  
NavSim是一个数据驱动的非反应式自动驾驶仿真与评测平台，相比开环评测（如nuScenes），它提供了更真实的闭环评估：
- 引入**反应式背景交通参与者**，能够根据自车行为做出动态响应
- 生成**逼真的合成多视角图像**，测试模型的泛化能力
- 评估指标包括安全性、舒适性、效率等多维度

**训练策略**：两阶段训练
- **第一阶段**：分别训练VLM引导模块和混合感知模块（稀疏分支）
  - 稀疏感知分支：batch size 128，学习率1e-4，100 epochs
  - VLM模块：batch size 192，学习率2e-5，1 epoch（利用预训练权重）
- **第二阶段**：冻结VLM和稀疏感知权重，联合训练密集感知分支和轨迹头
  - batch size 256，初始学习率1e-4，100 epochs
  - 使用AdamW优化器和余弦学习率衰减

这种分阶段训练策略确保了各模块能够充分学习，同时避免了梯度冲突和不稳定训练。

**评测指标**：EPDMS（End-to-End Planning Detection and Motion Simulation）
EPDMS是NavSim平台的核心评测指标，综合考虑了：
- 碰撞检测（Collision Checking）
- 可行驶区域违规（Drivable Area Compliance）
- 舒适性指标（舒适性加速度、转向平滑度）
- 效率指标（行程时间、路径长度）

### 4.2 实验结果

在NavSim v2私有测试集上，DiffVLA取得了**45.0 EPDMS**的优异成绩（在Autonomous Grand Challenge 2025中验证）。

虽然论文未提供与基线方法的详细对比表格，但这一成绩在当时的挑战赛中已属顶尖水平。考虑到NavSim v2引入了反应式交通参与者和合成图像，测试难度远高于传统开环评测，45.0 EPDMS充分证明了DiffVLA在闭环场景下的鲁棒性和泛化能力。

### 4.3 消融实验洞察

尽管论文未详细展开消融实验，但从架构设计中我们可以推断：

1. **VLM引导的消融**：去除VLM语义引导，仅依靠感知特征进行规划，预计会导致在需要语义推理的场景（如理解交通标志、遵循导航指令）性能显著下降。

2. **混合感知的消融**：仅使用密集BEV分支，计算成本会增加但性能未必提升；仅使用稀疏分支，在复杂场景下的上下文理解能力可能不足。

3. **扩散 vs 回归**：将扩散规划替换为传统的单模态回归，预计会损失行为多样性，在复杂交互场景表现下降。

## 五、局限性与未来展望

### 5.1 当前局限

作者在论文中坦诚地指出了DiffVLA的局限性：

**分阶段训练的局限**：由于时间和计算资源的限制，目前VLM、稀疏感知、密集感知和规划头是分阶段单独训练的，尚未实现真正的端到端联合训练。这种分离训练可能导致各模块之间的优化目标不一致，存在性能损失。

**未来改进方向**：探索所有模块的单阶段端到端训练，可能会带来进一步的性能提升。但这面临着梯度传播、训练稳定性、计算资源等多重挑战。

### 5.2 技术启示

DiffVLA的提出为VLA自动驾驶研究提供了重要启示：

**1. 架构设计比单纯堆砌更重要**：DiffVLA没有盲目追求更大的模型规模，而是通过精心设计的架构（认知-执行分离、稀疏-密集融合），实现了效率和性能的平衡。

**2. 多模态融合需要深度耦合**：VLM与扩散模型不是简单串联，而是通过结构化条件和层次化编码实现深度交互，这是性能提升的关键。

**3. 工程优化不可忽视**：2%的减速偏移这种看似简单的后处理，却能显著改善实际表现，体现了从研究到部署的完整思考。

### 5.3 行业影响

DiffVLA代表了VLA自动驾驶的一个重要技术方向：**将世界知识（VLM）与生成式规划（扩散模型）相结合**。这一思路预计将在以下方面产生深远影响：

- **长尾场景处理**：VLM的常识推理能力有助于处理罕见场景，提升系统安全性
- **人机交互体验**：VLM的自然语言理解能力为更自然的人车交互奠定基础
- **数据效率提升**：借助VLM的预训练知识，可能减少对大规模标注数据的依赖

## 六、总结：认知与执行的协奏曲

DiffVLA是博世与清华大学等机构的联合研究成果，代表了工业界与学术界在VLA自动驾驶领域的深度合作。它提出的混合稀疏-密集扩散策略，成功地将VLM的语义理解能力与扩散模型的生成能力相结合，构建了一个既具备世界知识、又能生成物理可行轨迹的端到端自动驾驶系统。

**核心贡献回顾**：
1. **VLM引导模块**：赋予系统高层次认知和推理能力
2. **混合感知模块**：稀疏与密集表示的互补融合
3. **扩散规划模块**：生成多样化、物理可行的驾驶轨迹
4. **闭环验证**：在NavSim v2上取得45.0 EPDMS的优异成绩

DiffVLA的设计哲学值得行业深思：**在自动驾驶这个安全关键领域，单纯的数据驱动可能不足以应对复杂多变的真实世界；将世界知识、推理能力与生成式规划相结合，可能是通往更可靠、更智能自动驾驶系统的必由之路。**

随着VLA技术的快速发展，我们有理由期待，未来的自动驾驶系统将不仅能"看见"世界，更能"理解"世界，像人类驾驶员一样做出富有智慧、灵活多变的驾驶决策。DiffVLA正是朝着这一愿景迈出的坚实一步。
