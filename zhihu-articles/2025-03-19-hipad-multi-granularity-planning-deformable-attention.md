# HiP-AD：Nullmax团队破局端到端自动驾驶，多粒度规划+可变形注意力刷新闭环控制SOTA

> **一句话总结**：Nullmax（岩山科技旗下自动驾驶公司）团队在ICCV 2025提出HiP-AD框架，用多粒度规划查询+可变形注意力机制突破端到端自动驾驶的闭环控制瓶颈，Bench2Drive成功率刷新SOTA，解决了困扰业界的"开环表现好、闭环表现差"难题。

---

## 一、端到端自动驾驶的尴尬现实：开环王者，闭环青铜

### 1.1 端到端自动驾驶的狂欢与隐忧

2023-2024年，端到端自动驾驶（End-to-End Autonomous Driving, E2E-AD）迎来了前所未有的热潮。

从特斯拉FSD V12的惊艳表现，到国内华为ADS、小鹏XNGP的端到端化改造，整个行业都在拥抱这一范式革命。学术界更是百花齐放：
- **UniAD**：首个将感知、预测、规划统一到一个框架的端到端方案
- **VAD**：用向量化表示简化场景，提升效率
- **SparseDrive**：稀疏查询范式，大幅降低计算量
- **DriveTransformer**：单Transformer完成所有任务

这些工作在nuScenes等开环评测数据集上取得了惊人成绩——**碰撞率低至0.1%**，看起来自动驾驶似乎已经近在眼前。

### 1.2 闭环测试：端到端的"照妖镜"

然而，当这些模型真正进入**闭环测试（Closed-loop Evaluation）**时，情况却急转直下。

什么是闭环测试？简单来说：
- **开环测试**：模型预测轨迹，与真值轨迹对比，计算偏差
- **闭环测试**：模型预测轨迹，车辆真正按预测行驶，观察是否会撞车、是否能完成路线

两者的区别就像是：
- 开环 = 考试抄答案，只要和参考答案相似就算对
- 闭环 = 真正上路开车，撞不撞车是唯一的评判标准

**残酷的对比数据**：

| 方法 | nuScenes碰撞率（开环） | Bench2Drive成功率（闭环） |
|-----|---------------------|------------------------|
| UniAD | 0.3% | ~25% |
| VAD | 0.1% | ~30% |
| SparseDrive | 0.1% | ~32% |
| **HiP-AD（本文）** | **0.08%** | **>45%** |

即使是专门针对闭环设计的CarLLaVA，在紧急制动场景下的成功率也**不到55%**。

### 1.3 为什么开环和闭环差距如此之大？

**根本原因：目标函数不匹配**

开环优化的目标是最小化预测轨迹与真值轨迹的偏差（L2距离）。但这存在两个问题：

**问题一：轨迹不唯一**

同样的场景，人类司机可能有多种合理开法：
- 方案A：平稳跟车，保持安全距离
- 方案B：加速超车，更加果断
- 方案C：稍微减速，更加保守

只要安全，都是正确答案。但开环评测只认"和训练数据一致"。

**问题二：累积误差**

开环测试每帧独立计算误差，不会累积。但闭环测试中，上一帧的小误差会导致下一帧的输入发生变化，误差像滚雪球一样越来越大。

**问题三：缺乏精细控制**

很多端到端模型只输出稀疏的轨迹点（如每秒2个），对于精确控制车辆的速度和方向远远不够。结果就是：模型能"大概"走对方向，但控车不稳，要么犹犹豫豫，要么猛打方向。

### 1.4 现有的闭环优化尝试及其局限

针对上述问题，业界有过一些尝试：

**尝试一：更密集的轨迹点**
直接把输出从2Hz提升到10Hz甚至更高。
*问题*：计算量暴增，且单纯增加点数并不能解决控制稳定性问题。

**尝试二：时空解耦**
CarLLaVA将轨迹解耦为时间条件（纵向控制）和空间条件（横向控制）分开预测。
*问题*：基于预训练大语言模型，缺乏中间感知结果，可解释性差，且没有充分利用轨迹的多样性。

**尝试三：强化学习优化**
用RL在闭环仿真中直接优化成功率。
*问题*：训练不稳定，样本效率低，难以scale。

**我们需要一个新的思路**

HiP-AD的核心洞察是：**规划查询的设计和交互机制，潜力远未被充分挖掘**。

## 二、HiP-AD核心创新：从"单粒度模仿"到"多粒度感知-规划协同"

### 2.1 三大核心贡献

HiP-AD（Hierarchical and Multi-Granularity Planning for Autonomous Driving）提出了三个关键创新：

**创新一：多粒度分层规划查询（Multi-Granularity Planning Queries）**

突破传统的单一时空轨迹表示，将规划解耦为**时序轨迹点（Temporal Waypoints）**、**空间轨迹点（Spatial Waypoints）**和**驾驶风格轨迹点（Driving-style Waypoints）**，每种类型内部再细分多个粒度。

**创新二：规划可变形注意力（Planning Deformable Attention）**

不再用全局注意力机制盲目扫描整张图像，而是利用规划轨迹的几何先验，在轨迹点附近动态采样图像特征，实现精准、高效的信息提取。

**创新三：统一解码器架构（Unified Decoder）**

感知、预测、规划任务在一个统一的Transformer解码器内迭代交互，规划查询既能与BEV空间的感知查询交互，也能与透视图空间的图像特征交互，信息流动更加充分。

### 2.2 多粒度规划查询：让轨迹表达更丰富

#### 2.2.1 传统方法的局限

传统端到端方法通常只预测一种轨迹点：

**方案A（稀疏时序）**：每隔0.5秒预测一个轨迹点，共预测6个点（3秒）
- 优点：计算量小
- 缺点：只有6个点，插值后控制不够精细

**方案B（CarLLaVA）**：将轨迹解耦为时序点和空间点
- 时序点：决定何时到达某位置（纵向控制）
- 空间点：决定走哪条路径（横向控制）
- 优点：时空解耦，控制更灵活
- 缺点：仍缺乏对速度变化的显式建模

#### 2.2.2 HiP-AD的三层轨迹表示

HiP-AD进一步将轨迹解耦为三个层次：

**第一层：时序轨迹点（Temporal Waypoints）**
- 在固定时间间隔采样（如0.2s, 0.4s, 0.6s...）
- 告诉车辆"何时"应该在哪
- 适合控制纵向运动（加速/减速）
- 支持多粒度：2Hz（稀疏）和5Hz（密集）

**第二层：空间轨迹点（Spatial Waypoints）**
- 在固定空间间隔采样（如每2米一个点）
- 告诉车辆"沿什么路径"行驶
- 适合控制横向运动（转弯/直行）
- 支持多粒度：2米间隔（密集）和5米间隔（稀疏）

**第三层：驾驶风格轨迹点（Driving-style Waypoints）**
- 结合时间和速度采样
- 例如：0.2s后速度应该是30km/h，0.5s后应该是40km/h
- 显式建模速度变化，适合超车、紧急制动等场景
- 支持多粒度：不同目标速度组合

#### 2.2.3 多粒度的威力：稀疏全局+密集局部

为什么要用多粒度？来看一个具体场景：

> **场景**：前方100米有路口，需要右转

**稀疏轨迹点**（5米间隔）：
- 提供全局路径信息：直行→路口→右转
- 帮助模型理解"我要去哪个方向"

**密集轨迹点**（2米间隔）：
- 提供局部精细控制：转弯时的精确路径
- 帮助模型"精确控制转向角度"

**多粒度融合**：
稀疏点和密集点互补——稀疏点提供全局导航信息，密集点确保局部控制精度。

#### 2.2.4 解决"犹豫不决"问题

端到端模型在闭环测试中经常出现"犹豫（Hesitation）"现象：
- 到达路口时，不知道该不该走
- 反复减速、加速，浪费时间
- 最终超时失败

HiP-AD的多粒度设计通过以下方式缓解这一问题：
1. **更丰富的监督信号**：训练时有更多轨迹点作为监督，学习更稳定
2. **显式驾驶风格建模**：模型学会"果断"或"谨慎"的不同风格
3. **全局+局部信息融合**：既有战略（去哪），又有战术（怎么去）

实验数据显示，HiP-AD将"Agent timed out"（因犹豫超时）的情况大幅降低。

### 2.3 规划可变形注意力：精准聚焦，拒绝盲目

#### 2.3.1 全局注意力的浪费

传统方法的规划查询如何与图像交互？

**典型做法（如DriveTransformer）**：
规划查询与**所有**多视角图像特征做全局交叉注意力。

这有什么问题？
- **计算浪费**：图像中大部分区域与规划无关（如天空、远处的建筑）
- **信息淹没**：真正重要的信息（如前方车辆、车道线）被淹没在海量无关特征中
- **缺乏几何先验**：模型不知道"我应该关注哪里"

#### 2.3.2 可变形注意力：从Deformable DETR到Planning

可变形注意力（Deformable Attention）最早来自Deformable DETR，核心思想是：
> 不关注所有位置，只关注参考点附近的几个采样点。

HiP-AD将其引入规划任务，称为**Planning Deformable Attention（PDA）**：

**步骤一：轨迹投影**
- 将预测的轨迹点投影到多视角图像上
- 例如：前方5米的轨迹点，在左前摄像头图像中的坐标是(x1, y1)

**步骤二：邻域采样**
- 以投影点为中心，采样周围K个点的特征
- 采样位置通过可学习的偏移量（offset）动态调整

**步骤三：特征聚合**
- 用注意力权重加权聚合采样点的特征
- 得到轨迹点对应的图像特征表示

#### 2.3.3 PDA的几何直观

想象你在开车：
- **全局注意力**：你同时看所有方向，包括后视镜、侧窗、天空——信息过载
- **PDA**：你主要关注行驶路径前方的一小块区域——专注、高效

具体来说，PDA会关注：
- 轨迹点正前方（路径本身）
- 轨迹点两侧（车道线、相邻车辆）
- 根据场景动态调整关注范围（高速时看更远，城市工况看更近）

#### 2.3.4 PDA与BEV的协同

HiP-AD的统一解码器中，规划查询既参与：
1. **BEV空间交互**：与感知查询（检测、地图）在鸟瞰图空间交换信息
2. **透视图空间交互**：通过PDA与原始图像特征交互

这种设计兼顾了两者的优势：
- BEV：空间关系清晰，适合与感知结果对齐
- 透视图：保留原始图像细节，纹理信息丰富

### 2.4 统一解码器：打破模块壁垒

#### 2.4.1 从Sequential到Unified

端到端架构的演进历程：

**Stage 1：Sequential（串行）**
- 代表：UniAD、VAD
- 流程：感知→预测→规划，逐阶段执行
- 问题：信息传递有损耗，后面阶段看不到原始图像

**Stage 2：Parallel（并行）**
- 代表：Para-Drive
- 流程：感知和规划同时做，但规划只与BEV交互
- 改进：减少等待，但规划与图像的直接交互仍不足

**Stage 3：Unified（统一）**
- 代表：DriveTransformer、HiP-AD
- 流程：所有任务在一个解码器内迭代交互
- 优势：信息流动最充分，每个任务都能看到所需信息

#### 2.4.2 HiP-AD解码器的三层交互

HiP-AD的统一解码器每层包含三个子模块：

**模块一：时序交互（Temporal Interaction）**
- 当前帧的查询与历史帧的查询交互
- 跟踪动态物体（车辆、行人）的历史轨迹
- 理解场景的时间演化

**模块二：协同交互（Collaborative Interaction）**
- 不同任务查询之间的交互
- 检测查询 ↔ 地图查询 ↔ 规划查询
- 使用几何注意力图（距离加权的注意力）聚焦相关元素

**模块三：任务可变形聚合（Task Deformable Aggregation）**
- 每个任务查询通过可变形注意力获取图像特征
- 检测查询：关注物体中心附近
- 地图查询：关注车道线区域
- 规划查询：关注轨迹附近（PDA）

#### 2.4.3 迭代优化的威力

解码器共L层（如6层），查询逐层迭代更新：
- 第1层：粗略估计
- 第2-3层：细化调整
- 第4-6层：精确预测

每层的规划查询都能：
1. 从感知查询获取最新的环境理解
2. 从图像获取轨迹附近的精细特征
3. 从历史查询获取时间连续性

这种迭代优化使得最终预测更加鲁棒。

## 三、技术细节详解：HiP-AD是如何工作的

### 3.1 整体架构

HiP-AD的网络结构可分为四个部分：

```
输入：多视角图像 {I_i}_{i=1}^V
  ↓
Backbone（ResNet/ViT）+ FPN
  ↓
多尺度图像特征 {F_i}_{i=1}^V
  ↓
统一解码器（Unified Decoder，共L层）
  - 输入：混合任务查询（检测 + 地图 + 规划）
  - 每层：时序交互 → 协同交互 → 可变形聚合
  - 输出：更新后的查询
  ↓
任务头（Detection/Motion/Map/Planning Heads）
  ↓
输出：检测结果 + 地图 + 多粒度轨迹
```

### 3.2 多粒度规划查询的数学表达

#### 3.2.1 三种轨迹点类型

**时序轨迹点（Temporal）**：
```
T = { (t_i, x_i, y_i) }_{i=1}^{n_t}
```
- t_i：时间点（如0.2s, 0.4s, ...）
- (x_i, y_i)：对应时刻的BEV坐标
- n_t：时序采样点数（多粒度：稀疏n_t^s=6，密集n_t^d=15）

**空间轨迹点（Spatial）**：
```
S = { (d_j, x_j, y_j) }_{j=1}^{n_s}
```
- d_j：沿路径的距离（如2m, 4m, ...）
- (x_j, y_j)：对应位置的BEV坐标
- n_s：空间采样点数（多粒度：稀疏n_s^s=10，密集n_s^d=25）

**驾驶风格轨迹点（Driving-style）**：
```
D = { (t_k, v_k, x_k, y_k) }_{k=1}^{n_d}
```
- t_k：时间点
- v_k：目标速度
- (x_k, y_k)：位置
- n_d：风格采样点数

#### 3.2.2 多粒度融合

对于每种模态（左/直/右），融合不同粒度的查询：

```
Q_fuse = Aggregation(Q_temporal_sparse, Q_temporal_dense, 
                      Q_spatial_sparse, Q_spatial_dense,
                      Q_driving_slow, Q_driving_fast)
```

融合后的查询用于预测所有粒度的轨迹点，通过多任务学习获得更丰富的监督。

### 3.3 规划可变形注意力的实现

#### 3.3.1 轨迹投影

将BEV坐标系的轨迹点投影到图像平面：

```
p_img = Project(p_bev, camera_params)
```

考虑多个高度层（如0m, 1m, 2m），因为车辆、交通标志在不同高度。

#### 3.3.2 可变形采样

对于每个投影点p_img，学习K个采样偏移：

```
Δp_k = MLP_offset(Q_planning, p_img)[k]  # k=1,...,K
w_k = MLP_weight(Q_planning, p_img)[k]
```

采样位置：p_sample = p_img + Δp_k

#### 3.3.3 特征聚合

```
F_planning = Σ_{k=1}^K w_k · F_image(p_sample) · φ(p_sample)
```

其中φ是双线性插值，用于获取非整数坐标的特征值。

### 3.4 训练目标

HiP-AD的训练损失包括：

**感知损失**：
- 检测损失：L1 + 分类
- 地图损失：向量化地图点回归

**规划损失**：
- 多粒度轨迹回归：L2 loss for each waypoint type
- 碰撞惩罚：与预测障碍物的距离约束
- 舒适度约束：加速度、 jerk 平滑性

**联合训练**：
```
L_total = λ_det * L_det + λ_map * L_map + λ_plan * L_plan
```

## 四、实验结果：刷新Bench2Drive SOTA

### 4.1 评测基准

**Bench2Drive**：
- 200+个驾驶场景
- 44种不同的交通场景类型
- 闭环测试：模型控制车辆，CARLA仿真器执行
- 关键指标：
  - **Success Rate（成功率）**：完成路线的比例
  - **Progress Rate（进度率）**：行驶距离/总距离

**nuScenes**：
- 真实世界数据集
- 开环测试：预测轨迹与真值比较
- 关键指标：
  - **Collision Rate（碰撞率）**
  - **L2 Error（L2误差）**

### 4.2 主实验结果

#### 4.2.1 Bench2Drive闭环测试

| 方法 | 成功率（SR） | 进度率（PR） |
|-----|------------|------------|
| ST-P3 | 12.5% | 35.2% |
| UniAD | 25.3% | 52.1% |
| VAD | 28.7% | 55.4% |
| SparseDrive | 32.1% | 58.9% |
| DriveTransformer | 38.5% | 63.2% |
| CarLLaVA | 42.3% | 65.8% |
| **HiP-AD** | **46.8%** | **71.5%** |

**关键发现**：
- HiP-AD刷新Bench2Drive成功率SOTA，比之前最好的CarLLaVA提升**4.5个百分点**
- 比DriveTransformer提升**8.3个百分点**
- 进度率同样领先，说明不仅完成更多路线，且行驶效率更高

#### 4.2.2 紧急制动专项测试

Bench2Drive中专门测试紧急制动（Emergency Brake）场景：

| 方法 | 紧急制动成功率 |
|-----|--------------|
| VAD | 42% |
| SparseDrive | 48% |
| DriveTransformer | 52% |
| CarLLaVA | 54% |
| **HiP-AD** | **67%** |

**为什么HiP-AD在紧急制动上表现更好？**

1. **驾驶风格轨迹点**：显式建模速度变化，模型学会"快速减速"
2. **多粒度密集采样**：密集轨迹点提供更精细的速度控制
3. **PDA机制**：动态关注前方障碍物，反应更及时

#### 4.2.3 nuScenes开环测试

| 方法 | L2 Error (m) | Collision Rate (%) |
|-----|-------------|-------------------|
| ST-P3 | 1.33 | 0.65 |
| UniAD | 0.48 | 0.30 |
| VAD | 0.46 | 0.10 |
| SparseDrive | 0.34 | 0.10 |
| **HiP-AD** | **0.32** | **0.08** |

**结论**：HiP-AD在闭 Loop 上大幅提升的同时，开 Loop 指标依然保持SOTA水平。

### 4.3 消融实验

#### 4.3.1 模块消融

| 配置 | 成功率 | 说明 |
|-----|-------|------|
| 基线（无PDA，单粒度） | 35.2% | 传统端到端设计 |
| + PDA | 40.8% | 加入可变形注意力 |
| + 多粒度（无驾驶风格） | 43.5% | 时序+空间多粒度 |
| + 驾驶风格轨迹 | 46.8% | 完整HiP-AD |

每个组件都有正向贡献，完整版效果最佳。

#### 4.3.2 多粒度策略消融

| 时序粒度 | 空间粒度 | 驾驶风格 | 成功率 |
|---------|---------|---------|-------|
| 稀疏 | - | - | 38.5% |
| 密集 | - | - | 40.2% |
| 稀疏 | 稀疏 | - | 41.8% |
| 稀疏+密集 | 稀疏+密集 | - | 43.5% |
| 稀疏+密集 | 稀疏+密集 | 有 | **46.8%** |

稀疏+密集的组合优于单一粒度，驾驶风格轨迹进一步提升。

#### 4.3.3 统一解码器 vs 串行解码器

| 架构 | 成功率 | FPS |
|-----|-------|-----|
| Sequential（串行） | 41.2% | 8.5 |
| Parallel（并行） | 43.6% | 9.2 |
| Unified（统一，本文） | 46.8% | 8.8 |

统一解码器在性能上显著优于串行和并行，速度上也有竞争力。

### 4.4 可视化分析

**场景：复杂路口右转**

传统方法（SparseDrive）：
- 轨迹稀疏（6个点）
- 转弯过程不够平滑
- 到达路口时犹豫，最终超时

HiP-AD：
- 多粒度轨迹（时序密集15点+空间密集25点+风格轨迹）
- 转弯平滑，速度控制合理
- 顺利通过路口

**PDA可视化**：
- 绿色点：轨迹投影位置
- 红色区域：可变形注意力采样区域
- 可以看到采样区域随场景动态调整（直道看远，弯道看近）

## 六、局限与展望

### 6.1 当前局限

**局限一：仿真依赖**
- 主要在CARLA仿真环境验证
- 真实世界部署仍需更多测试

**局限二：场景覆盖**
- Bench2Drive覆盖44种场景，但真实世界场景更复杂
- 极端天气（暴雨、大雪）未充分测试

**局限三：计算资源**
- 多粒度设计增加计算量
- 需要进一步优化以满足边缘设备要求

**局限四：可解释性**
- 相比VLM方法，缺乏自然语言解释
- 中间感知结果虽可解释，但规划决策过程仍较黑盒

### 6.2 未来方向

**方向一：真实世界闭环验证**
- 在封闭场地测试
- 逐步推进到开放道路
- 与车厂合作实车部署

**方向二：端到端可解释性**
- 引入VLM生成决策解释
- 可视化PDA关注区域
- 输出轨迹置信度

**方向三：多模态扩展**
- 融合激光雷达、毫米波雷达
- 结合高精地图
- 夜间和恶劣天气场景

**方向四：持续学习**
- 在线适应新场景
- 从人类司机反馈学习
- 快速迭代模型

**方向五：与VLM结合**
- 用HiP-AD做底层控制
- 用VLM做高层决策和解释
- 分层架构兼顾性能和可解释性

## 七、对行业的意义

### 7.1 技术层面的启示

**启示一：闭环性能才是真正的试金石**

HiP-AD再次证明，开环指标好不等于能开车。行业应该更关注闭环评测，Bench2Drive这样的基准会越来越重要。

**启示二：查询设计 matters**

规划查询的设计（单粒度 vs 多粒度）对性能影响巨大。这提示我们，端到端不是"一个网络搞定一切"那么简单，合理的任务分解和表征设计至关重要。

**启示三：注意力机制需要几何先验**

全局注意力虽然简单，但效率低下。利用任务的几何特性（如规划关注轨迹附近）设计专门的关注机制，是提升性能和效率的关键。

### 7.2 产业层面的影响

**影响一：端到端落地加速**

HiP-AD证明，不依赖大模型也能做好端到端自动驾驶，且更适合车端部署。这会降低端到端方案的落地门槛。

**影响二：Nullmax的技术实力展示**

作为岩山科技旗下的自动驾驶公司，Nullmax在ICCV 2025发表SOTA工作，展示了中国自动驾驶公司在全球的技术竞争力。

**影响三：学术界与工业界的协同**

HiP-AD从问题定义（闭环控制差）到解决方案（多粒度+PDA）都紧扣实际需求，体现了工业界驱动的研究价值。
