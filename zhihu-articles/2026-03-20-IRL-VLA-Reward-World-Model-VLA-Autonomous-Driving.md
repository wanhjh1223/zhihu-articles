# IRL-VLA：逆强化学习驱动的奖励世界模型，让VLA自动驾驶实现闭环进化

> 博世团队提出首个不依赖仿真器的闭环VLA强化学习框架，斩获CVPR 2025自动驾驶挑战赛亚军

---

## 引言：VLA自动驾驶的"阿喀琉斯之踵"

2024年以来，Vision-Language-Action（VLA）模型在自动驾驶领域掀起了一股热潮。从DriveGPT4到Senna，从ORION到GigaBrain-0，研究者们试图将大视觉语言模型（VLM）的丰富世界知识注入端到端自动驾驶系统。然而，一个根本性困境始终横亘在前：**模仿学习的上限与仿真器训练的成本**。

模仿学习（Imitation Learning）虽然能利用大规模人类驾驶数据快速训练，但本质上是"复读机"——它只能复现数据集中记录的行为，难以应对训练分布之外的长尾场景。更要命的是，驾驶是一个**多模态、多目标**的复杂任务：同一时刻可能存在多条可行轨迹，而评价标准涵盖安全性、舒适性、交通效率等多个维度。

转向强化学习（Reinforcement Learning）似乎是出路，但这条路布满荆棘。传统的闭环训练依赖高保真传感器仿真器，不仅计算成本高昂，还面临严峻的**Sim2Real领域差距**问题。3D Gaussian Splatting等新技术虽然提升了渲染质量，但离实时训练仍有距离。

在这个背景下，**博世联合清华AIR、上海交大等团队**提出了IRL-VLA（Inverse Reinforcement Learning for Vision-Language-Action），一个颠覆性的三阶段训练框架。他们的核心洞见是：**与其用物理仿真器生成奖励，不如用逆强化学习直接从数据中学习一个奖励世界模型**。这个轻量级的Reward World Model（RWM）不仅规避了Sim2Real差距，还让VLA模型首次实现了真正意义上不依赖仿真器的闭环强化学习。

**CVPR 2025 Autonomous Grand Challenge的1st runner up，NAVSIM v2 benchmark 45.0 EDPMS的SOTA成绩**——这些数字背后，是一套全新的技术范式。

---

## 一、技术背景：VLA自动驾驶的三重困境

### 1.1 模仿学习的"天花板"

当前的VLA自动驾驶系统，无论是Senna、ORION还是OpenDriveVLA，本质上都是模仿学习的产物。它们接收多视角摄像头输入，输出未来轨迹或控制信号，训练目标是最大化与人类驾驶行为的相似度。

这种方法的局限在于：**它假设人类驾驶数据是"最优"的**。但现实中，同一驾驶场景往往存在多条合理轨迹——你可以加速超车，也可以减速让行，两者都是正确的。模仿学习 forcing 模型选择数据集中出现的那一条，本质上是在**抹杀驾驶的多模态特性**。

更严重的是，当遇到训练数据中未充分覆盖的长尾场景（如极端天气、异形障碍物、复杂交叉口博弈）时，模仿学习模型的表现会急剧退化，因为它们从未"见过"这些情况的正确处理方式。

### 1.2 强化学习的"仿真器依赖"

强化学习的优势在于**探索**——让模型在环境中主动试错，发现超越人类演示的策略。但这需要一个能够提供即时反馈的训练环境。

传统的解决方案是CARLA、nuPlan等仿真器。它们可以渲染传感器数据、模拟物理交互、计算碰撞检测。然而，这条路的代价是：

- **计算成本**：高保真渲染需要巨大的GPU算力
- **领域差距**：仿真器渲染的图像与真实世界存在明显的分布差异
- **反馈延迟**：传感器渲染和物理模拟的延迟限制了训练效率

RAD等最新工作尝试用3D Gaussian Splatting替代传统渲染，虽然提升了视觉质量，但仍受限于离线策略学习，难以实现真正的在线闭环训练。

### 1.3 多目标优化的"平衡难题"

驾驶不是单一目标优化问题。一个好的驾驶策略需要同时考虑：

- **安全性**：无碰撞、遵守交通规则
- **舒适性**：平滑的加速度、合理的转向角
- **效率性**：合理的行驶速度、最小化行驶时间
- **合规性**：车道保持、交通灯遵守

这些目标之间存在复杂的权衡关系。传统方法往往需要手工设计复杂的奖励函数，或者使用多目标优化技巧，但效果往往差强人意。

## 二、IRL-VLA核心创新：逆强化学习+奖励世界模型

面对上述三重困境，IRL-VLA提出了一个优雅的三阶段解决方案：

```
阶段1: 模仿学习预训练 → 建立基础驾驶能力
阶段2: 逆环境学习 → 构建奖励世界模型(RWM)
阶段3: 闭环强化学习 → PPO微调突破模仿上限
```

### 2.1 阶段一：分层VLA架构的模仿学习

IRL-VLA首先构建了一个高效的VLA模型，其核心设计灵感来自**具身智能领域的双过程理论**——人类决策既有快速的直觉反应（系统1），也有慢速的推理思考（系统2）。

#### 2.1.1 语义推理模块

基于SennaVLM框架，IRL-VLA设计了一个VLM指令引导模块。它采用**多图像编码策略**和**多视角提示机制**，能够：

- 处理6-8个摄像头的多视角输入
- 理解高层语义指令（如"跟随前车"、"在下一个路口右转"）
- 提取场景级别的语义特征（道路类型、交通状况、天气条件）

这个模块负责"系统2"的慢速思考——理解场景语境，为后续规划提供高层指导。

#### 2.1.2 3D推理模块

为了获得精确的几何感知，IRL-VLA采用BEV（Bird's Eye View）编码器：

- 多视角图像通过视觉编码器转换为BEV空间特征图
- 使用**检测token**学习向量化表示的动态物体（车辆、行人、骑行者）
- 使用**地图token**学习向量化表示的静态地图元素（车道线、人行道、交通标志）

这种向量化表示相比传统的栅格化BEV，不仅计算效率更高，还能更好地捕捉物体间的结构化关系。

#### 2.1.3 统一扩散规划器

轨迹生成采用**扩散策略（Diffusion Policy）**，这是从机器人领域借鉴的先进技术：

- 从锚点提议轨迹开始，逐步添加高斯噪声
- 条件扩散模型学习去噪过程，将噪声轨迹恢复为合理的驾驶轨迹
- 在条件解码步骤中，层次化地融合BEV token、地图token、检测token等场景信息
- 最终通过轻量级MLP回归头生成多模态轨迹

扩散模型的优势在于**天然的多模态性**——它不会强行拟合单一轨迹，而是学习整个轨迹分布，从而能够生成多样化的候选轨迹。

训练使用标准的模仿学习目标：

```
L_IL = Σ[y_k * L_rec(τ̂_k, τ_gt) + λ * BCE(ŝ_k, y_k)]
```

其中包含L1重建损失和二分类交叉熵损失，前者保证轨迹精度，后者用于选择最优轨迹模式。

### 2.2 阶段二：逆强化学习构建奖励世界模型(RWM)

这是IRL-VLA最核心的创新。传统强化学习需要仿真器提供奖励信号，而IRL-VLA选择**从数据中直接学习一个奖励模型**。

#### 2.2.1 逆强化学习的核心思想

逆强化学习（Inverse Reinforcement Learning, IRL）的基本假设是：**人类驾驶行为是"最优"的，我们可以反过来推断出人类在优化什么样的奖励函数**。

具体来说，给定：
- 场景状态 S（多视角图像、BEV特征、自车状态）
- 人类轨迹 τ_human
- 替代轨迹集合 {τ_alt}

IRL试图找到一个奖励函数 R，使得：

```
R(S, τ_human) > R(S, τ_alt) 对所有替代轨迹成立
```

换句话说，人类选择的轨迹在学到的奖励函数下应该是最优的。

#### 2.2.2 奖励世界模型的架构

RWM的架构与主策略模型类似，输入包括：
- 多视角摄像头图像
- 候选轨迹（可以是人类轨迹，也可以是模型生成的替代轨迹）

输出是**细粒度的奖励分解**，而不是单一标量奖励：

```
r̂_m = MLP_m(f_traj),  m ∈ {NC, DAC, DDC, TLC, EP, TTC, LK, HC}
```

其中每个子奖励对应NAVSIM的评估指标：

| 子奖励 | 含义 | 取值范围 |
|--------|------|----------|
| NC | 无责任碰撞 | {0, 0.5, 1} |
| DAC | 可行驶区域合规 | {0, 1} |
| DDC | 行驶方向合规 | {0, 0.5, 1} |
| TLC | 交通灯合规 | {0, 1} |
| EP | 自车行驶进度 | [0, 1] |
| TTC | 碰撞时间 | {0, 1} |
| LK | 车道保持 | {0, 1} |
| HC | 历史舒适性 | {0, 1} |

最终的EPDMS（Ego-Pseudo Driving Metric System）分数是这些子奖励的加权和：

```
r̂_epdms = Σ_m w_m · r̂_m
```

这种分解式设计的好处是：
1. **可解释性**：我们可以知道模型在哪个维度表现好/差
2. **多目标平衡**：通过调整权重，可以灵活控制安全vs舒适vs效率的权衡
3. **稳定性**：某些子奖励的训练信号更稳定，可以帮助优化其他子奖励

#### 2.2.3 多样化轨迹数据收集

为了训练RWM，需要大量多样化的（轨迹，奖励）配对数据。IRL-VLA采用了三种策略：

**策略1：扩散过程采样**
在扩散规划器的每个去噪时间步都记录中间轨迹，这些轨迹代表了不同程度的"噪声"和"合理性"。

**策略2：K-means聚类采样**
从人类演示数据中采样多种轨迹模式，K值从32到8192不等，覆盖保守、激进、居中等多种驾驶风格。

**策略3：多初始姿态采样**
对同一场景，使用不同的自车初始姿态（位置、速度、朝向）进行仿真，生成更多样化的交互结果。

通过这些策略，RWM的训练数据涵盖了：高质量人类轨迹、中等质量模型轨迹、低质量噪声轨迹，从而让奖励模型学会**细粒度的质量区分**。

#### 2.2.4 奖励模型的训练

RWM的训练目标是预测真实仿真器给出的分数。针对不同取值范围的子奖励，使用不同的损失函数：

```
- 二分类指标（DAC, TLC等）：二元交叉熵损失
- 连续指标（EP）：均方误差损失
- 三分类指标（NC, DDC）：多分类交叉熵损失
```

总损失为：

```
L_RWM = Σ_{i,m} w_m · |r̂_i^m - r_i^m|
```

训练完成后，RWM就可以**替代真实仿真器**，为任意给定轨迹快速预测质量分数，速度比真实仿真器快数个数量级。

### 2.3 阶段三：PPO闭环强化学习微调

有了RWM，就可以进行真正的闭环强化学习了。IRL-VLA采用PPO（Proximal Policy Optimization）算法，这是因为在与学到的奖励模型交互时，**稳定性**至关重要——奖励模型的预测误差可能导致训练崩溃。

#### 2.3.1 扩散策略的马尔可夫决策过程视角

将扩散规划器建模为一个**内在MDP**（Markov Decision Process）：

- **状态**：当前噪声水平 x_t，条件信息 c（场景特征、语义指令）
- **动作**：去噪步骤 x_{t-1}
- **策略**：π_θ(x_{t-1} | x_t, c) = N(μ_θ(x_t, c, t), σ_t²I)

每个轨迹由τ个去噪步骤组成，整个扩散过程的概率可以分解为：

```
log π_θ(x_{0:T}) = Σ_{t=1}^{T} log π_θ(x_{t-1} | x_t, c)
```

#### 2.3.2 RWM驱动的奖励计算

在每个训练迭代中：

1. 从当前策略采样一批轨迹 {T_traj}
2. RWM对每个轨迹计算细粒度奖励分数 r_epdms
3. 结合critic网络的值估计，使用GAE（Generalized Advantage Estimation）计算优势函数

为了处理奖励模型的预测方差，IRL-VLA采用**分组标准化优势**：

```
Â_i = (R_epdms - mean(R_{1:T_trajs})) / std(R_{1:T_trajs})
```

这种标准化让优势值在批次内保持相对关系，同时减小绝对数值的波动。

#### 2.3.3 PPO策略优化

PPO的 clipped surrogate objective：

```
L_RL = -1/(T_trajs·T) Σ_{i=1}^{T_trajs} Σ_{t=1}^{T} γ^{t-1} log π_θ(x_{t-1}^{(i)} | x_t^{(i)}, c) · Â_i - DKL(π_θ || π_ref)
```

其中包含两个关键设计：

- **折扣因子γ**：对早期去噪步骤给予更高权重，因为初始步骤的决策对最终轨迹质量影响更大
- **KL散度约束**：防止策略偏离预训练策略太远，避免灾难性遗忘

#### 2.3.4 行为克隆混合训练

纯强化学习容易导致策略崩溃。IRL-VLA在优化目标中加入行为克隆项：

```
L = L_RL + w_IL · L_IL
```

实验发现，当w_IL = 0.5时达到最佳平衡点——既保留了人类驾驶的基本模式，又通过RL探索了更优策略。

## 三、实验结果：CVPR 2025挑战赛亚军的技术底气

### 3.1 NAVSIM v2 Benchmark性能

NAVSIM是学术界广泛认可的端到端驾驶benchmark，基于nuPlan数据集构建，提供了非反应式仿真环境和详细的驾驶指标评估。

| 方法 | EPDMS | NC | DAC | EP | LK | HC |
|------|-------|-----|-----|-----|-----|-----|
| DiffusionDrive | 63.2 | 97.1 | 93.5 | 74.2 | 97.5 | 96.3 |
| WOTE | 66.7 | 96.3 | 92.8 | 78.3 | 96.8 | 94.2 |
| GTRS-Dense | 72.1 | 97.8 | 94.2 | 76.5 | 97.2 | 95.1 |
| GTRS-Aug | 74.3 | **98.9** | 94.8 | 76.1 | 97.5 | 95.8 |
| **IRL-VLA-PT** | **74.4** | 98.3 | 94.5 | **83.9** | 97.8 | **97.2** |
| **IRL-VLA-RL** | **74.9** | 98.5 | 94.7 | **84.2** | 97.9 | **97.5** |

表1：IRL-VLA在NAVSIM Navhard-real benchmark上的表现

**关键发现**：

1. **预训练即超越**：仅用模仿学习的IRL-VLA-PT就已经达到74.4 EPDMS，超越所有对比方法。这证明了分层VLA架构的强大表达能力。

2. **RL进一步提升**：经过PPO微调后，IRL-VLA-RL达到74.9 EPDMS，刷新了SOTA。

3. **卓越的舒适性**：EP（ego progress）83.9 vs GTRS-Aug的76.1，HC（history comfort）97.2 vs 95.8——IRL-VLA在不牺牲安全性的前提下，显著提升了驾驶舒适性。

4. **平衡的NC得分**：98.3的NC（无责任碰撞）虽然略低于GTRS-Aug的98.9，但考虑到GTRS-Aug牺牲了EP和EC（extended comfort）来换取NC，IRL-VLA展现了更好的多目标平衡能力。

### 3.2 CVPR 2025 Autonomous Grand Challenge

在CVPR 2025的端到端自动驾驶挑战赛中，IRL-VLA获得了**1st runner up**（亚军）的成绩，在NAVSIM v2 benchmark上取得了**45.0 EDPMS**的分数。

这个成绩的意义在于：

- **真实传感器输入**：不同于某些方法使用完美感知信息（如ground truth BEV），IRL-VLA接收的是原始摄像头图像
- **闭环评估**：在NAVSIM的非反应式仿真环境中，模型需要处理多帧连续决策，而不是开环的单帧预测
- **泛化能力**：Navhard split专门挑选了困难场景（复杂交叉口、高密度交通、极端行为）

## 四、技术深度解析：为什么IRL-VLA能成功？

### 4.1 逆强化学习的"数据效率"优势

传统RL需要与环境交互数百万次才能学会一个好的策略。而IRL-VLA的RWM本质上是一个**监督学习模型**，它从已有的（轨迹，分数）配对中学习。这意味着：

- **样本效率**：RWM只需要几十万条轨迹-分数对就能收敛
- **离线学习**：RWM的训练完全离线，不需要实时仿真
- **可复用性**：一旦训练完成，RWM可以用于训练多个不同的策略

更重要的是，RWM通过**逆强化学习**学到的奖励函数，比手工设计的奖励函数更能捕捉人类驾驶的本质。它不是在优化某个具体的物理量（如加速度），而是在学习**隐式的人类偏好**。

### 4.2 奖励分解与多目标优化

IRL-VLA的奖励分解设计（8个子奖励）借鉴了多目标优化领域的思想。通过显式建模每个维度，系统可以：

- **诊断问题**：当模型表现不佳时，可以快速定位是哪个子奖励出了问题
- **灵活调整**：通过调整权重w_m，可以轻松实现"安全第一"或"效率优先"等不同驾驶风格
- **避免欺骗**：单一标量奖励容易被"黑客攻击"（找到意外的高奖励捷径），分解设计增加了欺骗难度

### 4.3 扩散模型+RL的"探索-利用"平衡

扩散模型天然适合与RL结合，因为：

- **探索**：扩散过程的随机性提供了内在探索机制，模型可以尝试不同的去噪路径
- **多模态**：扩散模型可以同时生成多条候选轨迹，RL只需要选择/优化最好的那些
- **连续性**：扩散模型的输出是连续轨迹，可以直接用于控制，避免了离散动作的离散化损失

PPO的KL散度约束在这里起到了关键作用——它确保RL微调后的策略不会偏离预训练策略太远，保留了模仿学习学到的基本驾驶常识。

### 4.4 第一个真正的"无仿真器"闭环VLA

IRL-VLA最重要的贡献是证明了：**VLA模型可以在不依赖高保真仿真器的情况下进行闭环强化学习**。

这打破了长期以来的一个假设——"没有仿真器就无法训练端到端自动驾驶"。RWM作为一个**轻量级的数据驱动替代方案**，不仅计算效率更高（快数个数量级），还从根本上消除了Sim2Real领域差距，因为RWM直接学习真实世界的奖励分布。

## 五、局限性与未来方向

### 5.1 当前局限

**局限1：RWM的泛化边界**

RWM是在NAVSIM数据集上训练的，其泛化能力受限于训练数据的分布。当遇到与训练数据差异巨大的场景（如完全不同国家的交通规则、极端罕见的天气条件）时，RWM的预测可能不准确。

**局限2：非反应式仿真**

NAVSIM的仿真器是非反应式的——周围车辆不会根据自车行为做出反应。这意味着IRL-VLA尚未在真正的"交互式"闭环环境中验证。在多智能体博弈场景（如无保护左转、汇入匝道）中，这种简化可能导致性能下降。

**局限3：计算资源需求**

虽然RWM比高保真仿真器高效，但IRL-VLA的整体训练仍然需要8张A100 GPU。对于资源受限的研究团队来说，这仍是一个门槛。

**局限4：缺乏真实世界验证**

当前所有结果都是在仿真benchmark上获得的。真实世界的传感器噪声、系统延迟、执行器特性等尚未被充分考虑。

### 5.2 未来研究方向

**方向1：跨数据集迁移学习**

如何让在NAVSIM上训练的RWM迁移到nuPlan、Waymo Open Dataset甚至真实世界？这需要研究领域自适应（Domain Adaptation）和元学习（Meta-Learning）技术。

**方向2：交互式多智能体强化学习**

将IRL-VLA扩展到反应式仿真环境，让模型学习与其他车辆的交互策略。这可能需要引入博弈论思想和多智能体RL算法。

**方向3：在线学习与持续改进**

当前IRL-VLA是离线训练的。未来的系统应该能够从真实驾驶数据或影子模式（shadow mode）运行中持续学习，不断更新RWM和策略。

**方向4：可解释性与安全性验证**

VLA模型的决策过程仍然是一个"黑盒"。需要开发更好的可视化工具和形式化验证方法，确保模型在关键安全场景中的行为符合预期。

## 六、产业影响与思考

### 6.1 对自动驾驶行业的启示

IRL-VLA的发表标志着VLA自动驾驶进入了**"闭环强化学习时代"**。在此之前，大多数VLA工作停留在模仿学习阶段；在此之后，我们有望看到更多结合RL的VLA系统。

对于自动驾驶公司来说，IRL-VLA提供了一个重要的技术路径：

1. **降低仿真器依赖**：不再需要投入巨资建设高保真仿真器
2. **数据驱动奖励**：利用积累的人类驾驶数据直接学习奖励模型
3. **快速迭代**：RWM的训练和更新比物理仿真器快得多

### 6.2 与特斯拉FSD的潜在联系

值得注意的是，特斯拉的FSD v12被广泛认为是第一个"端到端"量产系统，其训练也结合了模仿学习和强化学习。虽然技术细节不公开，但IRL-VLA的思想与特斯拉的技术路线有异曲同工之妙：

- 都强调从真实数据中学习效率
- 都使用某种形式的"世界模型"来替代或辅助仿真
- 都采用端到端的神经网络架构

IRL-VLA的发表，某种程度上为学术界提供了一个可复现的"特斯拉FSD技术替代方案"。

### 6.3 开源与 reproducibility

论文提到代码将发布在 https://github.com/IRL-VLA/IRL-VLA。如果这一承诺兑现，IRL-VLA将成为VLA自动驾驶领域的重要开源基准，推动整个社区的快速发展。

## 七、总结：从模仿到理解，从开环到闭环

IRL-VLA代表了端到端自动驾驶的一个重要里程碑。它的核心贡献可以概括为三点：

**第一，技术范式创新**：首次实现了不依赖仿真器的闭环VLA强化学习，通过逆强化学习构建的奖励世界模型，既保留了数据驱动的优势，又避免了Sim2Real领域差距。

**第二，卓越的性能**：CVPR 2025挑战赛亚军，NAVSIM v2 SOTA，这些成绩证明了方法的有效性。更重要的是，它在安全性、舒适性、效率性之间取得了出色的平衡。

**第三，可扩展的架构**：三阶段训练范式（模仿→逆强化学习→RL）具有很强的通用性，可以应用于其他VLA模型，甚至其他具身智能任务。

展望未来，随着VLA模型规模的扩大和RWM训练数据的积累，我们有理由期待：

- 更通用的驾驶策略，能够零样本迁移到新城市、新国家
- 更安全的决策系统，能够处理人类驾驶员也会感到棘手的极端场景
- 更自然的驾驶风格，让乘客感受到"这不是AI在开车，而是一位经验丰富的老司机"

IRL-VLA不是终点，而是一个新的起点。它向我们展示了：当逆强化学习遇见VLA，闭环自动驾驶的未来正在加速到来。
