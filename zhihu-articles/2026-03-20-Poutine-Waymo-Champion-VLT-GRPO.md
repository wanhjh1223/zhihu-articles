## 一、引言：长尾场景的自动驾驶难题

### 1.1 当自动驾驶遇到"奇葩"场景

自动驾驶领域有一个公开的秘密：**普通场景已经解决得差不多了，真正的难点在于长尾场景**。

想象以下情景：
- 施工区域临时改道，路锥摆放不规则
- 前方车辆突然抛锚，占用了半个车道
- 暴雨天视线受阻，路边有行人打着伞横穿
- 学校门口放学时段，大量儿童和家长在路边聚集
- 高速公路上有掉落的货物散落在路面

这些场景不常见，但每一个都可能致命。传统的模块化自动驾驶系统依靠人工编写的规则来应对，但规则永远无法覆盖所有可能的情况。端到端深度学习虽然能够学习数据分布，但在训练时没见过的场景（OOD，Out-of-Distribution）上往往表现糟糕。

### 1.2 大语言模型的启示

2022年底，ChatGPT的出现让整个AI界震惊。这个基于大语言模型（LLM）的对话系统展现出了惊人的**常识推理能力**和**泛化能力**——它能回答从未见过的问题，能在不同领域间灵活切换，能理解上下文中的细微含义。

研究者开始思考：如果自动驾驶系统也能具备这种通用推理能力，是不是就能更好地应对长尾场景？毕竟，驾驶本质上也是一种需要常识和逻辑的决策任务。

这个思路催生了**视觉-语言-动作（Vision-Language-Action, VLA）**模型在自动驾驶领域的兴起。VLA模型将图像理解、语言推理和动作控制统一在一个端到端框架中，试图让自动驾驶系统像人类驾驶员一样"看懂、想通、做对"。

### 1.3 复杂度的陷阱

然而，早期的VLA驾驶模型往往陷入一个误区：**过度工程化**。

为了适配自动驾驶任务，研究者们设计了各种复杂的组件：
- 手工设计的轨迹tokenizer
- 专门的感知模块（BEV编码器、对象检测器）
- 复杂的注意力机制
- 多阶段训练流程

这些组件的初衷是引入领域知识，但实际情况是：它们往往限制了基础VLM的能力发挥，增加了系统的复杂性，却未必带来预期的性能提升。

### 1.4 Poutine的极简主义宣言

来自加拿大Mila研究所的Poutine团队提出了一个大胆的假设：**也许我们不需要这些复杂的组件**。

他们的核心观点是：
1. 现有的开源VLM（如Qwen2.5-VL）已经具备了强大的视觉理解和推理能力
2. 与其添加复杂的适配组件，不如直接用简单有效的方式微调这些模型
3. 关键在于**训练数据的质量和规模**，以及**后训练的优化策略**

这个思路催生了Poutine——一个仅3B参数、没有任何额外组件的端到端自动驾驶系统，却在2025 Waymo视觉端到端挑战赛上以**RFS 7.99的成绩获得第一名**，远超第二名。

（题外话：论文名字"Poutine"是加拿大魁北克省的特色美食，由薯条、肉汁和奶酪凝乳组成。作者以此命名，可能是想表达"简单食材也能做出美味"的哲学。）

## 二、Poutine的技术架构：极简而不简单

### 2.1 整体架构：纯粹的基础模型

Poutine的架构简单得令人惊讶：

**基础模型**：Qwen2.5-VL-3B-Instruct  
**额外组件**：无  
**Tokenizer修改**：无  
**架构修改**：无

没错，Poutine直接使用了一个标准的3B参数视觉语言模型，没有添加任何专门的驾驶模块，没有设计复杂的轨迹编码器，没有引入BEV感知组件。

这种"拿来主义"看似偷懒，实则是一种更深层次的自信：**相信基础模型的能力，专注于训练策略的优化**。

### 2.2 输入输出设计

**输入**：
- 多视角历史图像序列（车载摄像头捕获）
- 高级导航指令（如"左转"、"直行"）
- 历史自车轨迹
- 任务描述（可选）

**输出**：
- 驾驶解释（自然语言描述）
- 未来轨迹（以文本形式表示）

Poutine支持两种输出模式：
1. **CoT模式**：先生成链式思考（Chain-of-Thought）推理，再输出轨迹
2. **直接模式**：直接输出未来轨迹

### 2.3 轨迹的文本化表示：关键的设计选择

Poutine最核心的设计决策之一是**将轨迹直接表示为文本**。

传统的端到端驾驶模型通常使用专门的轨迹编码器，将连续的轨迹点（x, y坐标序列）编码为特殊的token或向量。这种做法的问题是：
- 需要设计专门的tokenizer
- 限制了模型利用预训练语言知识的能力
- 增加了系统复杂性

Poutine的做法是直接把轨迹当作文本序列来处理：

```
轨迹示例：
"在未来3秒内，车辆将先轻微右转，然后保持直线行驶，
 预计在坐标 (10.2, 2.1) 处到达..."
```

或者更简单的形式：
```
"x: 0.0, 1.2, 3.5, 6.1; y: 0.0, 0.1, 0.3, 0.5"
```

这种文本化表示的优势：
1. **无需修改tokenizer**：直接使用VLM的词表
2. **充分利用预训练知识**：模型已经理解数字、坐标等概念
3. **可解释性**：人类可以直接阅读模型输出的轨迹
4. **灵活性**：可以轻松添加语言解释

**这是一个被验证的正确选择**。后续实验表明，这种简单的方法比使用复杂自定义tokenizer的AutoVLA（同场竞赛的参赛作品）表现好得多。

## 三、两阶段训练策略：VLT预训练 + GRPO后训练

Poutine的训练分为两个核心阶段，这个设计体现了对VLM能力释放的深刻理解。

### 3.1 第一阶段：VLT预训练（建立基础能力）

**VLT = Vision-Language-Trajectory（视觉-语言-轨迹）**

这一阶段的目标是让模型学会**将视觉输入、语言理解和轨迹规划统一起来**。

#### 训练数据

**CoVLA数据集**（83小时）：
- 日本地区的正常驾驶场景
- 包含多视角视频、自车轨迹
- 涵盖各种道路类型和城市环境

**WOD-E2E训练集**（11小时）：
- Waymo开放数据集的长尾场景
- 经过精心筛选的挑战性场景
- 包含施工区、异常物体、复杂交通交互等

#### 语言标注的自动生成

Poutine的一个关键创新是**完全自动化的语言标注流程**：

1. 使用一个72B参数的VLM（比Poutine本身大20多倍）自动生成驾驶场景的语言描述
2. 描述内容包括：
   - 场景描述（"前方有施工区域"）
   - 驾驶行为解释（"需要减速并变道"）
   - 推理过程（"因为左侧车道被占用..."）

3. 这种"大模型教小模型"的策略带来了几个好处：
   - **无需人工标注**：大幅降低数据成本
   - **规模可扩展**：只要有视频数据就能生成标注
   - **质量可控**：通过调整大模型的prompt可以控制标注风格和内容

#### 训练目标

标准的自监督下一个token预测：

```
输入：[图像序列] + [导航指令] + [历史轨迹] + "驾驶说明："
输出："前方车辆减速，需要保持安全距离。预测轨迹：x: 0.0, 2.1, 4.5..."
```

模型需要同时学习：
- 从图像中提取驾驶相关信息
- 理解导航指令的语义
- 将视觉-语言信息转化为轨迹
- 生成人类可读的驾驶解释

#### 训练细节

**CoVLA预训练**：
- 1个epoch
- batch size: 64
- 学习率: 1e-5（余弦衰减）
- 硬件: 4×NVIDIA A100
- 时间: 24小时

**WOD-E2E预训练**：
- 2个epoch
- batch size: 16
- 学习率: 1e-5
- 时间: 10小时

**训练后的Poutine-Base已经展现出了惊人的能力**：
- 验证集RFS达到8.12（满分10分）
- Waymo专家轨迹的RFS是8.13
- **也就是说，Poutine-Base几乎达到了人类专家的水平**

### 3.2 第二阶段：GRPO后训练（优化长尾表现）

虽然Poutine-Base已经很强大，但在长尾场景上还有提升空间。Poutine团队选择了**Group Relative Policy Optimization (GRPO)**进行后训练。

#### 为什么用GRPO？

GRPO是DeepSeek团队提出的一种强化学习算法，相比PPO有以下优势：
- **无需critic模型**：降低内存和计算开销
- **组内相对奖励**：使用同一问题的多个采样结果计算相对优势，减少方差
- **样本效率高**：适合数据量有限的场景

#### 训练设置

**数据**：
- WOD-E2E验证集
- 仅使用不到500帧人类偏好标注的数据
- 每帧都有人工评分员（rater）提供的质量分数（RFS）

**奖励函数**：
- 直接使用Rater Feedback Score（RFS）作为奖励信号
- RFS范围1-10，反映轨迹质量和驾驶安全性

**超参数**：
- 训练步数: 2000
- 学习率: 1e-6（线性衰减）
- 采样温度: 0.9
- β (KL惩罚系数): 0.04
- 每样本rollout数: 8
- batch size: 32
- 时间: 12小时（4×A100）

#### GRPO带来的提升

GRPO后训练带来了显著的性能提升：

| 模型 | 测试集RFS |
|------|----------|
| Waymo基线 | 5.21 |
| Poutine-Base | 7.91 |
| **Poutine (with GRPO)** | **7.99** |

**提升幅度：+0.08 RFS**

虽然数字看起来不大，但考虑到：
1. 这仅是不到500帧数据的微调结果
2. Poutine-Base本身已经很强（7.91）
3. 人类专家水平是8.13

这个提升实际上非常可观。Poutine-Base到Poutine的改进，主要体现在**长尾场景的鲁棒性**上。

## 四、实验结果：数据说话

### 4.1 Waymo 2025挑战赛冠军

2025 Waymo视觉端到端驾驶挑战赛吸引了全球顶尖团队参与。比赛任务是在长尾驾驶场景下进行端到端的轨迹规划。

**比赛结果**（按RFS排序）：

| 排名 | 方法 | RFS | 备注 |
|-----|------|-----|------|
| 1 | **Poutine** | **7.99** | 本文方法 |
| 2 | Poutine-Base | 7.91 | 无RL版本 |
| 3 | UniPlan | ~7.5 | 扩散模型方案 |
| 4 | AutoVLA | ~7.2 | 使用自定义tokenizer |
| ... | ... | ... | ... |
| - | Waymo基线 | 5.21 | - |
| - | Ground Truth | 8.13 | 人类专家轨迹 |

**Poutine不仅获得第一，而且是大幅领先**：
- 比第二名（自己的基础版本）高0.08分
- 比第三名高0.5分以上
- 比Waymo官方基线高2.78分

### 4.2 零样本跨域泛化：日本→美国

Poutine展现出了一个令人惊讶的能力：**仅用日本驾驶数据训练，就能在美国数据上表现良好**。

实验设置：
- 仅在CoVLA（日本数据）上进行VLT预训练
- 不进行任何WOD-E2E（美国数据）训练
- 直接在WOD-E2E验证集上测试

结果：
- **零样本RFS: 7.74**

这意味着什么？
- 日本是**左侧通行**，美国是**右侧通行**——这是根本性的交通规则差异
- 道路标志、交通信号灯、建筑风格、车辆类型都有显著差异
- 城市布局、驾驶习惯完全不同

尽管如此，Poutine-Base CoVLA仍然取得了7.74的RFS，接近在Waymo数据上训练过的模型。

**这个结果的启示**：
1. **VLT预训练学到了通用的驾驶原理**，而不仅仅是特定地区的驾驶模式
2. **大规模多样化的预训练数据**能够带来强大的泛化能力
3. **基础VLM的世界知识**确实能够帮助自动驾驶决策

### 4.3 消融实验：每个组件的价值

Poutine团队进行了详细的消融实验，验证每个设计决策的价值：

| 配置 | RFS (验证集) | 说明 |
|-----|-------------|------|
| Qwen 2.5-VL 3B (无预训练) | 5.59 | 基础模型直接测试 |
| Qwen 2.5-VL 7B (无预训练) | 5.40 | 更大的基础模型，反而更差 |
| Ego-status MLP | 7.47 | 仅用自车状态，无视觉得分 |
| Poutine-Base CoVLA | 7.74 | 仅用CoVLA预训练 |
| Poutine-Base No-CoVLA | 7.95 | 仅用WOD-E2E预训练 |
| Poutine-Base No Language | 7.94 | 去掉语言标注 |
| **Poutine-Base** | **8.12** | 完整配置 |
| Poutine-Base CoT | 8.08 | 生成CoT推理 |
| Ground Truth | 8.13 | 专家轨迹上限 |

**关键发现**：

1. **VLT预训练至关重要**
   - 无预训练的3B模型：5.59
   - 无预训练的7B模型：5.40（更大但表现更差，因为没有预训练）
   - 有预训练的3B模型：8.12
   - **结论：预训练比模型大小更重要**

2. **CoVLA和WOD-E2E数据都重要**
   - 仅用CoVLA：7.74
   - 仅用WOD-E2E：7.95
   - 两者都用：8.12
   - **结论：多样化的数据来源带来互补价值**

3. **语言标注有价值但非决定性**
   - 无语言标注：7.94
   - 有语言标注：8.12
   - **结论：语言监督带来约0.18的提升**

4. **CoT推理在测试时并非必需**
   - 无CoT推理：8.12
   - 有CoT推理：8.08
   - **结论：CoT训练有帮助，但推理时生成CoT并不能进一步提升性能**

### 4.4 与其他方法的对比

Poutine与AutoVLA的对比特别有意思，因为两者都是基于VLM的端到端驾驶方法：

| 特性 | Poutine | AutoVLA |
|-----|---------|---------|
| 基础模型 | Qwen2.5-VL 3B | Qwen2.5-VL |
| 轨迹表示 | 纯文本 | 自定义tokenizer |
| 额外组件 | 无 | 有 |
| RFS提升（相对基线） | +2.78 | +0.03 |

**这个对比验证了Poutine的核心假设**：复杂的自定义组件反而限制了VLM的能力发挥，简单的文本化表示效果更好。

## 五、深度解读：为什么Poutine能成功

### 5.1 极简主义的设计哲学

Poutine的成功首先是一种**设计哲学的胜利**。

在深度学习领域，有一个常见的误区：**为了解决问题，不断添加新组件**。这种做法的结果是：
- 系统越来越复杂
- 训练越来越困难
- 调试越来越痛苦
- 性能却不一定提升

Poutine反其道而行之：
- 做减法而不是加法
- 移除所有非必需的组件
- 专注于训练策略的优化

这种极简主义带来的好处：
1. **更低的复杂度**：更容易理解和复现
2. **更好的可扩展性**：可以专注于数据和计算规模的扩大
3. **更强的泛化性**：没有过拟合到特定组件的风险
4. **更快的迭代**：简单的系统更容易实验和改进

### 5.2 文本化表示的深层优势

将轨迹表示为文本，看似是一个工程选择，实则有深刻的理论基础。

**语言作为通用的表示空间**：
- 人类用语言描述世界，语言承载了丰富的常识和逻辑
- VLM在预训练阶段已经学习了大量的语言知识和推理模式
- 将轨迹嵌入语言空间，天然地利用了这些预训练知识

**对比自定义tokenizer**：
- 自定义tokenizer需要从零开始学习轨迹表示
- 它的嵌入空间与语言空间是割裂的
- 无法利用VLM的预训练知识

这解释了为什么AutoVLA（使用自定义tokenizer）的表现不如Poutine。

### 5.3 VLT预训练的数据效率

Poutine的另一个关键洞见是：**高质量的多样化预训练数据比海量单一数据更重要**。

CoVLA和WOD-E2E加起来只有94小时的驾驶数据，这听起来不多，但：
- CoVLA提供了**多样化的地理分布**（日本）
- WOD-E2E提供了**挑战性场景**（长尾）
- 两者结合覆盖了广泛的驾驶情况

相比之下，仅用WOD-E2E（11小时）的得分是7.95，仅用CoVLA（83小时）是7.74，但两者结合达到了8.12。

**这说明**：
1. 长尾场景的数据特别珍贵
2. 地理多样性带来泛化能力
3. 预训练的数据组合策略很关键

### 5.4 GRPO的高效样本利用

GRPO后训练仅用了不到500帧偏好数据，就带来了明显的性能提升。这体现了GRPO的优势：

1. **组内相对奖励**：不需要绝对准确的奖励估计
2. **采样效率**：通过多次采样和相对比较，从有限数据中提取更多信息
3. **稳定性**：相对优势估计减少了方差

对于自动驾驶这样的领域，高质量的偏好标注数据非常昂贵（需要专业驾驶员或安全员评估）。GRPO的高效样本利用意味着可以用更少的标注成本获得更好的性能。

### 5.5 基础模型的能力边界

Poutine的成功也给我们一个启示：**现代开源VLM（如Qwen2.5-VL）已经具备了足够强大的基础能力**。

3B参数的规模并不算大，但：
- 它已经在海量互联网数据上预训练
- 具备了视觉理解、常识推理、空间感知等能力
- 需要的只是**领域适配**（domain adaptation）

这意味着：
- 更多的研究资源应该投入到**训练策略**而非**架构设计**
- 开源VLM的潜力巨大，远未被充分挖掘
- 未来的自动驾驶VLA可能不需要专门设计的基础模型

---

## 六、局限性与未来方向

### 6.1 当前局限

尽管Poutine取得了突破性成果，但仍有一些局限值得注意：

**1. 开环评估的局限**
- Poutine在Waymo开环基准上测试，模型不会真正控制车辆
- 开环评估无法完全反映闭环驾驶中的时序一致性和安全约束
- 未来需要在闭环仿真或真实场景中进行验证

**2. 长尾场景的覆盖不足**
- 虽然比传统方法更好，但在某些罕见场景上仍有改进空间
- 例如，Poutine在Spotlight、Construction和Multi-lane maneuvers三类场景上不如Poutine-Base
- 这说明GRPO的后训练可能带来了一些过拟合

**3. 计算效率**
- 3B参数模型在边缘设备上运行仍有挑战
- 实时性要求（通常需要10-30Hz的推理频率）可能需要进一步优化

**4. 可解释性的局限**
- 虽然CoT机制提供了一定的可解释性，但深度学习模型的决策过程仍是黑盒
- 在安全关键领域，可解释性至关重要

### 6.2 未来研究方向

基于Poutine的洞见，以下几个方向值得探索：

**1. 更大规模的VLT预训练**
- Poutine仅用了94小时数据
- 如果有1000小时甚至10000小时的多样化驾驶数据，性能会如何？
- 可以考虑加入欧洲、中国等更多地区的驾驶数据

**2. 多模态感知融合**
- Poutine仅使用视觉输入
- 加入LiDAR、雷达、高精地图等信息可能进一步提升性能
- 挑战在于如何将这些模态与VLM有效融合

**3. 闭环强化学习**
- 在仿真环境中进行闭环RL训练
- 可以探索世界模型（World Model）用于数据生成
- 安全约束的引入是关键

**4. 更高效的模型架构**
- 研究如何在保持性能的前提下减小模型规模
- 知识蒸馏、量化、剪枝等技术的应用
- 专为边缘设备设计的轻量级VLM

**5. 跨任务迁移学习**
- Poutine的VLT预训练能否迁移到其他具身智能任务？
- 机器人操作、导航等领域是否适用类似方法？

---

## 七、对行业的深远影响

### 7.1 研发范式的转变

Poutine的成功可能推动自动驾驶领域的研发范式转变：

**从"堆组件"到"优训练"**：
- 过去：设计复杂的新模块来解决特定问题
- 未来：利用强大的基础模型，专注于训练策略优化

**从"专用模型"到"通用能力"**：
- 过去：为每个任务训练专门的模型
- 未来：一个强大的VLA模型处理多种驾驶场景

### 7.2 数据策略的重新思考

Poutine的结果强调了**数据多样性**和**质量**的重要性：

- 地理多样性（日本→美国）带来泛化能力
- 长尾场景的专门数据对性能至关重要
- 自动生成语言标注是 scalable 的解决方案

这可能改变自动驾驶公司的数据收集策略：
- 更多资源投入到长尾场景的采集
- 利用大模型自动生成高质量标注
- 构建全球化的多样化数据集

### 7.3 开源生态的繁荣

Poutine完全基于开源组件（Qwen2.5-VL），这也展示了开源生态的力量：

- 研究社区可以快速复现和扩展
- 不需要从头训练基础模型，节省大量计算资源
- 促进了知识的快速传播和累积

可以预见，未来会有更多基于开源VLM的自动驾驶解决方案涌现。

### 7.4 硬件部署的新思路

虽然3B参数对边缘设备仍有挑战，但Poutine证明了**小模型也能达到接近专家水平的性能**：

- 这降低了高性能自动驾驶系统的硬件门槛
- 通过进一步的模型压缩和优化，边缘部署成为可能
- 可能加速L3/L4级自动驾驶的量产落地

---

## 八、结语

Poutine是自动驾驶VLA领域的一个重要里程碑。它用最简单的方式——一个没有额外组件的3B参数VLM，配合精心设计的VLT预训练和GRPO后训练——击败了所有复杂架构的对手，拿下了Waymo 2025挑战赛的冠军。

这个结果传递了一个清晰的信号：**在拥有强大的基础模型时，训练策略比架构设计更重要**。

Poutine的极简主义哲学、文本化轨迹表示、高效的样本利用，都为未来的自动驾驶研究提供了宝贵的参考。更重要的是，它展示了开源VLM在物理世界决策任务中的巨大潜力。

当然，Poutine仍是一个开环评估的系统，距离真正的量产部署还有距离。但它指明了一条清晰的道路：**利用通用人工智能的能力，通过高效的领域适配，解决自动驾驶的长尾难题**。

在不远的将来，当我们看到一辆自动驾驶汽车优雅地处理各种复杂场景时，也许会想起这个来自魁北克的"Poutine"——那个用简单食材做出了顶级美味的系统。
