## 一、引言：当自动驾驶学会"主动观察"

想象一下，当你在高速公路上行驶时，前方车辆突然开始减速。作为人类驾驶员，你会怎么做？——你会下意识地眯起眼睛、身体前倾，试图看清前方到底发生了什么。可能是交通事故，可能是施工路段，也可能只是前车司机在接电话。无论哪种情况，**主动获取更清晰的视觉信息**是你做出正确决策的前提。

然而，当前的视觉-语言-动作（VLA）自动驾驶系统却存在一个根本性的局限：**它们都是"被动感知"的**。模型被喂入固定的摄像头画面，只能基于这些有限的信息进行推理。当默认视角不足以判断复杂情况时，模型没有机制去"凑近看一看"或"换个角度观察"。这就好比让一个人蒙着眼睛开车，只通过别人的口述来了解路况——即便这个人口述能力再强，也难免遗漏关键细节。

2025年7月，来自理想汽车、清华、同济等机构的研究团队发表了 DriveAgent-R1，首次将**主动感知（Active Perception）**引入自动驾驶的高级行为规划任务。这篇论文的核心理念很简单却极具颠覆性：**让自动驾驶 agent 像人类司机一样，在不确定时主动调用工具获取视觉证据，将决策建立在扎实的视觉基础之上。**

更令人惊叹的是，这个仅有 **30亿参数** 的模型，在性能上竟能与 **GPT-5** 和人类司机相媲美，同时保持了部署友好的轻量化优势。它是如何做到的？让我们深入剖析这篇论文的技术内涵。

## 二、研究背景：VLA 模型的困境与突破方向

### 2.1 从端到端到 VLA：自动驾驶范式的演进

自动驾驶技术的发展经历了多个阶段的范式转移：

**阶段一：模块化方案（Modular Pipeline）**
早期的自动驾驶系统采用分模块设计：感知（检测、跟踪、分割）→ 预测（他车轨迹预测）→ 规划（自车轨迹生成）→ 控制（执行器控制）。这种方案的每个模块都有明确的输入输出接口，便于调试和优化，但模块间的信息损失和误差累积成为性能瓶颈。

**阶段二：端到端学习（End-to-End Learning）**
受深度学习浪潮推动，研究者开始尝试直接从传感器输入（图像、点云）映射到控制输出（方向盘、油门、刹车）。2016年英伟达的 PilotNet 是这一方向的先驱。端到端方案消除了人工设计的中间表示，让数据自己说话，但黑盒特性带来了可解释性和安全性挑战。

**阶段三：视觉-语言-动作模型（VLA）**
2023年以来，大语言模型（LLM）的崛起为自动驾驶带来了新的可能性。VLA 模型将视觉编码器、语言模型和动作解码器统一在一个框架内，能够处理"看到什么→理解什么→做什么→为什么"的完整链条。代表性工作包括 Wayve 的 LINGO-2、清华团队的 DriveVLM 等。

### 2.2 被动感知的根本性局限

尽管 VLA 模型展现了强大的语义理解能力，但现有方法普遍面临一个核心问题：**被动感知范式（Passive Perception Paradigm）**。

具体而言，当前 VLA 系统的工作流程是：
1. 接收固定的多摄像头输入（前视、环视等）
2. 将所有视觉信息一次性编码为特征向量
3. 基于这些固定特征进行文本推理
4. 输出驾驶决策

这种"一锤子买卖"的感知方式存在两个相互矛盾的问题：

**问题一：信息不足时的无力感**
当默认视角无法提供足够信息时（如远处的事故、被遮挡的行人、模糊的 signage），模型缺乏机制去主动获取补充证据。它只能基于不完整的视觉输入"猜"一个答案，这可能导致灾难性后果。

**问题二：信息过载时的注意力分散**
为了解决信息不足的问题，一个直观的想法是"把所有摄像头的所有帧都喂给模型"。但这会带来巨大的计算开销，更重要的是，**冗余信息会稀释模型对关键线索的注意力**。人类司机不会同时盯着所有方向，而是根据情况灵活转移注意力——这正是当前 VLA 系统缺失的能力。

### 2.3 主动感知：通向人类级驾驶智能的钥匙

人类驾驶是一个**主动的信息获取和决策过程**。研究表明，人类司机平均每 1.5 秒就会进行一次眼球扫视（saccade），根据当前任务需求动态调整视觉注意力：
- 正常巡航时，主要关注远处路况和后视镜
- 接近路口时，快速扫视左右盲区
- 跟随前车时，观察前车的刹车灯和转向意图
- 遇到异常情况时，凑近观察、变换视角确认

这种**主动感知（Active Perception）**能力是安全驾驶的基石。它体现了智能体在开放环境中应对不确定性的核心能力：**当感知不确定时，采取行动减少不确定性**。

在机器学习领域，主动感知并非新概念。机器人学中的主动 SLAM、主动目标跟踪、主动抓取等任务都涉及类似思想。但在自动驾驶的高级行为规划任务中，**如何让 VLA 模型具备主动感知能力，一直是一个未被探索的空白地带**。DriveAgent-R1 正是填补了这一空白。

## 三、DriveAgent-R1 核心创新：三大技术突破

DriveAgent-R1 的核心贡献可以概括为三个层面：

1. **主动感知框架（Active Perception Framework）**：首次将主动感知引入自动驾驶高级规划，让 agent 能主动调用视觉工具获取证据
2. **混合思考框架（Hybrid Thinking Framework）**：自适应切换高效文本推理与深度视觉推理，平衡性能与效率
3. **级联强化学习训练策略（Cascaded RL）**：通过三阶段渐进训练，逐步培养模型的混合思考能力

下面逐一深入解析。

### 3.1 主动感知框架：从"被动接受"到"主动探索"

#### 3.1.1 问题建模：高级行为规划任务

DriveAgent-R1 聚焦于**高级行为规划（High-Level Behavioral Planning）**，而非低级的轨迹回归。具体而言，模型的输入输出定义如下：

**输入**：
- 初始视觉上下文 $I_0$：前视摄像头图像
- 文本上下文 $T_0$：当前车速、导航指令等结构化信息

**输出**：
- 8 秒的元动作序列 $A = (a_1, a_2, a_3, a_4)$，每 2 秒预测一个动作
- 每个元动作 $a_t = (s_t, j_t)$ 包含：
  - 速度 token $s_t \in V_s$ = {加速, 保持速度, 减速, 停车}
  - 轨迹 token $j_t \in V_j$ = {直行, 右转, 左转, 变道}

这种离散化建模的优势在于：
1. **可解释性强**：每个动作都有明确的语义含义，便于调试和安全验证
2. **与人类直觉对齐**：人类司机思考的是"减速右转"而非具体的轨迹坐标
3. **适合 VLM 的离散输出空间**：避免了连续轨迹回归的不稳定性

#### 3.1.2 视觉工具箱（Vision Toolkit）设计

为了实现主动感知，DriveAgent-R1 配备了一个强大的**视觉工具箱（Vision Toolkit）**，包含四类工具：

**工具一：视角检索（Retrieve View）**
- 功能：从任意摄像头获取清晰图像
- 特色：支持从历史记忆池（5 秒滑动窗口）检索关键帧
- 使用场景："让我看看 3 秒前左后方发生了什么"

**工具二：RoI 检查（RoI Inspection）**
- 功能：对指定感兴趣区域（Region of Interest）进行放大和裁剪
- 特色：从高分辨率原图中提取细节
- 使用场景："放大前方那个模糊的交通标志，看清楚到底是什么"

**工具三：深度估计（Depth Estimation）**
- 功能：生成深度图，提供 3D 空间感知
- 使用场景："判断那个物体离我有多远"

**工具四：3D 目标检测（3D Object Detection）**
- 功能：开放词汇的 3D 目标识别和定位
- 使用场景："识别并定位那个奇怪的障碍物"

这些工具的设计遵循一个核心原则：**按需获取（On-Demand Acquisition）**。模型不会被动接收所有可能的信息，而是根据当前推理需要，精确调用特定工具获取特定信息。

#### 3.1.3 多轮交互式推理流程

DriveAgent-R1 的推理过程是一个**多轮迭代**的交互循环：

```
初始化：H_0 = T_0 ⊕ I_0  (历史 = 文本上下文 ⊕ 初始视觉)

for k = 1, 2, ..., K:
    1. 基于当前历史 H_{k-1}，生成文本思考 T_k
    2. 决定是否调用工具：生成工具调用请求 T_k'
    3. 如果调用工具：
       - 执行工具获取新视觉信息
       - 编码为图像嵌入 I_k
       - 更新历史：H_k = H_{k-1} ⊕ T_k ⊕ T_k' ⊕ I_k
    4. 否则：H_k = H_{k-1} ⊕ T_k
    
    如果生成最终动作序列 A：终止循环
    如果达到最大交互轮数 K：强制终止
```

这个流程的核心优势在于**动态性**：模型不是一次性看完所有信息再做决策，而是**边思考边看，根据思考的需要决定下一步看什么**。这与人类司机的认知过程高度一致。

### 3.2 混合思考框架：平衡效率与深度的智能切换

#### 3.2.1 两种思考模式

虽然主动感知很强大，但**不分青红皂白地对所有场景都调用工具是计算上的浪费**。人类司机也不会对每一个决策都左顾右盼、反复确认——简单场景靠直觉，复杂场景才需要仔细观察。

基于这一观察，DriveAgent-R1 提出了**混合思考框架（Hybrid Thinking Framework）**，定义了两种互补的思考模式：

**模式一：基于文本的 M-CoT（M_text）**
- 触发方式：生成 `<think_text>` token
- 推理过程：仅基于初始输入 $(I_0, T_0)$ 和内部知识进行纯文本推理
- 适用场景：路况清晰、决策明确的常规场景
- 优势：快速、轻量、计算成本低

**模式二：基于工具的 M-CoT（M_tool）**
- 触发方式：生成 `<think_tool>` token
- 推理过程：迭代调用视觉工具，获取新证据辅助决策
- 适用场景：视觉模糊、情况复杂、需要额外确认的场景
- 优势：准确、可靠、有视觉证据支撑

两种模式遵循统一的思维链（CoT）框架：**描述（Description）→ 推理（Reasoning）→ 预测（Prediction）**。这种设计确保了模式切换时的一致性和可解释性。

#### 3.2.2 自适应模式选择

混合思考框架的核心挑战在于：**如何让模型学会在合适的场景选择合适的模式？**

DriveAgent-R1 的解决方案是**让模型自己学习选择**。在推理开始时，模型首先生成一个模式选择 token（`<think_text>` 或 `<think_tool>`），这决定了后续是采用纯文本推理还是工具增强推理。

这种自适应机制的优势：
1. **计算效率**：简单场景快速通过，复杂场景深度处理
2. **可解释性**：模式选择本身反映了模型对场景复杂度的判断
3. **灵活性**：模型可以根据实时情况动态调整策略

### 3.3 级联强化学习训练：三阶段渐进培养

要让模型掌握主动感知和混合思考的能力，需要精心的训练设计。DriveAgent-R1 采用**三阶段渐进训练策略**，遵循"打好基础 → 强化单模式能力 → 学习智能选择"的递进逻辑。

#### 3.3.1 阶段一：领域对齐（Domain Alignment）

在正式训练之前，研究团队发现了一个关键问题：**通用 VLM 在驾驶规划任务中存在"视觉忽视"（Visual Neglect）倾向**——它们倾向于依赖低维文本线索（如车速、历史轨迹）进行决策，而忽略高维视觉输入。这种"走捷径"（Shortcut Learning）行为使得模型并未真正"看懂"道路场景。

为了解决这一问题，DriveAgent-R1 首先进行**领域对齐（Domain Alignment）**训练：

**数据集构建**：
- 从真实交通场景采集图像
- 构建 53 万条驾驶领域 VQA（Visual Question Answering）数据
- 问题涵盖四个维度：
  1. 场景描述（Scene Description）
  2. 交通实体识别（Traffic Entity Recognition）
  3. 关键目标定位（Critical Target Localization）
  4. 交通常识与规则（Traffic Commonsense & Rules）

**训练过程**：
- 以 Qwen2.5-VL-3B 为基座模型
- 全参数微调（Full Fine-Tuning）
- 输出：DriveAlign-3B，一个对驾驶场景视觉证据高度敏感的领域对齐模型

实验表明，领域对齐不仅提升了模型在驾驶 VQA 上的性能（+11.7 分），还改善了通用 VLM 基准的表现（+2.0 分），说明领域知识的注入并未损害通用能力。

#### 3.3.2 阶段二：双模式监督微调（Dual-Mode SFT）

在领域对齐的基础上，DriveAgent-R1 进入**监督微调（SFT）**阶段，目标是让模型初步掌握两种思考模式的格式和语义边界。

**数据集构建自动化 pipeline**：

第一步：数据分区
- 使用 Qwen2.5-VL-3B 测试所有样本：如果不用工具就能正确预测，归为 $D_{text}$（无需工具集）
- 使用 Qwen2.5-VL-72B 测试剩余样本：如果用工具能带来性能提升，归为 $D_{tool}$（需要工具集）
- 否则：丢弃

第二步：CoT 标注生成
- 用 Qwen2.5-VL-72B 为 $D_{text}$ 和 $D_{tool}$ 分别生成模式特定的 CoT 标注

第三步：质量验证
- 使用独立的裁判模型（Judge Model）评估推理连贯性
- 低于质量阈值的样本自动重新生成

第四步：规则清洗
- 格式检查、逻辑一致性验证
- 最终获得平衡的 4000 条高质量 SFT 数据（$D_{text}$ 和 $D_{tool}$ 各 2000 条）

#### 3.3.3 阶段三：级联强化学习（Cascaded RL）

SFT 阶段让模型"知道"两种模式的存在，但要让模型**真正精通**并**智能选择**，需要强化学习（RL）的进一步打磨。DriveAgent-R1 设计了**两级级联 RL**训练：

**子阶段 3.1：强制对比模式强化学习（FCM-RL）**

目标：深化模型在每种模式下的独立能力，确保不会因为初始性能差异而产生模式偏见。

核心算法：**Mode-Partitioned GRPO（MP-GRPO）**

GRPO（Group Relative Policy Optimization）是 DeepSeek-R1 验证有效的 RL 算法。MP-GRPO 在此基础上增加了模式分区机制：

对于每个输入样本 $q$：
1. 强制模型生成 $G/2$ 个 $M_{text}$ 模式的回复（通过强制 `<think_text>` token）
2. 强制模型生成 $G/2$ 个 $M_{tool}$ 模式的回复（通过强制 `<think_tool>` token）
3. 将 $G$ 个回复作为一个统一集合进行奖励归一化和优势估计

这种设计创造了**多维对比学习信号**：
- **模式内对比**：同一模式下不同回复轨迹的比较
- **模式间对比**：不同模式在同一问题上的性能比较

**奖励函数设计**：

$$R = R_{acc} + R_{fmt}$$

- $R_{acc}$（准确率奖励）：与 ground truth 的加权 Levenshtein 距离，考虑位置-动作权重以缓解类别不平衡
- $R_{fmt}$（格式一致性奖励）：惩罚结构错误和错误的模式使用（如在 $M_{text}$ 模式中调用工具）

**子阶段 3.2：自适应模式选择强化学习（AMS-RL）**

目标：培养模型根据场景复杂度自适应选择最优思考模式的能力。

在 AMS-RL 阶段，模型不再被强制选择模式，而是**自己生成模式选择 token**，然后 commitment 到对应的推理路径。

**奖励函数设计**：

$$R = R_{acc} + R_{fmt} + \mathbb{I}(mode = M_{tool}) \cdot R_{tool}$$

新增 $R_{tool}$（条件工具使用奖励），鼓励**有选择性的工具使用**：
- 在每个样本的 $G$ 个回复中，计算所有纯文本轨迹的平均准确率 $\bar{Acc}_{text}$ 作为基线
- 只有当工具轨迹的准确率 $R_{acc}$ 超过基线一定 margin 时，才给予正奖励
- 这种对比机制**明确惩罚冗余的工具调用**，激励模型仅在工具能带来明显收益时才使用

## 四、实验验证：小模型的大作为

### 4.1 实验设置

**数据集**：
- **Drive-Internal**：理想汽车内部数据集，3.5 万条视频片段，专门针对长尾和复杂场景设计，涵盖 40 余种驾驶情境（施工、动物穿行等）
- **nuScenes**：公开数据集，2000 条验证集样本用于跨数据集泛化测试

**评估指标**：
- **准确率（Acc）**：首帧准确率和序列平均联合准确率
- **宽松速度匹配（Relaxed Speed Matching）**：为了鼓励更安全的行为，预测比 ground truth 更安全（更慢）时给予部分分数（0.5 或 0.2）
- **模式选择准确率（MSA）**：衡量模型自适应模式选择与最优模式选择的对齐程度

**对比基线**：
- 闭源 SOTA：GPT-5、GPT-4.1、Gemini-2.5-Flash
- 开源模型：Qwen2.5-VL-3B/7B/72B
- 人类驾驶员表现

### 4.2 主实验结果

表 1 展示了 DriveAgent-R1 在 Drive-Internal 测试集和 nuScenes 测试集上的性能：

| 模型 | 参数量 | Drive-Internal (无工具) | Drive-Internal (有工具) | nuScenes (无工具) | nuScenes (有工具) |
|------|--------|------------------------|------------------------|------------------|------------------|
| GPT-5 | - | 47.82% | 51.50% (+3.68%) | 43.20% | 45.14% (+1.94%) |
| GPT-4.1 | - | 45.40% | 49.12% (+3.72%) | 41.68% | 44.10% (+2.42%) |
| Gemini-2.5-Flash | - | 44.64% | 48.32% (+3.68%) | 40.96% | 43.38% (+2.42%) |
| Qwen2.5-VL-72B | 72B | 43.52% | 44.16% (+0.64%) | 39.84% | 40.42% (+0.58%) |

*表 1：主实验结果（序列平均联合准确率）*

**关键发现**：

**发现一：小模型的大突破**
DriveAgent-R1 仅用 30 亿参数，就达到了与 GPT-5 和人类驾驶员相当的性能。在 Drive-Internal 上，它的工具增强版本（51.90%）甚至超过了 GPT-5 的工具版本（51.50%）；在 nuScenes 上，它的序列准确率（47.10%）直接超过了 GPT-5（45.14%），逼近人类水平（48.56%）。

**发现二：视觉工具是双刃剑**
一个有趣的现象是：**并非所有模型都能从视觉工具中获益**。GPT-4.1 和 Gemini-2.5-Flash 在使用工具后性能提升约 3.7%，但 Qwen2.5-VL-3B 和 7B 在使用工具后性能反而下降（约 -1.6%）。这说明**有效的工具使用是一项非平凡技能**，需要针对性的专门训练。DriveAgent-R1 的成功恰恰证明了其训练策略的有效性。

**发现三：工具使用带来显著增益**
DriveAgent-R1 的工具增强版本相比无工具版本，在 Drive-Internal 上提升 +6.00%，在 nuScenes 上提升 +4.82%，这一增益幅度远超其他所有模型，说明 DriveAgent-R1 真正学会了如何利用主动感知来辅助决策。

### 4.4 定性分析：DriveAgent-R1 如何思考

图 1 展示了一个典型的 DriveAgent-R1 主动感知案例：

**场景描述**：
前方有一辆大货车，其后方似乎有异常情况，但从默认视角看不太清楚。

**推理过程**：
1. **初始观察**："前方有一辆大型货车，看起来车速较慢"
2. **不确定性感知**："货车后方似乎有车辆，但看不清楚具体情况"
3. **主动工具调用**：`<call_tool>RoI Inspection(x=320, y=180, w=120, h=80)</call_tool>`
4. **新证据获取**：放大后发现"货车后方有两辆车发生了轻微碰撞"
5. **修正决策**："前方有事故，需要减速并准备停车"
6. **最终输出**：减速 + 停车

这个案例生动地展示了主动感知的价值：如果没有 RoI 检查工具，模型可能会误判为普通的慢车跟随场景；通过主动放大观察，模型发现了关键的安全隐患，做出了更保守、更安全的决策。

## 五、深入讨论：DriveAgent-R1 的启示与局限

### 5.1 对自动驾驶 VLA 研究的启示

DriveAgent-R1 的成功为自动驾驶 VLA 研究提供了几个重要启示：

**启示一：从"看得多"到"看得准"**
传统思路倾向于"堆更多摄像头、喂更多数据"来提升感知能力。DriveAgent-R1 证明，**智能地选择看什么，比盲目地看所有东西更重要**。这种"主动注意力机制"可能是解决长尾问题的关键——不是让模型见过所有情况，而是让模型具备应对未知情况时主动获取信息的能力。

**启示二：推理深度应该是自适应的**
不是所有决策都需要同样复杂的推理。简单场景快速决策，复杂场景深度思考——这种**自适应计算**理念对实时性要求极高的自动驾驶尤为重要。DriveAgent-R1 的混合思考框架提供了一个可行的技术路径。

**启示三：领域对齐是 VLA 落地的关键步骤**
通用 VLM 虽然知识广博，但在特定领域（如驾驶）往往存在"视觉忽视"问题。通过针对性的领域对齐，可以显著提升模型的视觉敏感性和决策可靠性，这一步不应被跳过。

**启示四：小模型 + 好策略可以匹敌大模型**
DriveAgent-R1 仅用 3B 参数就达到了 GPT-5 级别的性能，这在算力受限的量产场景极具价值。它证明了**精心设计的算法可以弥补模型规模的差距**，为自动驾驶的端侧部署提供了新思路。

### 5.2 局限性与未来方向

尽管 DriveAgent-R1 取得了显著进展，但作者也坦诚地指出了一些局限：

**局限一：对工具的视觉证据过度依赖**
在某些失败案例中，模型过度依赖新获取的视觉证据，而忽视了已有上下文信息。例如，当 RoI 检查提供的信息与全局视角矛盾时，模型倾向于相信局部放大视图，可能导致误判。

**改进方向**：
- 加强模型对复杂道路拓扑和交通规则的基础理解
- 培养更细致的推理过程，能够识别和解决信息冲突

**局限二：工具调用的延迟开销**
虽然混合思考框架减少了不必要的工具调用，但每次工具调用仍会带来额外的推理延迟（约 100-200ms）。在极端时间敏感的场景（如紧急避障），这可能成为瓶颈。

**改进方向**：
- 开发更轻量级的工具实现
- 探索工具调用的预测性预取机制

**局限三：泛化到未见场景的能力**
虽然 DriveAgent-R1 在 Drive-Internal 的长尾场景上表现良好，但在完全未见过的场景类型（如极端天气、异国交通规则）上的泛化能力仍需验证。

**改进方向**：
- 构建更多样化的训练数据
- 探索与世界模型结合的仿真训练

### 5.3 产业落地前景

DriveAgent-R1 的研究成果对自动驾驶产业具有直接的参考价值：

**对主机厂的价值**：
- **低成本高性能路径**：3B 模型即可达到 GPT-5 水平，大幅降低算力成本
- **可解释性增强**：主动感知的过程天然提供了决策依据的可视化
- **安全冗余设计**：混合思考框架可作为现有端到端系统的安全备份

**对算法供应商的价值**：
- **技术路线验证**：证明了主动感知在自动驾驶规划中的可行性
- **数据闭环启发**：主动感知识别的不确定性场景，可作为数据采集的优先级参考

**对芯片厂商的价值**：
- **端侧大模型部署参考**：3B 参数+自适应计算的架构，为车端芯片设计提供了明确的目标场景

## 六、结语：迈向更智能的自动驾驶

DriveAgent-R1 代表了自动驾驶 VLA 研究的一个重要里程碑。它首次将**主动感知**引入高级行为规划任务，让自动驾驶 agent 具备了"不确定时主动观察"的人类级认知能力。同时，**混合思考框架**和**级联强化学习训练策略**的设计，展示了一条在有限算力预算下实现高性能的可行路径。

更重要的是，DriveAgent-R1 的研究理念——**从被动感知到主动探索、从固定计算到自适应推理**——可能预示着自动驾驶技术范式的下一次转移。当我们不再满足于让模型"记住"所有可能的情况，而是教会模型"学会学习"、"学会提问"，自动驾驶系统才能真正具备应对开放世界不确定性的能力。

30 亿参数匹敌 GPT-5，这个数字背后是算法创新的力量。在算力军备竞赛日益激烈的今天，DriveAgent-R1 提醒我们：**精巧的设计，往往比蛮力的堆叠更能触及智能的本质**。
